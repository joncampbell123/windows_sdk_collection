Filter Categories

If you use the GraphEdt sample tool that comes with the Microsoft® DirectShow® SDK, and choose Insert Filters from the Graph menu, you will notice that the DirectShow filters on the system are grouped into categories. The DirectShow Filters category is the generic category that most filters fall into. New filters will be registered in this category by default. Graphedt does not display empty categories.

However, if your filter is one of the following kinds of filters, you should make sure that your filter appears in the proper category.

  • Audio Capture Sources - generate audio data from an external device.
  • Audio Compressors - compress PCM audio.
  • Audio Renderers - play audio a data on a local set of speakers.
  • External Renderers - sends data to an external peripheral (eg: 1394 DV camera).
  • Device Control - controls the transport of a video tape machine (eg: 1394 DV camera).
  • Midi Renderers - play MIDI data on a local set of speakers.
  • Video Capture Sources - generate video data from an external device.
  • Video Compressors - turn RGB video into compressed video.

For information on how to register your filters by category, see the AMovieDllRegisterServer2 function topic, which contains sample code for registering a Video Compressor filter. The AMCap sample shows how to enumerate a special category. The VidCap and the VCR control samples both demonstrate registering a filter in a special category. For additional information about registering, see Register DirectShow Objects.
