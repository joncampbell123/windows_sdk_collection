DXBNDS Union

Describes either a continuous or a discrete boundary for a transform object.

Syntax

typedef union DXBNDS switch( DXBNDTYPE eType ) u
{
  case DXBT_DISCRETE:
    DXDBND D[4];
  case DXBT_DISCRETE64:
    DXDBND64 LD[4];
  case DXBT_CONTINUOUS:
    DXCBND C[4];
  case DXBT_CONTINUOUS64:
    DXCBND64 LC[4];
} DXBNDS;

Members

D[4]
Array of DXDBND discrete bounds, containing 32-bit LONG minimum/maximum values.
LD[4]
Array of DXDBND64 discrete bounds, containing 64-bit LONGLONG minimum/maximum values.
C[4]
Array of DXCBND continuous bounds, containing 32-bit FLOAT minimum/maximum values.
LC[4]
Array of DXCBND64 continuous bounds, containing 64-bit DOUBLE minimum/maximum values.
eType
Member of the DXBNDTYPE enumeration, describing which member of the union to use.

Remarks

You should never have to use this union directly. Instead, it is used by the CDXBnds template class (found in DXBounds.h) to create and manipulate image and mesh boundary objects. The most commonly used boundary objects are the discrete CDXDBnds object for two-dimensional (2-D) images and the continuous CDXCBnds object for three-dimensional (3-D) meshes.

The four members of this continuous or discrete array consist of an x-dimension, y-dimension, z-dimension, and t- (time) dimension. Specify a minimum value of zero and a maximum value of one for any dimensions you will not be using. For example, the z-dimension and t-dimension may be set this way for a 2-D image. If you use the CDXBnds template class, proper initialization will be done for you.

The time dimension is intended for special data types, such as video or GIF animations.
