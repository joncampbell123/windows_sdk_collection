Pixel
An indivisible element of an image used for image reproducion on hardware. Any computer image that is reproduced is comprised of an array of pixels of different colors. The color of a pixel is a mix of 3 amounts of red, green, and blue. This color can be represented in a variety of pixel formats.

A sample is a rectangular element of an image that is stored in computer memory. The terms 'pixel' and 'sample' are often used interchangably.

Pixel format
The size and arrangement of a pixel's color components in memory. It is specified by the total number of bits used per pixel, and the number of bits used to store the red, green, blue, and alpha components of the color. For example, the RGB24 pixel format uses 24 bits to store a pixel color, with eight bits each for red, green, and blue. The ARGB32 pixel format uses 32 bits, with 24 bits of color information and 8 bits of alpha channel information.
PMARGB32
One of the two native pixel formats used by DirectX Transform for image modification; the other is ARGB32. It uses 8-bit values for alpha, red, green, and blue, for a total of 32-bits per pixel. In this format, each of the colors are alpha premultiplied, which makes alpha blending operations more efficient.
Procedural surfaces
Procedural surfaces dynamically calculate the necessary pixel values when the surface is accessed. Regular surfaces (such as a DirectDrawSurface) store an array of discrete values in memory. Procedural surfaces only store a procedure which is used to calculate the values. This provides an effective way to conserve memory and to decrease network download time. Because these surfaces are the result of mathematical calculations, access to them is always read-only.
