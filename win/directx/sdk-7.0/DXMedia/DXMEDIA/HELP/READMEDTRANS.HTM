Transforms in This Release

  • Transforms in Microsoft® DirectX® Media 6: The redistributable package (Dxmedia.exe) includes the core transform support and some core image and three-dimensional (3-D) transforms.
  • Optional transforms: This release includes about 40 optional transforms that are installed only if the "Optional DirectX Transforms" item is chosen during installation of the DirectX Media software development kit (SDK). If this option is chosen, Dxtlipi.dll, Dxtmeta.dll, Dxtmeta2.dll, Dxtsqfx.dll, Dxtcr.dll, and Dxtmsftp.dll are installed for an Intel CPU, and Dxtsqfx.dll, Dxtcr.dll, and Dxtmsftp.dll are installed for an Alpha-based CPU. Dxtmsftp.dll includes property pages for the core transforms. These optional dynamic-link libraries (DLLs) are part of the redist\{x86, alpha}\dxtxtra.exe self-extracting exe. You can uninstall the optional transforms separately.

Optional transforms are available for redistribution according to the terms of the legal agreement. They are part of Dxtxtra.exe, which requires the core support in Dxmedia.exe.

Requirements and Dependencies

  • Image loading depends on Internet Explorer 4 or later: You need to have Internet Explorer 4 or later installed to use transforms (because image loading depends on certain Internet Explorer 4 or later components).
  • Three-dimensional transforms depend on DirectX Foundation 6: 3-D transforms depend on DirectX Foundation 6 (included on this CD), due to the dependency on Direct3DRMMeshBuilder3 from Microsoft Direct3D® Retained Mode.
  • 3-D transforms are not supported on Microsoft Windows NT® 4: 3-D transforms depend on DirectX Foundation 6, which is not supported on NT 4. Only image transforms will run on NT 4.
  • Building and running samples depends on Microsoft Developer Studio®: You need to have Microsoft Developer Studio Visual C++ 5® or Visual C++ 6 (with MFC and ATL options) installed to build and run the C++ samples.
  • Building samples requires certain header and library files: Before building the samples, you need to add the following to your directory settings in Visual C++ under the menu Tools/Options/Directories: 
    Include:
DXMEDIA\include

Lib:
DXMEDIA\lib

    Make sure these are at the top of their respective lists, or else you might pick up some older header files.

  • Compiling transforms depends on DirectX Foundation 6 header files: To develop image or 3-D transforms or to build the samples, you need to copy the DirectX Foundation 6 include and lib files from the CD Extras\DXF6.0\{include, lib} to Dxmedia\include and Dxmedia\lib, respectively.

Known Limitations and Issues

  • Problem with the MIDL compiler: If you get a compiler error from MIDL upon compiling an IDL file, such as: "MIDL error 0xc0000005: unexpected compiler problem. Try to find a work around," there is a known issue with the MIDL compiler that shipped with Visual C++ 5. If you can't find a work around, contact dxmbeta@microsoft.com.
  • Bogus directory names generated by Visual C++ 6: There is a known issue with Visual C++ 6 where the results of compilation/build are placed under directories with bogus names instead of "retail". It is okay to ignore these names or to rename these directories after compilation.
  • Uninstalling Chromeffects 1.0: If you install the DirectX Media SDK over Chromeffects 1.0 and then uninstall Chromeffects 1.0, Dxtmsft.dll will be removed and with it a number of 2-D transforms will be removed. Reinstalling the SDK fixes the problem.
  • DXETool and 8-bit mode: DXETool does not support 8-bit mode. It is recommended that you use 16-bit mode while using the tool.
  • The documentation for IDXARGBReadWritePtr::OverArrayAndMove states "After a call to this method, the positions of both the source and destination surfaces are advanced by cSamples." This should say "After a call to this method, the position of the write pointer is advanced by cSamples."

Using DXETool

DXETool is a tool for previewing all the installed transforms and is accessible through Dxmedia\bin\DxeTool.exe. The tool provides the following capabilities for testing transforms as they are developed or for previewing/demonstrating installed transforms.

  • Select the transform you are interested in browsing from the list.
  • If a transform has optional inputs, select a desired number of inputs by selecting the corresponding radio button under Inputs.
  • Provide the inputs to the transform by loading either images (JPEG, GIF, PNG, BMP) or 3-D meshes (.x files) through the File menu; for example, from the Dxmedia\Samples\Multimedia\Media\Image or Dxmedia\Samples\Multimedia\Media\Geometry directory. Furthermore, through the File menu you can select predefined 3-D meshes containing submeshes. This is intended to help you test the proper handling of submeshes by a transform you are developing (submeshes are new and are rarely found in .x files at this point).
  • Use the slider labeled Progress to drive the Progress property, which the majority of transforms support. You can set Progress on auto-animate by selecting Repeat or Bounce. You then need to press the ESC key to stop.
  • If a transform (such as CrShatter or Explode) has a 3-D output, the tool provides basic 3-D navigational capabilities as follows:
    • Click to rotate object
    • Right-click to zoom in and out
    • Hold down the CTRL key and click to move light source
    • Hold down the SHIFT key and click to move object in the x-y plane
  • The button labeled Settings displays the property page for the transform if one is supported. This helps you set the custom properties and shows the default values of these properties.
  • The pixel format in DXETool enables you to convert the input and output surfaces to the different supported pixel formats.
  • The Dirty A and Dirty B buttons modify the corresponding image or mesh inputs, and provide a handy way to check and make sure that transforms pay attention to changes in these inputs.
  • The Bounds Check check box (enabled by default) provides a means to disable bound checking. All transforms must pass the bounds checking test, and this check box should always be enabled while developing transforms.

DirectX Transform SDK Details

  • Dxmedia\Samples\Multimedia\Dtrans\Bin includes the DLLs for sample transforms and some sample tools for hosting these transforms.
  • Dxmedia\Samples\Multimedia\Dtrans\C++ provides source code for C++ samples of image transforms, 3-D transforms, procedural surfaces, DXSurface usage, and applications that host transforms.
  • Dxmedia\Samples\Multimedia\Dtrans\Danim provides Microsoft DirectAnimation™ samples in Microsoft JScript® that host the following transforms: Explode, Ruffle, Ripple, Wipe, and several optional transforms.
  • Dxmedia\Samples\Multimedia\Media includes geometry and image media files that are needed to demonstrate certain transform capabilities. This includes .png image files with an alpha channel, .fpx image files for Kodak Flashpix™ images to be used with the FlashPix transform, and .mts progressive geometry files for the MetaStream transform.
  • Dxmedia\Include and Dxmedia\Lib include DirectX Transform header and library files that need to be included as the first items in your Visual C++ directory settings for building the samples.
  • Dxmedia\Bin\Pstrans.8bf includes the Adobe Photoshop filter adapter for transforms. If you have Photoshop installed, you need to copy this file to your filters directory to enable image transforms as Photoshop filters.
  • Extras\Dtrans\X86\Dxtrm.dll includes additional transforms that were minimally tested. These include the following 3-D transforms: CrBend, CrFade, CrInflate, CrMelt, CrMorph, CrTense, CrTwist, CrWibble, CrWipe. If you're interested in exploring these transforms you need to copy the Dxtrm.dll to your local drive and register it by calling "regsvr32". You can then preview the transforms through DXETool.
  • The FlashPix transform is a DxSurface for loading Flashpix images. You can try it through DXETool by clicking the Settings button and providing a path to a .fpx file in the Dxmedia\Samples\Multimedia\Media\Image directory. You can also try the FlashPix transform with remote files on a site running Flashpix server from the Live Picture Image Server.
  • The MetaStream transform is used for loading progressive meshes. You can try it through DXETool by clicking the Settings button and providing a path to an .mts file in the Dxmedia\Samples\Multimedia\Media\Geometry directory. You can also try the MetaStream transform with remote files such as:

    http://www.metastream.com/gallery/models/greencap.mts

    http://www.metastream.com/gallery/models/skull.mts

Instructions to Transform Writers

It is very important that you thoroughly read and follow the instructions in the article titled "Author's Guide to Transforms" under "Writing Transforms" in the DirectX Transform documentation. It's very important to follow the steps in the article to produce a proper transform that conforms to the architecture and hence can be hosted in Microsoft DirectAnimation™ and Microsoft Chromeffects™ interactive media technology.

DXETool is a validation tool that helps to ensure that a transform behaves properly, including:

  • Correct basic behavior including setup and execution
  • Correct handling of the required and optional inputs
  • Correct bound reporting
  • Correct provisions for picking
  • Correct checking of changes to inputs (through generation IDs)
  • Correct processing of 3-D mesh inputs including meshes with submeshes
  • Proper property pages for setting custom properties
