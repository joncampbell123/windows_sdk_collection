IDAPreferences Interface

The IDAPreferences interface is the COM interface that defines preferences for DAView objects. You can use IDAPreferences to provide explicit rendering and viewing preference settings to Microsoft® DirectAnimation™, such as output audio format, the maximum frames per second, kind of texturing, and fill mode. See the Remarks section in the IDAPreferences::PutPreference method for a list of preferences and their values.

This interface inherits from IDispatch.

IDAPreferences Methods

GetPreference Retrieves the value for the specified preference for the view.
Propagate Propagates the preferences to the view they are attached to.
PutPreference Sets the specified preference to the specified value for the view.

IDAPreferences::GetPreference

IDAPreferences

Retrieves the value for the specified preference for the view.

Syntax

HRESULT GetPreference( BSTR preferenceName, VARIANT *value );

Parameters

preferenceName
[in] BSTR value containing the name of the preference whose value is to be retrieved. See the Remarks section in PutPreference for a list of preferences and their values.
value
[out] Current value of the specified preference.

Return Value

Returns an HRESULT value that indicates the success of the method or the reason why it failed. This HRESULT depends on the implementation of the interface.

See Also

IDAPreferences::PutPreference, IDAView::get_Preferences

IDAPreferences::Propagate

IDAPreferences

Propagates the preferences to the view they are attached to. Not all preferences can be propagated successfully at all times. For example, whether 3-D hardware will be enabled and whether an application can override registry entries are preferences that cannot be changed by the application as it runs.

Syntax

HRESULT Propagate( );

Return Value

Returns an HRESULT value that indicates the success of the method or the reason why it failed. This HRESULT depends on the implementation of the interface.

See Also

IDAView::get_Preferences

IDAPreferences::PutPreference

IDAPreferences

Sets the specified preference to the specified value for the view.

Syntax

HRESULT PutPreference( BSTR preferenceName, VARIANT value );

Parameters

preferenceName
[in] BSTR value containing the name of the preference to be set. A BSTR is a length-prefixed string. The length is stored as an integer at the memory location preceding the data in the string. See the following Remarks section for a list of preferences and their values.
value
[in] Value to set the specified preference to.

Return Value

Returns an HRESULT value that indicates the success of the method or the reason why it failed. This HRESULT depends on the implementation of the interface.

Remarks

The following table lists preferences and their values.

PreferenceDescriptionValue
COLOR_KEY_BLUESpecifies the value of the blue RGB component.An integer from 0 to 255. Default value is 245.
COLOR_KEY_GREENSpecifies the value of the green RGB component.An integer from 0 to 255. Default value is 254.
COLOR_KEY_REDSpecifies the value of the red RGB component.An integer from 0 to 255. Default value is 1.
DITHERINGDetermines whether dithering is enabled.A Boolean. Default mode is Enabled.
ENGINE_OPTIMIZATIONS Specifies whether there will be optimizations. Optimizations include such things as image caching and dirty rectangle optimization (optimization of updating rectangles that have changed since the animation was last drawn). A Boolean. Default mode is On.
FILL_MODE Specifies the fill mode. FILL_MODE_POINT, FILL_MODE_SOLID, or FILL_MODE_WIREFRAME. Default mode is FILL_MODE_SOLID.
OVERRIDE_APPLICATION_PREFERENCES Specifies whether the registry overrides any attempts by the application to set preferences.You can't set this value. It can only be retrieved with GetPreference. Default value is Off.
PERSPECTIVE_CORRECTSpecifies whether texturing will be adjusted for perspective.A Boolean. Default mode is On.
MAX_FRAMES_PER_SECSpecifies the maximum number of frames per second.A double. Default value is 30.
RGB_LIGHTING_MODESpecifies whether the lighting mode is RGB. A Boolean. Default mode is On.
SHADE_MODESpecifies what the shading mode will be.SHADE_MODE_FLAT, SHADE_MODE_GOURAUD, or SHADE_MODE_PHONG. Default mode is SHADE_MODE_GOURAUD. In the flat shade mode, the color of one vertex is duplicated across all the other faces. In Gouraud shading, the color and intensity of adjacent vertices is interpolated across the space that separates them. In Phong shading, the appropriate shade value is calculated for each pixel on a face. See the "Technical Background" article in the Direct3D Retained Mode documentation for more information.
TEXTURE_QUALITYSpecifies what the texture rendering quality will be.TEXTURE_QUALITY_LINEAR or TEXTURE_QUALITY_NEAREST. Default mode is TEXTURE_QUALITY_NEAREST.
USE_3-D_HWSpecifies whether 3-D hardware will be enabled. A Boolean. Default mode is Off. A read-only value that the application can't override.
USE_VIDEOMEMSpecifies whether the video memory will be enabled.A Boolean. Default mode is On.
