What's New Since DirectShow 5.1

The DirectX Media 5.2b run time has improvements in two specific areas - DVD Playback and WDM Capture support.

For DVD playback, the run time provides support for PAL DVD discs as well as improved interactive responsiveness. The run time has also been tested with many new DVD titles that were released in the last two months.

The Overlay Mixer, which is a key rendering component for DVD playback, has been improved in the following areas.

  • New interfaces added on each pin of Overlay Mixer.
    • IOverlay enables upstream filters to do the rendering themselves, leaving only the window management to the Overlay Mixer.
    • IMixerPinConfig2 has methods to set and get brightness, contrast, and other color control features, as if setting the color controls on the overlay surface.
    • IVPNotify2 has methods to make the video port the sync master of the VGA, and to determine if the video port is the sync master.
  • The IMixerPinConfig methods SetZOrder, GetZOrder, SetBlendingParameter, and GetBlendingParameter, which previously returned E_NOTIMPL, have been enabled now.
  • Overlay Mixer now tries to create the overlay surface in AGP memory if allocation in video memory fails.
  • Some important scaling bugs, including one that did not allow shrinking of overlay surfaces on certain video cards (like ATI rage pro), have now been fixed.
  • Previously, using IBasicVideo to zoom or stretch video did not do the correct thing. That has now been fixed.

The video and audio capture infrastructure has been enhanced to add support for more types of WDM streaming class capture hardware running under Microsoft® Windows® 98 or Windows NT® 5.0. If your application is going to work with this new hardware, you should re-read the capture interface descriptions and re-read the code for our sample capture application, Amcap.exe found in the Capture sample directory.

In addition, a new video control interface has been added, IAMVideoControl, that enables you to flip a picture horizontally and/or vertically, to set up a stream so that it can capture from an external trigger, to simulate an external trigger in software, and to list the current frame rate, maximum frame rate, and available frame rates.

A new filter, Smart Tee, has been added to aid preview on capture filters that only have a capture pin and not a preview pin.

Overview

This release of Microsoft® DirectX® Media contains the following:

  • Direct X Media 5.2b components:
    • DirectShow™ run time and SDK
    • DirectAnimation™ run time and SDK

    DirectShow

    DirectShow enables capture of multimedia streams from devices and playback of multimedia streams from local files or Internet servers. The streams can contain video and audio data compressed in various formats, including MPEG, Apple QuickTime, audio-video interleaved (AVI), and WAV files. Capture can be based on either Video for Windows or Windows Driver Model (WDM).

    The DirectShow run time consists of:

    • Amovie.ocx and Amovie.chm — the Microsoft ActiveMovie™ Control and its documentation help file
    • Quartz.dll and Quartz.vxd — the playback dynamic-link library (DLL) and its Microsoft Windows® 95 helper file
    • Amstream.dll — the multimedia streaming DLL
    • Devenum.dll — the PNP device enumeration DLL
    • Mciqtz32.dll and Mciqtz.drv — the MCI support DLL and its Windows 95 helper file
    • Vidx16.dll — the 16-bit capture DLL for Windows 95

    The DirectShow SDK consists of filters, C++ classes, header files, libraries, tools, samples, and documentation. See CD Layout for more information.

    DirectAnimation

    DirectAnimation provides unified, comprehensive support for animation, streaming, and integration of diverse media types, such as two-dimensional (2-D) vector graphics, three-dimensional (3-D) graphics, sprites, audio, and video. Because it is a Component Object Model (COM) API with an underlying engine/run time, DirectAnimation functionality can be accessed — in different ways — by a wide variety of programmers and authors:

    • HTML authors can animate their Web pages by using the DirectAnimation controls and setting the control parameters, without any programming.
    • Scripting authors using Microsoft Visual Basic® Scripting Edition (VBScript) or Microsoft JScript™ can integrate animations with Dynamic HTML to animate the properties of their Web pages.
    • Programmers using Java, Visual Basic, and Microsoft Visual C++® can develop Microsoft ActiveX™ controls or full applications that support multimedia and interactivity.

    The DirectAnimation run time consists of:

    • Danim.dll — the DirectAnimation DLL
    • Daxctle.ocx — the DirectAnimation viewer control
    • Mmefxe.ocx and Mmutilse.dll — the DirectAnimation multimedia Dynamic HTML controls and DLL
    • Dajava.cab — the DirectAnimation Java classes
    • Lmrt.dll — the Liquid Motion Pro run time DLL
    • Proctexe.ocx — filters and transitions for Microsoft Internet Explorer 4.x from Intel® Corporation

    The DirectAnimation SDK consists of libraries, header files for the C++ samples, JScript, VBScript, Java, Visual Basic, and C++ samples, media files, and documentation. See CD Layout for more information.

    How to Use the Documentation

    The DirectX Media SDK documentation is supplied in two ways: as an InfoViewer file and as HTML.

    The InfoViewer files, DXMedia.ivt and DXMedia.ivi, are in the subdirectory x86\Dxmsdk\help within the SDK and contain the DirectShow, DirectAnimation, DirectPlay, and Direct3D Retained Mode documentation. The InfoViewer files are installed only if InfoViewer is present on the computer. (InfoViewer ships with the MSDN library, Visual C++, and other Microsoft products.)

    The HTML documentation opening page, Start.htm, is in the \Dxmsdk subdirectory within the SDK. You can only read this file if you have installed Internet Explorer 4.x, which is included on the CD. You can also bypass this opening page and go directly to the start pages of documentation that interests you. These start pages can be viewed in any browser:

    • The DirectShow documentation start page is \dxmsdk\help\ds\default.htm.
    • The DirectAnimation documentation start page is \dxmsdk\help\da\default.htm.
    • The DirectPlay documentation start page is \dxmsdk\help\dplay\default.htm.
    • The Direct3D Retained Mode documentation start page is \dxmsdk\help\d3drm\default.htm.

    CD Layout

    The DirectX Media 5.2b SDK contains the following directories (Note: this CD includes only the x86 version of the SDK, not the alpha version):

    • x86 — contains the complete x86 version of the DirectX Media SDK
      • Redist — contains the DirectX Media redistributable run time.
      • DXMSDK — contains the DirectX Media SDK.
        • Bin — contains the DirectShow filters.
        • Classes — contains the DirectShow C++ classes. These are located in the subdirectory Classes\Base.
        • Help — contains the DirectShow and DirectAnimation documentation files and subdirectories:
          • start.htm — the starting page of the DirectX Media HTML documentation, including DirectShow, DirectAnimation, DirectPlay, and Direct3D Retained Mode.
          • Relnotds.htm and Relnotda.htm — the Readme HTML files for DirectShow and DirectAnimation, respectively.
          • DXMedia.ivt and DXMedia.ivi — the InfoViewer files for DirectShow, DirectAnimation, DirectPlay, and Direct3D Retained Mode. The InfoViewer files are installed only if InfoViewer is present on the computer. (InfoViewer ships with the MSDN library, Visual C++, and other Microsoft products.)
          • Ds subdirectory — contains the DirectShow HTML documentation files.
          • Da subdirectory — contains the DirectAnimation HTML documentation files. The documentation is in the \Contents subdirectory and the documentation samples are in the \Docsamps subdirectory.
          • D3drm subdirectory — contains the Direct3D Retained Mode HTML documentation files. Note that Direct3D Retained Mode ships with the DirectX Foundation SDK.
          • Dplay subdirectory — contains the DirectPlay HTML documentation files. Note that DirectPlay ships with the DirectX Foundation SDK.
        • Include subdirectory — contains the DirectShow C++ header files, and the header files needed by the DirectAnimation C++ samples.
        • Lib subdirectory — contains the DirectShow and DirectAnimation library files.
        • Media subdirectory — contains the DirectShow and DirectAnimation media subdirectories and the DirectAnimation Java class files. The media subdirectories include:
          • Geometry — contains 3-D images in DirectX file format (.x files).
          • Image — contains image files in GIF and JPG formats.
          • Midi — contains music files in MIDI format.
          • Movie — contains a video file in AVI format.
          • Sound — contains sound in MP2 and WAV formats.
        • Samples — contains the DirectShow and DirectAnimation samples. The DirectShow samples are written in C++ and are in the Samples\Ds directory. DirectAnimation includes C++, Java, JScript, VBScript, Visual Basic, and multimedia controls samples. The Samples\Da directory contains the following subdirectories:
          • C++ — the DirectAnimation C++ samples.
          • Controls — the DirectAnimation multimedia controls samples.
          • Java — the DirectAnimation Java samples
          • JScript — the DirectAnimation JScript samples.
          • VBScript — the DirectAnimation VBScript samples.
          • VisualBasic — the DirectAnimation Visual Basic samples.
        • Tools — contains DirectShow tools and compiled DirectShow samples. The Vc5kit subdirectory contains a Visual C++ version 5.x project with project settings needed by DirectShow filters and a document describing how to use the project to build your own filters.
        • Webcd — contains HTML pages used by the DirectAnimation documentation to display the DirectAnimation samples within documentation topics.

    Hardware Requirements

    You should have a 486 processor or faster, 16 megabytes (MB) of RAM, and 100 MB of hard disk space. If you want to hear the sound in the movies and samples, you need a sound card and speakers. If you want to do video or audio capture, you need capture equipment.
