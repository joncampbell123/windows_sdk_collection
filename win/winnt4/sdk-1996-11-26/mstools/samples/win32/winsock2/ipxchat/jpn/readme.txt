IPXCHAT サンプル アプリケーション

本プログラムは、IPX/SPX プロトコルを通じて通信するための Windows Sockets
の使用方法をデモンストレーションします。


セットアップ:

IPXCHAT は IPX プロトコルがロードされていなければなりません。"IPX/SPX 互換
ﾄﾗﾝｽﾎﾟｰﾄ" が本アプリケーションのテストに使用されました。会話を成立させたい
２台のマシンが互いに同一フレーム タイプにプロトコルが正しく構成されているか
を確認してください。


IPXCHAT の使用方法:

１つのソケット アプリケーションには、サーバー側(受け側)とクライアント側(呼出側)
の２つの側面があります。最初のマシンは、サーバーとして IPXCHAT をセットアップ
します。 

IPXCHAT を起動して、[Options]メニューから[Listen]コマンドを選択してください。

ダイアログ内にソケット番号を入力するためのエディットコントロールが表示されます。
デフォルト値として 2FFF が提供されますが、任意の値に変更することが可能です。
Novell の"IPX Router Specification"を参照してください。

OK を選択すると、IPXCHAT は Windows Sockets の初期化を行い、指定されたソケット
でリスニングを開始します。ダイアログのステータス ボックスには、
"Listening on <netnum>.<nodenum>.<socknum>"というメッセージが表示されます。
<netnum>, <nodenum>, <socknum> にはそれぞれネットワーク番号、ノード番号、ソケッ
ト番号が表示されます。接続するためには、クライアント側でこれらの番号を入力する
必要があります。


２台目のマシンにて(同一マシンでも可能):

IPXCHAT を起動して、[Options]メニューから[Connect]コマンドを選択してください。

ダイアログ内にネットワーク番号、ノード番号、ソケット番号を入力する為のエディッ
トコントロールが表示されます(サーバーの Listen ダイアログのステータス ボックス
で示された３つの番号です)。一旦接続が確立すると、上側のエディット コントロール
にタイプしてください。そしてタイプされた文字は対の片方のエディット コントロール
の下側に表示されます。
