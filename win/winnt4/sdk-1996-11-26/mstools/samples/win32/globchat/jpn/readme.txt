グローバル チャット Windows ソケット サンプル アプリケーションは、
新しい名前登録・解決API を使用しています。

このアプリケーションは、仲介役としてのサーバーを使用して２つのクライアント
でチャットをすることができます。これらは、新しい名前登録・解決API の恩恵の
おかげで、同種のプロトコルがロードされている必要はありません。サーバーは、
２つのクライアント間のゲートウェイとして動作し、どんなプロトコルが使用され
ているのかということをまったく知らずして、適切な Windows ソケット インター
フェースにのっとり、メッセージを転送しています。

グローバル チャットの実行方法:
1)Windows NT 3.5 server 上で GLOBCHAT.EXE を実行してください。このマシンで
動作するすべてのプロトコル(クライアントが実行する可能性のある)が得られます。

2)クライアント マシン上で GLOBCL.EXE を起動してください。
[オプション]メニューの[サーバーへの接続]を選択して、表示されるダイアログに
対してプロトコルを選択して、通知したい名前を入力してください。

3)いったんこの情報を入力すると、グローバル チャット クライアントはサーバー
を発見し、チャット可能な相手ユーザーのリストが表示されます。
いずれかのユーザーを選択してチャット要求が受け入れられるのを待ちます。
そして、タイプを開始してください。

グローバル チャット サーバーは、使用されているプロトコル・各々のプロトコル
を使用して接続されているクライアント マシンの数・現時点接続されているすべて
のクランアント名のリストとそれぞれの状態等を表示します。

サポートされているプロトコルの幾つかは、単にグローバル チャット サーバーが
最初に発見したものが使用されることに注意してください。つまりネットワーク上で
複数のグローバル チャット サーバーが動作している場合、期待していたものと接続
されない可能性があるということです。
