サンプル: DDEML API デモ

概要:

インストールされた SDK サンプルに、プログラム マネージャにプログラム
項目やグループを追加する、DDEML API の使用法の例が提供されています。
このプログラムは、対話形式でもバッチ モードでも実行でき、コマンド 
ラインに検索パスを指定できます。

詳細:

このプログラムはコマンド ラインから起動します。以下のフラグが
使用できます。

   (-/)(sS) <path>  検索パスを指定します。現在、この値は単一のパ
   スでなければなりません。アプリケーションは、"MSTOOLS" という
   環境変数を検索します。この変数が見つからなければ、デフォルト
   値 "c:\mstools\samples" を使います。

   (-/)g(G) <name>  項目が追加されるグループの名前を指定します。
   デフォルトの値は "Sample Applications" です。

   (-/)b(B)  プログラムをバッチ モードで実行するよう指定します。
   プログラムは、検索を繰り返して、指定されたパス内のすべての 
   .exe ファイルを見つけ出します。その後、指定されたグループを
   作成し、見つけた実行可能プログラムをすべてそのグループに追加
   します。その後、プログラムは終了します。

   (-/)(iI)  対話モードを指定します。これはデフォルトです。
   プログラムは、検索を繰り返して、指定されたパス内のすべて
   の .exe ファイルを見つけ出します。見つかった名前は一覧表
   示されます。表示に使われる名前も追加のリストとして表示さ
   れます。ドロップダウン コンボ ボックスに、現在利用可能
   なグループの一覧が表示されます。新しいグループの名前を入
   力することもできます。

プログラムが、50 より多くの実行可能プログラムを見つけた場合は、グループ
に 50 項目を追加した後、新しいグループを作成します。その名前は、現在
プログラム マネージャ グループ リストで選択されている名前に "Part <n>" 
を付け加えたものになります。ここで、<n> は 2 から始まり、新しいグループ
が追加される度に番号の値が増えます。
