GE1263: Microsoft ODBC Drivers Ver. 3.0 for 32-Bit Programs

 Last reviewed: October 14, 1997
Article ID: Q140255
The information in this article applies to:
  • Microsoft Open Database Connnectivity
  • Microsoft Access for Windows 95, version 7.0
  • Microsoft Access, versions 1.x and 2.0
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Project for Windows 95, version 4.1
  • Microsoft Visual FoxPro(TM) version 3.0
  • Microsoft Visual Basic(R) version 4.0
  • Microsoft Visual C++ version 2.0, 2.1, 2.2, or 4.0

SUMMARY

Microsoft has made available an Application Note titled "Microsoft ODBC Drivers for 32-Bit Programs" (GE1263). This Application Note contains the latest Microsoft Open Database Connectivity (ODBC) drivers.

NOTE: If you are using Microsoft Office 97, you do not need this Application Note. The Data Access Pack is located in the Valupack\Dataacc folder on the Microsoft Office 97 compact disc. The following drivers are installed with the Data Access Pack that is included with Microsoft Office 97:

  • Microsoft Access Driver
  • Microsoft Excel Driver
  • Microsoft Text and HTML Driver
  • Microsoft xBase Driver
  • Exchange Driver
  • Paradox Driver
  • Lotus 1-2-3 Driver

You can obtain this Application Note from the following sources:
  • Microsoft's World Wide Web Site on the Internet
  • The Internet (Microsoft anonymous ftp server)
  • Microsoft Download Service (MSDL)
  • Microsoft Technical Support

For complete information, see the "To Obtain This Application Note" section at the end of this article.

INTRODUCTION

This Application Note contains the latest Microsoft Open Database Connectivity (ODBC) drivers for the following data sources:

  • ORACLE(R) versions 7.0 and 7.1
  • Microsoft SQL Server versions 4.21 and 6.0
  • Microsoft Jet database engine version 3.0, which supports the following formats: 

        - Microsoft Access versions 1.x, 2.0, and 7.0


        - Microsoft FoxPro(R) versions 2.0, 2.5, and 2.6


        - dBASE III(R), dBASE IV(R), and dBASE(R) for Windows


        - Paradox(R) versions 3.x, 4.x, and 5.0


        - Microsoft Excel versions 3.x, 4.x, 5.x, and 7.0


          [NOTE: A 16-bit version of the Oracle 7 driver in this appnote is
      available from Visigenic, which is necessary for those customers with
      Excel 5.0x who encounter bugs in the 16-bit Oracle 7 driver (from the
      ODBC Driver Pack Fulfillment Kit 2.0) that are fixed in the new
      driver. Visigenic can be reached at (800)632-2864.]


        - Text

These ODBC drivers are 32-bit only and are designed for use by ODBC- compliant, 32-bit programs running under Microsoft Windows 95 or Microsoft Windows NT(R) 3.51. Note that the ORACLE 7.0 and 7.1 ODBC drivers are designed only for Windows NT 3.51 and not Microsoft Windows 95.

Installation Requirements

To set up the ODBC drivers, you must have one of the following 32-bit ODBC-compliant programs installed on your computer:

  • Microsoft Access for Windows 95
  • Microsoft Excel for Windows 95
  • Microsoft Word for Windows 95
  • Microsoft Office for Windows 95
  • Microsoft Project for Windows 95
  • Microsoft Visual FoxPro(TM) version 3.0
  • Microsoft Visual Basic(R) version 4.0
  • Microsoft Visual C++ version 2.0, 2.1, 2.2, or 4.0

To use the ORACLE 7.0 ODBC drivers, you must have a 32-bit version of SQL*Net(R) installed on your computer. For information about obtaining the 32-bit version of SQL*Net, please contact Oracle Corporation at (800) 345-DBMS.

Installation

To install the Microsoft ODBC drivers, do the following:

  1. If you received the disks with this Application Note, run Windows Explorer in Windows 95 (or File Manager in Windows NT) and create a new folder on drive C called Ge1263.

    If you downloaded this Application Note from an online service, skip to step 5. This procedure assumes that you have already downloaded and extracted the compressed file to a folder on drive C named Ge1263.

  2. Insert Disk 1 in drive A, copy the contents of Disk 1 to the Ge1263 folder, and double-click Disk1.exe to extract the files it contains.

  3. Insert Disk 2 in drive A, copy the contents of Disk 2 to the Ge1263 folder, and double-click Disk2.exe to extract the files it contains.

  4. If you are running Windows 95, click the Start button, and then click Run. (If you are running Windows NT, choose Run from the File menu in Program Manager or File Manager.)

  5. In the Open box (or in the Command Line box in Windows NT), type the following command, and then click OK: 

          c:\ge1263\setup

  6. Follow the setup instructions on the screen. Note that the Setup program also runs the ODBC Administrator (Odbcad32.exe), which enables you to set up data sources for the Microsoft ODBC drivers.

Help Files for ODBC Drivers

This Application Note includes the following three driver-specific help files:

  • The ODBC SQL Server help file (Drvssrvr.hlp) contains information about adding, modifying, and connecting to Microsoft SQL Server data sources, connection string attributes, and ODBC conformance levels.
  • The ODBC Desktop Database Drivers Help file (Odbcjet.hlp) contains information about adding and modifying Microsoft Jet database engine data sources, connecting to and deleting data sources, and levels of ODBC conformance. The Odbcjet.hlp file contains a "Product Support" topic with details about ODBC support offerings from Microsoft. The Odbcjtnw.hlp file is a subset of the Odbcjet.hlp file and describes new features of the ODBC drivers.
  • The Visigenic ODBC ORACLE Driver Help file (Oracle.hlp) contains information about adding, modifying, and connecting to ORACLE version 7.0 data sources, connection string attributes, and levels of ODBC conformance.
