PRB: "Can't Load Mspdox35.dll" Error Installing DDP 3.0

 Last reviewed: April 30, 1997
Article ID: Q161635
The information in this article applies to:
  • Microsoft Desktop Driver Pack 3.0

SYMPTOMS

When you install the Desktop Driver Pack 3.0 (Wx1220.exe), you may receive the following error: 

   Can't load mspdox35.dll (error code=1157)
   The configuration of the ODBC Microsoft Paradox Driver
   (*.db) driver failed.
Then you receive a message stating that Setup was not completed successfully.

CAUSE

You have one of the ODBC desktop drivers version 3.50.x installed, but not the Paradox ODBC driver version 3.50.x.

MORE INFORMATION

The Desktop Driver Pack 3.0 installs ODBC desktop drivers 3.40.x, and should not be used to install any of the ODBC desktop drivers if you have any of the ODBC desktop drivers version 3.50.x already installed. If you need to install the Visigenic Oracle driver from the Desktop Driver Pack 3.0, you need to install the Data Access Pack first. After installing the complete Data Access Pack, you can use the Desktop Driver Pack 3.0 to install the Visigenic Oracle driver by choosing the Oracle option only in the custom setup screen.

If you have any of the ODBC desktop drivers version 3.50.x installed, then you should use the Data Access Pack to install the ODBC desktop drivers from Office 97 or Microsoft Access 97. For information about obtaining and installing these drivers, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q159322
   TITLE     : INF: Paradox, Lotus, & MS Exchange/Outlook ISAMs in ValuPack
You may encounter the same problem if you run Setup for Microsoft Access 7.0 on a computer that already has Microsoft Access 97 installed. For information about Microsoft Access 7.0 installation problems, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q159793
   TITLE     : PRB: "Can't Load Mspdox35.dll" Error Installing MS Access
