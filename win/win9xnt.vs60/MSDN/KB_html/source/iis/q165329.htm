IDQ Files Overwritten When Published with FrontPage

 Last reviewed: January 22, 1998
Article ID: Q165329
The information in this article applies to:
  • Microsoft Index Server version 1.1

SYMPTOMS

Any IDQ files that have been customized for use with Index Server do not work after publishing files from FrontPage to your Web site.

CAUSE

FrontPage overwrites all IDQ files within the FrontPage Webs it manages. FrontPage attempts to do this whenever Index Server is detected.

WORKAROUND

There is no way to configure FrontPage to not overwrite IDQ files if Index Server is installed. To work around this problem:

  1. Create a new user account in Windows NT Server and sign on as that user account to edit the IDQ file.

  2. On the NTFS partition, assign write permission to only that account, with read permissions to the FrontPage accounts, Admins, and System.
