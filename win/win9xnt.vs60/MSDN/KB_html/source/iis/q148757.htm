IIS Published Directory Can Not Be Located on GSNW Share

 Last reviewed: April 22, 1997
Article ID: Q148757

The information in this article applies to:
  • Microsoft Internet Information Server version 1.0

SYMPTOMS

The following error may be displayed when Internet Information Server (IIS) tries to access published directories located on gateway shares on Microsoft Windows NT running Gateway Service for NetWare (GSNW) or on shares located on UNIX servers running SAMBA: 

   The system can not find the path specified.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Internet Information Server version 1.0. We are researching this problem and will post new information in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

IIS published directories including home and virtual directory may be located on local or network drives. However, when you try to specify GSNW share drive for the published directory, it will fail and generate the above error message and log an EventID 101 in Windows NT EventViewer: 

   The server was unable to add the virtual root '/NWshare' for the
   directory "\\computer\NWshare" due to the following error:

   The system can not find the path specified. The data is the error code.
The current version of IIS does not support locating the published directory on GSNW share drive.
