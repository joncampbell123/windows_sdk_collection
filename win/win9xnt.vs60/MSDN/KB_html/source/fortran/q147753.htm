BUG: Using CTRL-S with FL32.EXE Fails Under Windows 95

 Last reviewed: February 29, 1996
Article ID: Q147753
The information in this article applies to:
  • Microsoft Fortran PowerStation for Windows 95 and Windows NT, version 4.0

SYMPTOMS

Running the FL32.EXE compiler spawning utiility at the MS-DOS command prompt under Windows 95 and pressing the CTRL-S keys simultaneously, fails to pause the compilation process.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The problem only occurs in the MS-DOS command prompt in Windows 95, not in the DOS window in Windows NT version 3.51.
