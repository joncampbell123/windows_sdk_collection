KBDCTL16.DLL and CPMGR.DLL Errors Starting IntelliType

 Last reviewed: October 25, 1996
Article ID: Q120034
The information in this article applies to:
  • Microsoft Natural Keyboard with IntelliType software version 1.0

SYMPTOMS

When you start the IntelliType Manager, one of the following error messages is displayed:

  • Error, file not found KBDCTL16.DLL

    -or-

  • Error, file not found CPMGR16.DLL

CAUSE

These errors occur when you try to start the IntelliType Manager without restarting Microsoft Windows after initially installing the Microsoft IntelliType software version 1.0.

These errors can also occur if you bypass the KBDCPL and SET commands for the Microsoft IntelliType software and then try to start the IntelliType Manager. (These commands are loaded from the AUTOEXEC.BAT file.)

RESOLUTION

To correct these errors, restart the computer and Windows. Ensure that both the KBDCPL and SET commands for the Microsoft IntelliType software are valid within the AUTOEXEC.BAT file. The commands should appear as follows: 

   SET MSINPUT=C:\MSINPUT
   C:\MSINPUT\KEYBOARD\KBDCPL.EXE /S

MORE INFORMATION

The Task Manager does not generate these errors when the KBDCPL and SET commands are bypassed or when Windows is not restarted after first installing the Microsoft IntelliType software. However, in such cases, Task Manager does not reflect any of the following:

  • Pre-programmed buttons
  • Program and File Launcher
  • Application buttons other than the default Program Manager buttons
  • Applications in the StartUp group
  • Applications on the Load= or Run= lines of the WIN.INI file

The settings are not read from the MSINPUT.INI file because this file was bypassed during bootup and initialization of the AUTOEXEC.BAT statement for the KBDCPL and SET commands. The default Program and File Launcher application buttons are File Manager, IntelliType Manager, Paintbrush, Notepad, Write, and Calculator. The other application buttons are blank by default.

When you install the Microsoft IntelliType software from the MS-DOS level, you are reminded to restart the system before using IntelliType. However, the Windows installation of the Microsoft Natural Keyboard with IntelliType software version 1.0 does not directly state that the system must be rebooted. You are given the option to restart Windows or return to Windows.
