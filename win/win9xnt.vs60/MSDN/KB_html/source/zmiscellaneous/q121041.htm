PPT4: New PICSTORE.DLL Incompatible with PowerPoint 4.0

 Last reviewed: February 19, 1998
Article ID: Q121041
The information in this article applies to:
  • Microsoft PowerPoint for Windows, version 4.0
  • Microsoft ClipArt Gallery 1.1

SYMPTOMS

If you have Microsoft PowerPoint version 4.0 (not 4.0a or 4.0c) and you install the updated ClipArt Gallery Application Note (GE1031), you will get the following error message when you try to insert clip art 

  The ClipArt Gallery could not read the file <PATH><FILENAME.PCS>.
where <FILENAME.PCS> is the name of the .PCS image file installed by PowerPoint 4.0 and <PATH> is the location of the image file.

This occurs because the PICSTORE.DLL file that comes with the updated ClipArt Gallery Application Note is incompatible with the original .PCS image files from PowerPoint 4.0.

RESOLUTION

If the PICSTORE.DLL file is not the correct size for the version of the .PCS files you have (see the More Information section in this article below), follow these steps:

NOTE: The following steps assume you have Windows 3.1 installed in the WINDOWS directory on drive C, and your floppy disk drive is drive A. If Windows 3.1 is loaded in a different drive/directory, substitute the correct location. If you are using floppy disk drive B instead of drive A, substitute B for A.

  1. Exit Windows.

  2. Rename PICSTORE.DLL (in the Windows SYSTEM subdirectory) to PICSTORE.OLD.

  3. Place PowerPoint 4.0 Setup disk 1 in drive A.

  4. At the MS-DOS command prompt, type the following and press ENTER: 

          a:\decomp a:\picstore.dl_ c:\windows\system\picstore.dll

  5. Start Windows and try using ClipArt Gallery again.

MORE INFORMATION

The PICSTORE.DLL that comes with the Updated ClipArt Gallery Application Note (GE1031) is the same version that comes with PowerPoint 4.0a and 4.0c (15,872 bytes). It must be replaced with the original PICSTORE.DLL file that came with PowerPoint 4.0 (28,672 bytes) to read the .PCS image files that come with PowerPoint 4.0. This will not affect performance of the new ARTGALRY.EXE file.
