LAN Manager Version 2.2c Client DHCP Setup Changes May Fail

 Last reviewed: June 6, 1995
Article ID: Q122363
The information in this article applies to:
  • Microsoft LAN Manager version 2.2c

SYMPTOMS

On a LAN Manager version 2.2c MS-DOS client using TCP/IP, if Setup is used to disable Dynamic Host Configuration Protocol (DHCP) and then later to re- enable it, the re-enable fails. The client continues to use the manually- configured IP address.

CAUSE

When DHCP is disabled using Setup, the following entry is added to the PROTOCOL.INI file: 

   DISABLEDHCP = 1
Later, if Setup is used to re-enable DHCP, it fails to remove this line, or change it to DISABLEDHCP=0.

WORKAROUND

Manually edit the PROTOCOL.INI file by deleting the line containing DISABLEDHCP = 1.

STATUS

Microsoft has confirmed this to be a problem in Lan Manager version 2.2c. This problem was corrected in Windows NT version 3.51.
