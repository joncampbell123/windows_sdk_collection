Can Enter a Non-Existent Default Domain During Setup

 Last reviewed: February 3, 1998
Article ID: Q161676
The information in this article applies to:
  • Microsoft Commercial Internet System, version 1.0

SYMPTOMS

During an upgrade, the Microsoft Commercial Internet System (MCIS) Mail Server setup allows you to enter a non-existent Windows NT Server domain as the default domain. No warning is given, and no domain is actually created.

CAUSE

This problem only occurs when you are upgrading from an earlier Beta release of the MCIS Mail Server routing table.

STATUS

Microsoft has confirmed this to be a problem in MCIS version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
