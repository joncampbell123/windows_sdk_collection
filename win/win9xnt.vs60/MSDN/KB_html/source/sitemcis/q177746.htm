CRS Modifies Files

 Last reviewed: March 4, 1998
Article ID: Q177746
The information in this article applies to:
  • Microsoft Commercial Internet System version 1.0
  • Site Server version 2.0

SYMPTOMS

Replicated files may differ from source if Content Replication System (CRS) is configured to use Fastmode (Turn off framing for local replication) and the global flag TransactionMethod is set to No. The files can be different sizes or the same size with different content.

WORKAROUND

To work around this problem:

  • Configure CRS not to use Fastmode and to turn on TransactionMethod. You can do this from the command line with the following commands: 

          CRS setglobal TransactionMethod=Yes
      CRS editproj <ProjectName> Flags=ACLs
      CRS Stop
      CRS Start

STATUS

Microsoft has confirmed this to be a problem in Microsoft Commercial Internet System version 1.0 SP1 and Site Server 2.0.
