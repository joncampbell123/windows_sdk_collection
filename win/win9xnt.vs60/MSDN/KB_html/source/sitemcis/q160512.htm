X-Recipient Lines Are Not Being Added to the Message Drop

 Last reviewed: February 3, 1998
Article ID: Q160512

The information in this article applies to:

  • Microsoft Commercial Internet System, version 1.0

SYMPTOMS

Envelope data is put in the pickup and drop directory using x-sender and x- recipient headers. In MCIS version 1.0, the x-sender line is added to messages put in the drop directory, but the x-recipient lines are missing. This problem occurs because of changes in the envelope data itself. This causes the parsing code to make a mistake and assume that the recipient address strings are empty and therefore neglects to dump the x-recipient data to the drop file.

RESOLUTION

Envelope data is correctly parsed for recipient address strings, because of a modification to the Smtpsvc.dll file. Upgrade the server(s) with Service Pack 1 for MCIS version 1.0. This will update the Smtpsvc.dll file and correct the problem.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Commercial Internet System, version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
