CRS: Webadmin Cannot Take a Project Name Containing a Space

 Last reviewed: February 9, 1998
Article ID: Q176514
The information in this article applies to:
  • Microsoft Commercial Internet System version 1.0
  • Microsoft Site Server version 2.0

SYMPTOMS

The Content Replication System (CRS) Webadmin cannot start a project name that has a space in it; if the name contains a space, you receive the following error message: 

   crpr.pgi (46): error 8004005:unspecified error
   crpr.pgi (46): error 80020005: type mismatch

WORKAROUND

To work around this problem, do either of the following:

  • Do not use a space in the project name.

    -or-

  • Use CRS from a command prompt to start the project, as in the following example: 

          CRS startrep "<project_name>" /display

    NOTE: The quotation marks around the project name are required.
