Configuring Multiple Domains in IMS

 Last reviewed: February 3, 1998
Article ID: Q158422
The information in this article applies to:
  • Microsoft Commercial Internet System, version 1.0

SYMPTOMS

After a local domain and associated users are created in SMTP and POP3, a client is unable to access an inbox on a local POP3 domain that is not set as the default. The following error message is returned by the client: 

   ERR Couldn't open Inbox for <user>
The mail box is created when the first mail message is sent to the local user and is inaccessible via any POP3 client.

RESOLUTION

There are three workarounds for this issue:

  • Build a POP3 server for each local domain with associated users. See the "Internet Mail Server Operation Guide" for details on this procedure.

-OR-
  • Create a forwarding record for the local domain user (that is, the user that is not part of the "default" POP3 domain).

    1. Start Internet Service Manager.

    2. Double-click POP3.

    3. On the Aliases tab, click New User.

    4. Type the user name, the address domain, the virtual root. Click 

          the Auto Forward option.

    5. Type the forward name and forward domain. Click OK.

-OR-
  • Point both users to the same virtual root when these users are created.

    1. Start Internet Service Manager.

    2. Double-click POP3.

    3. On the Aliases tab, click new user.

    4. Type the user name, the default address domain, and the virtual root 

         (note this information for later use). Click OK.

    5. Click New User.

    6. Type the user name, the local address domain, and the virtual root 

         (specifying the same virtual root as you used for the first user).

    Note: if the users already exist you will have to delete the local domain user and re-create the account to perform this setup.
