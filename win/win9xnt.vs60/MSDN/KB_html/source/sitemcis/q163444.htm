ILS: NetMeeting 2.0 Cannot Connect to ILS Server

 Last reviewed: February 3, 1998
Article ID: Q163444
The information in this article applies to:
  • Microsoft Commercial Internet System, version 1.0

SYMPTOMS

NetMeeting 2.0 cannot connect to an Internet Locator Server (ILS) if only the Windows NT Server Challenge/Response box is checked under the LDAP service in Internet Service Manager. You must allow Anonymous Connections in order for NetMeeting 2.0 to connect successfully.

CAUSE

the ILS service does support security by means of the Windows NT Server Challenge/Response or Anonymous Connections to the LDAP database. However, the current beta version of the NetMeeting 2.0 does not support any form of security, and therefore you must use Anonymous Connections with NetMeeting.

STATUS

The final release version of NetMeeting 3.0 will include secure connections to an ILS. Currently, no third-party client supports secure connections to an ILS, so if security is an issue, you should investigate other methods (such as a firewall).
