Dr. Watson Error When Running a Scheduled Analysis

 Last reviewed: February 2, 1998
Article ID: Q173956
The information in this article applies to:

- Microsoft Site Server version 2.0 

  - Microsoft Usage Analyst 2.0

SYMPTOMS

When you are running an analysis by using the scheduler service that produces graphs as opposed to tables, if the scheduler service is configured to log in using any account other than the system account, the following error is returned: 

   Dr. Watson Error
   Analysis.exe
   Exception: access violation at (0xc0000005); address 0x0f03bbb8

WORKAROUND

If possible, configure the scheduler service to log in using the System Account. To do this, open Services in Control Panel. Double-click the scheduler service. In the Log On As window, click System Account.

STATUS

Microsoft has confirmed this to be a problem with Usage Analyst version 2.0, a component of Microsoft Site Server 2.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
