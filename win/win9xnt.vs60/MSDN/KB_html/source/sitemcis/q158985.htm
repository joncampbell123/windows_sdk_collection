Agent Installation Stops Responding at 92 Percent

 Last reviewed: February 3, 1998
Article ID: Q158985
The information in this article applies to:
  • Microsoft Commercial Internet System, version 1.0

SUMMARY

One of the more common problems with Commercial Internet Server's Agent installation is a failure to install at 92 percent. This problem may be caused by a lack of sufficient security credentials presented during the installation.

MORE INFORMATION

In the sequence of question dialog boxes presented during the installation, the sixth one, titled "Membership Agent Service Account Information" asks for a user name and password for the system to use during the installation. This information must be presented in <domain>\<username> format, regardless of whether or not this is a stand-alone server.

If this problem has occurred, do the following:

  1. Launch the installation again.

  2. Select "Remove all previously installed." This will remove all the old installation components, including the incorrect name and password.

  3. Run the installation again, using the correct data and security credentials.
