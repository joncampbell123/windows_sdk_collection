Server Does Not Support Simultaneous SSPI Security Providers

 Last reviewed: February 3, 1998
Article ID: Q164727

The information in this article applies to:

  • Microsoft Commercial Internet System, version 1.0

SYMPTOMS

The chat server does not support two SSPI security providers at once.

RESOLUTION

Chatsvc.dll has been modified to support 0, 1, or 2 security providers (such as DPA, RPA, and so forth). Upgrade the system to Service Pack 1 (SP1) for MCIS. This will update the Chatsvc.dll file. To specify two packages, enter them in the "SSPI Packages" field in the Administrator user interface and separate them with a single space (for example, DPA NTLM).

STATUS

Microsoft has confirmed this to be a problem in Microsoft Commercial Internet System, version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
