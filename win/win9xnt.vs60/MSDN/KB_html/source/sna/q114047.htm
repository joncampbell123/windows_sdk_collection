Bugs fixed:

1824 SNA Server Admin does not handle BLANK mode name correctly

SNA Server was not properly storing a BLANK mode name (8 space characters), so a CPI-C or APPC application which uses such a mode name would not work.

1579 Fix to interoperate with unreleased APPN Network Nodes (Patch 6)

In certain unreleased versions of APPN Network Node products, these products do not accept XID's which exceed 60 bytes in length. The default XID sent by SNA Server is now below 60 bytes in length.

1791 Admin: Error 3885 (Invalid rec) converting CS1.2.1 com.cfg 

       (Patch 15)
The SNA Server Admin program may fail to upgrade a DCA/MS Comm Server 1.2.1 configuration file, and display Error 3885 (invalid record).

Update Procedure:

- Stop SNA Server, then stop the SNABASE service

- Copy updated modules over the previous versions: 

     <snaroot>\SYSTEM\SNABASE.EXE
     <snaroot>\SYSTEM\SNACFG.DLL
     <snaroot>\CLIENTS\WINNT\SYSTEM\SNABASE.EXE
     <snaroot>\CLIENTS\WINNT\SYSTEM\SNACFG.DLL
- restart the SNABASE service and re-run the SNA Server administration 
  program and reconfigure the BLANK mode name. If a CPI-C application is
  using this mode, also reconfigure the symbolic destination name.  Be sure
  to resave the configuration changes.
