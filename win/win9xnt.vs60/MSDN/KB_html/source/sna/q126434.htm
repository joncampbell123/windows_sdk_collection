Configuring a Windows NT User as an SNA Server Administrator

 Last reviewed: June 17, 1997
Article ID: Q126434
2.10 WINDOWS kbnetwork

The information in this article applies to:

  • Microsoft SNA Server for Windows NT, versions 2.1, 2.11, and 3.0

SYMPTOMS

SNA Server Admin allows an authorized administrator to define administrative access to SNA Server. However, if a Windows NT user is granted Full Control using SNA Admin, the following error may occur when the user attempts to save the SNA Server configuration file: 

   Error: 4006
   Unable to get or set security information on the config file.

 -or-

   Unable to write file.
Windows NT administrators do not have this problem.

RESOLUTION

To allow a Windows NT user to fully administer the SNA Server:

  1. Run Windows NT User Manager for Domains.
2. Choose Policies and then choose User Rights. 3. Select "Manage Auditing And Security Log." 4. Grant the user this user right. 5. Exit Windows NT User Manager for Domains. 5. Run SNA Server Manager. 6. Select the SNA Subdomain and choose Propeties. 7. In the Config File Security Tab, select 'Take Ownership'. 8. Exit SNA Server Manager.

For more information on security, see Chapter 12 "Setting up security with SNA Server" of the "SNA Server version 2.1 Admin Guide."
