Bugs fixed:

1793 IBM PC Support users can't override default LU aliases

When using the SNA Server MS-DOS router with IBM PC Support, the default LU's configured in the CONFIG.PCS file are ignored if there are default Local and Remote APPC LU's assigned in the SNA Server user/group record in the SNA Server configuration file.

In order to allow the local CONFIG.PCS LU aliases to be used instead, the following keyword in the SNA.INI file has been implemented: 

   OverrideDefaultLu = YES   (default = NO)
If set to YES, the SNA Server router will always use the LU's specified on the RTLN line of the CONFIG.PCS file. Otherwise, the SNA Server user/group record settings will be used (default).

Update Procedure:

- Stop the SNA Server MS-DOS client software.

- Copy updated modules over the previous versions: 

     <snaroot>\INITRTR.EXE

  and, update the file at the server:

     <snaroot>\CLIENTS\MSDOS\INITRTR.EXE
