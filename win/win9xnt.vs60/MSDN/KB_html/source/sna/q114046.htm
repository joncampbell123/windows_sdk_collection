Bugs fixed:

1823 SLI_OPEN hangs when using LUA pool name.

- SLI_OPEN using an LUA pool name never returns. - occurs on Win16 and Win32

The SLI support was continuing to use the LU name on underlying RUI calls instead of the session identifier (lua_sid).

Also, the original Windows 3.x WINSLI.DLL included codeview symbols, which are no longer included.

Update Procedure:

- Stop the SNA Server client software on the client machine.

- Copy updated SLI DLL over the previous version: 

  For Windows NT clients:

  - Update the WINSLI.DLL located in the <snaroot>\SYSTEM directory with
    the patched version located in system\winsli.dll

  - Also, on the SNA Server installation server, update the WINSLI.DLL
    located in the <snaroot>\CLIENTS\WINNT\SYSTEM directory

  For Windows 3.x clients:

  - Update the WINSLI.DLL located in the <winroot>\SYSTEM directory with
    the patched version located in clients\win3x\winsli.dll

  - Also, on the SNA Server installation server, update the WINSLI.DLL
    located in the <snaroot>\CLIENTS\WIN3X directory

  For OS/2 clients:

  - Update the ACSSLI.DLL located in the <snaroot> directory with the
    patched version located in clients\os2\acssli.dll

  - Also, on the SNA Server installation server, update the ACSSLI.DLL
    located in the <snaroot>\CLIENTS\OS2 directory
Note: The SLI libraries are included here for completeness, though the SLI application do not have to be relinked with these new libraries.
