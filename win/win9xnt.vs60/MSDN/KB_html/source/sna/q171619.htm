SNA Server Does Not Detect Existing Host Security Domains

 Last reviewed: March 19, 1998
Article ID: Q171619

The information in this article applies to:

  • Microsoft SNA Server, versions 3.0 and 3.0 Service Pack 1 (SP1)

SYMPTOMS

During setup of SNA Server, when you choose to install host security, Setup presents a dialog box that asks for: 

   Domains providing a host account cache
You have to type the domain names; this box is not automatically filled in with a list of Windows NT domains detected on the network.

STATUS

Microsoft has confirmed this to be a problem in SNA Server versions 3.0 and 3.0 Service Pack 1 (SP1). This problem was corrected in the latest SNA Server version 3.0 U.S. Service Pack. For information on obtaining this Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K

MORE INFORMATION

With the fix applied, the dialog box presents a list of existing domains detected on the network.
