Deadlocks When Using Asynchronous Named Pipes

 Last reviewed: April 24, 1996
Article ID: Q121822
The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.5
  • Microsoft Windows NT Server version 3.5
  • Microsoft SNA Server for Windows NT, versions 2.0 and 2.1

SYMPTOMS

When Windows NT Workstation and Windows NT Server version 3.5 are stressed by asynchronous named pipes, such as by Microsoft SNA Server for Windows NT, the Windows NT redirector (RDR.SYS) doesn't create enough worker threads fast enough and system deadlock occurs.

STATUS

Microsoft has confirmed this to be a problem in Windows NT Workstation and Windows NT Server version 3.5. This problem was corrected in the latest U.S. Service Pack for Windows NT version 3.5. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
