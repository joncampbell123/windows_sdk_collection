TN3270 Service Appears Inactive if Started from Command Prompt

 Last reviewed: April 14, 1997
Article ID: Q165048
The information in this article applies to:
  • Microsoft SNA Server version 3.0

SYMPTOMS

The TN3270 service does not appear active in SNA Manager version 3.0 if it is started from a command prompt.

CAUSE

When the TN3270 Service is run as a normal process rather than a Windows NT service, the manage status is not properly updated.

RESOLUTION

The manage status is now properly updated for the TN3270 Service when it is run as a process.

STATUS

Microsoft has confirmed this to be a problem in SNA Server version 3.0. This problem was corrected in the latest Microsoft SNA Server 3.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
