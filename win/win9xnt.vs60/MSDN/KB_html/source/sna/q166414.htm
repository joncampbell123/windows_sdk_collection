DCA SDLC and DCA X.25 Links Don't Start When SNA Server Starts

 Last reviewed: April 14, 1997
Article ID: Q166414
The information in this article applies to:
  • Microsoft SNA Server version 3.0

SYMPTOMS

When the SNA Server Service starts, the DCA ISCA SDLC Link Service and/or DCA ISCA X.25 link service do not start automatically.

CAUSE

In the registry, the DependOnGroup and DependOnService are not set properly.

STATUS

Microsoft has confirmed this to be a problem in SNA Server version 3.0. This problem was corrected in the latest Microsoft SNA Server 3.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
