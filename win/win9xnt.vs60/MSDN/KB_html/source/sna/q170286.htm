Application Hangs if WinRUIStartup Called Before WinAPPCStartup

 Last reviewed: November 6, 1997
Article ID: Q170286
The information in this article applies to:
  • Microsoft SNA Server, versions 2.11 and 3.0

SYMPTOMS

If an application attempts to open the SNA Server Windows APPC and RUI interfaces from the same process, the application may hang if WinRUIStartup() is called before WinAPPCStartup().

NOTE: The RUI interface provides access to LU0-3 host applications, while APPC provides access to partner APPC transaction programs over LU6.2.

RESOLUTION

To workaround the problem, WinAPPCStartup should be called before WinRUIStartup.

STATUS

Microsoft has confirmed this to be a problem in SNA Server, versions 2.11 and 3.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
