HOWTO: Perform a RollBack Without Losing History

 Last reviewed: February 27, 1997
Article ID: Q153493
4.00 5.00 WINDOWS kbusage kbhowto

The information in this article applies to:

  • Microsoft Visual SourceSafe, 16-bit and 32-bit, for Windows, versions 4.0, 5.0
  • Microsoft SourceSafe for Windows, version 3.1
  • Microsoft SourceSafe for Macintosh, version 3.1
  • Microsoft SourceSafe for MS-DOS, version 3.1

SUMMARY

Use the following steps to return files to a particular version without losing their history. If you use the Rollback feature of SourceSafe or Visual SourceSafe, you lose the history of the file.

  1. Use GET on the version you want to roll back to in the working directory of the project.

  2. Highlight the latest version of the file in the SourceSafe Explorer, and hold the SHIFT key down while clicking the CHECKOUT button. Select the "Do not get local copy" option. This checks out the files but does not get them to the working directory.

  3. Check in the files that are checked out, and SourceSafe will bring in the older version. The history of the files will remain intact.
