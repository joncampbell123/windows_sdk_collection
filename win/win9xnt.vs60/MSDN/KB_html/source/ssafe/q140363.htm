FIX: Visual SourceSafe Slows as It Increases Processor Usage

 Last reviewed: August 25, 1997
Article ID: Q140363
The information in this article applies to:
  • Microsoft Visual SourceSafe for Windows, versions 4.0, 5.0
  • Microsoft Visual SourceSafe for Windows NT, versions 4.0, 5.0
  • Microsoft Visual SourceSafe for Windows 95, versions 4.0, 5.0

SYMPTOMS

Applications my run slower as system processor usage jumps to nearly 100% when the Visual SourceSafe Explorer is loaded.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article.

This problem has been fixed in Visual SourceSafe 5.0.

MORE INFORMATION

Steps to Reproduce Problem

  1. Load the Windows System Monitor.

  2. Make note of the current processor usage.

  3. Load the Visual SourceSafe Explorer, and minimize it. Notice that the processor usage jumps to about 100%.
