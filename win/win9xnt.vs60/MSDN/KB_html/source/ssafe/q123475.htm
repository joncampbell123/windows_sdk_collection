INFO: SourceSafe System Capacities and Specifications

 Last reviewed: May 23, 1997
Article ID: Q123475
The information in this article applies to:
  • SourceSafe for MS-DOS, versions 3.0x
  • SourceSafe for Windows, versions 3.0x
  • SourceSafe for Macintosh, versions 3.0x
  • SourceSafe for Windows NT, versions 3.0x
  • SourceSafe for UNIX, versions 3.0x

SUMMARY

This article contains information concerning the system capacities of SourceSafe.

MORE INFORMATION

General Specifications & Limits

  • Users running SourceSafe concurrently: Unlimited
  • Files and projects in a SourceSafe Database: approx. 4 billion
  • Files and projects directly in a single project: 8191 (however, for performance reasons, it is recommended there is no more than a few hundred files in one project)
  • Bytes in a file: 67 Mb
  • Lines in the Windows "View" Command: 64K
  • Versions in a file or project history: 32,767 (The count on a project history does not include file Updates in that project)
  • Bytes in the Windows history display: 32K
  • Files correlated in a project history: 8191

String Lengths

Project Path (for example, $/Word/Spelling): 63 Characters

  • Filename: 31 characters (all platforms except MS-DOS and Windows). A short (8.3) name is also stored for compatibility with the FAT file system. All names are case insensitive.
  • Username: 15 characters
  • Password: 15 characters
  • File or project Label: 15 characters
  • String to "Scan" for: 63 characters
  • Line in the SS.INI file: 255 characters
