BUG: SourceSafe for MS-DOS Hangs on Visual Difference

 Last reviewed: May 23, 1997
Article ID: Q129191
The information in this article applies to:
  • SourceSafe for MS-DOS, version 3.1

SYMPTOMS

Performing a Visual Difference in the SSD.EXE, version 3.1 causes SourceSafe to hang (stop responding) under MS-DOS. Under Windows or Windows NT, the MS-DOS session produces a memory error.

STATUS

Microsoft has confirmed this to be a bug in SourceSafe version 3.1 for MS-DOS. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Select a file to difference (make sure it is different from the working directory copy of the file).

  2. Select Difference from the Info menu.

  3. Make sure 'Visual' is selected.

  4. Click the Okay button.

  5. At this point, the computer hangs (stops responding) or a memory error occurs.
