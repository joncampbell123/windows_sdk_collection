HOWTO: Increase Performance in SourceSafe

 Last reviewed: February 25, 1997
Article ID: Q123473
The information in this article applies to:
  • Microsoft Visual SourceSafe, versions 4.0, 5.0
  • Microsoft SourceSafe, versions 3.0x

SUMMARY

Use the following steps to improve SourceSafe's speed (performance):

  1. Make the correct platform version of the SourceSafe executable local, as opposed to located on the network. Set the environment variable SSDIR to the network location of the SourceSafe database.

  2. Change the Comparison methods from the default of 'Full' to 'Checksum' by:

    - Choosing Command Settings... from the Setup menu and selecting the 

         pulldown labeled "Compare files by."

    - Adding Compare = Checksum to the top section of the SS.INI file.

  3. Set the TEMP_PATH variable to a local directory.
