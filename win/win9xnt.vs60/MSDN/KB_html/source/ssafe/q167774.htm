PRB: Enabling VSS4.x Integration with VB5 Must be Done Manually

 Last reviewed: September 5, 1997
Article ID: Q167774
The information in this article applies to:
  • Microsoft Visual SourceSafe, 16-bit and 32-bit, for Windows, versions 4.0, 4.0a
  • Microsoft Visual Basic Control Creation, Learning, Professional, and Enterprise Editions for Windows, version 5.0

SYMPTOMS

No SourceSafe integration features are available in the Visual Basic 5.0 IDE. There is no Source Code Control option available in the Add-In Manager. If Visual Basic 4.0 is on the same machine, its Visual SourceSafe integration works correctly.

RESOLUTION

You must manually add the following line to the vbaddin.ini in the [Add- Ins32] section to enable Visual SourceSafe integration for Visual Basic 5.0: 

   VBSCC=1

STATUS

This behavior is by design.
