Logon Occurs Using System Policy with Required Validation

 Last reviewed: January 7, 1998
Article ID: Q178979
The information in this article applies to:
  • Microsoft Windows 95

SYMPTOMS

If you are using a system policy that requires validation by the network for Windows access, you may be able to access the desktop even when the network does not validate your user account.

CAUSE

This behavior can occur if Tweak UI, a component of Windows 95 PowerToys, is installed and the Clear Last User At Logon feature is enabled.

RESOLUTION

To resolve this issue, disable the Clear Last User At Logon feature. To do so, follow these steps:

  1. Click Start, point to Settings, and then click Control Panel.

  2. Double-click Tweak UI, and then click the Paranoia tab.

  3. Click the Clear Last User Logon check box to clear it.

  4. Click OK.

NOTE: This change does not take effect until you restart Windows 95.

MORE INFORMATION

Windows 95 PowerToys is a suite of tools available for free download from the Microsoft Web site. Microsoft Technical Support Desktop Systems does not support the installation or usage of PowerToys or any of the components.
