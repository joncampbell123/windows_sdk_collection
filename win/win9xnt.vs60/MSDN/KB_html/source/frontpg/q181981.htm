FP: Cannot Remove Executable Property of Root Folder

 Last reviewed: March 18, 1998
Article ID: Q181981
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows
  • Microsoft FrontPage 97 for Windows with Bonus Pack

SYMPTOMS

You cannot remove the executable property on a root folder in your Web. Even if you attempt to remove this property by clearing the "Allow scripts or programs to be run" check box in the folder's Properties dialog box, the folder remains executable.

CAUSE

This behavior occurs if the FrontPage 97 Server Extensions are installed on a Microsoft Internet Information Server (IIS).

RESOLUTION

Use one of the following methods to resolve this behavior:

Method 1: Remove the Executable Property By Using Internet Service Manager

To remove the executable property from the root folder of your Web by using the Internet Service Manager, follow these steps:

  1. On the Windows Start menu, point to Programs, point to Microsoft Internet Services Manager (Common), and then click Internet Service Manager.

    NOTE: If you are using Microsoft Peer Web Services for Windows NT Workstation, point to Microsoft Peer Web Services (Common) and then click Internet Service Manager.

  2. On the View menu, click Services View.

  3. Click the WWW service and double-click the computer on which your webs are running.

  4. Click the Directories tab.

  5. Click the directory alias for your FrontPage web. For example, if your web is named "myweb," the directory alias will be "/myweb."

  6. Click Edit Properties.

  7. In the Directory Properties dialog box, click to clear the Execute check box.

  8. Click OK twice.

  9. On the Properties menu, click Exit.

Method 2: Upgrade the FrontPage Server Extensions

For additional information about installing the FrontPage Server Extensions for IIS, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q156842
   TITLE     : FP: How to Install FrontPage Server Extensions for IIS

STATUS

Microsoft has confirmed this to be a problem in the Microsoft FrontPage 97 Server Extensions for IIS. This problem was corrected in the Microsoft FrontPage 98 Server Extensions for IIS.
