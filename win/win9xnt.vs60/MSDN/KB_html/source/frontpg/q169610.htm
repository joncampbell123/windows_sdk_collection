FP: No Response When Attempting to Create New FrontPage Web

 Last reviewed: March 18, 1998
Article ID: Q169610
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows
  • Microsoft FrontPage 97 for Windows with Bonus Pack

SYMPTOMS

In Microsoft FrontPage 97, after you select a template or wizard in the New FrontPage Web dialog box and click OK, the following occurs:

  • The screen flashes

-and-
  • The New FrontPage Web dialog box reappears

-and-
  • The Web is not created.

In Microsoft FrontPage 98 after you select a template or wizard in the New FrontPage Web dialog box and click OK, you will receive the following error message: 

   HTTP/1.0 501 Not Supported.
For additional information about this error message, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q143225
   TITLE     : FP: HTTP Code 501 Error Opening, Creating Web

CAUSE

This problem occurs if either of the following is true:

  • The FrontPage Server Extensions are not installed on the Web server where you are attempting to create the web.
  • TCP/IP is not initialized. If this is the case, the server name is usually "Default" (without the quotation marks). For example, the host name is "Default" if the Client for Microsoft Networks is not installed or if the Domain Name Server (DNS) is not specified.

RESOLUTION

To resolve this problem, use one of the following methods:

Method 1. Install the FrontPage Server Extensions

Install the FrontPage Server Extensions to the Web server on which you are attempting to create the Web. For information about installing FrontPage Server Extensions, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q167931
   TITLE     : FP97: How to Install FrontPage Server Extensions for PWS

   ARTICLE-ID: Q156842
   TITLE     : FP1: How to Install FrontPage Server Extensions for IIS

Method 2. Install the Client for Microsoft Networks

Install the Client for Microsoft Networks in Control Panel. For information about how to do this, please see step 2 in the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q134304
   TITLE     : Troubleshooting Browsing with Client for Microsoft
               Networks
