FP: Server Administrator Loses Virtual Server Identities

 Last reviewed: March 18, 1998
Article ID: Q164282
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows
  • Microsoft FrontPage 97 for Windows with Bonus Pack

SYMPTOMS

When you attempt to install the FrontPage Server Extensions to additional identities on a multi-homed WebSite Pro 1.1f server, the additional server identities no longer appear in the Server Administrator.

All of the server's identities remain fully functional. The problem only affects how the identities appear in the FrontPage Server Administrator.

CAUSE

The FrontPage Server Extensions Setup program bases it's internal arrays on a different indexing system than the WebSite server does.

WORKAROUND

To see all of the identities in the FrontPage Server Administrator, use one of the following methods.

Method 1: Use Command Line Arguments to Specify Identities

Start the FrontPage Server Administrator from the MS-DOS command prompt to bypass the user interface that lists the identities. At the MS-DOS command prompt, type the following and then press ENTER: 

   CD c:\Program Files\Microsoft FrontPage\bin
Type the following command, and then press ENTER: 

   fpsrvwin -o install -t website -m <identity name>
The IP addresses will appear in the FrontPage Server Administrator.

Method 2: Reinstall the FrontPage Server Extensions Only

Download the FrontPage Server Extensions from the following Microsoft Web site: 

   http://www.microsoft.com/frontpage/softlib/fp_97_extensions.htm.
When you install to additional identities, simply select the additional identities from the list of identities. Identities that you have already installed the extensions to will continue to function correctly.

Method 3: Remove and then Reinstall the Extensions

Uninstall the server extensions for all of the multi-homed servers, and then quit the FrontPage Server Administrator. Restart the FrontPage Server Administrator and install the server extensions. All the IP addresses will reappear in the FrontPage Server Administrator.

Method 4: Rerun the FrontPage Setup Program

Rerun the FrontPage Setup program, click Custom Installation, and install the Server Extensions component. All the IP addresses will reappear in the FrontPage Server Administrator.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article.
