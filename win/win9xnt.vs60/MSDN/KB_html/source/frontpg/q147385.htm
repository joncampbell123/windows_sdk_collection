FP1: Change Made w/3rd-Party Editor Invisible When Browsing

 Last reviewed: March 17, 1998
Article ID: Q147385
The information in this article applies to:
  • Microsoft FrontPage for Windows, versions 1.0, 1.1

SYMPTOMS

After you edit an HTML page using a third-party editor configured with the FrontPage Explorer, you do not see updates to your page when you look at it in your browser. This problem occurs when you open a web and then open the page from within the FrontPage Explorer. It does not happen when you directly edit the original page with the third-party editor.

CAUSE

This problem occurs if you have not imported the page into your web.

When you specify a third-party application as your editor, the editor opens and saves your HTML page in the Vermeer\Temp folder. Changes are not made directly to the original file. The file needs to be imported into your web before you will see changes in your browser.

RESOLUTION

After you save and close the file in the third-party editor, do the following:

  1. Switch to the FrontPage Explorer.

  2. On the File menu, click Import.

  3. Click Import Now.

  4. Click Close.

  5. Switch to your browser and reload the page.

For information about the import list in FrontPage 1.1, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q153160
   TITLE     : FP1: Edited File Not Automatically Added to Import List
