FP98: Database Column Values Not Inserted Into New Table Cell

 Last reviewed: March 18, 1998
Article ID: Q175783
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows

SYMPTOMS

When you attempt to insert a database column value into a database table, the database column value is inserted into the current cell rather than a new cell in the table.

CAUSE

This behavior occurs if you select the "Show the query in a table " check box while you are creating the database table by using the Database Region Wizard. Once the Wizard inserts the table, you cannot add or remove column values.

WORKAROUND

To work around this problem, follow these steps:

  1. Insert a new table cell into the database table. To do this, 

        a. Click in any database column so that the Column Value command will be
       available when you point to Database on the Insert menu.


        b. On the Table menu, click "Insert Rows or Columns."


        c. In the "Insert Rows or Columns" dialog box, click Columns.


        d. In the Number Of Columns box, type "1" (without the quotation marks) or
       click the arrows to select one column.


        e. Click either Left Of Selection or Right Of Selection.


        f. Click OK.

  2. Insert a database column value into the new table cell: 

        a. Click in the cell you created in step 1.


        b. On the Insert menu, point to Database, and click Column Value.


        c. In the Database Column Value dialog box, type a query field name or
       select one from the list box.


        d. Click OK.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article.

REFERENCES

For more information about database column values, click the Index tab in FrontPage Help, type the following text 

   Database Column Value command
and then double-click the selected text to go to the "Database Column Value command" topic.
