FP: Publish Web Command Doesn't Copy Changed Pages in Subweb

 Last reviewed: March 18, 1998
Article ID: Q167861
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows
  • Microsoft FrontPage 97 for Windows with Bonus Pack

SYMPTOMS

In FrontPage 97, when you attempt to publish a FrontPage root Web after editing one or more subWebs but not editing the root Web, you receive the following error message and none of your pages, in either the root Web or the subWebs, are published: 

   No files have changed in your Web since you last published to
   http://<servername>
where <servername> is the name of the server.

Although this error message appears when you attempt to publish a root Web in FrontPage 98, all of your pages are published to the Web.

CAUSE

This error message appears if you changed the subWeb but not the root Web. and you select the "Publish changed pages only" option in the Publish dialog box (FrontPage 98), or the "Copy Changed Pages Only" option in the Publish FrontPage Web dialog box (FrontPage 97). In this case, FrontPage only checks for changes to pages stored in the root Web; it does not check the subWebs.

RESOLUTION

To resolve this problem in FrontPage 97, use one of the following methods:

Method 1. Publish the subweb only.

  1. Open the subweb in FrontPage Explorer.

  2. On the File menu, click Publish FrontPage Web.

  3. Click to select the "Copy Changed Pages Only" check box.

Method 2. Publish the root Web and the subwebs.

  1. Open the root Web in FrontPage Explorer.

  2. On the File menu, click Publish FrontPage Web.

  3. Click to clear the "Copy Changed Pages Only" check box.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in FrontPage 98 for Windows.
