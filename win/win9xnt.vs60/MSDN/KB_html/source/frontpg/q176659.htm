FP97: Applet Properties Not Auto Saved in FrontPage Editor

 Last reviewed: January 12, 1998
Article ID: Q176659
The information in this article applies to:
  • Microsoft FrontPage 97 for Windows with Bonus Pack

SYMPTOMS

When you close a page after editing Applet Parameters in the FrontPage 97 Editor, the changes that you made are not saved.

CAUSE

FrontPage may not correctly track changes to Applet Parameters as being changes to the file. If you save the file before closing it, the changes will be saved. If you have not saved the file after changing only the parameters of an applet, you will not be prompted to save the change and the change will not be saved.

RESOLUTION

To work around this problem, click Save on the File menu before you close the page.

STATUS

Microsoft has confirmed this to be a problem in the versions of Microsoft FrontPage listed at the beginning of this article. This problem was corrected in Microsoft FrontPage 98 for Windows.
