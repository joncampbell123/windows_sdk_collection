FP1: Anchor Tags That Do Not Surround Data Are Removed

 Last reviewed: March 17, 1998
Article ID: Q151045
The information in this article applies to:
  • Microsoft FrontPage for Windows, version 1.1

SYMPTOMS

When you save a pre-existing HTML document in the FrontPage Editor, the FrontPage Editor will remove all anchor tags (<a></a>) that do not have data between them.

CAUSE

FrontPage Editor attempts to remove unnecessary data from the HTML code. Whenever the FrontPage encounters an anchor tag surrounding no data, it detects that the tag is unused and, therefore, removes it from the file.

WORKAROUND

Place data between the <a> and </a> anchor tags to prevent FrontPage Editor from removing the anchor tags from the file.

MORE INFORMATION

This behavior occurs only when the file is created in an editor other than FrontPage Editor. FrontPage Editor versions 1.0 and 1.1 do not allow you to insert an empty anchor tag. Microsoft FrontPage versions 98 and 97 allow you to create empty anchor tags and so preserve bookmarks that do not surround data.
