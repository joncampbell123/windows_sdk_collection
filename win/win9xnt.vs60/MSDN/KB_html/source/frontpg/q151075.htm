FP1: Link to Second Results File Lost When Page Renamed

 Last reviewed: March 17, 1998
Article ID: Q151075
The information in this article applies to:
  • Microsoft FrontPage for Windows, version 1.1

SYMPTOMS

In FrontPage Editor, you can save results to two pages. If you rename a page to which you save results, the link to the second page is lost.

CAUSE

Links to the second result file from a WebBot Save Results component are lost when the page is renamed.

WORKAROUND

To work around this problem, reset the advanced properties of the WebBot Save Results after the page has been renamed.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft FrontPage 97 for Windows with Bonus Pack.

MORE INFORMATION

Normally, a page with a WebBot Save Results component saves results to one file. The author has the option to save the results to a second file for a backup file or to save an additional copy of the data to a second file.
