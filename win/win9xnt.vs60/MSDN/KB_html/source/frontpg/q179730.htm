FP98: "HTTP Error Code 405" Opening or Publishing a Web

 Last reviewed: March 18, 1998
Article ID: Q179730
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows

SYMPTOMS

When you open or publish a Web in Microsoft FrontPage 98, you may receive the following error message: 

   The server could not complete your request. Contact your Internet
   Service provider or web server administrator to make sure that the
   server has the FrontPage Server Extensions installed. For more
   specific information, click Details.
When you click Details, you will see the following additional information: 

   Error 405.
   HTTP Error 405.
   405 Method Not Allowed.
   The method specified in the request line is not allowed for the resource
   identified by the request. Please ensure that you have the proper MIME
   type set up for the resource you are requesting. Please contact the
   server's administrator if this problem persists.

CAUSE

This problem may occur if FrontPage cannot access the server through the extensions.

The FrontPage Server Extensions are a set of programs that are installed on the Web Server. They support authoring and administering FrontPage webs and provide browse-time functionality.

RESOLUTION

Make sure that the FrontPage Server Extensions are installed on the Web server. To do this, follow these steps:

  1. On the Windows Start menu, point to Find, and then click Files Or Folders.

  2. In the Name box, type Fpsrvwin.exe. In the Look In box, select the drive where you installed FrontPage. Click Find Now.

  3. When the search results appear, double-click the Fpsrvwin.exe file.

    FrontPage Server Administrator will start.

  4. When the FrontPage Server Administrator appears, check the server type and the version numbers of the installed FrontPage Server Extensions.
