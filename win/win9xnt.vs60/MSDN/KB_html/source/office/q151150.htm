OFF95: Incorrect Date Values in Client Billing Wizard

 Last reviewed: March 27, 1997
Article ID: Q151150
7.00 7.00a 7.00b WINDOWS kbtool kbusage

The information in this article applies to:

  • US Microsoft Office for Windows 95, version 7.0, 7.0b
  • German Microsoft Office for Windows 95, version 7.0 and 7.0a
  • US Microsoft Excel for Windows 95, version 7.0
  • German Microsoft Excel for Windows 95, version 7.0 and 7.0a
  • US Microsoft Binders for Windows 95, version 7.0 and 7.0b

SYMPTOMS

When you use the Client Billing Binder Wizard in Microsoft Office for Windows 95, the date formats may be displayed incorrectly or may be displayed with the text error code "#VALUE!" in the date column of the Invoice (Rechnung) Excel worksheet that is created by the Client Billing template (Kundenrechnung.obz).

CAUSE

The problem occurs when you install German Office for Windows 95, and then install US Microsoft Excel for Windows 95 over a US Microsoft Windows 95 system. In German Microsoft Excel 7.0 and 7.0a, the date is specified with the TT.MM.JJ German date format instead of the MM/DD/YY date format. The German date format is not recognizable as a valid date format in the US version of Microsoft Excel for Windows 95.

WORKAROUND

To avoid this problem, you must manually modify the date contained in cell M13 (the cell named "data9") in the Invoice (Rechnung) Excel worksheet section in the binder to properly represent the German date format. You must also format the date cell as text in order to avoid the "#VALUE!" error because you are working with US Excel for Windows 95. To do this, follow these steps:

  1. Select the cell containing the date (M13).

  2. Click Cells on the Format menu.

  3. Click the Number tab.

  4. In the Category list, click "Text," and then click OK.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
