MOM Button or Command Same Position or Duplicated After Edit

 Last reviewed: September 16, 1996
Article ID: Q117638
The information in this article applies to:
  • Microsoft Office for Windows, version 4.0, 4.2, 4.3

SYMPTOMS

If you edit the MSOFFICE.INI file to change the position of custom buttons on the Microsoft Office Manager (MOM) toolbar or to change the order of custom commands on the Microsoft Office menu, the items do not appear in the correct order, or the same item appears twice on the toolbar or menu.

CAUSE

This behavior occurs if you edit the MSOFFICE.INI file, and you change the numbers in the [ToolbarOrder] section or the [MenuOrder] for the UserApp entries. For example, if you change the UserApp1 entry to UserApp2, (by changing the number 1 to 2) and the UserApp2 entry to UserApp1, the items appear in the original location on the menu or toolbar the next time you run MOM. That is, the items are not reordered in the MSOFFICE.INI file according to the numbers you gave them.

If you change a UserApp entry to the same number as an existing entry, the entry you changed is replaced by another instance of the existing entry. For example, if you change the UserApp3 entry to UserApp2, and the UserApp2 entry exists, the UserApp3 icon or command is replaced with the UserApp2 icon or command, but maintains the UserApp3 name.

WORKAROUND

To avoid problems such as incorrect button or menu command order or duplicate items on the MOM toolbar, or the Microsoft Office menu, use the Customize command on the Microsoft Office menu to change the order of the buttons or the commands. Using the Customize dialog box is easier than editing the MSOFFICE.INI file, and this method also updates the information in the MSOFFICE.INI file correctly.

REFERENCES

For more information about how to Customize Microsoft Office Manager, choose the Search button in Help and type: 

   customize
