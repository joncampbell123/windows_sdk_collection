Filename and Directory Fields Empty in Notes Database

 Last reviewed: December 13, 1996
Article ID: Q125589
The information in this article applies to:
  • Microsoft Office 97 for Windows
  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Office for Windows, version 4.2c
  • Microsoft Excel 97 for Windows
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft Excel for Windows, version 5.0c
  • Microsoft PowerPoint 97 for Windows
  • Microsoft PowerPoint for Windows 95, version 7.0
  • Microsoft PowerPoint for Windows, version 4.0c
  • Microsoft Word 97 for Windows
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Word for Windows, version 6.0c

SUMMARY

When you insert an object created by Microsoft Excel, PowerPoint, or Word into a Lotus Notes database, the Filename and Directory fields in the database are empty.

CAUSE

This behavior is by design of Microsoft Excel, Microsoft PowerPoint, and Microsoft Word because an OLE object does not have a file name or directory. This is true even if you insert the object from a file, using the Choose File button in Lotus Notes.

The Help file for each program documents this behavior with the following information: 

   The Filename and Directory fields return empty values since there is
   no file name or directory for an embedded object.
