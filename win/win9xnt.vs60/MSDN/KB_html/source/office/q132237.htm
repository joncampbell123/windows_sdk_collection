OFF95: Error or Unexpected Value Entering Custom Date Property

 Last reviewed: March 27, 1997
Article ID: Q132237
7.00 WINDOWS kbother

The information in this article applies to:

  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft PowerPoint for Windows 95, version 7.0
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Project for Windows 95, version 4.1

SYMPTOMS

In any of the Microsoft applications listed above, when you enter a custom date value in the Properties dialog box for a document, one of the following may occur:

  • You receive the following error message: 

          The value entered does not match with the specified type. The value
      will be stored as text.

    -or-

  • The property is entered as Unknown data type, with no value.

    -or-

  • The property is entered as Date type but with a random date value.

CAUSE

This behavior occurs if you enter a date value earlier than 1/1/1601, between 9/15/30829 and 5/8/31779, or later than 5/8/31779.

Earlier than 1/1/1601

If you enter a date earlier than 1/1/1601 (for example 5/3/1600), you receive the error message described above, and the property is entered as a Text type.

Between 9/15/30829 and 5/8/31779

If you enter a date between 9/15/30829 and 5/8/31779, such as 4/3/30999, the property is entered as Unknown data type, and no value is entered in the Value field for this property; the Value field is null.

Later Than 5/8/31779

If you enter a date later than 5/8/31779, a "random" date value appears in the Value field for the property. For example, if you add a property as a Date type with a value of 1/1/99999, the value 9/1/78 appears in the Value field.
