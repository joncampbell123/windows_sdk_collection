Extra Page Printed from Microsoft Binder Running on Windows NT

 Last reviewed: March 27, 1997
Article ID: Q151061
7.00 WINDOWS kbinterop

The information in this article applies to:

  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Binder for Windows 95, version 7.0
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Excel for Windows 95, version 7.0 --------------------------------------------------------------------

SYMPTOMS

When you print a binder from Microsoft Office Binder running on Windows NT, a blank, extra page may be printed at the end of the file.

STATUS

Microsoft has confirmed this to be a problem with the products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

This problem occurs when printing the whole binder, regardless of its contents.

Steps to Reproduce Problem

  1. On Windows NT 3.51, create/open a binder containing more than one Microsoft Word or Microsoft Excel section.

  2. On the File menu, click Print Binder.
