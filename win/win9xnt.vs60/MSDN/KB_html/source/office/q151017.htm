OFF95: "Setup Cannot Access The Required Initialization Files"

 Last reviewed: March 10, 1997
Article ID: Q151017
7.00 WINDOWS kbsetup

The information in this article applies to:

  • Microsoft Office for Windows 95, version 7.0, 7.0b
  • Microsoft Windows for NT, version 3.51

SYMPTOMS

If you install Office Professional for Windows 95 over Office Standard for Windows 95 on a computer running Windows NT version 3.51, and you then attempt to run the Setup program for Microsoft Office Professional for Windows 95, you may receive the following error message 

   Setup cannot access the required initialization file:
   <path>\OFF95STD.INF
where <path> is the original location of Office Standard for Windows 95 .

CAUSE

This problem occurs when you install Office Professional for Windows 95 over Office Standard for Windows 95 because the Acme.lst file that is copied to the C:\MSOffice\Office\Setup location is not properly updated and continues to point to the Offstd.inf setup initialization file.

RESOLUTION

To work around this problem, copy the Setup.lst file from the original installation location to the C:\MSOffice\Office\Setup location. Rename the current Acme.lst file to Acme.old, and then rename the Setup.lst file to Acme.lst.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
