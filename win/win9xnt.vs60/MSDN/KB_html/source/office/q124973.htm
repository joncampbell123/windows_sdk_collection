MOFF: Office Setup Doesn't Remove All Files Using Remove All

 Last reviewed: February 12, 1998
Article ID: Q124973
The information in this article applies to:
  • Microsoft Office for the Macintosh, versions 4.2, 4.2.1, 4.2.1a, 4.2.1b

SYMPTOMS

When you run the Microsoft Office Setup program in maintenance mode, and you choose the Remove All button to remove all of the Microsoft Office files, some of the files are not removed.

CAUSE

In general, when you run an application's Setup program in maintenance mode, and you choose the Remove All button, a file is not removed if either of the following is true:

  • The file was not installed by the Setup program 

        -or-
  • The file was installed by the Setup program, but the file is used by another application.

WORKAROUNDS

To work around this behavior, you can manually delete any Microsoft Office files that are not removed by the Setup program by dragging the files to the trash. This will free up disk space on your hard disk drive.

For a list of the Microsoft Office files that are not removed when you use the Remove All button in maintenance mode of the Setup program, see the "Files Not Removed By Using Remove All" section at the end of this article.
