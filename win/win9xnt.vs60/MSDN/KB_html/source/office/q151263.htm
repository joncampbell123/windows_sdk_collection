OFF95: 32-Bit Lotus Notes/FX 4.0 Does Not Work with Office 7.0

 Last reviewed: March 10, 1997
Article ID: Q151263
7.00 WINDOWS kb3rdparty

The information in this article applies to:

  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft PowerPoint for Windows 95, version 7.0
  • Microsoft Word for Windows 95, version 7.0

SYMPTOMS

When you use the 32-bit version of Lotus Notes 4.0 or 4.0.2, Field Exchange (FX) does not work with Microsoft Office 7.0.

CAUSE

Although Microsoft Office 7.0 is a 32-bit program, it includes only the 16-bit versions of the Lotus Notes dynamic link (.dll) files.

The 32-bit version of Lotus Notes 4.0 does not support application programming interface (API) calls to these 16-bit .dll files. Lotus Notes 4.0 only supports API calls to .dll files that use the same bit version (16- bit or 32-bit) as Lotus Notes 4.0.

Microsoft Office 97 includes 32-bit .dll files for use with Lotus Notes, so you can use Field Exchange with Microsoft Office 97.

WORKAROUND

In order to use Lotus Notes 4.0 or 4.0.2 FX with Microsoft Office 7.0, you must use the 16-bit version of Lotus Notes 4.0 or 4.0.2.

MORE INFORMATION

The Lotus Notes/FX support in the applications listed above allows you to use one of these applications to write to fields in a Lotus Notes database.

Microsoft Office 7.0 uses the following .dll files to exchange data with Lotus Notes: 

   _Sdk.dll
   _Nsf.dll
   _Lib.dll.
