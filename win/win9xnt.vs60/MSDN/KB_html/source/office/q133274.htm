OFF95: Setup "Remove All" Doesn't Remove All Files

 Last reviewed: February 20, 1998
Article ID: Q133274
The information in this article applies to:
  • Microsoft Office for Windows 95, version 7.0

SYMPTOMS

When you run the Microsoft Office Setup program in maintenance mode, and you click the Remove All button, some of the Office files are not removed from your computer.

CAUSE

This behavior is by design of Microsoft Office Setup. Setup does not remove any files that may be used by other applications. To completely remove Microsoft Office, you must remove all of the files listed below. However, because some of the files may be shared you may need to reinstall other applications. In general, the following items are removed by Office Setup when you use the Remove All feature:

  • The Microsoft Office folder is removed, unless you have saved any personal files to folders inside the Microsoft Office folder. (Setup will not remove any of your personal data files.)
  • The Common Files folder (in the Program Files folder) is removed.
  • All files installed by Office Setup to the Microsoft Windows folder are removed.
  • Most of the Office entries in the Windows Registry are removed.

Setup does not remove files installed by Office to the Windows\System folder (where Windows is the folder to which you installed Microsoft Windows)

NOTE: In Windows 95, you may need to perform the following steps in order to see some of the files listed above:

   2. On the View menu, click Options. Click the View tab.

   3. Click the Show All Files option button. Also, clear the "Hide MS-DOS
      file extensions for file types that are registered" checkbox. Click
      OK.
NOTE: If you install Microsoft Excel for Windows NT after performing a "Remove All" maintenance mode installation of Microsoft Office version 7.0, you receive the following error message when you run Microsoft Excel: 

   An error occurred initializing the VBA libraries (-25346)
This error message occurs because the Office Setup program does not remove all of the .DLL files installed by Office, and these .DLL files are not compatible with Microsoft Excel for Windows NT.
