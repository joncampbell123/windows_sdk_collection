OFF95: Office Compatible Button Does Not Work After Admin Instal

 Last reviewed: July 15, 1997
Article ID: Q136165
7.00 WINDOWS kbsetup kbtool

The information in this article applies to:

  • Microsoft Office Standard for Windows 95, version 7.0

SYMPTOMS

After you perform an installation of Microsoft Office Standard version 7.0 from a network administrative setup, the Office Compatible button located on the Office Toolbar of the Office Shortcut Bar may not start the Microsoft Office Compatible Demos program; instead, it may open a window for the root directory of the administrative installation on your network server.

CAUSE

When an administrative installation of Microsoft Office version 7.0 is performed from the Office CD, the ValuPack files are not copied to the administrative folder.

RESOLUTION

If you want the Office Compatible button to be fully functional, do the following:

  1. From the Office CD, copy the Valupack folder to the network administrative folder. This may take a long time because the size of the Valupack folder is over 300 megabytes (MB).

  2. On your local installation of Microsoft Office, use the right mouse button to click the Office Compatible button on the Office Toolbar of the Office Shortcut Bar.

  3. On the shortcut menu, click Properties.

  4. Click the Shortcut tab.

  5. In the Target box, type the full path to the Browser.exe file which is located in the Valupack\Ofcompat folder. For example, if your administrative installation is located at F:\msoffice, the target should be the following: 

          F:\msoffice\valupack\ofcompat\browser.exe

  6. In the Start In box, type the path to the folder containing the Browser.exe file. Using the same example that you used in step 5, type: 

          F:\msoffice\valupack\ofcompat

  7. Click OK.

When you click the Office Compatible button, the Microsoft Office Compatible Demos program should start as expected.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
