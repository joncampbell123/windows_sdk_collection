OFF95: Copy/Cut/Paste Commands Don't Work in Open Dialog Box

 Last reviewed: March 10, 1997
Article ID: Q131791
7.00 WINDOWS kbother

The information in this article applies to:

  • Microsoft Office for Windows 95, version 7.0

SYMPTOMS

In the Open dialog box that appears when you click the Open A Document button on the Office toolbar, if you select a file and use the Copy or Cut command on the shortcut menu, the Paste command is not available on the shortcut menu in this dialog box.

Additionally, the Copy and Cut commands do not function in this dialog box, although they are available on the shortcut menu when you select a file.

CAUSE

This behavior occurs because Windows 95 does not support the Cut/Copy/Paste functionality for files unless the application that you are using has loaded all of the OLE files. By design, to improve performance, the Office Shortcut Bar does not load all of the OLE files. However, because the Copy and Cut commands appear on the shortcut menu when you select a file in the Open dialog box (using the Open A Document button), this behavior may be confusing.

Although the Cut and Copy commands appear on the menu when you click the right mouse button on a file in the Open dialog box (from the Office Shortcut Bar), these commands do not Cut or Copy the selected file. Additionally, the Paste command is not available in the Open dialog box even if you copied a file in another application.

RESOLUTION

To work around this behavior, you can use the Copy, Cut, and Paste commands on the shortcut menu for a file in the following locations:

  • Open dialog box in any Office application
  • Windows Explorer
  • My Computer

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
