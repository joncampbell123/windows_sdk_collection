XL: Can't Create Copy of Module Sheet On Netware Server

 Last reviewed: March 27, 1997
Article ID: Q150708
5.00 7.00 7.00a WINDOWS kbusage kbnetwork kb3rdparty

The information in this article applies to:

  • Microsoft Excel for Windows 95, versions 7.0 and 7.0a
  • Microsoft Excel for Windows NT version 5.0

SYMPTOMS

When a user attempts to create a copy of a module sheet or move a module sheet in a Microsoft Excel workbook that has been saved on a Novell NetWare version 3.x or version 4.x server, the user may receive one of the following error messages:

Microsoft Excel for Windows 95 version 7.0: 

   Permission Denied
Microsoft Excel for Windows NT version 5.0: 

   No Error Message
NOTE: This error message occurs only when the user is running Microsoft Windows NT version 3.5x and either the Microsoft Client for NetWare or Novell's NetWare Client for Windows NT.

CAUSE

The error message occurs only if the user has no inherited rights to the mapped drive's root on the NetWare server. The problem occurs, for example, if the user has mapped the f:\ drive to 

   \\novell_server\test
where the user has no inherited rights but only Read & File Scan (RF) rights.

WORKAROUND

There are currently two workarounds to this problem:

Method 1

Map root the f:\ drive to a folder on the server where the user has inherited rights mask of RWCEMF. 

    R - Read
    W - Write
    C - Create
    E - Erase
    M - Modify
    F - File Scan
For example, if the above user instead maps the f:\ drive to 

    \\novell_server\test\userhome
where the user has RWCEMF inherited rights to the user's home folder, the user will be able to create a copy or move a module sheet in the Microsoft Excel workbook.

Method 2

Have the user temporarily save the file to a drive where the user has full rights, for example, the c:\ drive, copy or move the module sheet, and then save the file back on to the NetWare server.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
