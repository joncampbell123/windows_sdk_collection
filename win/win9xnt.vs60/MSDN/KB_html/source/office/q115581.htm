Error Choosing Microsoft Access 2.0 Button from MOM Toolbar

 Last reviewed: September 16, 1996
Article ID: Q115581
The information in this article applies to:
  • Microsoft Office for Windows, version 4.2

SYMPTOMS

When you choose the Microsoft Access 2.0 button on the Microsoft Office Manager (MOM) toolbar, you receive the following error message: 

   Cannot find 'MSACCESS.EXE'. Please start the application another
   way. While the application is running, click on the tool bar
   button or choose the menu command again.

CAUSE

MOM version 4.2 does not find Microsoft Access version 2.0 on your computer. Therefore, the Microsoft Access button is not automatically added to the MOM toolbar. Additionally, if you add a button for Microsoft Access 2.0 to your MOM toolbar, you receive the above error message when you choose this button from the toolbar.

STATUS

Microsoft has confirmed this to be a problem in MOM version 4.2. This problem has been corrected in MOM version 4.2b.

WORKAROUND

To create a toolbar button for Microsoft Access that you can use without receiving this error message, do the following:

  1. From the MOM toolbar, choose the Microsoft Office button, and then choose Customize by right mouse clicking on the Microsoft Office Button.

  2. Select the Toolbar tab. Clear the Microsoft Access check box.

  3. Choose the Add button.

  4. In the Description box, type "MS Access" (without the quotation marks). In the Command Line box, type the full path to the MSACCESS.EXE file, for example "C:\ACCESS\MSACCESS.EXE" (without the quotation marks). In the Working Directory box, enter the drive and directory that you want to use as the default in Microsoft Access. From the Button Image list, select Application Icon, and choose OK.

  5. In the Customize dialog box, choose OK.

REFERENCES

For more information about the Customize dialog box, choose the Search button in Help and type: 

   customize
