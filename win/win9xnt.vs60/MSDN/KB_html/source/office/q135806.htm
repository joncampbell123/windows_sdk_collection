Error Saving with Long File Name to Network Folder

 Last reviewed: December 16, 1996
Article ID: Q135806
The information in this article applies to:
  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Access for Windows 95, version 7.0
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft PowerPoint for Windows 95, version 7.0
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Office 97 for Windows
  • Microsoft Access 97 for Windows
  • Microsoft Excel 97 for Windows
  • Microsoft PowerPoint 97 for Windows
  • Microsoft Word 97 for Windows

SYMPTOMS

When you try to save a file using a long filename to a network directory, you may receive the following error message: 

   Cannot Save. 'Long Filename.ext' is an invalid 8.3 file system name.

CAUSE

This behavior occurs if the network server is running Windows for Workgroups 3.11 or some other operating system that does not support long file names, and you save a file with a long file name to the server.

WORKAROUND

Use a filename that adheres to the 8.3 filename specification. That is, use a filename that is up to eight characters long and is (optionally) followed by a period and up to a three-letter extension.

MORE INFORMATION

This limitation may show up in other areas of Office in addition to the example presented in this article. When you troubleshoot this type of error message, the first step should always be to try a short filename.
