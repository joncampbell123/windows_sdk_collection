OFF98: Cannot Drag Clipping File to Office Program

 Last reviewed: February 2, 1998
Article ID: Q179580
The information in this article applies to:
  • Microsoft Office 98 Macintosh Edition
  • Microsoft Excel 98 Macintosh Edition
  • Microsoft PowerPoint 98 Macintosh Edition
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

When you are trying to drag a clipping file into a Microsoft Office 98 Macintosh Edition program, the clipping file may not be moved to the program, and the following error message may appear: 

   There is insufficient memory. Save the document now.

CAUSE

This behavior occurs when the following conditions are true:

  • You are using Mac OS 8.0.

    -and-

  • You create a clipping file from an Office program.

    -and-

  • You try to drag the clipping file into an Office program.

WORKAROUND

To work around this behavior, try using the Scrapbook. To do this, follow these steps:

  1. On the Apple menu, click Scrapbook.

  2. Drag the clipping file to the Scrapbook.

    The clipping file is placed in the Scrapbook.

  3. Start the Microsoft Office 98 Macintosh Edition program.

  4. Arrange both programs so that you can easily view them.

  5. Drag the picture clipping from the Scrapbook to the Office program.

  6. Switch to Scrapbook.

  7. To remove the picture clipping from the Scrapbook, click Clear on the Edit menu.

  8. Click Quit on the File menu.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The behavior a clipping file varies. The behavior depends on the object type of the clipping file and the Microsoft Office program that created the object.
