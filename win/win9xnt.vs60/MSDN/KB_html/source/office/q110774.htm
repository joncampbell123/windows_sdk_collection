Office Err Msg: General Error Decompressing <Filename>

 Last reviewed: September 9, 1996
Article ID: Q110774
The information in this article applies to:
  • Microsoft Office, versions 4.0, 4.2, 4.3

SUMMARY

When you install Microsoft Office, you may receive the following error message 

   General error decompressing <filename>
where <filename> is the name of the file that Setup cannot decompress.

MORE INFORMATION

This error message may appear if the file on the disk is corrupt.

To test to see if the disk is corrupt

Manually decompress the file using DECOMP.EXE from the Microsoft Office Disk 1. When you attempt to decompress the file, the problem may be a bad disk if you receive the following error message 

   Internal error during decompression
To use DECOMP.EXE, do the following:

  1. Copy DECOMP.EXE from the Microsoft Office Disk 1 to your hard drive.

  2. Change to the <drive>\<directory> that you copied DECOMP.EXE to.

  3. Place the disk with the file to be tested into your floppy disk drive.

  4. Attempt to decompress the file using the following syntax 

          DECOMP <drive>:\<filename> <outpath>\<newfilename>

    where:

    <drive> is the name of the floppy disk drive your disk is in <filename> is the name of the file you are trying to decompress <outpath> is the destination where you want the new file to be copied <newfilename> is the name of the file to expand to

For example, to decompress the file on drive a named TEST.DL_ to a file named TEST.DLL on drive c, you would use the following command: 

   decomp a:\test.dl_ c:\test.dll
If you are unable to successfully decompress the file, contact Microsoft Product Support Services to obtain a new set of disks.
