OFF95: Pipe Character in Comments Field in Properties Dialog

 Last reviewed: March 27, 1997
Article ID: Q129991
7.00 WINDOWS kbenv

The information in this article applies to:

  • Microsoft Windows 95
  • Microsoft Windows NT operating system version 3.51
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft Project for Windows 95, version 4.1

SYMPTOMS

In Microsoft Windows 95, when you view the file properties for a Microsoft Excel or a Microsoft Project file, the Comments field may contain a pipe character (|).

CAUSE

This behavior occurs with files that you create in Microsoft Project version 4.0 or Microsoft Excel version 5.0 if you enter multiple lines in the Comments field (by pressing ENTER between the lines) in the Summary Info dialog box. For example, if you enter the following comment in the Summary Info dialog box in a Microsoft Project version 4.0 file 

   Comment1
   Comment2
   Comment3
the comment appears as follows when you view the Comments field in Microsoft Windows 95: 

   Comment1|Comment2|Comment3
Multiple line comments in a Microsoft Project 4.0 file or a Microsoft Excel 5.0 file are displayed with the pipe character when you view the Comments field in the Properties dialog box from either a folder in My Computer, a folder in the Explorer, or by opening the file in Microsoft Project version 4.1 or Microsoft Excel version 7.0.

Note that this behavior does not occur with multiple line comments in Microsoft Word version 6.0 documents or Microsoft PowerPoint version 4.0 slides.

WORKAROUNDS

To work around this behavior, you can use any of the following methods.

Method 1: Open the file again in Microsoft Project version 4.0 or Microsoft 

          Excel version 5.0 and modify the Comments field so that it
          contains only one line of text.
Method 2. Open the file in Microsoft Project version 4.1, or Microsoft 
          Excel version 7.0, and modify the Comments field. You can delete
          the pipe character and enter a carriage return. The carriage
          return is displayed correctly in Windows 95 when you save the
          file in these applications.
