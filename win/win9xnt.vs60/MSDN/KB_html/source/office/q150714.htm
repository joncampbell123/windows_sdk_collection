Errors Using ODBC and MS Query After Custom Office Install

 Last reviewed: February 2, 1998
Article ID: Q150714
The information in this article applies to:
  • Microsoft Office for the Macintosh, versions 4.2.1b

SYMPTOMS

If you try to set up a data source using the ODBC Setup control panel, you may receive the following error 

   Sorry, a system error occurred.
   "Finder"
   Unimplemented Trap
and your only option is to restart your computer.

Also, if you try to start Microsoft Query, either by double-clicking the Query icon in the Microsoft folder or by clicking Get External Data on the Data menu in Microsoft Excel, you may receive either of the following errors 

   Couldn't load ODBC due to improper installation or memory shortage

   -or-

   Cannot launch MSQuery.exe
and Microsoft Query fails to start.

CAUSE

You may receive the above error messages if both of the following conditions are true:

  • You do not have a Power Macintosh computer. 

        -and-
  • You performed a Custom installation of Microsoft Office for the Macintosh, version 4.2.1b, and the only option selected was Microsoft Excel.

You receive these error messages under these conditions because the Office Setup program fails to install a resource that is needed for this Data Access component of Microsoft Office to function properly.

WORKAROUND

To work around this issue, use either of the following methods:

Method 1

If you already have Microsoft Office installed, add in the Microsoft Office Manager component of Microsoft Office through the maintenance setup program.

Method 2

If you do not have Microsoft Office installed, perform a Custom installation, and when you are in the Custom dialog box, select Microsoft Office Manager in addition to Microsoft Excel

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
