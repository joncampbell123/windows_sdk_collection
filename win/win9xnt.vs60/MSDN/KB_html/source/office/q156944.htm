OFF95: Setup Hangs on NT 4.0 Server With IIS Running

 Last reviewed: March 10, 1997
Article ID: Q156944
7.00 WINDOWS kbusage kbprb kbsetup The information in this article applies to:
  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Windows NT, version 4.0

SYMPTOMS

During the installation of Microsoft Office, after all files are copied, the Setup program displays the message: 

   Setup is updating your system files
and stops responding.

CAUSE

This may occur if all of the following conditions are true:

  • You are on a computer running Microsoft Windows NT 4.0 Server.

    -and-

  • You are performing a normal installation ("setup") of Microsoft Office, either from another server or from a CD-ROM. You are NOT performing an administrative installation ("setup /a").

    -and-

  • The computer has the Microsoft Internet Information Server (IIS) services installed and running.

WORKAROUND

To prevent this problem from occurring, turn off the IIS services before you run Microsoft Office Setup. To do this, use the following steps: 

   1. On the Start menu, point to Settings, and click Control Panel.

   2. Double-click Services.

   3. In the list of services, click "World Wide Web Publishing Service".
      Click Stop.

   4. When asked if you are sure you want to stop the service, click Yes.

   5. Click Close, and then close the Control Panel window.
After Microsoft Office Setup is complete, you can turn the services back on: 

   1. On the Start menu, point to Settings, and click Control Panel.

   2. Double-click Services.

   3. In the list of services, click "World Wide Web Publishing Service",
      then click Start.

   4. Click Close, and then close the Control Panel window.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem has been corrected in Microsoft Office 97 for Windows.

MORE INFORMATION

This error occurs only when you install Microsoft Office on a Windows NT 4.0 Server that is non-dedicated: that is, the computer also serves as a workstation. The error does NOT occur when you perform an administrative installation of Microsoft Office from a workstation onto a Windows NT 4.0 Server.
