OFF97: Installing Office Overwrites Web Find Fast Service

 Last reviewed: April 17, 1997
Article ID: Q166988
The information in this article applies to:
  • Microsoft Office 97 for Windows
  • Microsoft Windows NT Server versions 3.51, 4.0
  • Microsoft Windows NT Workstation version 4.0

SYMPTOMS

When you install Microsoft Office 97 and include the Find Fast utility, the Web Find Fast service that you previously installed from the Office Server Pack (on the Office compact disc) is overwritten, and all Web Find Fast functionality is removed.

CAUSE

Microsoft Office overwrites an existing Find Fast program even if you installed the program from the Srvpack folder on the Microsoft Office 97 compact disc.

WORKAROUND

To work around this problem, do not install Find Fast when you install or reinstall Microsoft Office 97 after installing the Office Server Pack.

NOTE: When you run Setup, the Find Fast option is under the Office Tools components. To prevent Setup from installing Find Fast, clear the Find Fast check box.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

When you run the Setup program in the Srvpack folder on the Microsoft Office 97 compact disc, a Find Fast service is installed in the Microsoft Windows NT Control Panel. This service creates indexes on Web servers, on network servers, or on your local hard disk. By default, Find Fast creates a single index on each hard disk on your computer.

For additional information about Web Find Fast, please see the Readme.txt file that is located in the Srvpack folder on the Microsoft Office 97 compact disc.
