WordPerfect Fonts Unavailable After Installing Office

 Last reviewed: March 27, 1997
Article ID: Q137708
4.20 4.30 7.00 7.00b WINDOWS kbdisplay kb3rdparty kbprint

The information in this article applies to:

  • Microsoft Office for Windows, versions 4.2, 4.3
  • Microsoft Office for Windows 95, versions 7.0, 7.0b

SUMMARY

When you install Microsoft Office or a Microsoft Office program, WordPerfect version 5.1 or 5.2 may be unable to properly display the correct fonts on the screen.

CAUSE

According to WordPerfect Corporation, when new fonts are installed, WordPerfect 5.1 or 5.2 for Windows will update the bitmap display files to correctly display fonts in WordPerfect. If these files are updated improperly after an application installs new fonts, then WordPerfect may print fonts that are different from what is displayed on the screen.

WordPerfect 5.1 or 5.2 for Windows comes with its own printer drivers. When the printers are installed, WordPerfect stores the screen fonts in a bitmap collection file (*.wrs). The characters sets are used to map and display selected fonts as they would appear on the screen. WordPerfect 5.1 or 5.2 only displays the fonts inaccurately. The fonts are printed correctly.

WORKAROUNDS

WordPerfect has recommended the following methods for updating the font files. If you have any questions regarding these methods, please contact WordPerfect technical support for assistance.

Method 1: Update the printer driver information. 

          1. Start WordPerfect.

          2. On the File menu, click Printer.

          3. Click the desired printer then click the Update button.
Method 2: Rebuild the character map files (*.wrs). 

          1. Close WordPerfect.

          2. In the WPC folder, rename all *.wrs files to *.old.

          3. Restart WordPerfect.
Method 3: Rebuild Setup preferences. 

          NOTE: This method will change preferences stored in WordPerfect.
          Contact WordPerfect technical support for more information about
          restoring these preferences.

          1. Close WordPerfect.

          2. In the WPWIN folder, rename the *.set file to *.old. (There
             should only be one *.set file).

STATUS

WordPerfect Corporation has confirmed this to be a problem with WordPerfect 5.1 or 5.2 for Windows and has made information available from WordPerfect technical support.

MORE INFORMATION

For more information about this problem, contact WordPerfect technical support at (800) 228-1029 and ask for Technical Info Document (TID) #626309.
