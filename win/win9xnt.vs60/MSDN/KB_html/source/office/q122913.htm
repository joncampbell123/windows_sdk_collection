Office Applications Hang Lanserver Running Under OS/2

 Last reviewed: September 16, 1996
Article ID: Q122913
The information in this article applies to:
  • Microsoft Project for Windows, version 4.00
  • Microsoft Excel for Windows, version 5.0
  • Microsoft Word for Windows, version 6.0
  • Microsoft PowerPoint for Windows, version 4.0
  • Microsoft Access, version 2.0
  • IBM OS/2 operating system, version 2.1 or 2.2

SYMPTOMS

If you are using IBM Lanserver 1.3 or 3.0 and the OS/2 2.1 or 2.2 operating system, your network system may hang when you attempt to start or run a Microsoft Office application such as Microsoft Project 4.0 for Windows. The server may crash and produce an error message similar to the following: 

   System 3175 Error program generated an access violation @
   00041385  Netservr.exe 0004:00001385
NOTE: Using IBM LanBridge may also cause the network to crash.

RESOLUTION

To correct this problem on the server and requester (workstation) side, you need the following files: 

   LS30REQ.ZIP
   LS30ESRV.ZIP
If you have a modem, you can download these files from a minimum security bulletin board at (512) 823-9538. The BBS may prompt you for a user name and a password, but you will be able to gain access to the BBS without them.

For more information about how to correct this problem, contact IBM at the appropriate phone number below.

  • If you have a support contract with IBM, and you have a customer number, call (800) 992-4777.
  • If you do not have a customer number, call (512) 339-5000.
