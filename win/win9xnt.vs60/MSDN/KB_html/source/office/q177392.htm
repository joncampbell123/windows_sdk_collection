OFF97: Setup Error 947 After Using NIW to Modify STF File

 Last reviewed: November 26, 1997
Article ID: Q177392
The information in this article applies to:
  • Microsoft Office 97 for Windows

SYMPTOMS

When you run the Microsoft Office 97 Setup program, you may receive the following error message multiple times: 

   Setup Error 947
   Network Log Location Object
   Your Setup files may be damaged. Try restarting the Setup program from
   where you originally ran it.
Next, you receive the following error message: 

   Setup Error 723
   Setup was unable to read configuration information from the first
   section of the setup file: C:\~MSSETUP.T\~msstfof.t\Off97Pro.stf.
Then, you receive the following error message: 

   Microsoft Office 97 Setup was not completed successfully.

CAUSE

This problem occurs if you do all of the following:

  • You use the Microsoft Office Network Installation Wizard to customize the installation of Microsoft Office.

    -and-

  • In the Network Installation Wizard, you specify a location that does not include the full path in the Network Log Location box, for example, you specify the following location for the log file: 

          Installed.log

RESOLUTION

To resolve this problem, follow these steps:

  1. Indicate the network server and share when you specify the location in the Network Log Location box.

  2. Open the customized LST file in the Network Installation Wizard and change the location for the network log to include a fully qualified path for the log file.

    For example, the path to the log file should be similar to the following 

          \\<server>\<share>\Installed.log

    where <server> is the path to a valid server and <share> is the share location on that server. To successfully install Microsoft Office, the user who is performing the installation must have both read and write permissions to this share point.

MORE INFORMATION

The Network Log Location box in the Network Installation Wizard should indicate the location for the Network Install Log file that is used to track the number of installations (and removals) from an administrative installation point. This log file maintains information for the Office administrator regarding the user name, computer, and the date on which Office is installed on the workstation from the administrative point.

The Network Log file is different from the Setup Log file (which is generated by adding the /g <filename.log> command-line parameter to the end of the setup command line. The Setup Log generates a log file that records details of the installation process on the workstation. For example, the following command 

   O:\Off97\setup.exe /L custom\custom.lst /B2 /Q1 /G "C:\off97.log"
creates a Setup Log file named Off97.log in the root directory of drive C.
