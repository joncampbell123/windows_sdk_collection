Novell's NETBIOS May Require TimerCriticalSection Line

 Last reviewed: November 21, 1994
Article ID: Q82694
The information in this article applies to:
  • Microsoft Windows operating system version 3.1, 3.11
  • Microsoft Windows for Workgroups version 3.1

SYMPTOMS

Microsoft MS-DOS NETBIOS applications run from MS-DOS Prompt under Windows in 386 enhanced mode may cause your system to stop responding (hang).

WORKAROUNDS

To correct this problem, add the following line to [386Enh] section of SYSTEM.INI: 

   TimerCriticalSection=1000
This switch is not added during Setup; it is required only in the above circumstance.

NOTE: The Novell NETBIOS.EXE file must be loading before you start Windows.
