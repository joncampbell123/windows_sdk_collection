DCA Irma Workstation for Windows and PCLP Network

 Last reviewed: November 30, 1994
Article ID: Q82400
The information in this article applies to:
  • Microsoft Windows operating system versions 3.1, 3.11

SYMPTOMS

The following information applies to Microsoft Windows running in enhanced mode.

When a DCA Irma Workstation for Windows is run using NETBIOS to connect to the IRMALAN gateway using PCLP version 1.34, the system stops responding (hangs) shortly after a NETBIOS connection is made.

CAUSE

This is not a problem with the MS-DOS based version of DCA IRMA Workstation.

This is not a problem with Windows, but with the applications use of page-locked memory.

WORKAROUND

Run Windows in standard mode or add more memory to the system.
