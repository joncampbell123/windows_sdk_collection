Second Print Job Fails after Deleting a Printer Driver

 Last reviewed: November 30, 1994
Article ID: Q50822
The information in this article applies to:
  • Microsoft Windows operating system versions 3.0, 3.0a, 3.1, 3.11

The Microsoft Windows version 3.0 Control Panel allows you to delete a printer while an application is printing to that printer.

The print job completes normally because it can still access the correct driver; however if you try to start another print job later with the same driver, it fails.
