XCLN: Client Setup Overwrites Localized Version of Ole32.dll

 Last reviewed: April 6, 1997
Article ID: Q165537
The information in this article applies to:
  • Microsoft Exchange Windows 3.x Client, version 4.0
  • Microsoft Exchange Windows 95 Client, version 4.0
  • Microsoft Exchange Windows NT Client, version 4.0

SYMPTOMS

When installing the Microsoft Exchange Client on localized operating systems that have a localized copy of Ole32.dll, the copy included with Microsoft Exchange overwrites the existing file. This causes problems displaying attachment labels.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Exchange Clients for Windows, version 4.0. This problem was corrected in the latest Microsoft Exchange 4.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces):
