XADM: Supporting Localized Exchange Clients

 Last reviewed: November 25, 1997
Article ID: Q150977
The information in this article applies to:
  • Microsoft Exchange Server, version 4.0

SUMMARY

The following error might be generated by the MSExchangeIS service and logged to the Application Log in EventViewer: 

   Event ID 1085

   Some data referred to code page <number> that is
   currently not supported by the operating system."
If any localized Microsoft Exchange clients have mailboxes on a Microsoft Exchange Server, the Microsoft Exchange Server computer must have the correct code pages installed for the appropriate character-mapping to occur. The code page must be installed using the International applet in the Control Panel on the Microsoft Exchange Server computer and the Locales property sheet for that Microsoft Exchange Server must list the code page (in other words, Language) under Selected Locales.

MORE INFORMATION

To configure the Microsoft Exchange Server to allow localized Microsoft Exchange clients to connect and see information that is properly sorted And formatted for their language, perform the following steps:

  1. Start the Microsoft Exchange Administrator program, select the Servers object, and select the Microsoft Exchange Server that is generating the error.
2. Click on Properties from the File menu and select the Locales tab. Add 
   the appropriate languages from the listbox on the left. If the
   appropriate code page has not been installed under Windows NT, a
   prompt will appear asking you to install the code page.
3. Per the procedure outlined in the Readme file provided on the Microsoft 
   Exchange Server CD:
     - Log on to the Windows NT Server computer as a local administrator.
     - In Program Manager, choose the Main program group, open the Control
       Panel, and open the International applet.
     - In Language, select the language support you want to add to the
       computer, and choose OK. It will prompt you to specify the location
       of the national language support files.
     - Restart the computer.
     - Reset the country and language settings back to their original
       settings.
This procedure may change with the release of Windows NT Server version 4.0.
