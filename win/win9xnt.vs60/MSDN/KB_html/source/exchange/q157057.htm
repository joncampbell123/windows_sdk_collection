XCLN: Display Name from GAL is not Added to PAB

 Last reviewed: April 3, 1997
Article ID: Q157057

The information in this article applies to:
  • Microsoft Exchange Macintosh client, versions 4.0 and 5.0
  • Microsoft Exchange Windows 3.x client, versions 4.0 and 5.0
  • Microsoft Exchange Windows 95 client, versions 4.0 and 5.0
  • Microsoft Exchange Windows NT client, versions 4.0 and 5.0

SUMMARY

When you add an entry from the Global Address List (GAL) to the Personal Address Book (PAB), the display name in the PAB will not be the same as the display name in the GAL.

MORE INFORMATION

This behavior is by product design. When you add an entry from the GAL into the PAB, the display name in the PAB will be a combination of the First Name field and the Last Name field.

One workaround is to modify the user's Last Name along with the Display Name when you make changes to a Microsoft Exchange GAL object.
