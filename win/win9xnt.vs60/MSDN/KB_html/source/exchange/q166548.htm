XADM: Administrators Cannot Add Accounts to Private IS

 Last reviewed: July 15, 1997
Article ID: Q166548

The information in this article refers to:

  • Microsoft Exchange Server, version 5.0

SYMPTOMS

If you are an administrator of Microsoft Exchange Server version 5.0, when you try to add Windows NT accounts to the private information store with the Admin Role the following error message appears: 

   Microsoft Exchange Administrator
   An internal processing error has occurred: the attribute or value
   exists.
   Try restarting the Administrator program or the Microsoft Exchange
   Server computer, or both.
   Microsoft Exchange Directory
   ID no: DS_E_ATTRIBUTE_OR_VALUE_EXISTS

CAUSE

Microsoft Exchange Server is not reading the Windows NT security descriptor for the store correctly when a modification is made.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Exchange Server version 5.0. This problem was corrected in the latest Microsoft Exchange Server 5.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
