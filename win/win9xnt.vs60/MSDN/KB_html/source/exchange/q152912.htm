XCLN: Delete Rule on Public Folder Treated as Error

 Last reviewed: April 15, 1997
Article ID: Q152912
The information in this article applies to:
  • Microsoft Exchange Windows 3.x client, version 4.0
  • Microsoft Exchange Windows NT client, version 4.0
  • Microsoft Exchange Windows 95 client, version 4.0

SYMPTOMS

When you try to synchronize an offline public folder with an online public folder, and the online public folder has a delete rule, the following error might occur: 

   This folder could not be synchronized. Consult the synchronization log
   in the deleted items folder for details.
The synchronization log will contain the following message: 

   Error synchronizing message '<message subject>'
   [80040800-3ED-0-324]
   Unknown Error.
   Microsoft Exchange Server Information Store
   Error synchronizing folder
   [80040800-3ED-0-324]
   Unknown Error.
   Microsoft Exchange Server Information Store
The message will stay in the offline folder.

CAUSE

The return code generated by the delete operation is treated as an error by the synchronization process.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Exchange Server version 4.0. This problem was corrected in the latest Microsoft Exchange Service Pack. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
