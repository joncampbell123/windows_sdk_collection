XADM: Internet Mail Service and IIS File Conflict

 Last reviewed: December 8, 1997
Article ID: Q177770
The information in this article applies to:
  • Microsoft Exchange Server versions 5.0 and 5.5

SYMPTOMS

It appears that Microsoft Internet Mail Service and Microsoft Internet Information Server 4.0 (IIS 4.0) cannot run on the same computer because of a conflict with the Isatq.dll file. Both services use the same name for a DLL. When installing the IMS on a machine with IIS 4.0 already installed, you will get a message informing you that Internet Mail Service cannot run on the same server as IIS 4.0. This message occurs after you restart the server after Internet Mail Service is installed.

STATUS

This problem only affects IIS 4.0. It is not a problem in earlier versions of IIS.
