XFOR: Lowercase Address Types in Dir-Sync with Exchange

 Last reviewed: April 3, 1997
Article ID: Q157712
The information in this article applies to:
  • Microsoft Exchange Server, version 4.0

The address type parameter (that is, SMTP, PCM, SNADS, and so forth) in a directory synchronization (dir-sync) transaction is not case sensitive, by definition.

A Microsoft Mail postoffice sends all its transaction address types in uppercase.

Microsoft Exchange Server sends all its transaction address types in lowercase.

Both Microsoft Mail and Microsoft Exchange Server read these transactions without sensitivity to case. Some third-party dir-sync requesters are sensitive to the case of the address type parameter and incorrectly expect it to be uppercase.
