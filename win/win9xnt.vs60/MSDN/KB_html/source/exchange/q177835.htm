XFOR: Extended Characters on Addresses Are Not Recognized

 Last reviewed: February 12, 1998
Article ID: Q177835
The information in this article applies to:
  • Microsoft Exchange Server version 5.0

SYMPTOMS

Incoming and outbound Internet mail passing through the Microsoft Exchange Server Internet Mail Service (IMC) may appear to have corruption in extended characters such as those used in International character sets in foreign languages (for example, ALT 0154,ALT 0225). Although the messages may arrive and be readable in an Exchange or Outlook client, the RTFHTML converter that allows the item to be readable in a Web access client displays corrupted characters.

STATUS

Microsoft has confirmed this to be a problem Microsoft Exchange Server version 5.0. This problem has been corrected in the latest U.S. Service Pack for Microsoft Exchange Server version 5.0. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): S E R V P A C K
