XCLN: Open Inbox Shows Old Inbox After Change

 Last reviewed: April 21, 1997
Article ID: Q147473
The information in this article applies to:
  • Microsoft Exchange Windows client, version 4.0
  • Microsoft Exchange Windows NT client, version 4.0
  • Microsoft Exchange Windows 95 client, version 4.0

SYMPTOMS

In the Microsoft Exchange Windows clients, the Open Inbox function opens the old Inbox folder after a change has been made to the default delivery options.

CAUSE

If the default delivery location is changed while the client is logged in, the Open Inbox function isn't updated to point to the new location.

RESOLUTION

Log out of the Exchange client, and log back in. Open Inbox will now function as expected.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft Exchange version 4.0 Windows 3.x, Windows 95, and Windows NT clients. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
