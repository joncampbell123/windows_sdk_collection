XFOR: MtaCheck Causes Access Violation Exception Error

 Last reviewed: April 9, 1997
Article ID: Q161932
The information in this article applies to:
  • Microsoft Exchange Server, version 4.0

SYMPTOMS

The MtaCheck utility causes a Dr. Watson access violation (AV) error with exception 0xc0000005, or the MTA causes the same error during its startup and recovery processing.

CAUSE

The database recovery processing done by MtaCheck and the MTA after a fatal error runs into a corrupt object, and gets crossed while searching for its referencing object.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Exchange Server version 4.0. This problem was corrected in the latest Microsoft Exchange 4.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
