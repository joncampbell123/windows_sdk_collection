XCLN: CHT Default Font Incorrect

 Last reviewed: April 15, 1997
Article ID: Q157710
This information applies to:

- Microsoft Exchange Windows NT client, version 4.0

SYMPTOMS

Using the Traditional Chinese (CHT) Exchange Microsoft Exchange Windows NT client, the default font type is "New New Ming Li," instead of "New Ming Li."

To view the default font type:

  1. Open a "Compose - New message" window.

  2. Set the focus on the message body.

  3. Look at the default font type in the tool bar.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft Exchange Windows NT client, version 4.0. A supported fix is now available, but has not been fully regression-tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Technical Support Services for more information.
