XADM: IS Stops Responding, Server-Specific Error 4294966766

 Last reviewed: December 11, 1997
Article ID: Q166688

The information in this article applies to:

  • Microsoft Exchange Server version 4.0

SYMPTOMS

When you start the Microsoft Exchange Server Information Store, the service may stop responding and server-specific error 4294966766 occurs.

CAUSE

The Priv.edb and Pub.edb files may be out of sync with the .log files.

RESOLUTION

To resolve this problem:

  • Remove all the .log files and the Edb.chk file from the <exchsrvr>\Mdbdata directory. This should allow the Information Store to start.

MORE INFORMATION

The following Event IDs may also be generated in the application log: 

   Event ID 37
   Source: EDB Type: Warning
   MSExchangeIS Unable to read header of database
   <exchsrvr\mdbdata>\PRIV.EDB
   Error -531

   Event ID 37
   Source: EDB Type: Warning
   MSExchangeIS Unable to read header of database
   <exchsrvr\mdbdata>\PUB.EDB
   Error -531

   Event ID 46
   Source: EDB Type: Error
   MSExchangeIS  The database signature does not match the log signature of
   the database <path to PRIV.EDB>

   Event ID 5000
   Source: MSExchangeIS Type: Error
   Unable to initialize the Microsoft Exchange Information Store service.
   Error -530.
