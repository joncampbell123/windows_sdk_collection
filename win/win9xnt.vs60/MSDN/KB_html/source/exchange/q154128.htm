XCLN: Long Filenames Converted to 8.3 Format

 Last reviewed: April 9, 1997
Article ID: Q154128

The information in this article applies to:
  • Microsoft Exchange Windows NT client, version 4.0
  • Microsoft Exchange Windows 95 client, version 4.0

SYMPTOMS

When you save more than one file attachment in a message at once, if these attachments are all using long filenames, the files saved are in 8.3 format rather than long filenames.

WORKAROUND

Save each attachment individually; the long filename will be preserved.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft Exchange Windows NT client, version 4.0, and the Microsoft Exchange Windows 95 client. This problem was corrected in the latest Microsoft Exchange 4.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces):
