XCLN: Address Book Stops with Modeless Address Book Applications

 Last reviewed: April 6, 1997
Article ID: Q165185
The information in this article applies to:
  • Microsoft Exchange Server, version 4.0

SYMPTOMS

When a modeless address book is opened by other MAPI applications, and the Microsoft Exchange Client tries to open and close the address book, it stops responding. Because the DBCS version of Microsoft Exchange Server uses a modeless address book, DBCS Microsoft Exchange Client stops responding with modeless address book MAPI applications.

CAUSE

This problem happens when RichEdit controls are used instead of normal edit controls for the edit control in the modeless Address Book. This used to be the case in the Far East version only. The failure happens because WMSUI is released before the address book dialog box is closed. When the RichEdit control on the dialog box is going to be removed, it fails because the control no longer exists.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Exchange Server 4.0. This problem was corrected in the latest Microsoft Exchange 4.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
