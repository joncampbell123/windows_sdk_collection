XADM: DIR Import from Text File Creates Addresses

 Last reviewed: April 3, 1997
Article ID: Q147518
The information in this article applies to:
  • Microsoft Exchange Server, version 4.0

SYMPTOMS

When you import Internet or X.400 custom recipients from a text file into Microsoft Exchange Server via Directory Import in the Microsoft Exchange Administrator program, addresses will be created for all three address types: X.400, Internet, and Microsoft Mail.

This occurs even if only one of these address types was defined in the text file. There is no way to override this feature when you import users from a text file. To remove the other addresses that are automatically generated, the user must go into the property sheet for each custom recipient and click the Remove button on each address. Inclusion of a blank Microsoft, X.400, or Internet address in the text file will generate an error, and enabling the Overwrite Multivalued Properties feature in the Directory Import dialog box has no effect on this behavior.

CAUSE

The Directory Import feature works as it was designed to, although the behavior is unexpected. The X.400 proxy address is necessary for Exchange messaging to function properly. The other addresses are created automatically, because disk space was not considered a major consideration when compared with the labor that would be involved in manually creating the proxy addresses for all users if a Microsoft Mail or Internet Connector were added to the system.

RESOLUTION

If the Microsoft Mail or Internet addresses will not be used, you can remove them manually by going into the property sheet for those custom recipients and selecting Remove.
