XCLN: Logging on to Local Machine Account on Exchange Server

 Last reviewed: April 3, 1997
Article ID: Q157039
The information in this article applies to:
  • Microsoft Exchange Windows NT client, versions 4.0 and 5.0

SUMMARY

When you login to a Windows NT 4.0 Local Machine account on a Microsoft Exchange Server computer and then try to start the Microsoft Exchange Windows NT client, the following error will be displayed: 

   The Set of folders could not be opened. You do not have permission to
   log on.
This is expected behavior when you do not login to a Windows NT Domain. You must be logged on with a valid NT Domain account in order to access a mailbox on a Microsoft Exchange Server.

MORE INFORMATION

Under Windows NT 3.51, the error message will be: 

   Microsoft Exchange could not be started. Exchange could not log on.
   Check to make sure you are connected to the network and are using the
   proper server and mailbox name.
