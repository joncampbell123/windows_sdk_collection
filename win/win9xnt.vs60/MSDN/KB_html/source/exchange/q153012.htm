XCLN: Wordmail Will Not Resolve Aliases With CTRL+K

 Last reviewed: April 15, 1997
Article ID: Q153012
The information in this article applies to:
  • Microsoft Exchange Windows NT client, version 4.0
  • Microsoft Exchange Windows 95 client, version 4.0

SUMMARY

When you use WordMail as your Microsoft Exchange client Editor, the CTRL+K keystroke option to resolve an alias name is not available. In WordMail, the keystroke is ALT+K.

MORE INFORMATION

You can also either include the Check Names button on your toolbar or use the Check Names option on the Compose menu to resolve alias names.

Both options are available with Microsoft Exchange 5.0 and Microsoft Word 97.
