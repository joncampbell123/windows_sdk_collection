XFOR: Routing Exchange DXA Updates Over SNADS Backbone

 Last reviewed: March 11, 1998
Article ID: Q167367
The information in this article applies to:
  • Microsoft Exchange Server versions 4.0 and 5.0
  • LinkAge Message Exchange

Routing Exchange Server DXA updates over a SNADS backbone requires custom configuration of LinkAge Message Exchange.

A new section, [Q-DXAMEX.IN] has been added to the LinkAge Configuration file (Linkage.ini) which allows the Exchange Server DXA to use the SNADS address type for routing in order to accommodate SNADS Backbone Routing. This is required if the LDE configuration includes any DXA that uses LinkAge Level 1 Message Exchange Format (MIF). LDE configurations that include only Level 3 MIF agents do not require the use of the SNADS address type for routing.
