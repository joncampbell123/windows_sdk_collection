XFOR: Starting Internet Mail Service, Store Causes Stack Fault

 Last reviewed: February 12, 1998
Article ID: Q161887
The information in this article applies to:
  • Microsoft Exchange Server version 5.0

SYMPTOMS

When you start the Microsoft Exchange Server Internet Mail Service, Store.exe causes a stack fault, halting store-related activities.

CAUSE

Multiple cascading embedded messages processed through a recursion routine is not limiting the number of recursions, which causes a stack overflow.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Exchange Server version 5.0. This problem has been corrected in the latest U.S. Service Pack for Microsoft Exchange Server version 5.0. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
