XFOR: Messages to 246 Character Addresses Return with NDR

 Last reviewed: July 18, 1997
Article ID: Q160284

The information in this article applies to:
  • Microsoft Exchange Server version 4.0

SUMMARY

Messages that are sent from one Microsoft Exchange Internet Mail Connector (IMC) to another IMC with addresses that have more than 246 characters (before and after the '@') might fail with Non-Delivery Report (NDR) from the Microsoft Exchange System Attendant as "Unknown Host" rather than traveling through and returning with a "User Unknown" error.

STATUS

Microsoft has confirmed this to be a problem in version 4.0 of Microsoft Exchange. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
