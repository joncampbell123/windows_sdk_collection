XADM: Troubleshooting the System Attendant Service

 Last reviewed: September 23, 1997
Article ID: Q147705
The Information in this article applies to:
  • Microsoft Exchange Server, version 4.0

SUMMARY

The System Attendant service can be started from Control Panel, Services applet or from Server Manager, Services or using NET START MSEXCHANGESA from the command prompt.

MORE INFORMATION

Steps to troubleshoot System Attendant service startup problem:

  1. Microsoft Exchange System Attendant Service, MSExchangeSA depends on the following services: 

                 EventLog
             OLE
             NtLmSsp
             RPCLocator
             RPCSS
             LanmanWorkstation
             LanmanServer

   Verify that all the dependency services have started.

  2. Check that the service account used to start the System Attendant and other services is a member of the Administrators group on the Microsoft Exchange Server.

  3. PVIEWER.EXE is a tool that comes with the Microsoft Windows NT resource kit. This tool can be used to check System Attendant activity. In PVIEWER, look for MAD, which is the System Attendant executable.

  4. Check the event log and filter using source MSEXCHANGESA to determine more information.
