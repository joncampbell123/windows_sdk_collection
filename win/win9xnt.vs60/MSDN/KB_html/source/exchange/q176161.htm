XFOR: Exchange 5.5 will Disallow Auto Forwarding to the Internet

 Last reviewed: November 14, 1997
Article ID: Q176161
The information in this article applies to:
  • Microsoft Exchange Server, version 5.5

SUMMARY

In Microsoft Exchange Server, versions 4.0 and 5.0, administrators were able to prevent users from sending Out-Of-Office messages and Auto Replies to internet addresses. However, administrators were unable to prevent users from Auto Forwarding messages to internet addresses.

A new feature of Microsoft Exchange Server, version 5.5 will allow administrators to configure the Microsoft Exchange Internet Mail Service (IMS) so that auto forwarded messages will not be delivered through the IMS. The IMS will return a Non-Delivery Receipt (NDR) to the user who is attempting to auto the message.
