XCLN: Only Default Store Deleted Items Get Deleted on Exit

 Last reviewed: April 2, 1997
Article ID: Q147294
The information in this article applies to:
  • Microsoft Exchange Windows 3.x client, versions 4.0 and 5.0
  • Microsoft Exchange Windows NT client, versions 4.0 and 5.0
  • Microsoft Exchange Windows 95 client, versions 4.0 and 5.0

SYMPTOMS

If a client has multiple Deleted Items folders, and the Empty 'Deleted Items' folder upon exiting option is selected, only the Deleted Items folder that exists on the default store is actually emptied.

CAUSE

This behavior is by design.

WORKAROUND

To empty a Deleted Items folder, right click it and select the Empty Folder option on the context menu. The Deleted Items folder will be purged immediately.

MORE INFORMATION

If a client has one or more local .PST files as well as a server mailbox, the Empty 'Deleted Items' folder upon exiting option affects only the Deleted Items folder on the store to which Exchange mail delivery is set. Therefore, if a client has its mail delivered to a particular local PST, only that .PST's Deleted Items folder will be purged. The messages that reside in Deleted Items folders on the server mailbox or other local .PSTs will remain when you exit.
