BUG: Add-in Causes Access Violation

 Last reviewed: October 10, 1997
Article ID: Q172068
The information in this article applies to:
  • Microsoft Transaction Server, version 1.0

SYMPTOMS

Microsoft Transaction Server provides an add-in for Visual Basic. However, it causes an access violation (AV) in Visual Basic 5.0 when you close one project and load another. This problem does not occur with Visual Basic 4.0.

CAUSE

The add-in was written to work with the add-in model of Visual Basic 4.0. This object model is not completely compatible with the add-in object model of Visual Basic 5.0.

WORKAROUND

Open Visual Basic 5.0's Add-in Manager and cancel the selection for the add- in titled "MTxServer Registry Refresh."

STATUS

Microsoft has confirmed this to be a problem in Microsoft Transaction Server version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

REFERENCES

For more information, visit the following Web sites:

  • http://support.microsoft.com/support/default.asp
  • http://www.microsoft.com/transaction
