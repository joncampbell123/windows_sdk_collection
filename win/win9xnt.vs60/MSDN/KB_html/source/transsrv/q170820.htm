BUG: Path May Prevent Creation of Client Installation Executable

 Last reviewed: October 10, 1997
Article ID: Q170820
The information in this article applies to:
  • Microsoft Transaction Server, version 1.0

SYMPTOMS

Microsoft Transaction Server Service Pack 2 introduced a new feature in its Package Export utility that creates a client installation executable file. This client installation executable file will not be created if the package name or path to the package contains spaces.

RESOLUTION

To resolve this problem, change the package name and path so that they do not contain spaces.

STATUS

Microsoft has confirmed this to be a problem in Transaction Server version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

REFERENCES

For more information, visit the Web sites at:

  • http://support.microsoft.com/support/default.asp
  • http://www.microsoft.com/transaction/
