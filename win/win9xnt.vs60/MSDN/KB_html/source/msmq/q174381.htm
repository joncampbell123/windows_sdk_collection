INFO: Using MS DTC Console to Administer MSMQ Transactions

 Last reviewed: January 28, 1998
Article ID: Q174381
The information in this article applies to:
  • Microsoft Message Queue Server version 1.0

SUMMARY

This article discusses how to use Microsoft Distributed Transaction Coordinator Admin Console to monitor Microsoft Message Queue (MSMQ) transactions.

MORE INFORMATION

MSMQ ships with Microsoft Distributed Transaction Coordinator Admin Console utility. This allows you to stop and restart MS DTC and monitor statistics for all transactions managed by DTC. Similar functionality is also available in Microsoft SQL Server Enterprise Manager and in Microsoft Transaction Server Explorer.

When testing MSMQ external transactions, the Statistics tab of DTC Admin Console provides very useful information on active, committed, and aborted transactions. You can stop and restart DTC service to recycle the count of transactions shown by the Admin Console.

MS DTC service must be running to perform external transaction, which also includes MTS (Microsoft Transaction Server) transactions.
