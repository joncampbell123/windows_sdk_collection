PRB: Borland Turbo Debugger Cannot Load WINDEBUG.DLL

 Last reviewed: February 15, 1996
Article ID: Q84844
The information in this article applies to:
  • Microsoft Windows Software Development Kit (SDK) for Windows version 3.1

SYMPTOMS

An attempt to run the Borland Turbo Debugger under Windows 3.1 generates a message stating that the WINDEBUG.DLL file is not found.

CAUSE

The Borland Turbo Debugger requires the WINDEBUG.DLL file to work with the Windows environment.

RESOLUTION

WINDBG is a file in the Software Library that contains the WINDEBUG.DLL. WINDBG is for registered owners of Borland language products that require the file to run the Borland Turbo Debugger under Windows 3.1. WINDBG may not be redistributed in any way.

Download WINDBG.EXE, a self-extracting file, from the Microsoft Software Library (MSL) on the following services:

  • Microsoft Download Service (MSDL) 

          Dial (206) 936-6735 to connect to MSDL
      Download WINDBG.EXE (size: 21115 bytes)
  • Internet (anonymous FTP) 

          ftp ftp.microsoft.com
      Change to the \softlib\mslfiles directory
      Get WINDBG.EXE (size: 21115 bytes)
