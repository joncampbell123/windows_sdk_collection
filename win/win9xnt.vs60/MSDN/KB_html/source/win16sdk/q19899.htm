Restoring Objects in a Metafile DC

 Last reviewed: July 22, 1997
Article ID: Q19899
3.00 3.10 WINDOWS kbprg

The information in this article applies to:

  • Microsoft Windows Software Development Kit (SDK) for Windows versions 3.0 and 3.1

A display context for a metafile does not have any default objects when it is created. When SelectObject() is passed the hDC of a metafile, SelectObject() will return 1 if the operation is successful and NULL if the operation fails.

Objects that have been selected into a metafile display context should be deleted directly by calling DeleteObject(hObject) after you have selected the object out of the DC. DeleteObject() is supported for Windows version 3.x metafiles only. Objects selected into other display contexts should be deleted by calling DeleteObject (SelectObject (hDC, hOldObject)).
