BUG:SEM Client Stops When Copying Preview Generate Sql Scripts

 Last reviewed: April 8, 1997
Article ID: Q153674

The information in this article applies to:
  • Microsoft SQL Server, version 6.5

SYMPTOMS

The 6.5 version of the SQL Enterprise Manager (SEM) client tool will stop responding on a Windows 95 computer when performing a copy (CTRL+C) of text in the preview screen of Generate SQL Scripts. Other applications can be accessed. The copied text can even be pasted into other applications like Notepad or Word.

This only happens with the 6.5 version of the client utilities on a Windows 95 machine. The 6.0 version of SEM does not stop responding on a Windows 95 computer. Also, this does not occur with SEM when copying text from the preview screen on a Windows NT 3.51 workstation or server.

WORKAROUND

If you are using the 6.5 client utilities on a Windows 95 computer, use the Save As button to save the script as a file. The text can then be copied using the CTRL+C or CTRL+INS keystrokes, without causing the application to stop responding.

STATUS

Microsoft has confirmed this to be a problem in Microsoft SQL Server version 6.5. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
