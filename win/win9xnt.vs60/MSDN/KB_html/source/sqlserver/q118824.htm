BUG: OS/2 Novell Clients Cannot Connect Without a File Server

 Last reviewed: April 29, 1997
Article ID: Q118824

The information in this article applies to: 

  - Microsoft SQL Server Programmer's Toolkit, version 4.2
BUG# 10073 (4.20.50)

SYMPTOMS

On a Novell network, OS/2 clients cannot connect to SQL Server for Windows NT if there is no Novell file server on the network. Attempting the connection in this scenario will return the message: 

   DB-Library: Unable to connect: SQL Server is unavailable or does not
   exist.
This error will occur if the connection was attempted using either the named pipes or the IPX/SPX net libraries.

WORKAROUND

The problem will not occur if a Novell file server is brought up on the network.

STATUS

Microsoft has confirmed this to be a problem in the DBMSSPXP Net Library versions 4.20.50 and 4.21. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
