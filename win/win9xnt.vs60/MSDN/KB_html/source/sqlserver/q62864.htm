INF: SQL Server Compatibility with OS/2 LAN Manager

 Last reviewed: April 25, 1997
Article ID: Q62864

The information in this article applies to:
  • Microsoft SQL Server version 4.2 for OS/2

Following is a list of rules that apply to running SQL Server and OS/2 LAN Manager:

  1. OS/2 LAN Manager version 2.0 requires OS/2 version 1.21 or later, and OS/2 version 1.21 requires OS/2 LAN Manager version 2.0 or later.

  2. Named pipes were changed in OS/2 LAN Manager version 2.0, which required a corresponding change in SQL Server DB-Library (DB-Lib). The corrected DB-Lib is available in the Software/Data Library in ONLINE, or in SQL Server version 1.1.

The following is a summary of the information: 

   OS/2       LAN Man        SQL Server         Interoperate?
   ----       -------        ----------         -------------

   OS/2 1.1   LM 1.1        SQL Server 1.0      y
   OS/2 1.1   LM 1.1        SQL Server 1.1      y
   OS/2 1.1   LM 2.0        SQL Server 1.0      n (rule 1)
   OS/2 1.1   LM 2.0        SQL Server 1.1      n (rule 1)
   OS/2 1.21  LM 1.1        SQL Server 1.0      n (rule 1)
   OS/2 1.21  LM 1.1        SQL Server 1.1      n (rule 1)
   OS/2 1.21  LM 2.0        SQL Server 1.0      n (rule 2)*
   OS/2 1.21  LM 2.0        SQL Server 1.1      y
   OS/2 1.1     -           SQL Server 1.0      y (SQL Server can run
   OS/2 1.1     -           SQL Server 1.1      y  stand-alone in any
   OS/2 1.21    -           SQL Server 1.0      y  version of OS/2 that
   OS/2 1.21    -           SQL Server 1.1      y  supports named pipes)
   OS/2 1.21  LM 2.0        SQL Server 1.11     y
   OS/2 1.3   LM 2.0        SQL Server 1.11     y
   OS/2 1.21  LM 2.0        SQL Server 4.2      y
   OS/2 1.3   LM 2.0        SQL Server 4.2      y
   OS/2 1.21  LM 2.1        SQL Server 1.11     y
   OS/2 1.3   LM 2.1        SQL Server 4.2      y
   OS/2 1.21  LM 2.1        SQL Server 1.11     y
   OS/2 1.3   LM 2.1        SQL Server 4.2      y


   * Note: Will interoperate if "fixed" DB-Lib from ONLINE is used in
     the client applications. No changes are required to SQL Server.
