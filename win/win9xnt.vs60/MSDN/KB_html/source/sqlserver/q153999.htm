BUG: Table Transfer Fails with Msg That Table Does Not Exist

 Last reviewed: April 9, 1997
Article ID: Q153999
The information in this article applies to:
  • Microsoft SQL Server version 6.5

SYMPTOMS

Sometimes when you use the transfer facility of SQL Enterprise Manager (SEM) to transfer a table to a different database, the transfer fails and the log file returns the following error: 

   Table does not exist.

WORKAROUND

To solve this problem, close SEM and run it again. The table should transfer successfully.

STATUS

Microsoft has confirmed this to be a problem in Microsoft SQL Server version 6.5. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The problem happens infrequently and only for tables created since the current instance of SEM was run.
