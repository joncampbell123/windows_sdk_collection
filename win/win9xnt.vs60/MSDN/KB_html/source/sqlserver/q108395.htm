INF: Update Statistics Causes Shared Table Lock

 Last reviewed: April 3, 1997
Article ID: Q108395

The information in this article applies to: 

  - Microsoft SQL Server version 4.2 for OS/2
  - Microsoft SQL Server version 4.2

Running the UPDATE STATISTICS command will cause a shared table lock to be placed on the table from which statistical information is being gathered. This will prevent data modifying queries (UPDATE, DELETE, INSERT) from being executed until the UPDATE STATISTICS completes. SELECT statements will execute normally.
