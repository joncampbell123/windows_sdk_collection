BUG: SQL Admin Query Window Does Not Timeout as Configured

 Last reviewed: April 29, 1997
Article ID: Q112833

The information in this article applies to: 

  - Microsoft SQL Server Programmer's Toolkit, version 4.2
BUG# 9903 (4.21)

SYMPTOMS

SQL Administrator provides a configuration option that allows the user to set the time-out for the query window. This option is provided under the menu Manage, Configure in the Microsoft SQL Administrator Configuration window within the timeouts group under the Query: single line edit.

Setting this configuration option may not have the desired effect in all time ranges. An example is provided below:

  1. Set the Query Configuration option for 1 minute.

  2. Open a Query window and execute the following command: 

          waitfor delay '01:00:00'

Under this configuration the user should be prompted, after 1 minute has transpired, with a message box stating that the query has timed out and be prompted to continue or cancel the operation. When tested this did not occur for the above scenario.

STATUS

Microsoft has confirmed this to be a problem in SQL Administrator for Windows NT and Windows version 4.21. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
