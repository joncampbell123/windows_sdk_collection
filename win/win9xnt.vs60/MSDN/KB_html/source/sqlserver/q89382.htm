BUG: Simultaneous Tape Dumps Cause SQL Server GP Fault

 Last reviewed: April 28, 1997
Article ID: Q89382

The information in this article applies to:
  • Microsoft SQL Server version 4.2 for OS/2
BUG# OS/2: 1352 (4.2)

SYMPTOMS

When the CONSOLE utility is used to dump a database to a tape dump device, the CONSOLE utility will prompt the user to remove the tape from the drive and press ENTER when the dump is complete. If another DUMP DATABASE or DUMP TRANSACTION command is issued to dump to that same tape device before ENTER is pressed to finish the previous dump, SQL Server may cause a general protection fault (GP fault) when ENTER is finally pressed.

CAUSE

SQL Server incorrectly handles subsequent dump requests when an initial dump request has not yet completed.

WORKAROUND

Before issuing a subsequent DUMP command, make sure that the first dump to the tape dump device is completed and ENTER is pressed in response to the final prompt from CONSOLE.

STATUS

Microsoft has confirmed this to be a problem in SQL Server version 4.2 for OS/2. We are researching this problem and will post new information here i the Microsoft Knowledge Base as it becomes available.
