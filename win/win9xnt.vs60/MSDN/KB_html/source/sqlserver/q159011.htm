BUG: LOAD HEADERONLY Does Not Report Spanned Dumps

 Last reviewed: March 28, 1997
Article ID: Q159011
The information in this article applies to:
  • Microsoft SQL Server, version 6.5
BUG #: 16197 (6.5)

SYMPTOMS

The DUMP DATABASE command can continue or span a dump across multiple tapes. If the LOAD HEADERONLY command is used to verify the dump, it will not report files on the second or following tapes.

STATUS

Microsoft has confirmed this to be a problem in Microsoft SQL Server version 6.5. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

For more information on LOAD HEADERONLY and dumping to multiple tapes, see the following article in the Microsoft Knowledge Base:
