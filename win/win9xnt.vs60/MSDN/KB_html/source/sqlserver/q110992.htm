BUG: Cannot Grant User Permissions When Logging in as DBO

 Last reviewed: April 29, 1997
Article ID: Q110992

The information in this article applies to:
  • Microsoft SQL Server Programmer's Toolkit, version 4.2
BUG# 9790 (4.2)

SYMPTOMS

Using SQL Administrator, when logging in as DBO or DBO alias to grant user permissions from the manage,database,user permissions list, you will receive error, 

   Only the system administrator may grant or revoke the create database
   permission
when checking any of the boxes in the permission list.

CAUSE

SQL Administrator does not handle granting permission as DBO or DBO alias to users correctly.

WORKAROUND

Use queries to grant the permissions; for example: 

   grant create table to userA

STATUS

Microsoft has confirmed this to be a problem in SQL Administrator version 4.2 for Windows and Windows NT. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
