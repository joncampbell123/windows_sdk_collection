BUG: Vines Clients Unresponsive When Querying ODS Apps

 Last reviewed: May 2, 1997
Article ID: Q152061
The information in this article applies to:
  • Microsoft SQL Server, version 4.21

SYMPTOMS

An Open Data Services (ODS) application fails to receive any queries from a Banyan Vines client. When the client application tries to send any queries, it may become unresponsive while waiting for the ODS application to respond.

Please note that this problem only occurs with ODS applications. It does not occur with SQL Server.

CAUSE

The server-side Net-Library, SSMSVINN.DLL, cannot successfully read the queries sent by the client under certain situations. The ODS waits on the Net-Library function, ConnectRead(), which in turn waits on VINES API vnssocketreceive(). The call to vnssocketreceive() never returns.

STATUS

Microsoft has confirmed this to be a problem in Microsoft SQL Server version 4.21. It does not occur with version 6.0 or 6.5. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
