PRB: SQL Server Error 1608 in Errorlog

 Last reviewed: April 25, 1997
Article ID: Q66936

The information in this article applies to: 

  - Microsoft SQL Server version 4.2 for OS/2
  - Microsoft SQL Server versions 4.2, 4.21, and 4.21a

SYMPTOMS

The following error occurred when the net cable was unplugged from a SQL Server client end while a SELECT query was being executed on a client machine: 

   Nov  8 1990  1:49PM kernel: Cannot read, host process disconnected:
        PC Client 36 spid: 1
   Nov  8 1990  1:49PM kernel: Cannot read, host process disconnected:
        PC Client 36 spid: 1
   Nov  8 1990  1:49PM server: Error: 1608, Severity: 21, State: 2
   Nov  8 1990  1:49PM server: Hard error encountered flushing network
        send buffer

CAUSE

In general, this error message is caused by the front end dying or otherwise going away without a clean close of its connection. In many cases, it is indicative of a network problem of some sort (such as the above instance, when the net cable is unplugged.)
