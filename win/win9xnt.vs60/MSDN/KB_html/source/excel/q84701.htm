Excel or Project: GP Fault in WIN87EM.DLL

 Last reviewed: October 7, 1997
Article ID: Q84701

The information in this article applies to:
  • Microsoft Excel for Windows, versions 4.0, 4.0a, 5.0
  • Microsoft Project for Windows, version 4.0

SUMMARY

Random problems may occur when you run Microsoft Excel for Microsoft Windows operating environment version 3.1 on a 386-based computer with an Intel i387 math coprocessor installed.

MORE INFORMATION

The following list is not all-inclusive. Since this problem is related to timing conditions, a wide and unpredictable variety of symptoms may be experienced. Following are some of the more common problems:

  • A General Protection Fault occurs in WIN87EM.DLL
  • When exiting Microsoft Excel, Windows exits to the MS-DOS prompt
  • Starting Microsoft Excel results in a General Protection Fault in an unknown module
  • Solver reports a hardware error with floating point calculations
  • Crosstabs fails in the middle of a calculation

These problems have all been traced to a timing problem between the WIN87EM.DLL and certain Intel i387 math coprocessors. There is a new version of WIN87EM.DLL that corrects this problem.

How to get the new version of this file

This file is available through any of these sources:

  • Intel bulletin board service at (503) 645-6275.
  • The Microsoft Download Service (425) 936-6735, in the APPNOTES Section.

To get to the APPNOTES section, choose the FILES section, the the WINDOWS section, then to APPNOTES.

The filename to download is: WIN87EM.EXE.

  • CompuServe, by doing the following:

    1. At the ! prompt, type "GO MSEXCEL" (without quotation marks).

    2. Choose LIBRARIES.

    3. Choose LIBRARY 3.

    4. Download WIN87EM.DLL.

  • As an appnote from Microsoft Product Support Services at (425) 635-7070.

The appnote (WW0548) is titled, "WIN87EM.DLL Patch for Intel 80387."
