Excel Err Msg: Help System Has Run Out of Memory

 Last reviewed: February 2, 1998
Article ID: Q86026

The information in this article applies to:

  • Microsoft Excel for the Macintosh, versions 4.0, 5.0

SUMMARY

When you add certain inits to your Macintosh system folder, you may receive the following error message: 

   The Microsoft Help system has run out of memory and cannot
   continue.
When you choose the OK button or press the RETURN key, you will receive the following error message: 

   Unable to display help.

MORE INFORMATION

You may receive one of the above Excel error messages with any of the following inits:

  • SAM Intercept

    NOTE: SAM versions 3.0.7 and 3.0.8 do not appear to create the problem.

  • Mobius Display System
  • StretchInit (Shareware)
  • DiskLock 2.1
  • QMS PaperHandler

For more information about SAM Intercept, contact Symantec Software Customer Service at (310) 449-4990 or Sales at (800) 441-7234.

For more information about Mobius Display System, contact Mobius Technologies (800) 669-0556.

The third-party contact information included in this article is provided to help you find the technical support you need. This contact information is subject to change without notice. Microsoft in no way guarantees the accuracy of this third-party contact information.

Inits may be located in the root level of the System 6.0.x System Folder. If you are using System 7.0 or 7.1, inits may be located in the Control Panels and Extensions folders within the System Folder, as well as in the root level.

For more information on troubleshooting system conflicts, query on the following words in the Microsoft Knowledge Base: 

   init and cdev and conflict
