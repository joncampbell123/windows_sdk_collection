MXL: Error Inserting Microsoft Excel Worksheet Object

 Last reviewed: February 2, 1998
Article ID: Q124668
4.00 5.00 MACINTOSH kbole kbinterop xlmac

The information in this article applies to:

  • Microsoft Excel for the Macintosh, versions 4.0, 5.0

SYMPTOMS

When you insert a Microsoft Excel Worksheet object in a client application, Microsoft Excel version 5.0 may start in the background, and you may receive the following error message: 

   The server application, source file, or item cannot be found. Make sure
   the application is properly installed, and that it has not been deleted,
   moved, or renamed.

CAUSE

This behavior occurs if you have Microsoft Excel versions 4.0 and 5.0 installed on your computer, and one of the following is true:

  • You installed version 4.0 most recently

    -or-

  • The Embedding Preferences file and the Registration Database file are missing from the Preferences folder in the System folder or these files are damaged.

This behavior occurs because Microsoft Excel version 4.0 and version 5.0 have the same creator type, and when a client application tries to start Microsoft Excel to insert or edit a worksheet object, it starts the most current version of Microsoft. When you install Microsoft Excel version 4.0, the Microsoft Excel Worksheet object is registered. When you install Microsoft Excel version 5.0 after installing version 4.0, the Microsoft Excel version 5.0 Worksheet object is registered, and the Microsoft Excel worksheet object entry is converted so that version 5.0 is used to edit and activate the object. If the Microsoft Excel version 4.0 registration information has not been "converted" by the Microsoft Excel version 5.0 Setup program, then the above symptoms occur when you insert a Microsoft Excel Worksheet object.

Note that if the Microsoft Excel version 5.0 information is not present in the Registration Database file, the information is not entered automatically when you start Microsoft Excel. You must run the Microsoft Excel version 5.0 Setup program again in order for these entries to be written correctly.

WORKAROUNDS

To work around this problem, run the Microsoft Excel version 5.0 Setup program again so that Microsoft Excel version 5.0 is registered correctly. When you insert a Microsoft Excel Worksheet object, a Microsoft Excel version 5.0 worksheet object is inserted.
