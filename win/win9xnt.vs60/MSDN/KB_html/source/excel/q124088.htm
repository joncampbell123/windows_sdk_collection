XL: GP Fault Opening Multiplan Files with External Links

 Last reviewed: April 25, 1997
Article ID: Q124088
5.00 5.00c 7.00 7.00a WINDOWS kbprb kbinterop

The information in this article applies to:

  • Microsoft Excel for Windows 95, versions 7.0, 7.0a
  • Microsoft Excel for Windows, versions 5.0, 5.0c
  • Multiplan versions 3.x, 4.2

SYMPTOMS

In the versions of Microsoft Excel mentioned above, if you open a Multiplan file that contains external links, your computer may appear to hang or the following error may be generated: 

   Excel has Caused  a General Protection Fault Error in Module
   XLCONVMP.DLL

CAUSE

This problem occurs when you open a Multiplan file containing external links. This problem occurs with Multiplan 3.x and 4.2 files.

NOTE: This problem does not occur if the file contains internal links and no external links.

WORKAROUNDS

To work around this problem, use any of the following methods.

Method 1. In Multiplan, remove the external links before you open the files 

          in Microsoft Excel version 5.0. After you open the files,
          reestablish the links in Microsoft Excel.
Method 2. In Multiplan, save the files in SYLK format. Open the file in 
          Microsoft Excel version 4.0, and save the file in Microsoft Excel
          version 4.0 format. The file can then be successfully opened into
          Microsoft Excel version 5.0.

          CAUTION: Do not open the SYLK formatted file directly in
          Microsoft Excel version 5.0. If you do this, groups of similar
          formulas may be converted into a single array formula that
          displays incorrect results.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
