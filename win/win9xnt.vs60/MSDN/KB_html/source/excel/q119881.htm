XL5: Opening Quattro Pro WQ1 File Causes GP Fault

 Last reviewed: August 16, 1996
Article ID: Q119881
The information in this article applies to:
  • Microsoft Excel for Windows, versions 5.0, 5.0c

SUMMARY

A file saved in Quattro Pro for DOS version 5.0 in a Quattro Pro WQ1 format will cause a general protection (GP) fault when you open it in Microsoft Excel for Windows.

MORE INFORMATION

Although Microsoft Excel 5.0 can convert files from the WQ1 format, there is an incompatibility in the format when the file is saved from Quattro Pro 5.0 for DOS. This conflict causes a GP fault when the file is opened in Microsoft Excel.

NOTE: The default format for Quattro Pro for DOS version 5.0 is WQ2.

A file saved in the WQ1 file format from Quattro Pro 4.0 for DOS or Quattro Pro 5.0 for Windows can be opened successfully in Microsoft Excel 5.0.

The following table lists the file formats that are available in Quattro Pro for DOS version 5.0 that can be opened in Microsoft Excel: 

     .WKS          Lotus 1-2-3 version 1a
     .WK1          Lotus 1-2-3 version 2.x
     .SLK          Multiplan
     .DBF          dBASE
     .WRK, .WR1    Symphony
Note: These file formats do not support multi-sheet files.

WORKAROUNDS

To work around this problem, follow the appropriate procedure below.

If you have a version of Quattro Pro earlier than 5.0 for DOS:

  1. Open the file into an earlier version of Quattro Pro, such as Quattro Pro 5.0 for Windows.

  2. Save the file in the WQ1 format or in a Lotus WK3 format. Close the file.

  3. Open the file in Microsoft Excel 5.0.

-or-

If an earlier version of Quattro Pro is unavailable, do the following:

  1. Using Quattro Pro 5.0 for DOS, save each sheet in the workbook in a Lotus WK1 format file. (Quattro Pro for DOS version 5.0 will not save in a WK3 format.)

  2. Open each WK1 sheet into Microsoft Excel, and drag the tabs for each sheet into one workbook.

  3. Save the file as a Microsoft Excel Workbook.
