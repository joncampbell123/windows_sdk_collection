Changes Are Not Saved to Original .DBF File in Q+E

 Last reviewed: October 31, 1994
Article ID: Q45958
Problem:

I opened a dBASE file, sorted it, and then closed it. Upon reopening the file, the information was not sorted. I was not prompted to save changes.

Response:

Q+E does not give a prompt to save any changes. Changes are not made to the original dBASE file if you sort, select records, or change the layout. Changes to the data occur only if Edit Allow Editing is selected. The changes in this case are saved when Edit Disallow Editing is selected.

To save the results of the changes made by the sort, selection, or layout changes, choose either File Save Results As or File Save Query As.
