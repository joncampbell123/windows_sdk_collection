MXL5 Err Msg: MS Excel Shared Resources Could Not Be Opened

 Last reviewed: March 13, 1998
Article ID: Q135003
The information in this article applies to:
  • Microsoft Excel for the Macintosh and Power Macintosh, versions 5.0, 5.0a

SYMPTOMS

If you are using Microsoft Excel version 5.0 for the Macintosh, and you double-click a Microsoft Excel file icon on the desktop, you may receive either of the following error messages:

  • The application program "MS Excel Shared Resources" could not be opened because an error of type -192 occurred. 

        -or-
  • The application program "MS Excel Shared Resources" could not be opened because an error of type -39 occured.

CAUSE

You may encounter this problem if both of the following conditions are true:

  • Microsoft Excel is not currently running. 

        -and-
  • There is a file called "MS Excel Shared Resources" on your hard disk.

This problem occurs because the desktop file tries to start "MS Excel Shared Resources" when you double-click the file's icon. "MS Excel Shared Resources" cannot be started because it is not a program.

RESOLUTION

To resolve this problem, follow these steps:

  1. Search the hard disk for a file called "MS Excel Shared Resources." More than likely, it will be in a folder called "Setup Temporary Items."

  2. When you find the file, delete it (drag the file into the Trash and then empty the Trash).

  3. Restart your Macintosh while holding down the COMMAND (Apple key) and OPTION keys until you receive a prompt to rebuild the Desktop. Choose OK to rebuild the Desktop file, and then release the COMMAND and OPTION keys.

MORE INFORMATION

When you double-click a file icon, the application that created the file should be started (if it's not already running) and the file should be opened.
