Wildcard Characters Cannot Be Used with Q+E's Select Find

 Last reviewed: October 31, 1994
Article ID: Q45887
Summary:

The DOS wildcard characters do not work with the Select Find command. The wildcard characters are the question mark (?), representing a position that any character can occupy, and the asterisk (*), representing a position that any character can occupy or any remaining positions in the string can occupy.

The dialog box that appears with the Select Find command finds an occurrence exactly matching the text string that is entered in the dialog. This string may be a substring of a longer string.
