SBFM: Accounting Data Files Don't Appear in Import Wizard

 Last reviewed: October 14, 1996
Article ID: Q157387
The information in this article applies to:
  • Microsoft Excel for Windows, version 7.0
  • Microsoft Small Business Financial Manager for Microsoft Excel, version 1.0

SYMPTOMS

In the Small Business Financial Manager for Microsoft Excel, accounting data files don't appear in step 3 of the Import Wizard.

CAUSE

This behavior occurs in either of the following cases.

The Wrong Data Folder Is Selected

  • In step 2 of the Import Wizard, you click "In specific directories."

    -and-

  • In "the following folders will be searched for accounting data" box, you specify a folder that does not contain the files.

The Driver Is Not Installed Correctly

The Small Business Financial Manager contains drivers (filters) to read the different accounting data file formats. To properly install these drivers, you must perform specific steps and place the files in the correct folders.

For additional information, please see the following articles in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q156663
   TITLE     : SBFM: Updated Accounting Program Drivers Available

RESOLUTION

When you select a folder to search for accounting data in the Import Wizard, use the following steps:

  1. Click "In specific directories."

  2. Select the drive that contains the accounting data.

  3. Click the folder that contains the accounting data. If the folder is contained within other folders, double-click the selected folder to "drill down" to folder that contains the data.

  4. With the folder selected in the box on the left, click "Add folder to list" to add the folder to "the following folders will be searched for accounting data" box on the right.

  5. Click Next to go to step 3.

The accounting data files should appear in the box in step 3.
