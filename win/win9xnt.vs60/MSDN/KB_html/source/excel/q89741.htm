Excel: Axes Inaccessible When Concealed by Other Axes

 Last reviewed: November 30, 1994
Article ID: Q89741
The information in this article applies to:
  • Microsoft Excel for Windows, versions 3.0, 4.0
  • Microsoft Excel for the Macintosh, versions 3.0, 4.0

SUMMARY

If one of an overlay's axes occupies the same position on the screen as one of the axes belonging to the main chart, that axis (either Axis 3 or Axis 4) may not be selected or manipulated until the main chart's axis is deactivated.

MORE INFORMATION

This error may occur when conditions are such that Axis 2 (the main X axis) and Axis 4 (the overlay's X axis) are located in the same position. It has also been observed to occur as a result of macro commands that relocated Axis 1 (the main Y axis) to the right-hand side of the chart, where Axis 3 (the overlay's Y axis) was already located. If a macro attempts to select or manipulate an axis that is "concealed" in this manner, a macro error results.
