MSQuery: Cannot Open Saved Query for Oracle Data Source

 Last reviewed: September 12, 1996
Article ID: Q139804
The information in this article applies to:
  • Microsoft Query for Windows, version 1.0

SYMPTOMS

In Microsoft Query, if you attempt to open a saved query that uses an Oracle data source, you may receive the following error message: 

   ORA-09301: osncon: local kernel only supported in standard mode

CAUSE

This error occurs if you attempt to open a query for an Oracle database and Microsoft Query has not yet logged you on to that data source.

WORKAROUND

To work around this problem, use the following steps to log on to the server before you open the query (*.qry) file:

  1. Start Microsoft Query.

  2. On the File menu, click New Query.

  3. Click your Oracle data source and click the Use button.

  4. You will be prompted to log in. Type your user id and password and click OK.

  5. Click Close in the Add Tables dialog box.

  6. On the File menu, click Close Query. If you are prompted to save the query, click No.

  7. On the File menu, click Open Query. Click your .qry file and click OK.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
