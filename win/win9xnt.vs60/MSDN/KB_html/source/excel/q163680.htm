SBFM97: 'Run-time Error 13' Using Buy vs. Lease Analysis

 Last reviewed: March 19, 1998
Article ID: Q163680
The information in this article applies to:
  • Microsoft Excel 97 for Windows
  • Microsoft Small Business Financial Manager 97 for Microsoft Excel 97

SYMPTOMS

When you attempt to perform a Buy vs. Lease analysis in the What-If Wizard in the Microsoft Small Business Financial Manager, the following error message occurs: 

   Run-time error '13':

   Type mismatch
When you click End, an hourglass appears. This hourglass does not disappear until you quit the Microsoft Excel program.

Note that you can still use the Microsoft Excel menus even though the pointer is an hourglass shape.

CAUSE

This problem occurs when you do either of the following in the Analyze Buy Versus Lease dialog box:

  • In the Canadian language version of the Microsoft Small Business Financial Manager, you use 0 (zero) as the estimated resale value. 

        -or-
  • The values you enter for "Length of Loan, Years" and "Payments per year" are both the value 1.

    NOTE: This problem occurs in both the U.S. English and Canadian language versions.

WORKAROUND

To work around this problem, use either of the following methods.

Method 1: Estimated Resale Value

Use a value of .0001 or another small value (other than zero). Doing this should not materially affect calculations in the analysis.

Method 2: Length of loan, Years and Payments per Year

Use a value that is greater than 1 for either (or both) of the values for "Length of Loan, Years" or "Payments per Year."

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem has been fixed in the updated versions of the Quickbooks driver and Msapp.dll.
