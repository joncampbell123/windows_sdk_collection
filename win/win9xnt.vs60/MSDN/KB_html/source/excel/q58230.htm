Excel: Unexpected Full or Fast Scrolling in Worksheet

 Last reviewed: November 2, 1994
Article ID: Q58230

SUMMARY

If a Microsoft Excel worksheet rapidly scrolls through many rows (full scrolling), as opposed to localized scrolling (row by row), the last cell on the worksheet is most likely set to a cell below the used area on the worksheet.

To delete unused rows and/or columns, do the following:

  1. From the Formula menu, choose Select Special. In the dialog box, choose Select Last Cell and click OK. This selects the blank last cell (that is, the incorrect last cell).

  2. Select all the rows from the row containing the blank last cell to the bottom row of the actual worksheet area. To easily do this, highlight the row containing the blank last cell, scroll to the last used row, and hold down the SHIFT key while clicking the row number for the row below the last used row in the worksheet.

  3. From the Edit menu, choose Delete.

  4. Select a cell in the active portion of the worksheet (for example, A1).

  5. Save the document.

  6. Close and reopen the document.

To verify that the last cell has been changed, choose Select Special from the Formula menu again.

Note: This procedure will not affect a worksheet if a formula is present that refers to a cell in an unused portion of the worksheet.
