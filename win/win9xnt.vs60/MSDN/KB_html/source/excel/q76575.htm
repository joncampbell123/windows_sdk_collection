Excel: Aliases in the Excel Startup Folder Do Not Open

 Last reviewed: November 29, 1994
Article ID: Q76575
The information in this article applies to:
  • Microsoft Excel for the Macintosh, versions 3.0, 4.0

SUMMARY

Upon startup, Microsoft Excel for the Macintosh do not open aliases that were created under System 7.0 and placed in the Excel Startup folder. If you place an alias in the Excel Startup folder, Excel displays the error message "Filename is Not Valid" and does not open the file.

To have a document open each time you start Excel, place the actual document, not its alias, in the Excel Startup folder, which is inside the Preferences folder of the System folder.

MORE INFORMATION
