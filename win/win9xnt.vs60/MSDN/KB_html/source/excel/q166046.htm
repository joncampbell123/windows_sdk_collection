SBFM97 ErrMsg: Illegal Operation Updating Quickbooks Data

 Last reviewed: February 12, 1998
Article ID: Q166046
The information in this article applies to:
  • Microsoft Excel 97 for Windows
  • Microsoft Small Business Financial Manager 97 for Microsoft Excel 97

SYMPTOMS

When you attempt to update your Quickbooks accounting data in the Small Business Financial Manager (SBFM) version 2.0, the following error message occurs: 

   This program has performed an illegal operation and will be shut down.
   If the problem persists, contact the program vendor.
If you click Details, the following error message appears: 

   EXCEL caused an exception e06d7363H in Module KERNEL32.DLL at
   0137:bff9a0c.
When you click Close, Microsoft Excel closes.

Note: the memory address (0137:bff9a0c ) may vary.

CAUSE

This behavior will occur when you do all of the following:

  • You import your accounting data

    - and -

  • You create a Financial Report such as a Trial Balance, then save and close the workbook

    - and -

  • You create a What-If Analysis, run an analysis such as Profitability and then save and close that workbook

    - and -

  • You update your accounting data

WORKAROUND

To keep avoid the illegal operation error do the following:

  1. Close Microsoft Excel 97 after you have saved and closed you analysis workbooks.

  2. Make any necessary updates in your accounting program.

  3. In the SBFM, Import your accounting data again. When the import is complete, click Cancel in Step 4 of the "Import your Accounting Data" dialog.

  4. On the File menu, click Open. Locate you workbook containing the saved SBFM scenario.

  5. In the What-If Overview workbook, click Save. This will update the analysis with the updated accounting information.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem has been fixed in the updated versions of the Quickbooks driver and Msapp.dll.
