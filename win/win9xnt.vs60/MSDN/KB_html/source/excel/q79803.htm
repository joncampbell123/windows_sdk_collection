Excel: Page Setup Orientation Doesn't Control Printer Fonts

 Last reviewed: November 4, 1994
Article ID: Q79803
The information in this article applies to:
  • Microsoft Excel for the Macintosh, versions 3.0, 4.0

SUMMARY

When you display the list of current printer fonts in Microsoft Excel, the fonts you see listed will be the fonts for the global orientation (landscape versus portrait), not the document-specific orientation, as would be expected.

MORE INFORMATION

You set document-specific settings in Microsoft Excel, with the Page Setup command in the File menu. Document-specific settings that differ from global settings (items that are set by using the Printer Setup command on the File menu) should take precedence in a Microsoft Excel worksheet.

You can display the list of current printer fonts in Microsoft Excel by choosing Font from the Format menu and selecting the Printer Fonts option. Any fonts that the printer is capable of printing in the current orientation are displayed in the Fonts box. This includes any hardware fonts, soft fonts, or fonts supplied by cartridges. However, Microsoft Excel incorrectly looks at the global orientation when displaying the list of fonts rather than the per document orientation.
