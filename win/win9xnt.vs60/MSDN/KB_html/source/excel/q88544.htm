XL: Print Reports Not Printing Correct Views

 Last reviewed: February 2, 1998
Article ID: Q88544

The information in this article applies to:
  • Microsoft Excel for Windows, versions 4.0, 4.0a, 5.0, 5.0c
  • Microsoft Excel for the Macintosh, versions 4.0, 5.0, 5.0a
  • Microsoft Excel for Windows 95, versions 7.0, 7.0a
  • Microsoft Excel 97 for Windows
  • Microsoft Excel 98 Macintosh Edition

SUMMARY

In Microsoft Excel, when you use the Report Manager to print a report, the wrong range may be printed if separate print areas have not been set for each view in the report.

WORKAROUND

To prevent this problem from occurring, set the print area before creating the custom view.

MORE INFORMATION

Microsoft Excel includes an add-in macro, called Report Manager (REPORT MANAGER for the Macintosh and REPORTS.XLA for Windows), that allows you to create a report from selected views of your worksheet. Report Manager uses views that you create with the View Manager/Custom Views add-in macro. To insure that your report prints correctly, you need to set the print area when you create each view. The Print Area must be set before defining a view, or the reports may print the wrong ranges.
