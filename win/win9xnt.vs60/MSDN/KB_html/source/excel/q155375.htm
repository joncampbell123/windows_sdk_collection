XL97: Unable to Rotate AutoShapes While in Edit Mode

 Last reviewed: February 18, 1998
Article ID: Q155375
The information in this article applies to:
  • Microsoft Excel 97 for Windows

SYMPTOMS

When you try to maneuver or position an AutoShape object, the Free Rotate toolbar button, and the "Rotate or Flip" and Nudge commands on the Draw menu may be disabled.

CAUSE

If you are working with an AutoShape object and you are in Edit mode (cursor is blinking within the object), the Free Rotate toolbar button and the "Rotate or Flip" and Nudge commands on the Draw menu will be disabled.

RESOLUTION

To be able to change the orientation of an AutoShape, you must not be in Edit mode, but the object still must be selected. Note that if the cursor is blinking within an AutoShape, you are in Edit mode. To be able to select an AutoShape object without going into Edit mode, follow these steps:

  1. If you are currently in Edit mode, click any cell within your worksheet.

  2. Place your mouse pointer on the border of the object, and click once to select the object.

    The Free Rotate toolbar button and the "Rotate or Flip" and Nudge commands on the Draw menu will now be available.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

When you rotate or flip an AutoShape object that already contains text, the text will not rotate or flip with the object. Also, some objects that you import into Microsoft Excel cannot be rotated or flipped. This is because they were not created within Microsoft Excel. You also cannot rotate or flip bitmaps that are pasted into your worksheet.

REFERENCES

For more information about rotating objects, click the Index tab in Microsoft Excel Help, type the following text 

   rotating objects
and then double-click the selected text to go to the "Rotate or flip drawing object" topic.
