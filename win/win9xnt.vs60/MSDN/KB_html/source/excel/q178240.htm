XL97: AutoCalculate Is Inconsistent with Hidden Rows or Columns

 Last reviewed: December 18, 1997
Article ID: Q178240
The information in this article applies to:
  • Microsoft Excel 97 for Windows
  • Microsoft Excel for Windows 95, versions 7.0, 7.0a

SYMPTOMS

In Microsoft Excel, the AutoCalculate feature may display a sum on the status bar that appears to be incorrect.

NOTE: The AutoCalculate allows you to view the total value for a selected range of cells in the status bar.

CAUSE

This problem occurs when the cell range you select contains cells that are in hidden rows. The AutoCalculate feature ignores values in selected cells that are in hidden rows. Note that AutoCalculate does not ignore values in selected cells that are in hidden columns.

WORKAROUND

To work around this problem, unhide the hidden rows. To do this, follow these steps:

  1. Select cells in the row above and the row below the hidden rows.

  2. On the Format menu, point to Row, and then click Unhide.

NOTE: If the first row of a worksheet is hidden, click Go To on the Edit menu. Type A1 in the Reference box, and click OK. Point to Row on the Format menu, and then click Unhide.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

REFERENCES

For more information about unhiding rows, click the Office Assistant, type "How do i unhide rows?," click Search, and then click to view "Display hidden rows or columns."

NOTE: If the Assistant is hidden, click the Office Assistant button on the Standard toolbar. If Microsoft Help is not installed on your computer, please see the following article in the Microsoft Knowledge Base:
