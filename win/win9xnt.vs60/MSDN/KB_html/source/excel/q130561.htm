XL: Data Map Toolbar Buttons Don't Release

 Last reviewed: May 21, 1997
Article ID: Q130561
7.00 97 WINDOWS kbole kbwwt 

 --------------------------------------------------------------------
The information in this article applies to:
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft Excel 97 for Windows --------------------------------------------------------------------

SYMPTOMS

In Microsoft Excel, when you edit a map on your worksheet using the buttons on the Data Map toolbar, the button may not release when you complete the edit; that is, when you click the button, it may remain depressed until you click another button on the Data Map toolbar. For example, when you click the Label button, a new label is added to your map each time you click the map.

CAUSE

This behavior is by design. The following buttons are "sticky" and remain selected until you click another button on the Data Map toolbar:

In Microsoft Excel 97

Select Objects Grabber Center Map Map Labels Add Text Custom Pin Map Show/Hide Microsoft Map Control

In Microsoft Excel 7.0

Select Objects Zoom In Zoom Out Grabber Map Labels Add Text Custom Pin Map Show/Hide Data Map Control

This may cause undesired behavior to occur when you click your map.

Note that this behavior occurs even if you deactivate the map object and select another object on your worksheet, and then reactivate the map. The button that was selected when you deactivated the object is selected when you activate the object again.

RESOLUTION

To "turn off" a map feature, such as the labeler, click another button on the Data Map toolbar, such as the Select Object button. Note that pressing the ESC key does not turn off the selected map editing button.

Note that the Show/Hide Data Map Control button is also "sticky." However, this button controls the display of the Data Map Control dialog box and does not interfere with editing objects on your map.
