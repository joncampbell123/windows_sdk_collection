SBFM97: Error Updating Accounting Data

 Last reviewed: February 12, 1998
Article ID: Q164747
The information in this article applies to:
  • Microsoft Excel 97 for Windows
  • Microsoft Small Business Financial Manager 97 for Microsoft Excel 97

SYMPTOMS

When you update accounting data in the Microsoft Small Business Financial Manager (SBFM) for Microsoft Excel, version 2.0, the following error message appears 

   Unable to update the database in <path> <filename.mdb>... the
   database is in use. Close all reports, and start again.
where <path> is the location of your SBFM accounting database and <filename.mdb> is the SBFM accounting database that is created by the SBFM.

CAUSE

The problem occurs when the following conditions are true:

  • You import an accounting dataset, click What-If Analysis, and then save and close the Overview. 

        -and-
  • You attempt to update the accounting dataset by clicking Update in the "Import your Accounting Data" dialog box.

WORKAROUND

To workaround this problem, do either of the following before you update the accounting data:

  • Close all open workbooks and quit Microsoft Excel. 

        -or-
  • In the What-If Analysis workbook, use a function, such as Profitability, that recalculates the workbook.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem has been fixed in the updated versions of the Quickbooks driver and Msapp.dll.
