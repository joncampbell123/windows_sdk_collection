Excel 2.20: Printing Colored Text to Mirus Spool File

 Last reviewed: October 31, 1994
Article ID: Q48160

SUMMARY

When a Microsoft Excel version 2.20 chart created with colored text is printed to a spool file using the Mirus Film Printer Driver, the text prints as black.

A chart created in Excel 2.20 with colored text that is then sent to a Mirus Spool File to be printed on the Mirus Film Recorder loses the color information for the fonts. Everything else transfers, but the font information is sent in such a way that the color information is not included.

This problem does not occur in Excel version 1.50 because it sends out information in a 72 dots-per-inch (dpi) format; that is, it sends out colored dots instead of colored font information.

Microsoft is researching this problem and will post new information here as it becomes available.
