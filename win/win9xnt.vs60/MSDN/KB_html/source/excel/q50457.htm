Excel 2.2: Date Incorrect When Opening a Text File

 Last reviewed: October 31, 1994
Article ID: Q50457

SUMMARY

If a Microsoft Excel text file is opened into Excel version 2.20 from the Finder (either by double-clicking on the file from the Finder or selecting the file and choosing Open from the Finder's File menu), all valid dates may be off by four years and one day. To display the correct date values, turn on the 1904 date system under Options Calculation.

The dates will be correct if the file was opened by choosing Open from Excel's File menu with a worksheet already open that is using the 1904 date system, or if the 1904 date system was not used when the file was created.

MORE INFORMATION

Worksheets in Excel can be created using either the Macintosh (1904-based) date system, or the PC (1900-based) date system. The date system should be selected before any dates are entered into the worksheet. When the worksheet is saved as Normal, Excel saves the preference of 1900 or 1904 date system.

When a worksheet is saved out as text, the document must contain only the ASCII information stored within the file. Thus, preferences such as the default date system cannot be saved in a text file. When Excel opens a text file, it uses the date system preference of the currently active document. If a document has not already been open using the 1904 date system, Excel will default to the 1900 date system when opening a text document. Therefore, if the document was created with the 1904 date system selected (default), it will open with all dates off by four years and one day when opened without the 1904 flag set.
