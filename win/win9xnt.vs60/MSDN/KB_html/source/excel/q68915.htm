FaceLift Fonts Not Printing Using Deskjet or Laserjet III

 Last reviewed: November 1, 1994
Article ID: Q68915
The information in this article applies to:
  • Microsoft Excel for Windows, version 3.0 and 4.0

SUMMARY

You may experience a problem when printing from Microsoft Excel 3.00 using any fonts created with FaceLift on the Hewlett-Packard (HP) DeskJet, DeskJet Plus or LaserJet III printers. The output displays correctly when you select Print Preview to view the document, but it will not print. Check to ensure that the following line "prtresfac=0 " is entered into the corresponding printer section of the WIN.INI file.

Example of the WIN.INI Listing for the DeskJet

[DESKJET,LPT1] prtresfac=0 tray=1 FontSummary=C:\WIN3\SYSTEM\FSLPT1.

NOTE: prtresfac=0 is case sensitive and must be entered in lowercase.

MORE INFORMATION

Microsoft has confirmed this to be a problem with the HP DeskJet Driver version A.03.00 and the HP LaserJet Series III Driver version 3.42. We are researching this problem and will post new information here as it becomes available.
