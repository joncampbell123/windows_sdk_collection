XL: ODBC Icon and Data Access Files Not Removed on Uninstall

 Last reviewed: June 3, 1997
Article ID: Q108816
5.00 7.00 97 WINDOWS kbusage kbsetup

The information in this article applies to:

  • Microsoft Excel for Windows, version 5.0
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft Excel 97 for Windows

SYMPTOMS

When you run the Microsoft Excel Setup program and choose to remove the Data Access components, the open database connectivity (ODBC) icon is not removed from the Control Panel, and the ODBC files and data access files are not removed.

CAUSE

Because the ODBC files are shared files and other applications may use them, they are not removed by Microsoft Excel Setup. ODBC files are located in the Windows directory and in the System subdirectory of the Windows directory.

MORE INFORMATION

When you choose the Data Access option in Microsoft Excel Setup, and choose Options, the following individual components are listed: 

   Microsoft Query
   Help For Microsoft Query
   Microsoft Query Cue Cards
   Access Driver
   dBASE Driver
   FoxPro Driver
   Paradox Driver
   SQL Server Driver
When you run Microsoft Excel Setup, and choose the Data Access option, all of these components are installed, along with ODBC files and an ODBC Control Panel icon. The ODBC Control Panel icon allows you to add, modify, or delete ODBC drivers and data sources.

When you run Microsoft Excel Setup, and choose to remove the entire Data Access option, the Microsoft Query files are removed, but the ODBC icon, ODBC files, and the data access driver files are not removed. However, when you run Microsoft Excel Setup, after removing the entire Data Access option, the Data Access component options are clear, indicating that the component has been removed, unless it was installed by another Setup program.

When you remove an individual data access driver, such as the Paradox driver, the Paradox driver files are not removed. In this case, when you run Microsoft Excel Setup again, the Paradox Driver option is not clear, indicating that it has not been removed.

For additional information, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q134745
   TITLE     : OFF: Add/Remove Does not Remove Shared Applications
