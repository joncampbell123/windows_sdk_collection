XL98: Text in AutoShapes Is Not Rotated

 Last reviewed: February 26, 1998
Article ID: Q181657
The information in this article applies to:
  • Microsoft Excel 98 Macintosh Edition

SYMPTOMS

In Microsoft Excel 98 Macintosh Edition, if you create an AutoShape that contains text, and then rotate the AutoShape, the text is not rotated with the AutoShape.

CAUSE

This behavior occurs because Microsoft Excel 98 has a limited capacity to rotate text in an AutoShape.

WORKAROUND

To work around this behavior, try changing the orientation of the text in the AutoShape. To do this, use the following steps:

  1. Select the AutoShape.

  2. On the Format menu, click AutoShape.

  3. Click the Alignment tab.

  4. In the Orientation box, click the text orientation you want for the AutoShape and click OK.

NOTE: The text in the AutoShape may not appear in the center. If you experiment with the text and alignment settings in the Alignment tab you may be able to center the text in the AutoShape.

STATUS

This behavior is by design of Microsoft Excel 98.

MORE INFORMATION

In Microsoft Excel 98, an AutoShape is a drawing shape that you can insert into a worksheet. You can also attach text to an AutoShape. When you attach text to an AutoShape, it is similar to a text box in function.

Unlike text boxes, you can rotate an AutoShape to any angle. If you rotate an AutoShape, text in the AutoShape is not rotated. Instead, text is repositioned with respect to the AutoShape. In some cases, this positioning causes the text to be moved outside of the AutoShape, which makes the text appear that it is not part of the AutoShape.
