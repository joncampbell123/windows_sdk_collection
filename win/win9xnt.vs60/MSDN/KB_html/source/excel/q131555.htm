XL7: Underline Font Formatting Incorrectly Applied

 Last reviewed: September 13, 1996
Article ID: Q131555
The information in this article applies to:
  • Microsoft Excel for Windows 95, version 7.0

SYMPTOMS

In Microsoft Excel, when you use the Format Cells dialog box to format a cell on your worksheet, if you click the Underline drop-down list arrow on the Font tab to view the Underline formatting items, an incorrect underline format may be applied to the selected cell.

CAUSE

This behavior occurs if you move the mouse pointer over the items in the Underline list, and then cancel the list by clicking the dialog box. When you move the mouse pointer over items in the Underline drop-down list, the items are highlighted. If you click another location on the dialog box, the list is closed; however, when you do this, the last item that was highlighted in the list is selected (the expected behavior is that the selected item is canceled and the previously selected item is displayed). When this occurs, the wrong underline formatting may be applied to a cell on your worksheet.

An item in the drop-down list should not be selected unless you click in the drop-down box. Additionally, if the Normal Font check box is selected in the Format Cells dialog box, this check box is not cleared when an Underline formatting item is selected in the manner described above.

RESOLUTION

To work around this problem, when an incorrect underline formatting is applied to a cell on your worksheet, use the Format Cells dialog box to apply the correct underline formatting. Note that you can also click Undo Format Cells on the Edit menu (if you have not performed any other actions in Microsoft Excel).

To avoid this problem when you click the Underline drop-down arrow, be sure to select the correct item before you cancel the drop-down list when you move the mouse over any of the items in the Underline list.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
