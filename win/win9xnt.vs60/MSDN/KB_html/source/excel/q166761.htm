XL97: Created Date Is Not Updated When Saving as a New File

 Last reviewed: February 12, 1998
Article ID: Q166761
The information in this article applies to:
  • Microsoft Excel 97 for Windows

SYMPTOMS

When you click the Statistics tab in the OLE Properties dialog box, the Created date for a workbook may display an incorrect or older date than you expected.

CAUSE

This problem occurs when you save an existing file with a new file name. The new file does not update the Created date value.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

You can enter property information for your files so that it is easier to find the files and information about them. Properties can include a descriptive title, the name of the author, the subject, and keywords that identify topics or other important information. You can also create your own custom properties and link a custom property to specific content in the file. Statistics, such as file size and the dates on which files are created and last modified, are automatically maintained for you.
