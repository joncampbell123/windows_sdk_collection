Excel: Picture Prints Incorrectly in PageMaker 4.01

 Last reviewed: November 2, 1994
Article ID: Q70975

SUMMARY

If a picture (graphic) of a Microsoft Excel worksheet is created, either by choosing Copy Picture from the Edit menu or by using the Camera tool in Excel version 3.00, and the picture is pasted into Aldus PageMaker version 4.01, the picture may not print as expected.

Once in PageMaker, the text in the graphic may flow into adjacent cells, whether or not there is information in the other cells. This may result in text printing over the top of the text in adjacent cells.

To avoid this printing problem, hold down the OPTION key when choosing Print from the File menu in PageMaker. This forces PageMaker to use the Apple printer drivers, and the Excel graphic will print properly.

Aldus is researching this problem. We will post new information here as it becomes available. For more information about this problem, call Aldus Technical Support at (206) 628-2040.
