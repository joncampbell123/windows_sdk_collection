XL97: Errors Appear When Opening Web Query File on FTP Server

 Last reviewed: January 15, 1998
Article ID: Q157309
The information in this article applies to:
  • Microsoft Excel 97 for Windows

SYMPTOMS

In Microsoft Excel 97, when you run a Web query, you may receive the following error messages: 

   EXCEL caused an invalid page fault in module EXCEL.EXE at
   014f:303d4818.

   -and-

   EXCEL caused an invalid page fault in module KERNEL32.DLL at
   014f:bff9a07c.

   -and-

   NETSCAPE caused an invalid page fault in module USER.EXE at
   0007:0000177c.
NOTE: The above error messages appear in the order given.

CAUSE

This behavior occurs when the following conditions are all true:

  • You are using a computer on which you installed the NCompass ScriptActive plug-in. 

        -and-
  • You view an embedded Microsoft Excel 97 workbook with Netscape Navigator 3.0. 

        -and-
  • You attempt to run a Web query, where the query .iqy file is stored on a File Transfer Protocol (FTP) server.

NOTE: This problem does not occur when you use Microsoft Internet Explorer version 3.0. When you use Microsoft Internet Explorer 3.0, you can view embedded Microsoft Excel 97 workbooks and run Web queries without receiving any errors, even if the Web query files are stored on an FTP server.

WORKAROUND

If you use Netscape Navigator 3.0 to view embedded Microsoft Excel 97 workbooks, do not run a Web query file that is stored on an FTP server.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The NCompass ScriptActive plug-in for Netscape Navigator 3.0 allows you to view Microsoft Excel workbooks (and other types of documents) directly from Netscape Navigator.
