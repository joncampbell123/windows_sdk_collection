MSQuery: Can't Escape Query Using dBASE Data Source

 Last reviewed: September 12, 1996
Article ID: Q121597
The information in this article applies to:
  • Microsoft Query for Windows, version 1.0
  • Microsoft Query for Windows 95, version 2.0

SYMPTOMS

In Microsoft Query, when you create a query using a dBASE data source, you cannot stop the query by pressing ESC.

CAUSE

This behavior occurs whenever you perform an action in your query that runs an SQL statement (for example, when you add criteria to a query or when you add a field to the data pane).

When you perform an action in your query, the hourglass appears, and you cannot perform another action until the hourglass disappears. Normally, you can stop the query, for example if you used an incorrect criteria, by pressing ESC. However, when you perform one of these actions in a query that you created using a dBASE data source, you cannot break out of the query by pressing ESC.

Note that there are some actions that you can cancel in a query that you created using a dBASE data source. For example, when you press CTRL+END to select the last record in the query, you can cancel this action by pressing ESC.
