XL5: GP Fault or Unexpected Results Using Object Dialog Box

 Last reviewed: September 12, 1996
Article ID: Q121326
The information in this article applies to:
  • Microsoft Excel for Windows, version 5.0

SYMPTOMS

In Microsoft Excel, when you use the Object dialog box to insert an object from a new file, using the Create New tab, one of the following symptoms occurs:

  • The client application starts, and it then closes without giving you a chance to edit the object. Additionally, the object is inserted on your worksheet as a picture, and if you double-click the object, the Format Object dialog box is displayed.

    -or-

  • The client application starts, and you receive a general protection (GP) fault.

CAUSE

This behavior occurs when the following conditions are true:

  • You use the Create New tab in the Object dialog box to create the object.

    -and-

  • The Link To File check box is selected on the Create From File tab.

If the Display As Icon check box is selected on the Create New tab under the above conditions, the client application starts as expected, and then closes without giving you a chance to edit the object. The object is inserted as a picture instead of as an embedded object.

If the Display As Icon check box (on the Create New tab) is not selected, under the above conditions, the client application starts as expected, but you receive a GP fault in Microsoft Excel.

WORKAROUND

To avoid this behavior when you insert an object using the Create New tab in the Object dialog box, make sure that the Link To File check box on the Create From File tab is not selected.

When you use the Create New tab in the Object dialog box, you do not need to use the check boxes on the Create From File tab, and vice versa.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft Excel version 5.0c for Windows.
