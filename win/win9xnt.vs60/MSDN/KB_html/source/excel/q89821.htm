Excel: Print Report Incorrectly Numbers Pages

 Last reviewed: November 30, 1994
Article ID: Q89821
The information in this article applies to:
  • Microsoft Excel for Windows, version 4.0

Summary:

In version 4.0 of Microsoft Excel for Windows, Reports Manager may number pages incorrectly when printing a multiple-page report with the Continuous Page Numbers option selected. This results from a problem with GET.DOCUMENT(50) which returns an incorrect result if the Print Area for a view covers only a portion of the worksheet.

For additional information on GET.DOCUMENT(50), query on the following words in the Microsoft Knowledge Base: 

   get.document(50) returns incorrect result
Microsoft has confirmed this to be a problem in version 4.0 of Microsoft Excel for Windows. We are researching this problem and will post new information here as it becomes available.

This is corrected in Microsoft Excel 4.0a which is now available.
