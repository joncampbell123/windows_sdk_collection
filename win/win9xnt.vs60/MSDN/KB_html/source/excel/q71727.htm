FLATFILE.XLA Does Not Retain 1904 Date Format in Excel

 Last reviewed: November 2, 1994
Article ID: Q71727

SUMMARY

FLATFILE.XLA is an add-in library macro that allows you to import or export data between "flat" text files (files in which data is not separated by tabs). If you use FLATFILE.XLA to export information from a spreadsheet that utilizes the 1904 date format option, you will find that the dates in the text file created are four years earlier than the dates in the original spreadsheet.

MORE INFORMATION

FLATFILE.XLA does not recognize that the spreadsheet you are exporting from uses the 1904 date system. It will display dates based on the normal 1900 date system in the destination text file.

Steps to Reproduce Problem

  1. From the Options menu, choose Calculation.

  2. Select the 1904 date System option and choose OK.

  3. Enter "1/1/90" (without the quotation marks) in cell A1.

  4. Open FLATFILE.XLA from the Library directory.

  5. Select cell A1 and choose Export from the Data menu.

  6. Enter a filename when prompted and choose OK.

  7. Open the file in Excel. The date will now display as "1/1/86."

Workaround

Either remove the 1904 date system option from your spreadsheet and update your dates or add four years to every date that you export.
