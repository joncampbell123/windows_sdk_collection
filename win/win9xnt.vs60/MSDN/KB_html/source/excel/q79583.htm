Excel: Relative Names Convert Incorrectly from Excel 2.2

 Last reviewed: November 1, 1994
Article ID: Q79583

SUMMARY

Names defined as relative external references do not convert properly when files created in Microsoft Excel for the Macintosh version 2.2 are opened in Excel version 3.0. The name will appear in the Define Name dialog, but the cell address given in the "Refers to:" field will always be relative to cell A1, not the active cell.

Workaround:

The following steps will redefine relative external names to their correct references:

  1. Make A1 the active cell on the dependent document.
2. From the Formula menu, choose Define Names. 3. Choose the defined name that contains the relative external 
   reference.
4. Click anywhere in the formula contained in the "Refers to:" 
   field.
5. Click OK or Add.

The relative reference will be reentered for the defined name and will behave normally (will be relative to the active cell). All references to the defined name on the spreadsheet will update to their correct relative external references.

Important: Cell A1 must be the active cell on the spreadsheet to use the above method. The formula contained in "Refers to:" after conversion to Excel 3.0 will be the proper external reference relative to cell A1 on the dependent document. If the active cell is not currently A1, the formula in "Refers to:" must be edited to reflect a correct external reference relative to the active cell on the worksheet.
