Using WIN32S Windows Sockets Applications with TCP/IP-32

 Last reviewed: September 9, 1996
Article ID: Q119369
The information in this article applies to:
  • Microsoft TCP/IP-32 for Windows for Workgroups, version 3.11
  • Microsoft Windows for Workgroups version 3.11

SUMMARY

Windows Sockets applications that use the WIN32S API (such as the Trumpet WinSock Point to Point Protocol [PPP]) do not function properly under Windows for Workgroups version 3.11 with the Microsoft TCP/IP-32 VxD. In order for these applications (for example, NCSA's Mosaic Alpha 3) to operate properly with TCP/IP-32, an updated version of WIN32S is required.

MORE INFORMATION

Windows Sockets applications that use WIN32S have memory locking issues with Win32S and the TCP/IP-32 VxD that are corrected in version 115a or later of the WIN32S distribution. The updated distribution of WIN32S can be found on the Internet. The file is located in the following path: 

   ftp://ftp.microsoft.com/SoftLib/MSLFiles/PW1118.EXE
