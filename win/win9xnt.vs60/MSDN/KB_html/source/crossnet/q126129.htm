Err Msg Installing IntelliType: Unable to show Program...

 Last reviewed: September 9, 1996
Article ID: Q126129
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1
  • Microsoft Windows NT Workstation version 3.5
  • Microsoft Windows NT Server version 3.5
  • Microsoft Natural Keyboard and IntelliType software version 1.0

SYMPTOMS

When you install IntelliType software under Windows NT version 3.1 or 3.5, the following error appears: 

   Unable to show Program Manager group: 'Microsoft Input Devices'
When you choose OK, the following message appears: 

   Setup failed! Contact Microsoft Produce Support Services for help.

CAUSE

This problem occurs when you have 40 common Program Manager groups, and the IntelliType software attempts to add one more common group. Windows NT Program Manager does not support more than 40 common groups and 40 personal groups.

WORKAROUND

To work around this problem:

  • Delete one or more of the existing common groups and re-install the IntelliType software.

    -or-

  • Delete one or more of common groups and manually create a Common Group called Microsoft Input Devices. Manually create a program item for \MSINPUT\KEYBOARD\KBDX32.EXE. Manually create another program item for %SYSTEMROOT%\TASKSW32.EXE.

    -or-

  • Manually create programs items for \MSINPUT\KEYBOARD\KBDX32.EXE and %SYSTEMROOT%\TASKSW32.EXE in an existing program group.

STATUS

Microsoft has confirmed this to be a problem in IntelliType version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
