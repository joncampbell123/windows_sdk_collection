Security Options Not Functional in Server Tools for Windows 95

 Last reviewed: September 9, 1996
Article ID: Q137859
The information in this article applies to:
  • Microsoft Windows NT Resource Kit version 3.51
  • Microsoft Windows 95

SYMPTOMS

After you install Windows NT Server Tools for Windows 95, included in the Windows NT Resource Kit version 3.51, and view the Properties of a Windows NT NTFS share, Security options, such as Permissions, Auditing, and Ownership, are not functional.

RESOLUTION

To correct this problem:

  1. Run Notepad and open the AUTOEXEC.BAT file.

  2. Add the Server Tools directory (SRVTOOLS) to the PATH.

  3. Save the file and quit Notepad.

  4. Restart Windows 95.
