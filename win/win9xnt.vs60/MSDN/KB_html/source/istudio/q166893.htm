BUG: Hang Previewing OLE Document in Web Project with Infoviewer

 Last reviewed: December 11, 1997
Article ID: Q166893
The information in this article applies to:
  • Microsoft Visual InterDev, version 1.0

SYMPTOMS

When previewing an OLE document, such as a Word Document or Excel Worksheet, in a Web project with Infoviewer, the document may hang the Visual InterDev Shell or you may experience screen painting problems.

CAUSE

This is a known problem with Infoviewer, Mshtml.dll, version 4.70.1215.

RESOLUTION

Any one or combination of the three possible workarounds listed below to resolve the problem:

  • Use a browser other than InfoViewer.
  • Save the OLE document object outside of Visual InterDev. You must then open the OLE document with any valid container besides Visual InterDev.
  • For the screen painting problems described above, resizing the "InfoViewer Topic" window correctly repaints the screen.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. We are researching this bug and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Open Visual InterDev 1.0.

  2. From the File menu, click New, Projects, and Web Project Wizard.

  3. Create a valid Visual InterDev Web project.

  4. From the File menu, click New, and Other Documents.

  5. Select any OLE document (such as Word, Excel).

  6. Click OK.

  7. Right-click the OLE document in the Project pane.

  8. Select Browse With.

  9. Select Infoviewer.

  10. Click Set as Default Browser.

  11. Click OK.

  12. Right-click the OLE document in the Project pane.

  13. Select Preview in Browser.
