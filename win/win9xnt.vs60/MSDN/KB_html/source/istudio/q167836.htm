FIX: DataForm Wizard Not Sizing Column Width Properly for DBCS

 Last reviewed: December 18, 1997
Article ID: Q167836
The information in this article applies to:
  • Microsoft Visual InterDev, version 1.0

SYMPTOMS

The following display problem occurs after inserting new records into a table that contains field names created with Double Byte Character Set (DBCS) characters: 

   The "Feedback" form confirming the new record will have a "Field" column
   of width 1 thereby compressing the field names to an unreadable size.

RESOLUTION

  1. Open <xxxAction.asp>.

  2. Move to the bottom of the page.

  3. Find the following line (the line looks like this): 

          <TD  ALIGN=Left  BGCOLOR=Silver><FONT SIZE=-1><B>
      &nbsp;&nbsp;Field</b></FONT></TD>

    Change it to the following: 

          <TD  WIDTH=30%  ALIGN=Left  BGCOLOR=Silver><FONT SIZE=-1><B> ....
      </b></FONT></TD>

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. This bug has been corrected in Visual Studio 97 Service Pack 1.

For additional information about the Visual Studio 97 Service Pack 1, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q170365
   TITLE     : INFO: Visual Studio 97 Service Packs - What, Where, and Why

MORE INFORMATION

Steps to Reproduce Behavior

  1. Start Visual InterDev.
2. Create a Web project. 3. Add a data connection to a database that includes DBCS named fields. 4. On the File menu, choose New. Choose File Wizards, and then click the 
   DataFormWizard tab. The following are necessary items on the Wizard:

   Step 1: 'Database that included DBCS named fields'
   Step 3: 'DBCS fields'
   Step 4: 'Insert New Records' and 'Return a feedback page after an
            update'

  • Finish the DataForm Wizard. 6. Browse xxForm.asp and click the New button. 7. Add valid record information and click the Insert button.

    Note the result—the field column has only one line width, so field names are compressed to an unreadable size.
