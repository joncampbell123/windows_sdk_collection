PRB: Uninstalling FrontPage 97 Causes Visual InterDev Error

 Last reviewed: December 9, 1997
Article ID: Q166675
The information in this article applies to:
  • Microsoft Visual InterDev, version 1.0

SYMPTOMS

It is not possible to create new or access existing Web projects on a Web server using the Visual InterDev Client, after Microsoft FrontPage 97 has been uninstalled from that Web server using Add/Remove Programs in the Control Panel.

The following error is displayed when a new project is created with the Web project wizard or an existing Web project is opened on the Web server in question: 

   Server error: Cannot open "temp\frontpg.lck": no such file of directory.

CAUSE

The problem is that by uninstalling Microsoft Front Page 97, the FrontPage Server Extensions are removed along with the FrontPage Editor and Explorer. The server extensions are shared between the FrontPage and Visual InterDev project model. Without the FrontPage Extensions, you will not be able to create or open a project on the IIS Server.

RESOLUTION

Re-install the FrontPage Server Extensions under the Server Components from the Visual InterDev CD-ROM.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Install the Visual InterDev Server components on a Web server.

  2. Install Microsoft FrontPage 97 on the same Web server.

  3. Uninstall Microsoft FrontPage 97 through Add/Remove programs in the Control Panel and select the Remove All option. It does not matter whether shared DLLs are removed or not.

  4. Launch the Visual InterDev Client and attempt to create a Web Project on this Web server.
