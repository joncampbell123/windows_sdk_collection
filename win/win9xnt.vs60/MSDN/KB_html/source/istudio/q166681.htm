BUG: Visual Basic 4 Custom Controls Return Error After install

 Last reviewed: December 11, 1997
Article ID: Q166681
The information in this article applies to:
  • Microsoft Visual InterDev, version 1.0

SYMPTOMS

After installing Visual InterDev 1.0, when selecting Custom Controls from the Visual Basic 4.0 Tools menu, you receive the following error several times: 

   Object server not correctly registered.

CAUSE

Visual InterDev causes this problem by registering some of the files it installs in a way that is incompatible with Visual Basic 4.0. This problem does not harm the machine, Visual InterDev, or Visual Basic.

RESOLUTION

You may download a RegClean tool provided by Visual Basic group. After running this registry cleaning tool the problem will not occur. You may obtain more information on this tool at http://www.microsoft.com/kb/articles/q147/7/69.htm. The self-extracting file on the Microsoft Software Library is called RegCln41.exe.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. We are researching this bug and will post new information here in the Microsoft Knowledge Base as it becomes available.

Steps to Reproduce Behavior

  1. Install Visual InterDev 1.0.

  2. Install Visual Basic 4.0.

  3. Open Visual Basic.

  4. From the Tools menu, select Custom Controls.

REFERENCES

The following article in the Microsoft Knowledge Base describes the registry clean update tool from the Visual Basic group for cleaning the registry: 

   ARTICLE-ID: Q147769
   TITLE     : "Updated Version of RegClean 4.1 Available"
For the latest Knowledge Base articles and other support information on Visual InterDev and Active Server Pages, see the following page on the Microsoft Technical Support site: 

   http://support.microsoft.com/support/vinterdev/
