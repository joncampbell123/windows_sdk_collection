INFO: Programmer's Guide to Using ADO in Visual InterDev

 Last reviewed: December 11, 1997
Article ID: Q172589
The information in this article applies to:
  • Microsoft Visual InterDev, version 1.0

SUMMARY

This article presents information useful to anyone wanting to take advantage of the ActiveX Data Objects (ADO) within Visual InterDev. The following topics are covered:

  • How to find ADO Knowledge Base Articles
  • ADO Resources / Points of Information

MORE INFORMATION

How to Find ADO Knowledge Base Articles

ADO is not a Visual InterDev or Active Server Pages (ASP) specific technology. ADO is also used with other products such as Visual Basic and Visual C++. Therefore, ADO is treated as its own product in the Microsoft Knowledge Base. To find ADO KB articles on the Web, you would connect to the following site: 

   http://www.microsoft.com/kb/default.asp
Then choose "ActiveX Data Objects" as the product.

For the MSDN or other non-Web-based Knowledge Base search mechanisms, you can use the keyword adobj, which all ADO articles contain. Other keywords that offer greater granularity when searching are documented in the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q168337
   TITLE     : HOWTO: Searching for ADO Articles by KBSubcategory

ADO Resources / Points of Information

ADO originally shipped with the OLE-DB SDK, so you will want to check both the OLE-DB Web page as well as the ADO Web site. 

   http://www.microsoft.com/ado/
   http://www.microsoft.com/oledb/

REFERENCES

For the latest Knowledge Base articles and other support information on Visual InterDev and Active Server Pages, see the following page on the Microsoft Technical Support site: 

   http://support.microsoft.com/support/vinterdev/
