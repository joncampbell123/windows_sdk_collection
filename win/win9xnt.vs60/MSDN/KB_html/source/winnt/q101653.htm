File Manager Uses Wrong Drive Letter in Net Connection

 Last reviewed: May 7, 1997
Article ID: Q101653

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you connect to a network share in a Command Prompt and connect to another network share in File Manager, File Manager uses the same drive letter.

CAUSE

File Manager does not automatically refresh the list of drive letters in use.

RESOLUTION

Before making a network connection, press F5 so File Manager refreshes its directory and network connection information.
