Inaccessible Boot Device STOP Message with Micronics 486DX/66

 Last reviewed: May 13, 1997
Article ID: Q108417
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

Windows NT does not install properly on Micronics 486DX/66 computers with Ultrastor 34F.

When you start Windows NT on a Micronics 486DX/66 computer, the NTLOADER starts, the screen clears to blue, NTLOADER queries the hard disk and then you receive the following STOP message: 

   0x0000007B Inaccessible Boot Device.

CAUSE

This problem is related to the combination of Ultrastor 34F disk controller and the Micronics board (09-00183-06 486DX/66).

WORKAROUND

To work around this problem, do the following.

  1. Make sure the Micronics computer has the latest Phoenix BIOS (0.01 JX30- 06, Rev. B3).

  2. Replace the Ultrastor 34F with a Bustec BT-445S.

Other workarounds using different hardware may also work.
