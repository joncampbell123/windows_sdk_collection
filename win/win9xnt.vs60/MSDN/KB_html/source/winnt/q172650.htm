Utility Returns Runtime Error when Run with AT command

 Last reviewed: September 17, 1997
Article ID: Q172650
The information in this article applies to:
  • Microsoft Windows NT Workstation version 4.0
  • Microsoft Windows NT Server version 4.0

SYMPTOMS

When you try to run certain third-party utilities, such as DumpACL from Somarsoft, using the AT Scheduler in Windows NT 4.0, you may receive a Visual C++ runtime error or nothing will happen when the scheduled job runs.

This problem does not exist under Windows NT 3.51.

CAUSE

Some utilities, like DumpAcl, make too many interactive API calls to be used under the Windows NT 4.0 AT Scheduler service.

RESOLUTION

Run the utility without using the AT Scheduler.
