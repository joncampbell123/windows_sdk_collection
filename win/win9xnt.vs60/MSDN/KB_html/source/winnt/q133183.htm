STOP 0x7A with Adaptec 2740A SCSI and 3Com Etherlink III

 Last reviewed: July 26, 1995
Article ID: Q133183
The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.51
  • Microsoft Windows NT Server version 3.51

SUMMARY

When you use an Adaptec 2740A SCSI controller and a 3Com Etherlink III network interface card (NIC) in Windows NT 3.51, the following error message appears: 

   STOP: 0x0000007a (0x00000103, 0xC0000185, 0x00000000, 0x00043000)
   Kernel_Data_Inpage_Error
   CPU ID:Genuine Intel 5.2.5  IRQ1:1F  Sys Version:0xF0000327

CAUSE

This problem occurs if the Adaptec 2740A SCSI controller and the 3Com Etherlink III NIC are using the same IRQ. The Adaptec 2740A SCSI controller uses a default IRQ of 10.

RESOLUTION

To correct this problem, configure the IRQ of the 3Com Etherlink III NIC or the Adaptec 2940A SCSI controller to another, unused IRQ.
