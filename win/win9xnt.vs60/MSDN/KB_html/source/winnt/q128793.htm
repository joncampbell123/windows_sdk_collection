Doc Err: STOP Error Log File Stated as MEMORY.LOG

 Last reviewed: May 25, 1995
Article ID: Q128793
The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.5
  • Microsoft Windows NT Server version 3.5

Page 157 of the Windows NT Server version 3.5 "System Guide" and page 158 of the Windows NT Workstation version 3.5 "System Guide" states the following: 

   Notice that Windows NT writes the log file to the same filename
   (MEMORY.LOG, by default) each time a STOP error occurs; to
   preserve log files, you should copy them to a new filename each
   time a STOP error occurs.
Page 158 of the Windows NT Server "System Guide" and page 159 of the Windows NT Workstation "System Guide" display a diagram of a Recovery dialog box which states the following for the Write Debugging Information To entry: 

   %SystemRoot%\MEMORY.LOG
These two statements are incorrect. By default, Windows NT writes to MEMORY.DMP each time a STOP error occurs.
