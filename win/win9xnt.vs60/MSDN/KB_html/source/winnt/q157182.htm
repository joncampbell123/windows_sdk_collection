FPNW Causes STOP 0x50 When Connection Is Closed Twice

 Last reviewed: November 5, 1997
Article ID: Q157182
The information in this article applies to:
  • Microsoft Windows NT Server version 4.0
  • Microsoft File and Print Services for NetWare version 4.0

SYMPTOMS

Your computer may sporadically stop responding (hang) while running File and Print Services for NetWare (FPNW).

NOTE: This problem is more likely to occur on multiprocessor computers and when the network experiences high load or network errors.

CAUSE

This problem occurs when FPNW closes a connection twice.

RESOLUTION

To resolve this problem, obtain the following fix or wait for the next Windows NT service pack.

This fix should have the following time stamp: 

   09/22/97  4:36pm              222,688 Fpnwsrv.sys (Intel)
   09/22/97  4:34pm              437,552 Fpnwsrv.sys (Alpha)

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 4.0. A supported fix is now available, but has not been fully regression-tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Technical Support for more information.
