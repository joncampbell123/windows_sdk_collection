Dumpel.exe Fails with DrWatson or Error Message

 Last reviewed: September 10, 1996
Article ID: Q154412
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1
  • Microsoft Windows NT Workstation versions 3.5 and 3.51
  • Microsoft Windows NT Server versions 3.5 and 3.51

SYMPTOMS

You can use the Dumpel utility, available in the Microsoft Windows NT Resource Kit, to write out the contents of the Windows NT Event logs to a standard ASCII text file. However, it might fail with one of the following errors:

  • DrWatson log after generating an Access Violation.
  • Display a pop-up dialog with the following error message: 

          LookupAccountSid failed with 1332.

CAUSE

The Access Violation might be caused by an improperly formatted Date/Time string in the Event log.

The LookupAccountSid failure is caused by a SID not mapping correctly to a valid username.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.51. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
