DISKCOPY Displays Confusing Message: Cannot Lock the Drive..

 Last reviewed: May 7, 1997
Article ID: Q101709

The information in this article applies to:
  • Microsoft Windows NT operating system, version 3.1
  • Microsoft Windows NT Advanced Server, version 3.1

Because Windows NT is a multi-tasking operating system, the DISKCOPY command locks the destination floppy drive while copying files. If the destination disk is the same as the source disk or if their serial numbers match (a very rare coincidence) then it will not complete the disk copy operation. The following error message is displayed: 

   "Cannot lock the drive. The volume is still in use."
