REG: System Components Without Additional Registry Entries

 Last reviewed: May 8, 1997
Article ID: Q102968
The information in this article applies to:
  • Windows NT operating system version 3.1
  • Windows NT Advanced Server version 3.1

The article contains information on system components without additional registry entries. It is intended as a complement to other articles which list all registry values.

SYSTEM COMPONENTS WITH NO ADDITIONAL ENTRIES

The following system components do not write entries in the Registry, besides the basic information shown in CurrentControlSet\Services: 

Abiosdsk               NetDDE
Beep                   Null
ClipSrv                Printing (all keys)
Cpqarray               Schedule service
Dellarry               SCSI class drivers
Delldsa                Simbad
Diskperf               Videoprt
File system drivers    Windbg
Floppy driver MUP utility

HOW WINDOWS NT MAPS LAN MANAGER PARAMETERS

This section lists LAN Manager network service parameters and their Windows NT equivalents. LAN Manager parameters that relate to MS- DOS are not listed. Also, because Windows NT dynamically allocates memory for network operations, many LAN Manager parameters are no longer relevant.

LAN Manager vs. Windows NT Settings for Services

In LAN Manager, service parameter settings are found in the file LANMAN.INI. In Windows NT, these settings are stored in the Registry.

In Windows NT, these entries are in the Registry in HKEY_LOCAL_MACHINE \SYSTEM\CurrentControlSet\Services\Service name\Parameters.

LAN Manager vs. Windows NT Settings for Adapter Cards

In LAN Manager, these entries are found in the [adaptercard] section of the PROTOCOL.INI file.

In Windows NT, these entries are in the Rgistry in HKEY_LOCAL_MACHINE \SYSTEM\CurrentControlSet\Services\adapter name#\Parameters. However, you can change most of settings for network adapter cards by using the Network option in Control Panel.
