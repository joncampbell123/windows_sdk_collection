Having 300+ Print Queues Causes Access Violation in Localmon

 Last reviewed: September 12, 1996
Article ID: Q150410
The information in this article applies to:
  • Microsoft Windows NT Workstation versions 3.5 and 3.51
  • Microsoft Windows NT Server versions 3.5 and 3.51

SYMPTOMS

If you create 300 or more printer queues on a single server, the computer will return a message indicating an access violation in the Localmon.dll.

CAUSE

The access violation occurred the initialization of the Localmon.

STATUS

Microsoft has confirmed this to be a problem in Windows NT versions 3.5 and 3.51. This problem was corrected in the latest Windows NT 3.51 U.S. Service Pack. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
