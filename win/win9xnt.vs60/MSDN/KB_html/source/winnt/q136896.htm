Stop 0A in Windows NT 3.51

 Last reviewed: March 21, 1997
Article ID: Q136896

The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.51
  • Microsoft Windows NT Server version 3.51

SYMPTOMS

The computer halts with the following STOP code: 

   Bugcheck 0000000a : 0000006c 00000002 00000001 804029cc

CAUSE

AFD.SYS assumes there is synchronization between TDI abortive disconnects and send completions. TDI guarantees no such synchronization. This can cause TDI to free one of its endpoint objects while sends are still pending. When the send finally completes, TDI calls AfdRestartBufferSend() with an invalid pointer.

This failure is most likely to occur on multiprocessor systems using Windows Sockets applications in high-stress environments.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.51. This problem has been corrected in the latest U.S. Service Pack for Windows NT version 3.51. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
