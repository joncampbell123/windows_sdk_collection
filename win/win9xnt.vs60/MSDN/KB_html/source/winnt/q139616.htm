Upgrading LAN Manager to Windows NT Overwrites Duplicate Account

 Last reviewed: March 21, 1997
Article ID: Q139616
The information in this article applies to:
  • Microsoft LAN Manager to Windows NT Server Upgrade
  • Microsoft Windows NT Server versions 3.5 and 3.51

SYMPTOMS

When you upgrade from LAN Manager 2.1x and 2.2x Server to Windows NT 3.5x Server, the duplicate user accounts on the Windows NT Server are overwritten by the account information found in NETACC.BKP of the LAN Manager Server.

NOTE: This problem occurs when you use the Upgrade Manager (WNTUPMGR) or PORTUAS /V as a standalone utility. If the same account name is found on both servers, the upgrade should pause and query the user on how to proceed.

STATUS

Microsoft has confirmed this to be a problem in Windows NT versions 3.5 and 3.51. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
