Err Msg: The Home Directory...Could Not Be Created...

 Last reviewed: May 6, 1997
Article ID: Q100321
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When creating a new user account profile in User Manager, you may receive the following error message: 

   The home directory <Directory Path> for <User Name> could not be
   created. The user account has been updated. You must create the
   Home Directory manually.
(The text between the <> brackets reflects the user information just entered.)

CAUSE

This message may be caused by entering a directory name that goes outside of the 8.3 naming convention (8 characters for the name, a period, 3 characters for the extension) of a FAT partition.

This error is also caused by the \\SERVER\SHARE directory not being shared. For example, if you specify 

   Connect to: X:\\NTSRV1\USERS\%USERNAME%
the \USERS directory must be shared with appropriate permissions (usually everyone - full control).

STATUS
