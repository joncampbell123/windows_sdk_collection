Windows NT Explorer May Not Indicate Status of Shared Folders

 Last reviewed: December 1, 1997
Article ID: Q177444
The information in this article applies to:
  • Microsoft Windows NT Workstation version 4.0
  • Microsoft Windows NT Server version 4.0
  • Microsoft Windows NT Server Enterprise Edition version 4.0

SYMPTOMS

If you stop or start the Server service, Windows NT Explorer may not indicate the current status of your shared folders.

RESOLUTION

To work around this behavior, refresh Windows NT Explorer after stopping or starting the Server service. To do so, follow these steps:

  1. Click Start, point to Programs, and then click Windows NT Explorer.

  2. On the View menu, click Refresh.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
