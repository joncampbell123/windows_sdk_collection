Windows NT 4.0 Installations Cannot Be Copied or Moved

 Last reviewed: February 4, 1998
Article ID: Q155717
The information in this article applies to:
  • Microsoft Windows NT Server version 4.0
  • Microsoft Windows NT Workstation version 4.0

SYMPTOMS

After copying or moving your Windows NT 4.0 installation into a new tree (folder or drive), some system functions and programs behave unpredictably or do not work at all.

CAUSE

The Windows NT 4.0 registry contains some hard-coded paths (for example, "C:\WINNT\SYSTEM32\" instead of "%SystemRoot%\SYSTEM32\"). In addition, some programs may uses hard-coded paths.

RESOLUTION

Moving or copying your Windows NT 4.0 installation to a different drive or folder is not supported.
