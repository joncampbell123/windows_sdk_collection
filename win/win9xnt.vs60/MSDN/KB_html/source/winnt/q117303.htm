Trap 0x0A with Future Domain 1680 SCSI Controller

 Last reviewed: May 16, 1997
Article ID: Q117303
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you attempt to copy, delete, or back up data on a hard disk or tape drive attached to a Future Domain 1680 SCSI controller, the following message may appear: 

   Trap 0x0000000A

CAUSE

Some board revisions of the Future Domain 1680 SCSI controller may conflict with the FD16_700.SYS driver shipped with Windows NT version 3.1.

WORKAROUND

Future Domain has a new version of the FD16_700.SYS driver available. You can obtain the file PCI_NT.EXE from Future Domain's BBS at (714) 253-0432.

STATUS

Microsoft has confirmed this to be a problem with Windows NT operating system and Windows NT Advanced Server version 3.1. This problem was corrected in the latest U.S. Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
