Compaq DeskPro XL 566 Not Automatically Rebooting Upon Failure

 Last reviewed: May 24, 1995
Article ID: Q130482
The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.5
  • Microsoft Windows NT Server version 3.5

SYMPTOMS

If you use an integrated on-board PCI SCSI-2 controller on a Compaq DeskPro XL 566 computer and configure Windows NT 3.5 to automatically reboot the computer when a STOP Message occurs, the system stops responding (hangs) and does not reboot.

NOTE: To force Windows NT to automatically reboot the computer when a STOP Message occurs, do the following:

  1. Run the System component of Control Panel and choose Recovery.

  2. Select the Automatically Reboot check box.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.5. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
