File Write to Network Share Using MS-DOS 21h Function Fails

 Last reviewed: May 14, 1997
Article ID: Q111027

The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.5
  • Microsoft Windows NT Server version 3.5

SYMPTOMS

When you use MS-DOS function 21h to write to a file on a network share, the following error appears in the event log: 

   System Process - Lost Delayed-Write Data

   The system was attempting to transfer file data from buffer to <network
   share>. The write operation failed, and only some of the data may have
   been written to the file.

STATUS

Microsoft has confirmed this to be a problem in Windows NT Workstation and Windows NT Server version 3.5. This problem was corrected in the latest U.S. Service Pack for Windows NT version 3.5. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
