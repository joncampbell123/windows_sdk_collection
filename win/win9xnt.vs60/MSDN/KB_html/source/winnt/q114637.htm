Windows NT Hangs on Start Up When No Keyboard is Connected

 Last reviewed: May 15, 1997
Article ID: Q114637

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you do not have a keyboard plugged in to your computer and you start Windows NT, it will stop responding (hang) at the blue startup screen.

RESOLUTION

This has been fixed in the new version of the binary file, I8042PRT.SYS. When this file is installed, Windows NT will start without a keyboard, but if you attach a keyboard at a later time, Windows NT will not respond to it.

STATUS

Microsoft has confirmed this to be a problem in Windows NT and Windows NT Advanced Server version 3.1. This problem has been corrected in the latest U.S. Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
