Some OS/2 Character-Based Apps Use Presentation Mgr APIs

 Last reviewed: May 8, 1997
Article ID: Q103101

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

In Windows NT, the OS/2 subsystem supports OS/2 1.x character-based applications. Although they are character-based, some OS/2 applications may use OS/2 Presentation Manager application programming interface (API) calls and thus will not run with Windows NT.

One of the following error messages may appear:

  • The system cannot find the file OS2SM
  • Error ordinal not found DOSCALLS.XXX

These errors indicate that execution of a Presentation Manager file was attempted.
