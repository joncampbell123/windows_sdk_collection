Logon Scripts for WfWG Clients Connecting to a WinNT RAS Server

 Last reviewed: August 22, 1996
Article ID: Q120287
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1
  • Microsoft Windows for Workgroups version 3.11

To implement logon scripts for Windows for Workgroups version 3.11 clients dialing into a Windows NT Remote Access Service (RAS) server follow these steps:

On the Windows NT RAS Server

  1. Create a logon script on the server for client workstations.
2. Place the script file in the SYSTEM32\REPL\IMPORT\SCRIPTS directory.

On the Windows for Workgroups 3.11 client

  1. Go to the Network option in the Control Panel, and select the Startup button.
2. Change the name of the Windows NT-based domain to the RAS server 
   domain (unless both are already the same).
3. Make sure that the Log On at Startup box is empty. 4. Restart Windows for Workgroups. 5. Start the RAS program and call into the RAS server. 6. Once the connection has been made and the user has been authenticated, 
   select the Log On/Off icon in the Network group.
7. Give the correct password for the RAS Domain. 8. The logon script is now functional.
