Error 692 Using 3Com Impact ISDN Modem with RAS

 Last reviewed: September 3, 1997
Article ID: Q151232
The information in this article applies to:
  • Microsoft Windows NT Workstation versions 3.5, 3.51, and 4.0
  • Microsoft Windows NT Server versions 3.5, 3.51, and 4.0

SYMPTOMS

When you use Remote Access Service (RAS) and attempt to dial out with your 3Com Impact modem, you receive the following error message: 

   Error 692: Hardware failure in port or attached device.
NOTE: You can use one of 3Com's line of Impact ISDN modems with Windows NT Remote Access Service (RAS) after you obtain a Modem.inf file from 3Com. These devices are not listed on the current Windows NT Hardware Compatibility List (HCL).

CAUSE

The <autodial> macro specified in the COMMAND_DIAL line of the 3Com Impact Digital Data section of 3Com's Modem.inf file is not resolved properly.

RESOLUTION

Edit the 3Com Impact Digital Data section of the Modem.inf file supplied by 3Com. Change the following line 

   COMMAND_DIAL=<autodial><phonenumber><cr>
to: 

   COMMAND_DIAL=ATD <phonenumber><cr>
You must restart RAS in order for this change to take effect.
