Err Msg on File Server: XXXXXX Target Directory is Corrupt...

 Last reviewed: May 16, 1997
Article ID: Q118476
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

If you receive a dialog box with an "Abort Retry Ignore" message and you choose Retry, the following message appears on File Server: 

   XXXXXX target directory is corrupt or unreadable.
If you choose Abort, it appears that no files in the destination directory are corrupt.

If the corruption is extensive enough, CHKDSK will not correct the corruption and you may receive a STOP message (trap error 0x0a/0x01E) when you access any of the corrupted directories or files.

STATUS

Microsoft has confirmed this to be a problem in Windows NT and Windows NT Advanced Server version 3.1. This problem has been corrected in the latest U.S. Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
