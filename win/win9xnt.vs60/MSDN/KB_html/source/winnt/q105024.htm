COMPAQ ProSignia Hangs While Scanning For NCR

 Last reviewed: April 10, 1997
Article ID: Q105024
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you run Setup for Windows NT from CD-ROM discs or floppy disks on a COMPAQ ProSignia, text-mode setup hangs while scanning for SCSI adapters. There may also be a noticeable delay of about one and a half minutes during the SCSI detection.

CAUSE

The disk drive firmware of newer COMPAQ ProSignia computers appear to cause this problem.

The delay is a separate problem under investigation, which still occurs when the fix below is applied.

RESOLUTION

A fixed SCSI driver is included for this particular problem on the retail CD. To use it, perform the following steps:

NOTE: Read the STATUS section below before proceeding.

  1. Copy the file DRIVER.SYS from the directory <cd drive>:\DRVLIB\MISC\DRIVER\X86\ to NCRC710.SYS on the install media. For CD setup, this is the CD startup floppy, or to Disk 1 of the 1.44MB floppy disks, or to both Disk 1 and Disk 2 of the 1.2MB floppy disks.

  2. Complete text-mode Setup using the modified media.

  3. Restart MS-DOS.

  4. Type "ATTRIB -h -s -r C:\NTBOOTDD.SYS" (without the quotes).

  5. Copy the driver to C:\NTBOOTDD.SYS and to %SystemRoot%\SYSTEM32\DRIVERS\NCRC710.SYS

  6. Restart your computer and continue with graphical-mode Setup.

STATUS

Microsoft has confirmed this to be a problem in Windows NT and Windows NT Advanced Server version 3.1. This problem was corrected in the latest U.S. Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
