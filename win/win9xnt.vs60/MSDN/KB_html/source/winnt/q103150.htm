WinMSD.EXE May Cause an Exception: Access Violation Error

 Last reviewed: May 8, 1997
Article ID: Q103150

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

In WinMSD, a diagnostic utility included with Windows NT, if you select a file with either a .SYS or .DRV extension that displays a version number and then use the File Info function, the following error message may occur: 

   An application error has occurred and an application log is being
   generated.

   WINMSD.exe

   Exception: access violation (0xc0000005), Address: 0x024973c9

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.1 and Windows NT Advanced Server version 3.1. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Open WINMSD.EXE.

  2. From the File menu, choose Find File.

  3. Search for *.DRV in the \WINNT\SYSTEM32 subdirectory.

  4. In the list of files that appears, select a file with a version number, such as WINSPOOL.DRV.

  5. Choose the File Info button.

The above error message occurs.
