ALR Business Veisa Problem with CD-ROM Drive Toshiba 3401

 Last reviewed: May 9, 1997
Article ID: Q104700
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

Windows NT Setup stops on Disk 1 with the following error message: 

   Setup could not be started. Could not open multi(0)disk(0)
   fdisk(0)partition(0) ARC Status=13 Loader exited.

   Setup was unable to load file system support. The distribution disk
   or CD-ROM may be damaged.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.1. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

This error message is generated with the following hardware configuration: 

   ALR Business Veisa 486/33
   Ide hard drive
   SCSI 1542C controller
   CD-ROM Toshiba 3401

Steps to Reproduce Problem

  1. Start Windows NT from a floppy disk. Setup reads the TXTSETUP.INF file.

  2. Select Express Setup and press ENTER. The SCSI controller is located, and the following error message is displayed: 

          Error: Setup was unable to load file system support. The
      distribution disk or CD-ROM may be damaged.


          ENTER or CONTINUE

  3. Press ENTER. Setup locates drive C and asks about keeping the existing file format or repartition and format, and so on.

  4. Press ENTER twice to continue.

The following error message appears: 

   Error: Setup was unable to invoke or complete CHKDSK on the partition
   onto which you have chosen to install Windows NT. The disk may be
   damaged or corrupted beyond repair or physically damaged.
NOTE: Other CD-ROM drives such as Sony or NEC work fine with the above system configuration.
