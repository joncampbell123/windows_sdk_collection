Nonpaged Pool Size Incorrectly Displayed in Performance Monitor

 Last reviewed: February 18, 1998
Article ID: Q177647
The information in this article applies to:
  • Microsoft Windows NT Server versions 3.51 and 4.0
  • Microsoft Windows NT Workstation versions 3.51 and 4.0

SYMPTOMS

When displaying the Pool Nonpaged Bytes counter from Object:Memory in Performance Monitor, it may appear too high.

CAUSE

The nonpaged pool counter is calculated incorrectly. This has no effect on the operation of the Memory Manager.

RESOLUTION

To resolve this problem, obtain the following fix or wait for the next Windows NT service pack.

This fix should have the following time stamp: 

   10/29/97  06:37p               936,000 Ntkrnlmp.exe (Intel)
   10/29/97  06:36p               916,160 Ntoskrnl.exe (Intel)

   10/29/97  07:34p             1,395,392 Ntkrnlmp.exe (Alpha)
   10/29/97  07:34p             1,368,000 Ntoskrnl.exe (Alpha)

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 4.0. A supported fix is now available, but has not been fully regression tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Technical Support for more information.

Microsoft has confirmed this to be a problem in Windows NT version 3.51. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
