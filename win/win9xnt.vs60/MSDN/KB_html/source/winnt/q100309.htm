No Files in FTP Directory

 Last reviewed: May 6, 1997
Article ID: Q100309
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1

SYMPTOMS

You log into an FTP (file transfer protocol) server and issue the DIR command to see the directory listing. No files are listed, even when you know several files exist.

CAUSE

Allow Read in the FTP Server portion of Control Panel is not checked and thus you do not have the proper security access to read files.

RESOLUTION

Start Control Panel and FTP Server. Choose Allow Read so that it is checked.
