Nwconv Fails "Error Saving User" With FPNW or GSNW

 Last reviewed: November 11, 1997
Article ID: Q138574
The information in this article applies to:
  • Microsoft Windows NT Server versions 3.5, 3.51, and 4.0

SYMPTOMS

When you have Microsoft File and Print Services for NetWare (FPNW) or Gateway Services for NetWare (GSNW) installed, and you use the FPNW or GSNW NWCONV migration utility to move accounts from a NetWare 3.12 server to a Windows NT 3.51 domain controller (DC), the files are migrated but an error occurs for user account. The error log shows Error Saving User Name for each user account.

WORKAROUND

To work around this problem:

  1. In User Manager for Domains, select Account from the Policies menu.

  2. Change the minimum password length from at least 6 characters to permit blank password.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.51. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
