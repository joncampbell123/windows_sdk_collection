NCP Client Stops Responding When Saving File

 Last reviewed: March 24, 1997
Article ID: Q142618
The information in this article applies to:
  • Microsoft File and Print Services for NetWare version 3.51

SYMPTOMS

When a NetWare Core Protocol (NCP) client tries to save a file that the user does not have write permissions to through File and Print Services for NetWare (FPNW), the client machine will stop responding.

CAUSE

FPNW returns an incorrect error code to the client.

WORKAROUND

Obtain the fix mentioned below.

STATUS

Microsoft has confirmed this to be a problem in File and Print Services for NetWare version 3.51. This problem was corrected in the latest Windows NT 3.51 U.S. Service Pack. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
