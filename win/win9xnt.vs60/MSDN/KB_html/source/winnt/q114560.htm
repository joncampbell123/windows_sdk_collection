SMC8000N.SYS Driver from SMC Does Not Work Correctly

 Last reviewed: May 15, 1997
Article ID: Q114560

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

The SMC8000N.SYS driver and OEMSETUP.INF that are supplied by SMC with the SMC 8000 series Ethernet adapters do not work properly. Installing these drivers for use with SMC adapters result in the error message: 

   5728: Unable to Bind the Protocol.
The date and time stamps of the non-functioning files from SMC are: OEMSETUP.INF 1-21-94 and SMC8000n.SYS 1-11-94.

RESOLUTION

SMC has confirmed the driver shipped with these ethernet adapters to be defective. An updated driver is available from SMC, and can be downloaded from SMC's BBS or from MS Download Service. The file containing the fix is named SMC16.EXE and is dated 3-11-94.

The file, SMC8000N.SYS that is provided with Windows NT setup operates properly.

Contact SMC Technical Support at (800) 992-4762 for more information.

SMC 8000 series Ethernet adapters are manufactured by SMC, a vendor independent of Microsoft; we make no warranty, implied or otherwise, regarding this product's performance or reliability.
