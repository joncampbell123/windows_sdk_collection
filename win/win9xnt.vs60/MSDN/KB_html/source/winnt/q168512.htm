IBMTRP.SYS Causes Stop 0x0000000A During Shutdown.

 Last reviewed: August 4, 1997
Article ID: Q168512
The information in this article applies to:
  • Microsoft Windows NT Workstation versions 3.51 and 4.0
  • Microsoft Windows NT Server versions 3.51 and 4.0

SYMPTOMS

During a shutdown and restart from within Windows NT, you may receive the error Stop 0x0000000A (0xc18c313b, 0x00000002, 0x00000000, 0x8012f72d) IRQL_NOT_LESS_OR_EQUAL.

NOTE: The parameters in parenthesis are specific to your computer configuration and may be different.

CAUSE

This problem is specific to the IBM PCI Token Ring Card 16/4, reference 75H9800, or 75H9802, using the following driver: 

   Driver properties : IBMTRP.SYS, 89728 bytes, 5/21/96, 4:31p.

RESOLUTION

To correct this problem, obtain a new version of the Ibmtrp.sys driver from IBM at the following location: 

   http://www.networking.ibm.com/nes/nestoken.htm
Download the following image (PCI Token Ring Adapter family): 

   PCI Token Ring NDIS Drivers Diskette
   Diskette 2 of 5
   [TRP1642.EXE | Version 1.2 | 1-30-97 | 518,330 bytes]
The updated driver is dated 11/04/96, and its size is 90,944 bytes. Any version of the driver dated before this date causes the error to occur.
