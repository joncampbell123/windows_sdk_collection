WINS and DHCP Dynamic Database Compaction in Windows NT 4.0

 Last reviewed: September 8, 1997
Article ID: Q167812
The information in this article applies to:
  • Microsoft Windows NT Server version 4.0

SUMMARY

Microsoft Windows NT Server 4.0 Windows Internet Name Service (WINS) and Dynamic Host Protocol Configuration (DHCP) Server Service have a new feature that dynamically compacts their databases. This is in addition to the jetpack utility which was also in previous versions of Windows NT and is not a replacement for jetpack.

Windows NT 4.0 dynamic compacting decreases the frequency for which the jetpack utility is needed. The jetpack program is still a feature of Windows 4.0. For more information, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: 145881
   TITLE     : How to use JETPACK.EXE To Compact a WINS or DHCP Database.
Microsoft recommends running the Jetpack utility when the WINS.mdb file approaches 30 megabytes (Windows NT 4.0 Server Networking Guide, Page 471) and when the Dhcp.mdb file approaches 20 megabytes.
