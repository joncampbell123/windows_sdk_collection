Event Log Reports Previously Loaded Mouse Driver Not Starting

 Last reviewed: May 8, 1997
Article ID: Q102623
The information in this article applies to:
  • Microsoft Windows NT operating system, version 3.1
  • Microsoft Windows NT Advanced Server, version 3.1

SYMPTOMS

After you install a new mouse driver, you receive an error message in the event log stating that the old mouse driver failed to start.

CAUSE

This problem occurs because the old driver is not always disabled by the new driver's installation.

WORKAROUND

To work around this problem, choose the Devices icon in Control Panel and change the old driver's Startup value to DISABLED.
