Draw Can't Leave Data in Clipboard After Closing Application

 Last reviewed: April 10, 1997
Article ID: Q102477

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you close Microsoft Draw, you may receive an error message telling you that there is not enough memory to store the data left in the Clipboard.

CAUSE

When you close Microsoft Draw, it uses different formats in an attempt to render the data in the Clipboard. The error noted above occurs because Microsoft Draw attempts to use a 16-bit driver to render the image. Windows on Windows (WOW) does not support 16-bit drivers.

NOTE: Even though you receive the error noted above, the image is placed in the Clipboard in the CF_BITMAP format.
