Bad Logins From Workstation on Domain Not in Event Viewer

 Last reviewed: September 23, 1996
Article ID: Q124154
The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.5
  • Microsoft Windows NT Server version 3.5

SYMPTOMS

Bad logins do not get logged into the Event Viewer when a user logs in from a workstation in the domain. If the user logs into the domain at the console of the primary domain controller (PDC), bad logins get recorded as designed.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.5. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
