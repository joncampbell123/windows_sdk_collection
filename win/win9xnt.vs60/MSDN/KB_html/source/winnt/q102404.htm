Err Msg Running MS-DOS-Based Application: Packed File Corrupt

 Last reviewed: May 8, 1997
Article ID: Q102404

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you run an MS-DOS-based application with Windows NT, you may receive the following error message: 

   packed file is corrupt

CAUSE

This error occurs when you run an .EXE file packed with the EXEPACK program. This behavior is a result of an error in the code that unpacks the .EXE file.

RESOLUTION

To work around this problem, use the Windows NT LOADFIX program. For example, if you want to run a program called MYPROG.EXE, type the following at the Windows NT MS-DOS command prompt: 

   loadfix myprog
