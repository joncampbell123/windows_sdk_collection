STOP: 0x0000000A After Installing Cheyenne ArcServe 1.0

 Last reviewed: April 11, 1997
Article ID: Q139633
The information in this article applies to:
  • Microsoft Windows NT Workstation versions 3.5 and 3.51
  • Microsoft Windows NT Server versions 3.5 and 3.51

SYMPTOMS

After you install the Cheyenne ArcServe 1.0 tape backup software, the following STOP error message appears: 

   STOP: 0x0000000A
   IRQL_NOT_LESS_OR_EQUAL

RESOLUTION

To correct this problem, do one of the following:

  • Upgrade to Cheyenne ArcServe 2.0 or greater.

    -OR-

  • Request ECN074.ZIP by sending an electronic mail to *SYSOP in the Cheyenne Software Forum (GO CHEYENNE). You must include your ArcServe software serial number.

For more information, contact Cheyenne Software, Inc., Technical Support at (800) 243-9832.
