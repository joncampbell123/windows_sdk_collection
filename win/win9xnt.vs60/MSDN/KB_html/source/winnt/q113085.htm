User Manager Auditing not Logging All File Accesses

 Last reviewed: May 14, 1997
Article ID: Q113085

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you use the User Manager's Policies-Audit utility in Windows NTAS after installing Service pack 2, some files accessed on the server are not logged.

CAUSE

While User Manager turns on and off auditing in general, only those individual files that have audit set will be audited.

WORKAROUND

Individually set auditing on those files you wish to audit.

STEPS TO START AUDITING

  1. Start User Manager.
2. Choose Policies, then Audit. 3. Choose Success and Failure under File and Object Access. 4. Choose OK.
