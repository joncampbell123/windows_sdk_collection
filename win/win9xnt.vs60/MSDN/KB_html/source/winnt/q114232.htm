User Attempt to Change Password Causes Error 2242

 Last reviewed: May 15, 1997
Article ID: Q114232

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1
  • Microsoft Windows for Workgroups version 3.11

SYMPTOMS

When you log on to a Windows for Workgroups 3.11 workstation and attempt to respond to the domain controller prompt to change their password, you receive error 2242.

CAUSE

A Backup Domain Controller (BDC) is incorrectly checking the local registry for user password expiration when pass-through validation to the primary domain controller is performed.

RESOLUTION

MSV1_0.dll was updated to prevent the BDC from checking the local registry for user password expiration during pass-through validation.

STATUS

Microsoft has confirmed this to be a problem in Windows NT and Windows NT Advanced Server version 3.1. This problem has been corrected in the latest U.S. Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces):
