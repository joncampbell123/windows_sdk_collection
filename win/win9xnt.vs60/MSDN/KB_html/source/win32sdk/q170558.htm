PRB: Console IME Support for Hangul Windows NT 3.51

 Last reviewed: June 25, 1997
Article ID: Q170558
The information in this article applies to:
  • Microsoft Win32 Software Development Kit (SDK)

SYMPTOMS

Korean Windows NT 3.5 supports IME (Input Method Editor) for Console applications. However, Korean Windows NT 3.51 does not support IME for Console applications running on DOS box.

RESOLUTION

In Korean Windows NT 3.5, we support DBCS character input using Input Method Editor(IME) for console applications, so that the users can type Hangul characters on console applications running on DOS Box. In Windows NT 3.51, we dropped this support of Hangul input on console. However, Hangul Windows NT 4.0 supports IME for console applications.

STATUS

This behavior is by design.
