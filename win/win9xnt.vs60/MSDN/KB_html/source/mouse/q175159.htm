IntelliPoint Setup Error Message When Files are Read-Only

 Last reviewed: October 15, 1997
Article ID: Q175159
The information in this article applies to:
  • Microsoft IntelliPoint software version 2.0

SYMPTOMS

When you run Setup for Microsoft IntelliPoint software version 2.0, you receive the following error message: 

   Setup has detected that your Autoexec.bat or Config.sys files are
   marked read-only or write protected. Set these files to Read/Write
   privleges and re-run Setup to install IntelliPoint.

CAUSE

This error message occurs if the Autoexec.bat, Config.sys, or the Dosstart.bat file attributes are set to read-only.

RESOLUTION

Disable the read-only attribute for these files and run the IntelliPoint software Setup again.

To turn off the read-only file attribute for these files, do the following:

  1. Click Start, point to Find, and then click Files or Folders.

  2. In the Named box type the name of one of the files listed earlier in this article (Autoexec.bat, Config.sys, or Dosstart.bat).

  3. In the Look In box select drive C:. If Windows is installed on a drive other than drive C, select the drive where Windows is installed.

  4. Click Find Now.

    The file that you typed in step 2 will be listed at the bottom of the Find dialog box.

    NOTE: You may have multiple copies of these files. Follow steps 5 through 8 for each copy listed.

  5. Click the first file in the list to select it.

  6. On the File menu, click Properties.

  7. Check to see if the Read-only check box in the Attributes section is selected. If it contains a check mark, click the check box to clear it.

  8. Click OK.

    Repeat steps 1 through 8 for all three files (Autoexec.bat, Config.sys, or Dosstart.bat).

  9. Run the IntelliPoint software version 2.0 Setup again.
