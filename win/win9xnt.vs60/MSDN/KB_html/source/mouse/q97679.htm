Mouse Cursor Moves Slowly in MS-DOS Sessions

 Last reviewed: September 12, 1996
Article ID: Q97679
The information in this article applies to:
  • Microsoft Mouse Driver for MS-DOS, version 9.0
  • Microsoft Mouse Driver for Windows, version 9.0
  • Microsoft IntelliPoint software version 1.0

SUMMARY

If your mouse sensitivity is slower in MS-DOS-based applications than it is in Windows or Windows-based applications, one of the following may the cause:

  • You may have different sensitivities set for the MS-DOS environment and the Windows environment. (The difference is most obvious in full MS-DOS sessions.)
  • If you are running MS-DOS virtual machines (that is, not full MS- DOS sessions), both the Windows mouse driver and MS-DOS mouse driver are accessed; this slows the responsiveness of the mouse.

MORE INFORMATION

The sensitivity used in an MS-DOS virtual machine (VM) is the Windows sensitivity. The sensitivity used in a full MS-DOS session is the MS-DOS sensitivity.

To set the sensitivity for the MS-DOS environment, use the mouse driver CPANEL command (for versions 8.20 and earlier) or the MS-DOS portion of MOUSEMGR (for version 9.0).

To set the sensitivity for the Windows environment, use POINT (for versions 8.20 and earlier) or MOUSEMGR (for version 9.0).
