MS-DOS Mouse Driver Does Not Use Primary Mouse Button Setting

 Last reviewed: April 7, 1997
Article ID: Q165751
The information in this article applies to:
  • Microsoft IntelliMouse version 1.0
  • Microsoft IntelliPoint software version 2.0

SYMPTOMS

If you use the IntelliPoint software to configure your IntelliMouse to use an alternate button as your primary mouse button, the change may not be applied to the MS-DOS mouse driver.

CAUSE

The Msinput.ini file is not updated to reflect the new configuration.

RESOLUTION

To resolve this problem, edit the Msinput.ini file in the Msinput folder to reflect the appropriate primary mouse button. The "Primarybutton=x" setting in the [mouse] section of the Msinput.ini file determines which button the MS-DOS mouse driver uses as the primary button. Use one of the following settings:

  • x=1 sets the left mouse button
  • x=2 sets the middle (wheel) button
  • x=3 sets the right mouse button
