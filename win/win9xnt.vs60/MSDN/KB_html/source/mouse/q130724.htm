Microsoft Mouse Version 8.2a Does Not Modify MOUSETYPE=

 Last reviewed: September 16, 1996
Article ID: Q130724
The information in this article applies to:
  • Microsoft Mouse driver versions 8.2 and 8.2a

SYMPTOMS

On a computer with more than one mouse installed on different ports, one mouse is activated in MS-DOS and another is activated in Microsoft Windows.

CAUSE

If the Microsoft Mouse driver is loaded with a switch indicating the port to be searched for a mouse, the MOUSETYPE= line in the MOUSE.INI file is not updated.

RESOLUTION

To work around this problem, use one of the following methods:

  • Upgrade the mouse driver to version 9.0 or later.
  • Edit the MOUSETYPE= line in the MOUSE.INI file to match the mouse you want to use.
  • Do not specify the port on the mouse command line.
  • Disconnect the other mice from the computer.
