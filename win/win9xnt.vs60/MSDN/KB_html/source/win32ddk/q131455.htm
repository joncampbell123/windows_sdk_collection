BUG: Image Does Not Print Correctly on Postscript Printer

 Last reviewed: October 9, 1995
Article ID: Q131455
The information in this article applies to:
  • Microsoft Win32 Device Development Kit (DDK) versions 3.5 and 3.51

SYMPTOMS

An image will not print correctly on a level 1 postscript printer with color extensions and mapped color images with BPC equal to 2 or 4.

RESOLUTION

Modify Postscript's IMAGE.CPS file as follows:

In the section "Adobe_WinNT_Co_Images_L1" ... "doclutimage" (4th image) change the parameter "bpc" to "8."

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
