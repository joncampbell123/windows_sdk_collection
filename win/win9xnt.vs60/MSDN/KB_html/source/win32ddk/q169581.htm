PRB: Windows 95 DDK Won't Install After Installing Platform SDK

 Last reviewed: June 4, 1997
Article ID: Q169581
The information in this article applies to:
  • Windows 95 Device Driver Kit (DDK), version 4.0

SYMPTOMS

The Platform SDK contains components that have been distributed separately in the Win32 SDK, the BackOffice SDK, and the ActiveX/Internet Client SDK, as well as preliminary documentation for newer technologies. The Platform SDK integrates these components into functional groups within one SDK providing one source of information for all application programming interfaces (APIs).

However, after installing the Platform SDK from the Microsoft Developer Network (MSDN), the Windows 95 DDK may not install.

CAUSE

Installing the Platform SDK by itself will not allow installation of the Windows 95 DDK.

RESOLUTION

The Windows 95 DDK requires prior installation of the Win32 SDK; the Platform SDK will not suffice. The Win32 SDK is provided on a separate CD-ROM from the Platform SDK.
