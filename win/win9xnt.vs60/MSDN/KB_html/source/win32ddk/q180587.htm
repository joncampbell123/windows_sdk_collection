PRB: Spurious Interrupts Cause Exception 0Fh on Pentium Pro

 Last reviewed: February 16, 1998
Article ID: Q180587
The information in this article applies to:
  • Windows 95 Device Driver Kit (DDK), version 4.0

SYMPTOMS

On some versions of the Intel Pentium Pro Processor, a spurious hardware interrupt may cause an unhandled exception 0Fh to occur under Windows 95.

CAUSE

This is a problem caused by some of the Pentium Pro Processor steppings, and is not a problem caused by Windows 95.

MORE INFORMATION

Exception 0Fh is defined as "Reserved" for current implementations of the Pentium Pro Processor, and it should not occur under normal operating conditions when you are running Windows 95. Windows 95 does not install an exception handler for exception 0Fh. Therefore, if this exception occurs, it results in a blue screen error.

This is a known problem with the Pentium Pro Processor. Please refer to errata number 5AP of the "Pentium Pro Processor Specification Update" from Intel for more information on the cause and resolution of this problem.
