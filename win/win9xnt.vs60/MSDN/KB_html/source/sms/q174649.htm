SMS: Remote Control Viewer Is Blank for a Busy WinNT Client

 Last reviewed: December 4, 1997
Article ID: Q174649
The information in this article applies to:
  • Microsoft Systems Management Server version 1.2

SYMPTOMS

When an administrator tries to use Remote Control on a Windows NT client in a remote domain, and that client computer's system is running near 100 percent CPU usage, the Quick Windows Viewer displays a blank white screen instead of the remote client video display. The Viewer window does not refresh when CPU usage drops to lower levels.

STATUS

Microsoft has confirmed this to be a problem in Systems Management Server version 1.2. This problem was corrected in the latest Microsoft Systems Management Server 1.2 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
