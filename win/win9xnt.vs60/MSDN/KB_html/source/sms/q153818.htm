Running SMSVIEW Removes Permissions On Existing SMS Views

 Last reviewed: April 24, 1997
Article ID: Q153818
The information in this article applies to:

- Microsoft Systems Management Server versions 1.0, 1.1 and 1.2

After you create Systems Management Server views, set user permissions for them, and re-run SMSVIEW, the views are re-created and the permissions are lost.

To work around this problem, you must manually reset the desired permissions.
