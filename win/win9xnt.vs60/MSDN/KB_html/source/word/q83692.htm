WinWord: MoreFonts Upgrade Required for Windows 3.1

 Last reviewed: February 5, 1998
Article ID: Q83692
The information in this article applies to:
  • Microsoft Word for Windows versions 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c, 6.0, 6.0a, 6.0c

SUMMARY

MicroLogic Software has confirmed that version 3.051 (M300-00-3051) of the MoreFonts soft font package is required for compatibility with the Microsoft Windows operating system version 3.1. If you have a version prior to 3.051, contact MicroLogic to request version 3.051.

If you are running an older version of MoreFonts with the above listed versions Microsoft Word for Windows and Windows version 3.1, you may experience the following problems:

  • Tool bar buttons and some dialog box buttons appear to be completely black.
  • The MoreFonts Tiempo font prints as garbage.
  • Words wrap incorrectly.

MORE INFORMATION

The version 3.051 maintenance release of MoreFonts is compatible with Windows version 3.1. If the last four digits of the serial number on your MoreFonts disk are 3050 or lower, you need the maintenance release of MoreFonts.

To determine what version of MoreFonts you have, use the following steps:

  1. Activate the MoreFonts control panel (the MoreFonts icon is a large "M").

  2. Press ALT+SPACEBAR to select the Control Box.

  3. Choose About More Fonts.

The version numbers have the format M300-00-30XX, where "30XX" corresponds to version 3.0XX (for example, version 3.051).

MoreFonts Version History

  • Versions 3.022 and earlier require an update to version 3.023 for compatibility with Windows version 3.0 and Word for Windows version 2.0.
  • Version 3.01 requires an upgrade for operation in Windows version 3.0.
  • MoreFonts version 2.0 is an MS-DOS-based font manager that works in Microsoft Windows version 3.0.
  • MoreFonts version 3.0 is a Windows-based font manager originally written for Windows version 3.0.
