WD98: Text Overflows Text Box Boundary When Printed

 Last reviewed: February 17, 1998
Article ID: Q178005
The information in this article applies to:
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

When you print a document that contains a text box, the text will print outside of the text box even though it is displayed correctly in all views, including Print Preview.

CAUSE

This behavior occurs when both of the following conditions are true:

  • The text direction is set for vertical.

-and-
  • The text within the text box overflows the boundaries of the text box

NOTE: To insert a text box, click Text Box on the Drawing toolbar.

WORKAROUND

To work around this problem, resize the text box so that the text does not overflow the boundaries of the text box.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
