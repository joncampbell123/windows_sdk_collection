Unable to Change Source for a Microsoft Excel Chart Object

 Last reviewed: February 5, 1998
Article ID: Q144774
The information in this article applies to: 

 – Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
 – Microsoft Excel for Windows, versions 5.0, 5.0c
 – Microsoft Windows for Workgroups, version 3.11
  • Microsoft Windows 95, version 7.0

    SYMPTOMS

    When you change the link source for an existing Microsoft Excel chart object that is embedded in a Word document, the object fails to link to the source object that you specify.

    CAUSE

    This problem occurs when

    • You use a computer that is running Microsoft Windows for Workgroups, version 3.11, or Microsoft Windows 95.

      -and-

    • You paste a Microsoft Excel chart as an embedded linked object in a Word document.

      -and-

    • You change the link source for the object in the document while Microsoft Excel is still open.

    STATUS

    Microsoft has confirmed this to be a problem when using Microsoft Word version 6.0, 6.0a, or 6.0c for Windows. This problem was corrected in Microsoft Word for Windows NT and Microsoft Word for Windows 95.

    WORKAROUND

    Method 1: Close the Microsoft Excel worksheet before you change the source 

              of the link.
    Method 2: Paste the chart as an object rather than as a picture.

    Method 3: If you are using Windows or Windows for Workgroups, obtain the 

              Application Note titled "WW1116: OLE Version 2.03." This
          Application Note contains an upgraded version of OLE for
          Microsoft Windows for Workgroups and Microsoft Windows versions
          3.1 and 3.11. For additional information, please see the
          following article in the Microsoft Knowledge Base:

          ARTICLE-ID: Q123087
          TITLE     : OLE Version 2.03
