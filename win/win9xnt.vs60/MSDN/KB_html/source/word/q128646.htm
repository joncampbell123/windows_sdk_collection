Summary of Bookshelf Integration with Word for Windows

 Last reviewed: August 5, 1997
Article ID: Q128646
The information in this article applies to:
  • Microsoft Word for Windows versions 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a

SUMMARY

Different releases of Microsoft Bookshelf integrate differently with the different versions of Word for Windows. Following is a summary of how the various versions of Bookshelf accomplish the integration with Word. Instructions about how to remove Bookshelf are also included.

MORE INFORMATION

Bookshelf 1992 and Word 2.0

   Files Used:

           Filename:      Location:

           BOOKSHLF.DLL   \WINDOWS\SYSTEM
           BOOKSHLF.DOT   \WINWORD
           NORMAL.DOT     \WINWORD
           WORDDDE.DAT    \WINWORD
           DDEAPP.EXE     \WINWORD

           Note: The macros in BOOKSHLF.DOT are copied into NORMAL.DOT by
           running the wabSetup macro.

   WIN.INI changes:

           [Bookshelf]
           Bookspath=D:\BOOKS
           Viewerpath=C:\VIEWER\VIEWER.EXE
           Integration=1
           LastQuote=0

           [Microsoft Word 2.0]
           programdir=C:\WINWORD

   Removal: Remove the WAB and AUTOEXEC macros from NORMAL.DOT

Bookshelf 1992 and Word 6.0

   Files Used:

           Filename:      Location:

           BOOKSHLF.DLL   \WINDOWS\SYSTEM
           BOOKSHLF.DOT   \WINWORD\STARTUP
           WORDDDE.DAT    \WINWORD
           DDEAPP.EXE     \WINWORD

   WIN.INI changes:

           [Bookshelf]
           Bookspath=D:\BOOKS
           Viewerpath=C:\VIEWER\VIEWER.EXE
           Integration=1
           LastQuote=0

           [Microsoft Word 2.0]
           programdir=C:\WINWORD

           Note: The [Microsoft Word 2.0] section must be in the WIN.INI
           file. If it does not exist, it must be added because Bookshelf
           looks at the programdir= setting to find the Word for Windows
           files.

   Removal:  Remove BOOKSHLF.DOT from the WINWORD\STARTUP subdirectory.
NOTE: These products were not designed to integrate. You must perform a manual integration.

   Removal: Remove the WAB and AUTOEXEC macros from NORMAL.DOT.

Bookshelf 1994 and Word 6.0

   Files Used:

           Filename:      Location:

           BOOKSHELF.DLL  \WINDOWS\SYSTEM
           BSWORD.HLP     C:\BOOKS94
           BSHELF94.DOT   \WINWORD\STARTUP

   WIN.INI changes:

           [Bookshelf]
           Integration = 2
           LastQuote = 1
           BSWordHelp=C:\BOOKS94\BSWORD.HLP

   Removal: Remove BSHELF94.DOT from the \WINWORD\STARTUP directory.

            - or -

           1. Run Bookshelf Setup, choose Add/Remove.

           2. Clear the "Integration with Microsoft Word" check box.

           3. Choose Continue.

Office 4.3 and Bookshelf 1994 (also known as "Office and Bookshelf")

See "Word 6.0 and Bookshelf 1994."

Word 6.0 and Bookshelf 1994

Both "Office 4.3 and Bookshelf 1994" and "Word 6.0 and Bookshelf 1994" use an "integrated" version of Bookshelf that cannot be used with non- integrated applications, such as Word version 2.0. Both programs place files in the same locations as each other. 

   Files Used:

           Filename:      Location:

           BOOKSHELF.DLL  \WINDOWS\SYSTEM
           BSWORD.HLP     C:\BOOKS94
           BSH.WLL        \WINWORD\STARTUP

           Note: BSH.WLL in the integrated version replaces BOOKS94.DOT
           from the nonintegrated version.

   WIN.INI changes:

           [Bookshelf]
           Integration = 3
           LastQuote = 1
           BSWordHelp=C:\WINWORD\BSWORD.HLP

   Removal: Remove BSH.WLL from the \WINWORD\STARTUP directory.

            - or -

           1. Run Setup and choose the Add/Remove button.

           2. Clear the "Integration with Microsoft Word" check box.

           3. Choose Continue.

Bookshelf 1995

Bookshelf 1995 is integrated with Word 6.0. However, Bookshelf 1995 will not integrate with Word 2.0 for Windows because it uses a WLL file for the integration. Word 2.0 does not recognize WLL files. 

   Files Used:

           Filename:      Location:

           BSH.WLL        \WINWORD\STARTUP
           BS95CTW.EXE    C:\BOOKS95

   WIN.INI changes:

           [Bookshelf]
           Integration = 2
           LastQuote = 1
           BSWordHelp=D:\BOOKS\BSHELF95.HLP

   BSHELF95.INI file (created by this installation):

           [WORDOTD]
           TipContext=0x8327
           TipMVB=wordtips.m11

   Removal: Remove BSH.WLL from the \WINWORD\STARTUP directory.

             - or -

            1. Run Setup, choose Add/Remove.

            2. Clear the "Integration with Microsoft Word" check box.

            3. Choose Continue.

Bookshelf 96-97 and Word 7.0

On Windows 95 or Windows NT 3.51, Bookshelf integrates with Word 7.0. There is a Lookup Reference command on the Tools menu and a shortcut menu for Define.

Removal:

  1. Run Setup, and click Add/Remove.

  2. Click the "Integration with Microsoft Word" check box to clear it.

  3. Click Continue.

Bookshelf 96-97 and Word 6.0

On Windows 3.x, Bookshelf can be integrated with Word 6.0. The only feature is a shortcut menu for the Define command. 

   Removal: Remove BSH.WLL from the \WINWORD\STARTUP directory.

            - or -

            1. Run Setup, and click Add/Remove.

            2. Click the "Integration with Microsoft Word" check box to
               clear it.

            3. Click Continue.
