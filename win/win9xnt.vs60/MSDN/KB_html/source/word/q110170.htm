Printing Envelopes Using the IBM 4029 Print Accelerator

 Last reviewed: February 5, 1998
Article ID: Q110170
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c

SYMPTOMS

When you print envelopes with the IBM 4029 Print Accelerator version 1.0 using the Word 6.0 for Windows Envelopes And Labels command, some sizes of envelopes may not print at all or may print in the wrong position.

CAUSE

If you try to print envelope sizes other then #10, #9, DL, C5, B5, or Monarch, the printer resets and nothing is printed.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Envelope sizes of #10, #9, DL, C5, B5, and Monarch are included in the printer driver and print correctly once you set the correct feed method using the envelope feed method icon. For example:

  1. From the Tools menu, choose Envelopes And Labels.

  2. Choose Options.

  3. Choose the Printing Option tab.

  4. Choose the Right (instead of the default choice of Center) feed method icon to match the way the 4029 accepts envelopes.

  5. In the Feed From list, select Envelope if you are using the 4029 envelope feeder.

Any other size of envelope will cause the printer to reset and nothing will be printed. Refer to the workaround below for information on how to print other size envelopes.

WORKAROUND

To work around this problem:

  1. Choose the next largest size of envelope from the envelope sizes mentioned above.

  2. Add the envelope to a document.

  3. Increase the margins to create a workspace the size of the envelope you want to print.

  4. Print page 0 to print the envelope.
