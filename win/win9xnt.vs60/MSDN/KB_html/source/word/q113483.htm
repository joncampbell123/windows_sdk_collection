Word 6.0a Patch Fails After Installing Office 4.2

 Last reviewed: February 6, 1998
Article ID: Q113483
6.00 6.00a WINDOWS

The information in this article applies to:

  • Microsoft Word for Windows, versions 6.0, 6.0a
  • Microsoft Windows operating system version 3.1

SYMPTOMS

When you run the Word 6.0a Patch setup program, you may receive the following error message 

   Unable to find the .STF file in the WIN.INI.
if you have Microsoft Office version 4.2 for Windows on your computer and if you installed it in the following specific sequence:

  1. You installed the full retail Office 4.0 package--not the Office 4.0 upgrade package.

  2. You installed the Office 4.2 upgrade--not the full retail Office 4.2 package.

  3. You tried to upgrade to Word 6.0a for Windows using the Word 6.0a patch.

NOTE: This problem does not occur if you installed Office 4.0 using the upgrade package or if you installed Word 6.0 before you installed Office 4.0. Similarly, it does not occur if you installed Office 4.2 using the full retail package.

CAUSE

When you upgrade to Office 4.2, Setup replaces your old SETUP.STF file with a new SETUP.STF file; the new .STF file does not contain the Word 6.0 information the Word 6.0a Patch needs to upgrade your Word for Windows program. The Word 6.0a Patch searches in the [MS Setup (ACME) Table Files] section of your WIN.INI file for either of the following two entries: 

   WINWORD.EXE@v6.0.0.2226(1033)=C:\WINWORD\SETUP\SETUP.STF

   MSOFFICE.EXE@v1.0.0.0000(1033)=C:\MSOFFICE\SETUP\SETUP.STF
In the installation sequence described above, there is no WINWORD.EXE line in the [MS Setup (ACME) Table Files] section of your WIN.INI file (only Word 6.0 Setup, not Office Setup, writes the WINWORD.EXE line to your WIN.INI). As a result, the Word 6.0a Patch only looks at the SETUP.STF file specified by the MSOFFICE.EXE line, which is the .STF file installed by Office 4.2 that does not contain any Word 6.0 information.

In other words, this problem occurs when:

  1. There is no WINWORD.EXE line in the [MS Setup (ACME) Table Files] section of your WIN.INI file.

    -and-

  2. The Word 6.0a Patch looks at the SETUP.STF file installed by the Office 4.2 upgrade, which does not contain any Word 6.0 information.

STATUS

We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

  1. Rename the SETUP.STF file installed by the Office 4.2 upgrade (for example, rename the file to SETUP.OLD).

  2. Reinstall Office 4.0, but choose to only install Word 6.0. You do not need to reinstall Microsoft Excel or PowerPoint if you have already upgraded to Microsoft Excel 5.0 and PowerPoint 4.0. This restores the SETUP.STF file that contains Word 6.0 information.

  3. Run the Word 6.0a Patch again.

  4. Upgrade to Office 4.2.
