WD6X: Underlined Tab Extends to Margins When Printed

 Last reviewed: February 5, 1998
Article ID: Q154307
The information in this article applies to:
  • Microsoft Word for the Macintosh, versions 6.0.1a

SYMPTOMS

In Microsoft Word for the Macintosh, if you apply underline formatting to a tab character and then print your document, the underline extends from the left margin to the right margin instead of from the last character before the tab to the tab stop position. However, the underline appears to be correct on screen.

For example, if you type the word "Name" (without the quotation marks), press the TAB key, click the Underline button and then press the TAB key again, the underline appears under the second tab, but is printed beginning at the left margin and extending to the right margin.

STATUS

We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

To work around this problem, use any of the following methods.

Method 1: Use a Tab Leader Instead of the Underline Format:

To do this, follow these steps:

  1. Select the tab character to which you applied the underline format and remove the underline format.

  2. On the Format menu, click Tabs.

  3. In the Tab Stop Position box, select the tab to which you want to apply the leader.

  4. Click the underline tab leader (number 4) and click OK.

Method 2: Change the Font to a TrueType Font:

Select the tab characters to which you applied the underline formatting, and change the font to a TrueType font, such as Arial. To do this, follow these steps:

  1. On the Edit menu, click Replace.

  2. Place the insertion point in the Find What box.

  3. Click the Special button, and then click Tab Character.

  4. Place the insertion point in the Replace With box.

  5. Click the Special button, and then click Tab Character.

  6. Click the Format button and select Font. Change the font to a TrueType font (such as Arial) and then click OK.

  7. Click Replace All.

  8. Click Close.

NOTE: When changing to a TrueType font, make sure that the font does not have a bitmap (screen) version installed for the point size you are using. For example, although Times is a TrueType font, it usually has corresponding bitmap versions installed for popular sizes (9, 10, 12, 14, 18, 24 points). If you choose one of these sizes, the problem still occurs. To avoid this problem, choose a size for which no corresponding bitmap equivalent is installed (such as 13 or 15 points) and apply it to the tab characters. Changing the size of the underlined tab character by a few points will not make a significant difference in the thickness of the printed line.

Method 3: Remove All the Bitmap (Screen) Fonts from the System

This is the preferred method in cases where a large number of documents are affected because you don't have to edit each individual document.

  1. Quit all programs.

  2. Open the System Folder.

  3. Open the Fonts Folder.

  4. On the View menu, click By Icon.

  5. Open the suitcase for a specific font, such as Helvetica, and drag the bitmap fonts to the desktop (or a separate folder on the desktop).

    Bitmap fonts can be identified as follows: 

        - A single "A" in the center of a plain document icon.


        - The font name includes the point size, for example, "Helvetica 9."

  6. Restart your computer.
