No Blank Line Suppression When Printing Mail Merge Main Doc

 Last reviewed: February 5, 1998
Article ID: Q110304
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows NT, version 6.0
  • Microsoft Word for Windows 95, versions 7.0, 7.0a

SYMPTOMS

If you create a MailMerge Main document, use the View Merge Data option when one of the mergefields is blank, and then print the main document, you get a blank line where the blank mergefield is, even though it looks correct on the screen.

STATUS

Microsoft has confirmed this to be a problem in the Word versions listed above. This problem was corrected in Word 97 for Windows

WORKAROUND

To work around this problem, disable background printing. To do so:

  1. From the Tools menu, choose Options.

  2. Select the Print tab.

  3. Clear the Background Printing option.
