Unexpected Results Inserting FreeHand 5.0 Objects into Word

 Last reviewed: February 6, 1998
Article ID: Q155112
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a

SYMPTOMS

Case 1

When you attempt to insert a FreeHand object into a Word document, you get one of the following results:

  • FreeHand opens but does not maximize.
  • The hard drive runs continuously.
  • An icon displays.

Case 2

When you paste a FreeHand object into a Word document, you get the following error messages:

  • There is insufficient memory. Save the Document now.
  • There is not enough memory to complete the operation.

STATUS

We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

To work around this problem, paste the FreeHand object using one of the following methods in Word:

  • Click Paste Special on the Edit menu, select Bitmap, and click OK.
  • Click Paste Special on the Edit menu, select Device Independent Bitmap, and click OK.
