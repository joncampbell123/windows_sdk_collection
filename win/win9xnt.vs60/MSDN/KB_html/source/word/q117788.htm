Merge Revisions Error: "Original Text...Is Different"

 Last reviewed: February 5, 1998
Article ID: Q117788
6.00 6.00a WINDOWS

The information in this article applies to:

  • Microsoft Word for Windows, versions 6.0, 6.0a

SYMPTOMS

If you merge revisions from one version of a document to the original, the following error message occurs if the revised document contains any unmarked revisions: 

   The original text of the merged document is different.
   Merge first part of document?
Note: To merge revisions, choose Revisions from the Tools menu and choose the Merge Revisions button.

CAUSE

This can happen if your document contains a DATE or TIME field that you update while revision marking is turned off. The updated field result is an unmarked change between the two versions of the document.

WORKAROUND

Method 1: Lock the fields before you send a document out for revisions.

Method 2: Leave Revision Marking turned on when you update the fields 

          in either version of the document.

REFERENCES

"Microsoft Word User's Guide," version 6.0, pages 564-565
