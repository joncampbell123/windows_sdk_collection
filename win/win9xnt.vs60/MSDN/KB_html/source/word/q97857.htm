"Use As Default" Causes WordBasic Error in Letter Template

 Last reviewed: July 30, 1997
Article ID: Q97857
The information in this article applies to:
  • Microsoft Word for Windows, versions 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c

SUMMARY

You may get one of the following WordBasic error messages while using one of the letter templates supplied with Word for Windows (that is, LETBLOCK.DOT, LETMDSEM.DOT, LETMODBK.DOT, and LETPERSN.DOT): 

   WordBASIC Err=102
   Command failed

   WordBASIC Err=124
   Undefined SUB or FUNCTION

   WordBASIC Err=509
   Command Unavailable
These errors occur if:
  • The letter template itself is open.

-and-
  • You have chosen Set Letter Option from the Format menu; chosen Letterhead, Salutation, or Closing; and then chosen Use As Default.

When the error message appears, if you choose OK, the error closes, and the NORMAL.DOT file is left open on the screen. To avoid making any changes to the Normal template, close it and choose No in the dialog box that prompts you to "save global, glossary, and command changes."

Workarounds

If you want to modify a letter template, do one of the following:

  • Make your changes in the open template file, then choose Save from the File menu. Do not choose the Save As Default button.
  • In a document based on the template, choose the Save As Default button. (Keep the template file itself closed.)

Both procedures will save changes to the original template file without generating error messages.

If you want to modify a document based on a template without modifying the template itself, do not choose the Save As Default button.

MORE INFORMATION

These errors occur because one of the Set Letter Options macros tries to save the default setting in the Normal template if the letter template itself is open.

The Save As Default option is designed to let you save a desired change to the letter template itself while you are working in a document based on the template. For this option to work correctly, the letter template itself must be closed.
