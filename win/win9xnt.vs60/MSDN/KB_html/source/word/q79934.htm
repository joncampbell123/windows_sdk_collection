WinWord: Soft Fonts Installed But Cannot Print

 Last reviewed: February 5, 1998
Article ID: Q79934
The information in this article applies to:
  • Microsoft Word for Windows, versions 1.0, 1.1, 1.1a, 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c, 6.0, 6.0a, 6.0c

SUMMARY

If you install soft fonts on your computer and they appear in the available fonts list, but they do not print, you can force Hewlett-Packard (HP) printer drivers to re-create the font database; this enables the fonts to print.

MORE INFORMATION

To edit a font that appears on your list, but does not print, use the following steps to cause the printer driver to re-create the font database:

  1. From the File menu, choose Print Setup.

  2. Select the HPPCL or HPPCL5A driver, and choose the Setup button.

  3. Choose the Fonts button.

  4. Select one of the installed fonts from the left box.

  5. Choose the Edit button.

  6. Do not make any changes, and choose the OK button.

  7. In the HP Font Installer dialog box, choose the Exit button.

  8. In the HP Setup box, choose the OK button.

  9. In the File Print Setup box, choose the OK button.

A dialog box stating that the font database is being rebuilt appears presently. The rebuilt font database allows printing of soft fonts.

For more information on installing soft fonts, query on the following words in the Microsoft Knowledge Base: 

   adding and deleting and soft and font and installer and winword.ini

REFERENCES

"Microsoft Word for Windows Printer Guide," version 1.x, pages 24-28

"Microsoft Word for Windows User's Guide," version 2.0, pages 122-123
