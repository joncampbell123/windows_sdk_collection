WD98: Pressing RETURN Doesn't Activate OK Button in Dialog Box

 Last reviewed: February 3, 1998
Article ID: Q178063
The information in this article applies to:
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

When you press RETURN in the following dialog boxes, the OK button fails, even though the OK button may appear to have focus: 

   Insert Word Field: Ask
   Insert Word Field: Fill-In
   Insert Word Field: If
   Insert Word Field: Set
(To open these dialog boxes click Insert Word Field on the Mail Merge toolbar and then click Ask, Fill-in, If...Then...Else, or Set Bookmark, respectively, on the menu that appears.)

CAUSE

This behavior occurs because the text box accepts carriage returns. For example, the Value box in the "Insert Word Field: Set" dialog box allows you to enter text including carriage returns. If the pointer is in this box when you press RETURN, a carriage return is inserted in the Value box. The "Insert Word Field: Set" dialog box is not closed.

WORKAROUND

To work around this behavior, use either of the following methods:

  • Press ENTER on the Numeric keypad.

-or-
  • In the dialog box, press TAB, so that another control in the dialog box has focus and then press RETURN. For example, in the "Insert Word Field: Set" dialog box press TAB until the Bookmark box has focus and then press RETURN.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
