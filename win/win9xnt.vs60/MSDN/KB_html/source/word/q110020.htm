CTRL+HOME Moves Insertion Point to Framed Object

 Last reviewed: February 5, 1998
Article ID: Q110020
The information in this article applies to:
  • Microsoft Word for Windows, version 6.0

SYMPTOMS

Pressing the CTRL+HOME key combination may move the insertion point to a framed object instead of moving the insertion point to the beginning of the document.

CAUSE

This problem occurs when an anchor is attached to the first paragraph of the document. If you view the document in normal view, you can see that the framed object is really the first item in the document.

STATUS

Microsoft has confirmed this to be a problem in Word for Windows version 6.0. This problem was corrected in Word for Windows version 6.0a.

RESOLUTION

To work around this problem:

  • Move the anchor to a paragraph other than the first one in the document, like the second paragraph. To do this, drag the anchor to another paragraph.

    -or-

  • Move to the beginning of the document by pressing the Page Up key enough times and insert a bookmark. You can then use the F5 key to move to the bookmark.

MORE INFORMATION

A frame is always anchored to a paragraph. As you move the frame, the anchor moves to the closest paragraph. Locking an anchor to a specific paragraph ensures that the frame always appears on the same page as the paragraph.

To view anchors of framed objects, choose Options from the Tools menu, select the View tab, and select All in the Nonprinting Characters list, or click the Show/Hide button on the Standard Toolbar. Then click the framed object.
