cc:Mail Address Book Sorted and Listed by First Name in Word

 Last reviewed: August 4, 1997
Article ID: Q117704
6.00a WINDOWS kb3rdparty kbprb

The information in this article applies to:

  • Microsoft Word for Windows, version 6.0a

SYMPTOMS

In Word for Windows, with Lotus cc:Mail as your electronic mail (E-mail) program, if you use the Send or Add Routing Slip commands on the File menu and access the address book, the names may not appear in the same order as they display when you access the address book directly in cc:Mail.

For example, in Word, the names may display in "FirstName LastName" format, sorted by either first or last name, while they appear in cc:Mail in "LastName, FirstName" format, sorted by last name.

This discrepancy makes it difficult to look up names, especially in large address books.

Note: The problem described in this article is not the same as the Word 6.0 MAPIVIM.DLL problem in which Word 6.0 hangs when you send E-mail via cc:Mail. For information about this problem, which was corrected in Word 6.0a, query on the following words in the Microsoft Knowledge Base: 

   mapivim.dll and hang and cc:mail and hourglass

CAUSE

This discrepancy occurs only if you are using a version of the cc:Mail VIM.DLL file with a creation date later than 09/09/93 and earlier than 03/22/94.

WORKAROUNDS

Method 1: Lotus has an updated version of the VIM.DLL file, dated 03/22/94, 

          that corrects this problem. You can obtain this file from any of
          the following sources:

           - Phone cc:Mail Technical Support at (415) 966-4900.

           - Download the file VDLWIN.ZIP from the cc:Mail Technical
             Support BBS (bulletin board system) at (415) 691-0401.

           - Download the file VDLWIN.ZIP from CompuServe, in the cc:Mail-
             VIM section.
Method 2: As a temporary workaround, you can install an earlier version of 
          the VIM.DLL file, dated 09/09/93 or earlier.
