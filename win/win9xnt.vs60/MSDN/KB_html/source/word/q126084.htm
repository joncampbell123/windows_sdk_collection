SGML: Inference Parser Error Annotations in Feedback File

 Last reviewed: August 5, 1997
Article ID: Q126084
The information in this article applies to:
  • Microsoft SGML Author for Word, version 1.0
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c

SYMPTOMS

After a Word document is saved as SGML, the Feedback (.FBK) file contains annotations for Inference Parser Errors such as: 

   Could not find external document type "DOC"
   No definition for DOC document type; "DOC O O ANY" assumed
   Possible attributes treated as data because none were defined

CAUSE

The Path statement in the Entity Catalog (ENTM.CAT) is incorrect or the DTD, DCL, and MAP files are not in the same directory.

RESOLUTION

Check the Path statement in your Entity Catalog and verify that it points to the location where SGML Author is installed.

If the Path statement in your Entity Catalog file is correct, make sure that your DTD and DCL files are located in the same directory as the association mapping (*.MAP) file you are using.

You can open and edit the Entity Catalog with any text editor.
