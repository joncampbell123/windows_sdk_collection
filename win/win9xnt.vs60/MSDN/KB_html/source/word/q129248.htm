WD6: Files Not Overwritten Running IA Setup over Old Version

 Last reviewed: February 5, 1998
Article ID: Q129248
The information in this article applies to:
  • Microsoft Internet Assistant for Word, version 1.0
  • Microsoft Word for Windows, versions 6.0a, 6.0c

SYMPTOMS

If you manually remove the Internet Assistant Setup files and then install a more recent version of Internet Assistant, some files, such as WORDHTML.INI and IWIA.INI, may not be overwritten. This behavior occurs even when you choose to have the Setup program overwrite existing files.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Internet Assistant version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUNDS

Use one of the following methods to remove and install WD6: Internet Assistant

Method 1: Remove the previous version of Internet Assistant by rerunning 

          the original Setup program. Choose the Remove All button to
          remove all of the files. Then, run Setup for the updated
          version.
Method 2: If you do not have the original Setup program, you can remove 
          Internet Assistant by deleting the following files (where
          <WINWORD> is usually C:\WINWORD, and <WINDOWS> is usually
          C:\WINDOWS):

           - HTML.DOT and WEBVIEW.DOT, located in the <WINWORD>\TEMPLATE
             subdirectory

           - WORDHTML.WLL, located in the <WINWORD>\STARTUP subdirectory

           - All files in the <WINWORD>\INTERNET subdirectory

           - WORDHTML.INI, located in your <WINDOWS> directory

           - IWPS.DLL, located in the <WINDOWS>\SYSTEM directory

          For more information about removing files before installing an
          updated version, please see the README.DOC file, which is
          contained in the compressed WORDIA.EXE file.
