WD: ImageWriter Error, "Can't Open Printer..." from MacWord

 Last reviewed: February 5, 1998
Article ID: Q106119

The information in this article applies to:
  • Microsoft Word for the Macintosh, versions 4.0, 5.0, 5.1, 6.0

SYMPTOMS

When printing to an ImageWriter, the Macintosh gives the error message: 

   "Can't open printer. Use the Chooser command on the Apple
    menu to select another printer."
The error occurs when printing in faster or best quality, but not when printing in draft quality.

CAUSE

This problem occurs when using FileGuard, by ASD Software, to lock the root of the active startup drive. When printing to an ImageWriter in faster or best mode, Word creates a spool file in the root directory, and if FileGuard locks the root, Word cannot print.

When printing in draft mode, the ImageWriter uses internal fonts (fonts built into the ImageWriter), so Word does not need to create a spool file, and the problem does not occur.

NOTE: Other programs may behave differently because they may not put the spool file in the root directory.

WORKAROUND

In FileGuard, set the permissions for the root level of the startup drive to allow files to be written. For assistance, consult the FileGuard manual or contact FileGuard technical support.

MORE INFORMATION

FileGuard limits access to files, folders, and volumes on the Macintosh.
