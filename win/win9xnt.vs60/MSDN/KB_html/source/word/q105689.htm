"Disk Is Full..." Saving File Under Windows for Workgroups

 Last reviewed: February 6, 1998
Article ID: Q105689
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Windows for Workgroups, version 3.1

SYMPTOMS

When you save a document in Word for Windows, the following error message may occur if you run Word under Windows for Workgroups in standard mode: 

   The disk is full or too many files are open.
   (C:\WINWORD6\DOC1.DOC)
A similar error message may occur when Word performs an autosave operation: 

   The disk is full or too many files are open.
   (C:\WINWORD6\~WRANNNN.ASD)

CAUSE

This happens when you run Word under Windows for Workgroups in standard mode if the parameters for the SHARE.EXE utility are set too low or if no parameters are set (which causes Share to use its default settings). The Word Setup program adds the following line to your AUTOEXEC.BAT file to load Share: 

   <path>\SHARE.EXE /L:500 /F:5100
where <path> is the drive and directory where the SHARE.EXE file is located (usually the MS-DOS directory).

This may also occur if you are using Windows for Workgroups version 3.11 and the line, DEVICE=IFSHLP.SYS is missing or misspelled in CONFIG.SYS file.

RESOLUTION

If your Share settings are lower than the ones listed above, you must modify them in your AUTOEXEC.BAT file.

Note: If your Share settings are higher than those listed above, do not reduce the settings; other applications on your system may require the higher settings.

For more information on loading Share, query on the following words in the Microsoft Knowledge Base: 

   share.exe and config.sys and autoexec.bat and word and 6.0

WORKAROUND

From the File menu select Save As. From the Save File as Type drop-down scroll box select, Rich Text Format and choose OK.

MORE INFORMATION

To run Word 6.0 under Windows version 3.1 or Windows for Workgroups version 3.1 in standard mode, you must run the MS-DOS SHARE command before starting Windows. Word uses Share for object linking and embedding (OLE) version 2.0 operations. Specifically, Share provides the range-locking functionality that Word uses in compound files. Word 6.0 will not start under Windows 3.1 or Windows for Workgroups 3.1 in standard mode unless SHARE.EXE is loaded.

You do not need SHARE.EXE if you run Word under Windows for Workgroups 3.1 in enhanced (386) mode. Windows for Workgroups uses a comparable file-sharing utility called VSHARE.386. Similarly, you do not need SHARE.EXE if you run Word under Windows NT because this operating system includes built-in range-locking functionality.
