WD: Incorrect Smart Quotes Inserted in Protected Form Field

 Last reviewed: February 3, 1998
Article ID: Q125249
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for the Macintosh, version 6.0
  • Microsoft Word for Windows NT, version 6.0

SYMPTOMS

You do not get an opening quotation mark in a protected form field when you have the "Change 'Straight Quotes' to 'Smart Quotes'" option selected. (To locate this option, choose AutoCorrect from the Tools menu.) This behavior occurs when the first character in the form field is the opening quotation mark. It does not occur when other text precedes the opening quotation mark.

WORKAROUNDS

Method 1

Clear the "Change 'Straight Quotes' to 'Smart Quotes'" option before typing the form field text.

Method 2

When "Change 'Straight Quotes' to 'Smart Quotes'" is turned on, insert a space before typing the opening quotation mark in the protected form field.

STATUS

This problem was corrected in Word 7.0 for Windows 95; note also that Word 97 for Windows and Word 98 Macintosh Edition do not use smart quotes in form fields.
