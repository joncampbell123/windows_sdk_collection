WD: Compare Versions Does Not Mark Some Modified Field Results

 Last reviewed: February 5, 1998
Article ID: Q109532
The information in this article applies to:
  • Microsoft Word for Windows, version 6.0
  • Microsoft Word 97 for Windows

SYMPTOMS

The Compare Documents (Compare Versions) command may not mark some field results with revision marks, even though the fields are different between the two documents. The following message may be displayed: 

   Compare Documents encountered one or more fields with content longer
   than 750 characters. Word could not track changes for these fields.
NOTE: To locate the Compare Documents (Compare Versions) command, point to Track Changes (Revisions) on the Tools menu and click Compare Documents (Compare Versions).

CAUSE

The Word Compare Documents (Compare Versions) command skips field results that contain more than 750 characters.

Work Around

Convert problem fields in the document to text.

  1. Place your cursor in the field

  2. Press Ctrl + Shift + F9

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

REFERENCES

"Microsoft Word User's Guide," version 6.0, Chapter 25, "Annotating, Revising, and Routing Documents," "Comparing Versions of a Document" section
