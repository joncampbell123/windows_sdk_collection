WD: Avery Label 5267 Mail Merge Prints Incorrectly

 Last reviewed: February 3, 1998
Article ID: Q170874
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for the Macintosh, versions 6.0, 6.0.1, 6.0.1a
  • Microsoft Word for Windows 95, versions 7.0, 7.0a

SYMPTOMS

When you perform a mail merge using Avery 5267, Return Address label, as your main document, if you have more than 80 records in your data source, the labels are printed incorrectly with an extra row of labels added to each page.

CAUSE

The 5267 Return Address label main document format contains 4 columns and 20 rows, so when it is merged, each page of labels should consist of 20 rows. The bottom margin of the mail merge main document for the 5267 Return Address label is set to 0 inches, so when the merge is performed, it prints the twenty-first row on page one instead of moving it to the second page.

WORKAROUND

To work around this problem, follow these steps to set the bottom margin of the mail merge main document to .25 inches:

  1. In the main document, click Page Setup on the File menu.

  2. Click the Margins tab.

  3. Change the Bottom Margin from 0" to .25" and then click OK.

STATUS

This problem was corrected in Microsoft Word 97 for Windows and Word 98 Macintosh Edition.

MORE INFORMATION

The details of the Avery 5267, Return Address label, are: 

   Top Margin = .5"
   Side Margin = .28"
   Vertical Pitch = .5"
   Horizontal Pitch = 2.06"
   Label Height = .5"
   Label Width = 1.75"
   Number across = 4
   Number down = 20
