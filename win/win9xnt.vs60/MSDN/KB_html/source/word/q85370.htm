Footnotes Lost After Applying Change Case in Word 5.0

 Last reviewed: July 30, 1997
Article ID: Q85370
The information in this article applies to:
  • Microsoft Word for the Macintosh, version 5.0

SUMMARY

In version 5.0 of Microsoft Word for the Macintosh, if you apply the Change Case command to text containing auto-numbered footnote reference marks, the link between the selected footnote reference marks and their corresponding footnotes may be lost.

If this occurs, the footnote text associated with footnote reference marks in the selected body text will disappear, and unselected footnote reference marks will be renumbered. The footnote reference marks in the case-changed text will remain in the body text, but they will be renumbered in various ways. If you double-click these unlinked footnote reference marks, additional footnote reference marks and footnotes numbers will be added.

Microsoft is researching this problem and will post new information here as it becomes available.
