IA: "Copy Hyperlink" from Unsaved Document Is Invalid

 Last reviewed: February 5, 1998
Article ID: Q130766
The information in this article applies to:
  • Microsoft Internet Assistant for Word, versions 1.0, 1.0z
  • Microsoft Word for Windows, versions 6.0a, 6.0c

SYMPTOMS

Selecting a pasted hyperlink does not jump you to the original hyperlink, and you do not receive an error message. This behavior occurs when using the following steps:

  1. In a new HTML document, you insert a hyperlink, then copy the hyperlink (by choosing Copy Hyperlink from the Edit menu).

  2. You save the original document.

  3. You open a second HTML document and paste the hyperlink.

  4. You switch to Web Browse view and select the hyperlink.

CAUSE

The hyperlink is invalid. An invalid hyperlink will result if the source document has not been saved prior to copying the hyperlink. This occurs because Internet Assistant will paste the hyperlink like this: 

   LOCAL:C/MSOFFICE/WINWORD/DOCUMENTx
where x represents the number of the Word document created during the current session of Word.

When the original document is saved, Internet Assistant will no longer be able to reference it as DOCUMENTx.

STATUS

Microsoft has confirmed this to be a problem in Internet Assistant for Word, versions 1.0 and 1.0z. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

RESOLUTION

Save the file from which you copy a hyperlink before copying the hyperlink. The pasted hyperlink will correctly identify the document's path and filename.
