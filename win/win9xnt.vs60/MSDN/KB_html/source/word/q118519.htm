WD: Framed Object After Manual Page Break Moves to Previous Page

 Last reviewed: February 5, 1998
Article ID: Q118519
The information in this article applies to:
  • Microsoft Word for Windows, versions 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c, 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Word 97 for Windows
  • Microsoft Windows operating system versions 3.0 and 3.1

SYMPTOMS

If you insert a frame on a page containing a single paragraph mark that immediately follows a blank page, the frame may position itself either at the bottom of the previous blank page or in the last paragraph that contains text preceding the blank page. This behavior also occurs when you select an existing object--such as a picture, text, or equation--and apply a frame to the selection.

STATUS

We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUNDS

Method 1: Insert an extra paragraph mark on the page following the blank 

          page before inserting the frame.
Method 2: Insert a Continuous Section Break immediately following the 
          manual page break.
Method 3: Insert a Next Page Section Break instead of a manual page break.

REFERENCES

"Microsoft Word for Windows User's Guide," version 2.0, pages 390-419
