SGML Converted Files Show Incorrect Page Numbers

 Last reviewed: August 5, 1997
Article ID: Q126292
The information in this article applies to:
  • Microsoft SGML Author for Word, version 1.0
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c

SYMPTOMS

When you open an SGML document in Word for Windows, all page numbers take on the Continue From Previous Section property. In other words, page numbers that were supposed to restart at the beginning of a section now continue the previous section's numbering.

Even though the page numbers appear incorrectly, the SGML file does contain the correct information.

STATUS

Microsoft has confirmed this to be a problem in SGML Author for Word, version 1.0. Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
