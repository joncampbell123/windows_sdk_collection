Labels Print Incorrectly with the Generic Text Only Driver

 Last reviewed: February 5, 1998
Article ID: Q123699
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c

SYMPTOMS

If you perform a mail merge to print labels to a Panasonic KX-P1124 printer while using the Generic Text Only version 3.10 printer driver, your labels will feed incorrectly:

Case 1: If No Page Break is selected in the printer driver setup, the 

        labels will not form feed correctly to the next page of labels.
        The first line that should print on the second page prints at
        the bottom of the last label on the first page.
Case 2: If you clear the No Page Break option, a full sheet of labels 
        will form feed before continuing to print.
This functionality worked correctly in Word 2.x.

STATUS

We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

Switch to the Panasonic KX-P1124 printer driver, change the Print Quality setting to Draft, and use a printer font such as Courier.

To change the Print Quality setting:

  1. From the File menu, choose Print.

  2. Choose the Printer button.

  3. Select the Panasonic KX-P1124 printer and choose the Set As Default Printer button.

  4. Choose the Options button.

  5. From the Print Quality list, choose Draft.

  6. Choose the OK button.

  7. Choose the Close button.

  8. Choose the OK button to print the labels or choose Close to return to the document.
