WD95: Error Setting Document Location to Banyan or Pathworks

 Last reviewed: July 31, 1997
Article ID: Q152771
The information in this article applies to:
  • Microsoft Word for Windows 95, version 7.0

SYMPTOMS

When you change the default document location to a Banyan Vines server or DEC Pathworks server, you get the following error message: 

   Directory not valid.

CAUSE

Banyan Vines and DEC Pathworks do not support the Universal Naming Convention (UNC) required by Word to locate the default file location. Therefore, Word will not identify the server location as a valid directory.

WORKAROUND

To work around this problem, first set the default document path in Microsoft Word to a folder that contains a shortcut to the Banyan Vines or DEC server. To do this, follow these steps:

  1. Create a shortcut on your desktop to the Banyan Vines or DEC server.

  2. In Word, use the following steps to establish a path to the shortcut:

    a) Start Word.

    b) On the Tools menu, click Options and then select the File 

          Locations tab.

    c) Select Documents, and click the Modify button.

    d) From the Look in list, select Desktop.

    e) Select the shortcut you created in step 1.

    f) Click the OK button.

    g) Click the Close button.

When you save or open a document, click the shortcut to select the location for your document and then save or open your document by:

  1. To open a document:

    a) On the File menu, click Open.

    b) The Look In area should point to the Banyan Vines or DEC 

          server; if it does not, select Desktop and then double-click
      the shortcut folder to the Banyan Vines or DEC server.

    c) Click the file you want to open and then click Open.

  2. To save a document:

    a) On the File menu, click Save As.

    b) The Save In area should point to the Banyan Vines or DEC 

          server; if it does not, select Desktop and then double-click
      the shortcut folder to the Banyan Vines or DEC server.

    c) Enter a file name and then click Save.

STATUS

Microsoft has confirmed this to be a problem in the products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
