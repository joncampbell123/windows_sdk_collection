WD: Case-Sensitive Custom Styles Lost in RTF Conversion

 Last reviewed: July 31, 1997
Article ID: Q165741
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word 97 for Windows

SYMPTOMS

If you create a user-defined style that has the same style name as a built- in style, only with different capitalization in the name, the user-defined style will be lost if you save the document in Rich Text Format (RTF).

NOTE: This behavior also occurs if you save a document in the Word 6.0/95 format, because Word 97 creates an RTF file (with a .doc extension) when you save a document in this format.

RESOLUTION

Do not give the user-defined styles the same names as the built-in styles. To rename a style, follow these steps:

  1. Position the insertion point on a paragraph that is formatted with the style you want to rename.

  2. On the Format menu, click Style.

    The style you want to rename should be selected on the left.

  3. Click the Modify button.

  4. Type a new name for the style.

  5. Click OK and then click Close.

Your style will now be renamed throughout the document.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
