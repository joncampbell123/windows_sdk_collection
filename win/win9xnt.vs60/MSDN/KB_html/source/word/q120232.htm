Word Unexpectedly Quits While Modifying Font Name

 Last reviewed: February 5, 1998
Article ID: Q120232
The information in this article applies to:
  • Microsoft Word for the Macintosh, version 6.0

SYMPTOMS

When you modify a font name, Word for the Macintosh unexpectedly quits (that is, the application closes) or hangs (stop responding).

CAUSE

Word may quit or hang if you try to paste more than 31 characters in the Font dialog box. When you type (rather than paste) in the box, Word beeps and prevents you from typing more than 31 characters, so this problem does not occur when you modify the Font entry this way. However, Word does not prevent you from trying to insert more than 31 characters in this box when you paste the characters from the Clipboard.

WORKAROUND

If you paste text in Font box in the Format Font dialog box, be sure you do not paste more than 31 characters for the Font name.

STATUS

Microsoft has confirmed this to be a problem Word version 6.0 for Macintosh. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
