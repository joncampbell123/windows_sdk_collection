WD: Date and Time Won't Update in HTML Document

 Last reviewed: July 31, 1997
Article ID: Q167374
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word 97 for Windows

SYMPTOMS

When you insert a Date And Time field into a HyperText Markup Language (HTML) document using Word 97 and select the Update Automatically check box (click Date And Time on the Insert menu), the date and time does not update as expected.

CAUSE

The Date And Time field is converted to plain text when the document is saved as HTML.

WORKAROUND

Method 1: Reinsert the Date And Time field

Delete the previous date and time and reinsert the date and time by clicking Date And Time on the Insert menu.

NOTE: If you select the Update Automatically check box, a Date and Time field will be inserted into your document; however, when you save the document as HTML, the Date And Time field will be saved as plain text.

Method 2: Add a Web counter

Add a Web counter to your HTML page. Most Web counters include a feature to display the current date and time using a CGI script. For more information, contact your Internet service provider.

MORE INFORMATION

The Update Automatically option should not be available when you edit an HTML document, since the Date and Time field is saved as plain text.
