WD: Can't Find/Change a Table Column or OPTION+Drag Area

 Last reviewed: November 17, 1997
Article ID: Q58168
The information in this article applies to:
  • Microsoft Word for the Macintosh, versions 3.0, 3.01, 3.02, 4.0, 5.0, 5.1, 6.0

SYMPTOMS

In Microsoft Word for the Macintosh a Find or Replace cannot be performed on a selected column(s) in a table or on an area selected by holding down the OPTION key and dragging with the mouse. If a column in a table is selected for a Find or Replace, Word 6.0 does not clear the column selection however, the selection is ignored. The Find or Repace will look through the whole document to find what is in the find box. In Word 5.0 the highlighted area clears when Replace is chosen from the Edit menu in Word for the Macintosh 5.0 or 5.1 (or when Change is chosen from the Utilities menu in Word version 4.0, or Change is chosen from the Search menu in Word versions 3.0, 3.01, and 3.02). The same result occurs if an area is selected by dragging over the text with the mouse while holding down the OPTION key.

WORKAROUND

To perform a Find and Change on a column in a table, follow these steps:

  1. Select the column and click Copy on the Edit menu.

  2. On the File menu, click New and open a new document.

  3. On the Edit menu, click Paste.

  4. Make the necessary changes.

  5. Highlight the column and click Copy on the Edit menu.

  6. Switch back to the original document and select the column to paste it to.

  7. On the Edit menu, click Paste.

This method also works for selections made outside a table. There is no workaround for an OPTION+drag area.

This feature is under review and will be considered for inclusion in a future release.
