WD: InsertTableOfContents .Outline=0 Macro Doesn't Use TC Fields

 Last reviewed: February 2, 1998
Article ID: Q110465
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows NT, version 6.0
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word for the Macintosh, versions 6.0, 6.0.1

SYMPTOMS

When you run a WordBasic macro that inserts a table of contents using the InsertTableOfContents command, Microsoft Word includes the heading styles in the table of contents, even if you use the .Outline=0 argument.

CAUSE

The "InsertTableOfContents .Outline=0" macro command inserts the following field in your Word document: 

   { TOC }
By design, a TOC field with no parameters collects table of contents entries from heading levels 1 through 9.

RESOLUTION

Use the .Fields=1 InsertTableOfContents argument, as shown in the sample macro below, to create a table of contents that collects entries only from TC (Table of Contents Entry) fields. 

   InsertTableOfContents .Fields = 1
Tip: You can record this macro command to easily set all the parameters at once, as shown in the sample macro below: 

   Sub MAIN
     InsertTableOfContents .Outline = 0, .From = "", .To = "", \
     .Fields = 1, .TableID = "C", .RightAlignPageNumbers = 1, .Replace = 0
   End SUB
