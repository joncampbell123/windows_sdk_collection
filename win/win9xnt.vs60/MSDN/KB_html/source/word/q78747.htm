LASTSAVEDBY Field Not Updating Correctly in Word for Windows

 Last reviewed: February 6, 1998
Article ID: Q78747
The information in this article applies to:
  • Microsoft Word for Windows, versions 1.0, 1.1, 1.1a, 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c, 6.0, 6.0a, 6.0c

SUMMARY

In Microsoft Word for Windows, if you change the name in the Author box in the Summary Info dialog box and then insert the LASTSAVEDBY field into the document, the Author box displays the incorrect name.

MORE INFORMATION

To display the new author's name, do the following:

Word version 6.0 for Windows

  1. From the Tools menu, choose Options, and select the User Info tab.

  2. Change the Name text box to reflect the new author's name. Choose OK.

  3. Save the document.

  4. Select the LASTSAVEBY field in your and press F9 to update the field.

Word version 2.x for Windows

  1. From the Tools menu, choose Options, and select User Info from the Category box.

  2. Change the Name field to reflect the new author's name.

  3. Select the LASTSAVEDBY field in your document, and press F9 to update the field.

Word version 1.x for Windows

  1. From the Utilities menu, choose Customize.

  2. Change the Your Name field to reflect the new author's name.

  3. Select the LASTSAVEDBY field in your document and press F9 to update.

Steps to Reproduce Problem

  1. Type some text into a blank document.

  2. Save the document as C:\TEST.DOC.

  3. When the Summary Info dialog box appears, type "Unknown" (without the quotation marks) in the Author box. Choose the OK button, and choose Yes to save the change.

  4. On a new line within TEST.DOC, press CTRL+F9 to insert a field, and type LASTSAVEDBY inside the field braces.

  5. Select the field, and press the F9 key to update the field.

The Author field still reflects the name that was in the Author field before you changed it to "unknown." In order for the field to update correctly, it will be necessary to save the file and then update the field.
