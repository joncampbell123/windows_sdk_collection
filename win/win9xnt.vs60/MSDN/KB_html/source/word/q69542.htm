Cannot Use WinWord Symbol Fonts with LaserMaster Controller

 Last reviewed: July 30, 1997
Article ID: Q69542
The information in this article applies to:
  • Microsoft Word for Windows, versions 1.0, 1.1, 1.1a, 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c, 6.0, 6.0a, 6.0c

SYMPTOMS

When using a LaserMaster printer controller card, LaserMaster's symbol screen and printer fonts must be installed. Word for Windows symbol fonts will not be accessed with this device installed.

In Word version 6.0 you will receive the following error: 

   You are already printing in another application
if LaserMaster's symbol screen and printer fonts are not installed.

RESOLUTION

Installing LaserMaster's symbol font corrects the problem. LaserMaster's symbol fonts are compatible with Word and its equation field.

If symbol fonts are not an option from Word's font list, they can be installed by running LaserMaster's Setup program. Typically Setup can be found in: C:\LM\UTIL\SETUP.EXE.

This LaserMaster product has a card installed in the computer and a card installed in the printer. The cards are directly connected by using a cable. When LaserMaster's driver version 3.2 is installed, it modifies the [Windows], [Ports], [Printerports], and [Devices] sections of the Windows WIN.INI file as follows: 

   [Windows]
   DEVICE=LaserMaster,LMLX6,Direct:

   [Ports]
   Direct:=

   [PrinterPorts]
   LaserMaster=LMLX6,Direct:,15,45

   [Devices]
   LaserMaster=LMLX6,Direct:
Because LaserMaster does not create an analogous section when its driver is installed, there is no place for Word to write information about the symbol fonts when this driver is used. As a result, you are limited to the fonts provided with the LaserMaster controller device.

For further assistance with LaserMaster fonts and drivers, contact LaserMaster technical support at (612) 944-9331.
