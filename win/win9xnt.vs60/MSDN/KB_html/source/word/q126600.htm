Word for Windows NT Returns Error When Starting WordArt

 Last reviewed: February 6, 1998
Article ID: Q126600
The information in this article applies to:
  • Microsoft Word for Windows NT, version 6.0

SYMPTOMS

After you start WordArt in Word for Windows NT, the following error message appears: 

   Cannot find server application.

CAUSE

The registration for WordArt is incorrect in WRDART32.REG.

RESOLUTION

Double-click on WRDART32.REG in File Manager, and the WordArt file locations should re-register and correct the problem. If this does not resolve the problem, open WRDART32.REG with a text editor and check the file paths. For example, if NT is installed in a directory called C:\WINNT35, a typical configuration would look like: 

   HKEY_CLASSES_ROOT\MSWordArt.2\protocol\StdFileEditing\server =
   C:\WINNT35\MSAPPS\WORDART\WRDART32.EXE

   HKEY_CLASSES_ROOT\CLSID\{000212F0-0000-0000-C000-
   000000000046}\LocalServer32 = C:\WINNT35\MSAPPS\WORDART\
   WRDART32.EXE

   HKEY_CLASSES_ROOT\CLSID\{000212F0-0000-0000-C000-
   000000000046}\Defaulticon = C:\WINNT35\MSAPPS\WORDART\WRDART32.EXE, 0
NOTE: The lines may not necessarily wrap as indicated above.
