WD: TOC Style Tabs Redefined When Updating Table of Contents

 Last reviewed: February 2, 1998
Article ID: Q109884
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word for the Macintosh versions 6.0, 6.0.1, 6.0.1a

SYMPTOMS

In Word, tab settings in TOC styles that you have manually changed can reset themselves to their standard defaults when you update a table of contents. This occurs when you choose Update Entire Table in the Update Table Of Contents dialog box.

CAUSE

This problem occurs when you remove the tab setting for the page number from the TOCx style and add a new one. Word fails to retain the new setting.

WORKAROUND

When redefining a TOCx style, add the new tab setting without removing the existing 6-inch tab setting.

STATUS

Microsoft has confirmed this to be a problem products listed at the beginning of this article. This problem has been corrected in Microsoft Word 97 for Windows and Word 98 Macintosh Edition.
