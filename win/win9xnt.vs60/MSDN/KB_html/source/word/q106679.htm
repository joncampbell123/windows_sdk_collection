WordBasic Error When Adding a Symbol Font to a Menu

 Last reviewed: July 30, 1997
Article ID: Q106679
The information in this article applies to:
  • Microsoft Word for Windows, version 6.0, 6.0a, 6.0c

SYMPTOMS

If you record or create a WordBasic macro that adds the Symbol font to a menu, keystroke, or toolbar, Word generates the following WordBasic error message: 

   WordBasic Err=24
   Bad parameter

CAUSE

This problem occurs because Symbol is the name of a font and the name of a command.

STATUS

Microsoft has confirmed this to be a problem in Word for Windows. This problem was corrected in Word version 6.0 for the Macintosh, Word for Windows 95 and Word for Windows NT.

WORKAROUND

This problem does not occur if you add the Symbol font to a menu, keystroke, or toolbar without using a macro.
