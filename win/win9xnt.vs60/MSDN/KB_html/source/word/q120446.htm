Font Problems Using WinWord 6.0 with MS Multimedia Viewer

 Last reviewed: February 5, 1998
Article ID: Q120446
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Multimedia Viewer Publishing Toolkit, versions 2.0 and 2.0a

SYMPTOMS

If you author Microsoft Multimedia Viewer titles in Word 6.0 for Windows and then compile them with the Viewer compiler supplied with the Microsoft Multimedia Viewer Publishing Toolkit version 2.0, the fonts are not correct in the compiled Viewer file.

CAUSE

Word 6.0 modifies the way it saves font information when you save a file in Rich Text Format (RTF). The Viewer compiler supplied with the Microsoft Multimedia Viewer Publishing Toolkit version 2.0 does not process the new font information correctly.

RESOLUTION

You need an updated version of the Viewer compiler, WMVC.EXE. This updated compiler handles the new font information. It is available as a self- extracting file titled WMVCUP.EXE from the Microsoft Software Library:

~ WMVCUP.EXE

For more information about downloading files from the Microsoft Software Library, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q119591
   TITLE     : How to Obtain Microsoft Support Files from Online Services
When you run WMVCUP.EXE, the file WMVC.EXE will be extracted. You should then copy this file to your MVPUBKIT\MVTOOLS subdirectory.

MORE INFORMATION
