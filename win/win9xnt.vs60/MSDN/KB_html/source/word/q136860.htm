WD6X: Table Shading Prints Incorrectly on LaserWriter

 Last reviewed: February 5, 1998
Article ID: Q136860
The information in this article applies to:
  • Microsoft Word for the Macintosh, versions 6.0, 6.0.1

SYMPTOMS

When you apply a pattern such as Lt. Horizontal to a cell of a table that precedes a cell formatted with shading, such as 12.5 percent, some of the horizontal lines from the previous cell replace the shading of the current cell when you print the document. The document is displayed correctly in normal view, page layout view, and print preview.

CAUSE

This problem occurs when you use the LaserWriter 7.1.2 driver. Enabling or disabling Background Printing does not affect the printed results.

WORKAROUNDS

Method 1:

If you are using the LaserWriter 7.1.2 driver, select (enable) color/greyscale.

Method 2:

Use the version 8.1.1 or later LaserWriter printer driver.
