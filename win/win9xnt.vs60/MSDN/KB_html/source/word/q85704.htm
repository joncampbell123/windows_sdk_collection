"Word Cannot Save or Create File" Using LANtastic Network

 Last reviewed: July 30, 1997
Article ID: Q85704
The information in this article applies to:
  • Microsoft Word for Windows, versions 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c

SUMMARY

If you are using a LANtastic version 3.x network and you do not have READ, WRITE, and CREATE access controls in the network directory where you are trying to save a file, version 2.0 of Microsoft Word for Windows will return the following error message: 

   Word Cannot Save or Create this File
Note: LANtastic refers to network user rights as access controls.

MORE INFORMATION

To save a file under all conditions to a LANtastic network directory, you must have READ, WRITE, CREATE, RENAME, and DELETE access controls in that directory. Other error messages occur when you don't have RENAME or DELETE access controls.

To check a user's LANtastic version 3.x access controls, use the following steps:

  1. At an MS-DOS prompt, type "NET_MGR" (without the quotation marks) and press ENTER.

  2. Select Shared Resources Management and press ENTER.

  3. Select the drive or directory and press ENTER.

A line of letters is displayed next to the user's name. These letters represent the access controls currently assigned to the user or group. The following is a sample access control list: 

   ACCESS CONTROL LIST
   JOHN_SMITH      RWCMLDKNEAIP
   JILL_JONES      RWCMLD N  I
To modify the access controls, continue with the following steps:

  1. Select the name of the user or group.

  2. Type the first letter of the access control you want to add. For example, if you want to add the Create access control, type "C" (without the quotation marks). Access controls are toggle settings; therefore, if the Create access control is currently on, typing "C" will turn this access control off.

  3. Press ESC three times to exit.
