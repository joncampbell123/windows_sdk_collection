WD6X: Shading and OLE Objects Don't Print to QuickDraw Printers

 Last reviewed: February 6, 1998
Article ID: Q125802
The information in this article applies to:
  • Microsoft Word for the Macintosh, versions 6.0

SYMPTOMS

When you print from Word to a QuickDraw printer or Portable Digital Document printer from a Macintosh running QuickDraw GX, paragraph shading, table shading, and OLE objects do not appear on the printout.

WORKAROUNDS

To properly print shading and OLE objects, use any the following methods:

Method 1

Use a PostScript printer when using QuickDraw GX.

Method 2

Disable QuickDraw GX when using a QuickDraw printer.

Method 3

Select the OLE object and turn it into a picture by pressing COMMAND+SHIFT+F9. You will no longer be able to edit it in its original application, but you will be able to print it.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Word version 6.0 for the Macintosh. This problem was corrected in Word version 6.0.1 for the Macintosh.

MORE INFORMATION

OLE Objects

OLE objects include, but are not limited to, the following:

Microsoft ClipArt Gallery Microsoft Equation 2.0 (Equation Editor) Microsoft Excel 5.0 Chart Microsoft Excel 5.0 Macrosheet Microsoft Excel 5.0 Worksheet Microsoft Graph 5.0 Microsoft Movie Microsoft Organizational Chart 1.0 Microsoft Word 6.0 Document Microsoft Word 6.0 Picture Microsoft WordArt 2.0

QuickDraw GX

QuickDraw GX is the Apple printing architecture that ships with System 7.5. It replaces the original display and printing engine (called QuickDraw) that shipped with the Macintosh in 1984.

The following printers are QuickDraw-based:

Apple ImageWriter Apple ImageWriter II Apple LaserWriter Select 300 Apple LaserWriter LS Apple Personal LaserWriter 300 Apple Personal LaserWriter II SC Apple StyleWriter Apple StyleWriter II Apple Portable StyleWriter Apple Color StyleWriter Pro Apple Color StyleWriter 2400 HP DeskWriter HP DeskWriter C HP DeskWriter 550C GCC PLP II GCC WriteMove II
