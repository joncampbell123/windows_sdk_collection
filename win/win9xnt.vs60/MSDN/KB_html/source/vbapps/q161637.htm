PPT97: F1 Key May Not Start Help When in the Visual Basic Editor

 Last reviewed: December 2, 1997
Article ID: Q161637
The information in this article applies to:
  • Microsoft PowerPoint 97 for Windows

SYMPTOMS

When you are working in the Visual Basic Editor, Visual Basic Help may not start if you press the F1 key. This may happen even if Help does start when you click Visual Basic Help on the Help menu.

CAUSE

If there is a dialog box (such as the Print dialog box) open in PowerPoint, the F1 key does not start Visual Basic Help.

RESOLUTION

Switch to PowerPoint and close any open dialog boxes. You now should be able to start Visual Basic Help by pressing the F1 key.
