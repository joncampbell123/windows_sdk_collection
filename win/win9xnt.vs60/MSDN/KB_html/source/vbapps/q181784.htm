WD98: Type Mismatch Error Running Converted WordBasic Macro

 Last reviewed: March 3, 1998
Article ID: Q181784
The information in this article applies to:
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

When you run a macro that was converted from WordBasic to Microsoft Visual Basic for Applications, you receive the following error message: 

   Type Mismatch Error

CAUSE

If you converted a WordBasic macro that contains statements similar to the following: 

   x = Val(FontSize())
the converted code may appear as: 

   x = WordBasic.Val(WordBasic.FontSize())
This converted statement causes the macro to halt with the "Type Mismatch" error message.

To resolve this problem, remove "WordBasic." from the Val function as in the following example: 

   x = Val(WordBasic.FontSize())
-or-

Convert the statement to the Visual Basic equivalent: 

   x = Selection.Font.Size

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
