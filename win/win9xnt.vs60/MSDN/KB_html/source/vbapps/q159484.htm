WD97: Simulating a Macro Pause in Word 97

 Last reviewed: February 11, 1998
Article ID: Q159484
The information in this article applies to:
  • Microsoft Word 97 for Windows

SUMMARY

In WordPerfect there is a macro command called PAUSE that suspends the macro for user input until the user presses ENTER, at which point the macro continues. There is no equivalent command in Microsoft Visual Basic for Applications. You can simulate this behavior using a macro that calls a second macro when you press a specific key, such as the ENTER key.

You can simulate a pause in Visual Basic for Applications, by using two macros: one macro runs the commands up to the "pause" and includes a key assignment that initiates the second macro, which runs all commands that occur after the pause. In the first macro (BeforePause), the ENTER key is reassigned to run the second macro (AfterPause). 

   Sub BeforePause()
      '<code>
      '<exit point> - These commands should be placed outside of any error
      'traps.
      'Sets the Normal.dot to hold keyboard changes.
      CustomizationContext = NormalTemplate

      'This sets the ENTER key to run the "AfterPause" macro.
      KeyBindings.Add KeyCode:=BuildKeyCode(wdKeyReturn), _
      KeyCategory:=wdKeyCategoryCommand, Command:="AfterPause"

   End Sub
The second macro runs after you press ENTER and then restores the functionality of the ENTER key. 

   Sub AfterPause()

      'Sets the Normal.dot to hold keyboard changes.
      CustomizationContext = NormalTemplate

      'Disables the command associated with the ENTER key in the
      'BeforePause macro.
      FindKey(BuildKeyCode(wdKeyReturn)).Disable

      'Displays a message box.
      MsgBox "Hello"

   End Sub
