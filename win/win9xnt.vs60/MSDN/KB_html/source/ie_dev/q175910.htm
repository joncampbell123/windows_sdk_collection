BUG: IDocHostShowUI::ShowHelp Not Called for IE4/MSHTML Host

 Last reviewed: February 16, 1998
Article ID: Q175910
The information in this article applies to:
  • Microsoft Internet Explorer (Programming), version 4.0
  • Microsoft Internet Client SDK, version 4.0

SYMPTOMS

MSHTML never calls IDocHostShowUI::ShowHelp in a host application.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. We are researching this bug and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The WebBrowser control and Mshtml.dll components of the Internet Explorer 4.0 can be hosted by applications other than the Internet Explorer. These "hosts" can implement several interfaces used for customizing the behavior of IE4/MSHTML objects. A host can supply mechanisms for message boxes and Help to be shown by implementing the IDocHostShowUI interface.

REFERENCES

The Internet Client SDK; search on Internet Tools and Technologies /Reusing the WebBrowser and MSHTML.
