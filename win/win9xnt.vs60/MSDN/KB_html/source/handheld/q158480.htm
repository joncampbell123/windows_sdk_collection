Pocket Word MRU List Displays File Name Incorrectly

 Last reviewed: April 14, 1997
Article ID: Q158480
The information in this article applies to:
  • Microsoft Pocket Word version 1.0

SYMPTOMS

A document's name may be displayed incorrectly in the Pocket Word MRU (most recently used) list. The document name is displayed correctly in all other locations (including the Documents list on the Start menu, the folder in which the file was saved, and on the taskbar when the file is open).

CAUSE

This problem occurs if you save a document with a name that includes an ampersand (&) character. When you use this character in a document's name, the name is displayed in the Pocket Word MRU list with an underscore under the character following the ampersand and the ampersand is not displayed.

RESOLUTION

To avoid this problem, do not use the ampersand character in document names.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Pocket Word 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The Pocket Word MRU list displays the names of the last three documents you opened in Pocket Word. To access the MRU list, tap the File menu in Pocket Word.
