PRB: Visual Workbench Open File or Save As Dialog Box Problems

 Last reviewed: July 17, 1997
Article ID: Q97111
1.00 1.50 WINDOWS kbtool kbprb

The information in this article applies to:

  • The Visual Workbench included with: Microsoft Visual C++ for Windows, versions 1.0 and 1.5

SYMPTOMS

In the Visual Workbench, when the user chooses Open or Save As from the File menu, various problems occur. These problems will also occur if the user selects the corresponding toolbar buttons.

CAUSE

These problems are most likely to occur when an incorrect version of COMMDLG.DLL is installed on the computer. COMMDLG.DLL should be installed only in the Windows system directory (by default, C:\WINDOWS\SYSTEM) and it should have a file date no earlier than March 10, 1992.

If there is another copy of COMMDLG.DLL in the Windows directory (by default, C:\WINDOWS), be sure to delete it after placing the correct version in the Windows system directory.

RESOLUTION

Install one of the following versions of COMMDLG.DLL into the Windows system directory: 

   provided with Microsoft Windows version 3.1,
   file date: March 10, 1992, size: 89,248 bytes

   provided with Microsoft Windows for Workgroups version 3.1,
   file date: October 1, 1992, size: 97,984 bytes

   provided with Microsoft Visual C++ version 1.0,
   file date: February 8, 1993, size: 89,248 bytes
Remove any other copy of COMMDLG.DLL installed on your system.

To install the correct version of COMMDLG.DLL, use the Setup program provided with the application or refer to the packing list provided with the application to locate the file on one of the installation disks and use the EXPAND.EXE utility to copy the file to your hard disk.
