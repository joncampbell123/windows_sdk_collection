BUG: Pressing CTRL+ALT+SYSRQ in Debugger Reboots Windows 95

 Last reviewed: July 18, 1997
Article ID: Q140358
1.00 1.50 1.51 1.52 WINDOWS NT kbtool kbbuglist

The information to this article applies to:

  • The integrated debugger included with: Microsoft Visual C++ for Windows, versions 1.0, 1.5, 1.51, 1.52

SYMPTOMS

Pressing CTRL+ALT+SYSRQ to switch from application window to debug source code window results in the appearance of this message: 

   Microsoft Visual C++. System is unstable. Debugging
   session will terminate. OK
Clicking OK causes Windows 95 to reboot the computer.

RESOLUTION

If you want to switch from application to debugg source code while debugging, set a break point in the source code and press ALT+TAB to switch between the application window and Visual Work Bench source code window.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

To reproduce the problem, make a debug build of the GENERIC sample from the MSVC samples directory. Press the F5 key to start the debugger. When the application window appears, press CTRL+ALT+SYSRQ.
