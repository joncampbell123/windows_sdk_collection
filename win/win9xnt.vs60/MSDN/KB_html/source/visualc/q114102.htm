BUG: CL /? Only Displays 25 Lines at a Time

 Last reviewed: July 22, 1997
Article ID: Q114102
7.00 | 1.00 1.50 MS-DOS | WINDOWS kbtool kbbuglist

The information in this article applies to:

  • The Microsoft C/C++ Compiler (CL.EXE), included with: 

        - Microsoft C/C++ for MS-DOS, version 7.0
    - Microsoft Visual C++ for Windows, versions 1.0 and 1.5

SYMPTOMS

Running CL.EXE with the /? option displays a list of all the available compiler options with a brief explanation of what each option does. When running CL.EXE with the /? option, the output is only displayed 25 lines at a time regardless of the number of lines available on the screen.

STATUS

Microsoft has confirmed this to be a problem with the C/C++ compiler version 7.0, 8.0, and 8.0c. This problem does not occur when using the C/C++ 32-bit compiler version 8.0. We are researching the problem and will post new information in the Microsoft Knowledge Base as soon as it becomes available.
