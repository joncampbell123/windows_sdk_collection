DOCERR: CodeView 4.10 Does Not Support Remote Debugging

 Last reviewed: July 18, 1997
Article ID: Q125781

MS-DOS              | WINDOWS
kbtool kbdocerr

The information in this article applies to:

  • The Microsoft CodeView Debugger for MS-DOS, included with: Microsoft Visual C++ for Windows, versions 1.0, 1.5, 1.51, 1.52
  • The Microsoft CodeView Debugger for Windows, included with: Microsoft Visual C++ for Windows, versions 1.0, 1.5, 1.51, 1.52

SUMMARY

CodeView version 4.1 for MS-DOS and Windows does not support remote debugging, although it appears to be possible. The user is still able to access the Configure dialog box, and the online help (CV.HLP) still includes information about remote debugging. The information was removed only from the printed documentation and Books Online.

This configuration is no longer supported in CodeView, beginning with version 4.1.

MORE INFORMATION

The user can get to the Configure dialog box by selecting Configure Remote in the Load dialog box. CodeView displays the Load dialog box if you start it without specifying the name of the .EXE to debug or when the user selects the Load command from the Run menu.

Visual C++ version 1.x for Windows no longer includes the alternate transport DLL, and the remote monitor program required for remote debugging. If the user tries to follow the instructions in the online help and specifies an alternate transport DLL in the TOOLS.INI file or in the Configure dialog box when running CodeView, it will fail. The files are not shipped with the product, and are no longer available.

Visual C++ version 1.0 for Windows NT ships Codeview version 4.25 to allow users to debug Win32s applications. However, Codeview 4.25 does not support remote debugging either.
