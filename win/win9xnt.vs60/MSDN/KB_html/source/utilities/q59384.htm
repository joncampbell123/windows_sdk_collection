BUG: NMAKE /D /C Switches Suppress Modification Date

 Last reviewed: July 17, 1997
Article ID: Q59384
1.01 1.10 1.11 1.12 1.13 1.20 1.30 1.40 | 1.11 1.12 1.13 1.21 
MS-DOS                                  | OS/2
kbtool kbbuglist

The information in this article applies to:

  • Microsoft NMAKE Utility for MS-DOS, versions 1.01, 1.1, 1.11, 1.12, 1.13, 1.2, 1.3, and 1.4
  • Microsoft NMAKE Utility for OS/2, versions 1.11, 1.12, 1.13, and 1.21

SYMPTOMS

When using the NMAKE /D and /C switches together, the modification date of each file will not be displayed when the date is checked. A short description of each switch is described as follows: 

   /D  Displays the modification date of each file when the date is
       checked.

   /C  Suppresses the NMAKE copyright message and prevents nonfatal
       error or warning messages from being displayed.

STATUS

Microsoft has confirmed this to be a problem with NMAKE versions 1.01, 1.1, 1.11, 1.12, 1.13, 1.2, 1.21, 1.3, and 1.4 for MS-DOS and versions 1.11, 1.12, 1.13, and 1.21 for OS/2. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
