BUG: Very Large Line Number Displayed When Opening File

 Last reviewed: July 17, 1997
Article ID: Q86818
2.00 MS-DOS kbtool kbbuglist

The information in this article applies to:

  • Microsoft Programmer's Workbench for MS-DOS, versions 2.0 and 2.1.49

SYMPTOMS

Under Programmer's WorkBench version 2.0, which is supplied with Microsoft C/C++ 7.0, a very large number may be displayed in the lower right corner of the screen when opening a file.

CAUSE

If you choose Open from the File menu, and you enter a non-existent file, a dialog box will be displayed asking if the file should be created. While the dialog box is being displayed, a very large line number will be displayed in the lower right corner of the screen.

RESOLUTION

The line number display returns to normal after the "File does not exist!" dialog box disappears. The large line number that is temporarily displayed does no harm.

STATUS

Microsoft has confirmed this to be a problem in PWB versions 2.0 and 2.1.49. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
