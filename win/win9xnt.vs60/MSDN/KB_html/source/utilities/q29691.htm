Conditions Where /PACKCODE is Not the Default

 Last reviewed: July 17, 1997
Article ID: Q29691
5.01 5.02 5.03 5.05 5.10 5.11 5.13 5.15 5.20 5.30 MS-DOS kbtool

The information in this article applies to:

  • Microsoft LINK for MS-DOS, versions 5.01, 5.02, 5.03, 5.05, 5.1, 5.11, 5.13, 5.15, 5.2, and 5.3

/PACKCODE is normally the default when linking. However, if either of the following is true, /NOPACKCODE is the default: 

   1. Segments are defined in the .DEF file

      -or-

   2. The /INCR[EMENTAL] switch is given
