PRB: CVW 3.05 Incompatible with Windows 3.1

 Last reviewed: July 17, 1997
Article ID: Q83100
3.05 WINDOWS kbtool kbprg kbfile kbprb

The information in this article applies to:

  • Microsoft CodeView for Windows, version 3.05

SYMPTOMS

Under Windows 3.1, when the user double-clicks CodeView for Windows (CVW) version 3.05, which is provided with the Windows Software Development Kit (SDK) version 3.0, the mouse cursor momentarily changes to an hourglass, then changes back to an arrow. The CVW main window does not appear. Under the debugging version of Windows 3.1, a fatal exit message appears on the debug terminal.

CAUSE

Version 3.05 of CodeView for Windows requires the WINDEBUG.DLL file to work with version 3.1 of the Windows graphical environment.

RESOLUTION

The best solution is to update to CodeView for Windows version 3.07 included with version 3.1 of the Windows SDK.

As a temporary measure until the CodeView update is available, WINDBG is a file in the Software/Data Library that contains the WINDEBUG.DLL. WINDBG is for use by registered owners of the Windows Software Development Kit (SDK) version 3.0 who require the file to run CVW version 3.05 with Windows 3.1. WINDBG may not be redistributed in any way.

Download WINDBG.EXE from the Microsoft Software Library (MSL) on the following services:

  • Microsoft Download Service (MSDL) 

          Dial (206) 936-6735 to connect to MSDL
      Download WINDBG.EXE (size: 21115 bytes)
  • Internet (anonymous FTP) 

          ftp ftp.microsoft.com
      Change to the \SOFTLIB\MSLFILES directory
      Get WINDBG.EXE (size: 21115 bytes)

MORE INFORMATION

Applications that require the functionality of WINDEBUG.DLL must use the TOOLHELP.DLL file provided with version 3.1 of the Windows SDK. Under the terms and conditions of the Microsoft Windows SDK Licensing Agreement, TOOLHELP.DLL may be redistributed with an application.
