BUG: Wrong Help Topic Context for IntegralHeight Property

 Last reviewed: October 16, 1996
Article ID: Q150212
The information in this article applies to:
  • Standard, Professional, and Enterprise Editions of Microsoft Visual Basic, 16-bit and 32-bit, for Windows, version 4.0

SYMPTOMS

Choosing the F1 key when the IntegralHeight property of a list box is selected causes a jump to the incorrect Help topic. The Sorted property topic appears rather than the IntegralHeight property topic.

STATUS

Microsoft has confirmed this to be an issue in the Microsoft products listed at the beginning of this article. Microsoft is researching this issue and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

From the Help menu, choose Search. Then choose Find and display the IntegralHeight property.

Steps to Reproduce Problem

  1. Start a new project in Visual Basic. Form1 is created by default.

  2. Place a list box on Form1.

  3. Choose the IntegralHeight property in the Properties window for the list box, and the press F1 key.

Notice that Help is launched for the Sorted property topic instead of the IntegralHeight property topic.
