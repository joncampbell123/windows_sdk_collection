PRB: Error TDS Buffer Length too Large

 Last reviewed: October 30, 1995
Article ID: Q137026
The information in this article applies to:
  • Standard, Professional, and Enterprise Editions of Microsoft Visual Basic, 16-bit and 32-bit, for Windows, version 4.0
  • Standard and Professional Editions of Microsoft Visual Basic for Windows, version 3.0

SYMPTOMS

You may receive the following error when connecting to SQL Server under Microsoft Windows NT version 3.5 via named pipes from a Microsoft Windows for Workgroups version 3.11 client over the IPX/SPX-Compatible transport (NWLink): 

   [Microsoft][ODBC SQL Server Driver]TDS buffer length too large(#0)

RESOLUTION

For a workaround to this problem, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q121258
   TITLE     : PRB: Connect to SQL on NT 3.5 from WFW 3.11 via Named Pipes

STATUS

This behavior is by design.
