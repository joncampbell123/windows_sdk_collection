ACC: Empty Field List in Report or Form's Design View

 Last reviewed: May 28, 1997
Article ID: Q106286
The information in this article applies to:
  • Microsoft Access versions 1.0, 1.1, 2.0, 7.0, 97

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

When you view an unbound form or report (a form or report not based on any table or query) in Design view, a blank field list appears.

CAUSE

A field list will appear automatically if it was displayed in the form or report you last designed. If the current form or report is unbound, the field list will be empty.

RESOLUTION

Close the field list. The Field List command on the View menu will become unavailable until the form or report in Design view is bound.

STATUS

This behavior is by design.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Open the sample database Northwind.mdb (or NWIND.MDB in versions 1.x and 2.0).

  2. Open the Categories form in Design view.

  3. On the View menu, click Field List. Note that the field list is displayed.

  4. Close the form.

  5. Open a new, unbound form in Design view. Note that an empty field list is displayed.

REFERENCES

For more information about the field list, search the Help Index for "field list," or ask the Microsoft Access 97 Office Assistant.
