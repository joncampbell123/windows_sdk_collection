ACC95: Overflow Error in Option Group Wizard on Windows NT 4.0

 Last reviewed: May 14, 1997
Article ID: Q155670
The information in this article applies to:
  • Microsoft Access version 7.0

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

When you use the Option Group Wizard to create an option group, you may receive the following error message 

   Overflow
and then the following error message one or more times: 

   Application-defined or object-defined error

CAUSE

You are running Microsoft Access 7.0 with Microsoft Windows NT version 4.0.

RESOLUTION

Create the option group on your form without using the Option Group Wizard.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access 7.0. This problem no longer occurs in Microsoft Access 97.

MORE INFORMATION

Steps to Reproduce Problem

  1. Open the sample database Northwind.mdb.

  2. Create a new form in Design view.

  3. Create an option group using the Option Group Wizard.

  4. In the Option Group Wizard box, under Label Names, type Label1, and then press TAB.

  5. Click Finish. Note that you receive the error messages mentioned in the "Symptoms" section.

REFERENCES

For more information about using option groups, search for "option groups," and then "What is an option group?," using the Microsoft Access 7.0 Help Index.
