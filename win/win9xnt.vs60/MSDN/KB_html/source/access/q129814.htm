ACC: "Couldn't Find Object '<Name>'" Err Attaching DB2 Table

 Last reviewed: June 23, 1997
Article ID: Q129814
The information in this article applies to:
  • Microsoft Access version 2.0, 97

SYMPTOMS

Advanced: Requires expert coding, interoperability, and multiuser skills.

When you try to attach a DB2 table containing an underscore (_) or a percent sign (%) in its name, you receive the following error message: 

   Couldn't find object "<tablename>"
NOTE: This error message has been found to occur in Microsoft Access 97.

CAUSE

The Microsoft Jet database engine misinterprets the underscore or the percent sign. This causes the Jet database engine to pass an incorrect table name to the DB2 ODBC driver.

STATUS

This problem no longer occurs with the 04/18/95 version of the Microsoft Jet database engine (MSAJT200.DLL). This file is available with the Microsoft Access version 2.0 Service Pack dated 05/01/95.

For information on how to obtain the updated Service Pack, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q122927
   TITLE     : WX1124: Microsoft Access Version 2.0 Service Pack

MORE INFORMATION

To find out which version of the Microsoft Jet database engine is installed on your computer, follow these steps:

  1. Open File Manager.

  2. From the View menu, choose All File Details.

  3. Select the Windows\SYSTEM directory to view its contents.

  4. Scroll through the contents of the window to find the MSAJT200.DLL file.

You may have one of the following versions of the Microsoft Jet database engine (note the version number and the size):
  • If you never installed the Microsoft Access version 2.0 Service Pack: 

          MSAJT200.DLL   2.00   994496
  • If you installed the Microsoft Access version 2.0 Service Pack dated 12/02/94: 

          MSAJT200.DLL   2.50.1108   995056
  • If you installed the Microsoft Access version 2.0 Service Pack dated 05/01/95: 

          MSAJT200.DLL   2.50.1606   995136
