ACC1x: Microsoft Excel Custom Date Format Not Detected by Import

 Last reviewed: June 6, 1997
Article ID: Q92651
The information in this article applies to:
  • Microsoft Access versions 1.0 and 1.1
  • Microsoft Excel version 4.x for Windows

SYMPTOMS

When you import a Microsoft Excel version 4.0 spreadsheet containing custom date formats into a new Microsoft Access version 1.x table, Microsoft Access gives the fields Double data types, instead of Date/Time data types.

STATUS

This behavior no longer occurs in Microsoft Access version 2.0.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Create a new Microsoft Excel version 4.0 spreadsheet.

  2. Enter a date in cell A1.

  3. Assign a date format, such as mmmm/dd/yyyy, to the cell.

  4. Import the spreadsheet to Microsoft Access. Note that the date is formatted as a Number data type instead of a Date/Time data type.

REFERENCES

For more information about importing Microsoft Excel data, search for "Excel" using the Microsoft Access Help menu.
