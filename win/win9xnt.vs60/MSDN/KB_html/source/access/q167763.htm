ACC97: New Pages Added to Tab Control Don't Appear as Expected

 Last reviewed: April 30, 1997
Article ID: Q167763
The information in this article applies to:
  • Microsoft Access 97

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

When you insert a new page onto a tab control, either by clicking Tab Control Page on the Insert menu, or by using the right mouse button to click the Tab control, and then clicking Insert Page on the tab control shortcut menu, the new page is added before the active Page instead of after it.

RESOLUTION

If you want new pages to appear after the active Page tab instead of before it, copy and paste new pages onto the tab control. When you copy and paste new pages onto the tab control, the new pages will appear after the active Page tab.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Open the sample database Northwind.mdb in Microsoft Access 97.

  2. Create a new form in Design view, and add a tab control to the form.

  3. With the tab control selected, on the Insert menu, click Tab Control Page.

    Note a new page is added on top of the active Page.

REFERENCES

For more information about creating tab controls, search the Help Index for "Tab controls, creating," or ask the Microsoft Access 97 Office Assistant.
