ACC1x: Incorrect Install Directory with Workstation Setup

 Last reviewed: May 14, 1997
Article ID: Q109372
The information in this article applies to:
  • Microsoft Access versions 1.0, 1.1

SYMPTOMS

When you run the Microsoft Access version 1.0 Setup program with the workstation switch (SETUP /N) from a workstation on a network, Microsoft Access lists :\ACCESS as the default directory for the installation directory. If you choose this directory for the installation, you receive an error message.

If you run the Microsoft Access version 1.1 Setup program with the workstation switch, Microsoft Access lists C:\ACCESS as the default directory for the installation directory even if the workstation does not have a drive C.

CAUSE

These errors occur if you run the Microsoft Access Setup program with the workstation switch from a diskless workstation.

The Microsoft Access Setup program creates a list of locally available hard disks. Since, in the case of a diskless workstation, there are no local hard disks, a null list is created, which leads to the incorrect default directories listed above.

RESOLUTION

Do not run the Microsoft Access Setup program with the workstation switch (SETUP /N) from a diskless workstation.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access versions 1.0 and 1.1. This problem no longer occurs in Microsoft Access version 2.0.
