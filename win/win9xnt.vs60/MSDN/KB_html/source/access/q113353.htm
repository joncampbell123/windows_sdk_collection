ACC2: Datasheet Header and Footer Print at Constant Locations

 Last reviewed: May 14, 1997
Article ID: Q113353
The information in this article applies to:
  • Microsoft Access version 2.0

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

The default top and bottom margin value for table and query Datasheet views is 1 inch. Decreasing these margins does not move the location of the title and date header, or of the page number footer. Instead, the header and footer is overwritten by the column data when you print or preview the data.

CAUSE

By default, the title and date header is placed at a constant 0.5 inches from the top of the page, and the page number footer is placed at 0.5 inches from the bottom of the page.

RESOLUTION

Create a form or report, instead of using a table or query's Datasheet view, if you want to decrease the default top and bottom margins.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access version 2.0. This problem no longer occurs in Microsoft Access version 7.0.

MORE INFORMATION

Steps to Reproduce Problem

  1. Open the sample database NWIND.MDB.

  2. Open the Customers table in Datasheet view.

  3. Preview the table.

  4. From the File menu, choose Print Setup. Change the top and bottom margins to 0.25 inches, and then choose OK.
