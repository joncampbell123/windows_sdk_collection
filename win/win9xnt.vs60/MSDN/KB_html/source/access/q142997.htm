ACC: Setting Height Property in Page Setup Is Ignored (95/97)

 Last reviewed: May 14, 1997
Article ID: Q142997
The information in this article applies to:
  • Microsoft Access versions 7.0, 97

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

When you set a form or report section's height in the Page Setup dialog box under Item Size on the Columns tab (or the Layout tab in Microsoft Access 7.0), the height of the form or report's section does not change. The Page Setup Height setting is ignored, and the section remains at the height specified in the form or report section's Height property setting.

RESOLUTION

  1. Open the form or report in Design view.

  2. Click the form or report's detail section and open the detail section's property sheet.

  3. Set the detail section's Height property to the number of inches or centimeters you want (22" or 55.873cm is the maximum).

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access 7.0 and 97. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Open the sample database Northwind.mdb.

  2. Click the Reports tab, and then select the Customer Labels report.

  3. On the File menu, click Print Preview. Note the current white space between the labels on the Customer Labels report.

  4. On the File menu, click Page Setup.

  5. In the Page Setup dialog box, click the Columns tab (or the Layout tab in Microsoft Access 7.0.)

  6. If it is checked, click to clear the Same As Detail check box, and then, under Item Size, set Height to 4 inches.

  7. Click OK. Note that the white space between the labels hasn't changed.

  8. Close the Customer Labels report and re-open it in Design view.

  9. Change the detail section's Height property to 4 inches.

  10. Preview the report. Note the significant white space between the labels.

REFERENCES

For more information about customizing reports, search the Help Index for "customizing reports," or ask the Microsoft Access 97 Office Assistant.
