ACC95: Updated Jet DLLs Available on MSL

 Last reviewed: September 25, 1997
Article ID: Q151186
The information in this article applies to:
  • Microsoft Access version 7.0
  • Standard, Professional, and Enterprise Editions of Microsoft Visual Basic, 32-bit only, version 4.0 for Windows
  • Microsoft Windows NT version 4.0

SUMMARY

Novice: Requires knowledge of the user interface on single-user computers.

The Msjtwng.exe file contains an update of three .dll files that are included with Microsoft Access for Windows 95 version 7.0, Microsoft Visual Basic version 4.0, and Microsoft Windows NT version 4.0. The problems fixed by this update are listed below.

On Windows NT:

  • Slow performance when you run large queries or execute operations that read Long Value data types (OLE or Memo fields).
  • Microsoft Jet database engine version 3.0 may stop responding (may crash) when used with Microsoft Internet Information Server version 2.0.
  • You cannot create a Dropbox on the local computer in Replication Manager.

On Windows 95 or Windows NT: 

   Msjt3032.dll prevents a rare bug that can allow duplicate indices to be
   created on a primary or unique index. When this happens, you receive the
   following error message when you try to open your database:

      <Database Name> isn't an index in this table.

   All users of Microsoft Jet database engine version 3.0, which includes
   Microsoft Access 7.0 and Microsoft Visual Basic version 4.0, should
   download the updated .dll files to avoid this potentially serious error.
   Once you receive the error, the database is unrecoverable, and you must
   restore a backup copy.
