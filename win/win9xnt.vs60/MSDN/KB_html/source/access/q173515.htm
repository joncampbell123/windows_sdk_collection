ACC: "Couldn't Decrypt File" Message Linking Paradox Table

 Last reviewed: September 9, 1997
Article ID: Q173515
The information in this article applies to:
  • Microsoft Access versions 2.0, 7.0, 97

SYMPTOMS

Moderate: Requires basic macro, coding, and interoperability skills.

When you link to a Paradox table that has an auxiliary password, you may receive the following message: 

   Couldn't decrypt file.

CAUSE

You entered the auxiliary password in the Password Required dialog box. Microsoft Access does not recognize the auxiliary password.

RESOLUTION

Enter the master password for the table, rather then the auxiliary password. If the table needs to be read only, you can lock the table with the "write lock" feature in Paradox.

REFERENCES

For more information about linking Paradox tables, search the Help Index for "Paradox," or ask the Microsoft Access 97 Office Assistant.
