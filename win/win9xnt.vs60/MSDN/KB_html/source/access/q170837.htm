ACC97: Status Bar Continues to Display "Calculating..."

 Last reviewed: October 13, 1997
Article ID: Q170837
The information in this article applies to:
  • Microsoft Access 97

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

After you have opened an object, such as a query or a form, in Microsoft Access 97, the status bar continues to display the message "Calculating..."

CAUSE

You are using the Quiet Logo Microsoft Office Assistant.

RESOLUTION

There are two ways to refresh the text in the status bar:

  • Click outside the Microsoft Access application window; then return to the Microsoft Access window by clicking on it. For example, you can accomplish this by clicking the taskbar, and then clicking the Microsoft Access window.
  • Minimize and then restore the Microsoft Access application window.

You can avoid this problem entirely by using a different Microsoft Office Assistant.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access 97. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Download the Quiet Logo Microsoft Office Assistant from the Microsoft Web site. You can find the additional Office Assistants at: 

          http://www.microsoft.com/OfficeFreeStuff/office/assistants.htm

  2. Run tofflogo.exe to install the Quiet Logo Office Assistant, but do not change your default Assistant.

  3. Start Microsoft Access, and open the sample database Northwind.mdb.

  4. Open the Categories form in Form view. Note that the status bar briefly displays the message "Calculating..." and then displays the message "Name of food category."

  5. Right-click the Office Assistant, and then click Choose Assistant. Click Next or Back until you see the Office Logo Assistant described as "Absolutely No Moving Parts." Click OK.

  6. Close the Categories form and reopen it.

  7. Note that the status bar displays the message "Calculating..." and that this message does not change.
