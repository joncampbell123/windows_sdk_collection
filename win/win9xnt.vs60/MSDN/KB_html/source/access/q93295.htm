ACC1x: "Couldn't Open File" Error Message

 Last reviewed: May 7, 1997
Article ID: Q93295
The information in this article applies to:
  • Microsoft Access versions 1.0, 1.1

SYMPTOMS

Attempting to open a database exclusively when another user already has it open results in the error message: 

   Couldn't open file <database name>.

CAUSE

The Help topic says in part "You must have ReadData permission for the file in order to view its data." The error message and Help text are misleading because the problem is that the file cannot be opened exclusively.

RESOLUTION

To open the file successfully:

  1. Press ESC or choose the OK button to exit the error message.

  2. From the File menu, choose Open Database.

  3. Select the database you want to open.

  4. Clear the Exclusive check box.

  5. Choose OK.

STATUS

This behavior no longer occurs in Microsoft Access version 2.0.
