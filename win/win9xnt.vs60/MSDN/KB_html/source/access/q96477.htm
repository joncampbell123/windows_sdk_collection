ACC1x: Changes in Subform Design Not Updated in Main Form

 Last reviewed: May 28, 1997
Article ID: Q96477
The information in this article applies to:
  • Microsoft Access versions 1.0, 1.1

SYMPTOMS

After editing a subform with the main form in Design view, the changes you make to the subform are not committed until you close and reopen the main form.

RESOLUTION

After editing the subform with the main form in Design view, close and save the subform. Click in the subform control and press ENTER. Your main form now reflects the changes that you made to its subform.

STATUS

This behavior no longer occurs in Microsoft Access version 2.0.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Open the sample database NWIND.MDB.

  2. Open the Categories form in Design view.

  3. Select the Category ID control.

  4. Double-click anywhere inside the Categories Subform control to open it in Design view.

  5. From the View menu, choose Field List. Drag the English Name field to the Category Subform control.

  6. Exit and save the changes to the Categories Subform control. This action returns you to the the Categories form in Design view.

  7. View the form in Form view.

Note that the field you added (English Name) is not in the subform.

REFERENCES

Microsoft Access "User's Guide," version 1.0, page 274
