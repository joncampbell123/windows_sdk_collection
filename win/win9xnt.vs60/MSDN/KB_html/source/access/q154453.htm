ACC95: Invalid Page Fault When Selecting the Format or Other Tab

 Last reviewed: May 14, 1997
Article ID: Q154453
The information in this article applies to:
  • Microsoft Access version 7.0

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

In the Properties window of a form or report, when you click a value, such as General Number, from the drop list of the Format property, and then click either the Format tab or the Other tab, you may receive the following error message: 

   MSACCESS caused an invalid page fault in module <unknown>
If you are running Microsoft Access for Windows 95 within a Windows NT environment, you may receive a Dr. Watson error message as follows: 

   An application error has occurred and an application error log is
   being generated.

RESOLUTION

After you click the drop list of the Format property with the mouse pointer, use the UP/DOWN arrow keys to select the desired value, and then press ENTER. Do not use the mouse pointer to make the selection.

You can now safely click either the Format tab or the Other tab.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access 7.0. This problem no longer occurs in Microsoft Access 97.

MORE INFORMATION

The page faults occur only when you use the mouse pointer to click the desired Format property value and then the Format tab or the Other tab.

Steps to Reproduce Problem

  1. Open the sample database Northwind.mdb.

  2. Open the Products form in Design view.

  3. Click in the UnitPrice text box.

  4. If the Properties window is not already open, click Properties on the View menu.

  5. Click the All tab in the Properties window.

  6. Click the Format property, and then click the drop-down arrow.

  7. Click Currency in the list.

  8. Click the Format tab or the Other tab.

    NOTE: Under Windows 95, a message box appears first: 

          This program has performed an illegal operation
      and will be shut down.

    When you click Details, you see the actual error: 

          MSACCESS caused an invalid page fault in
      module <unknown>
