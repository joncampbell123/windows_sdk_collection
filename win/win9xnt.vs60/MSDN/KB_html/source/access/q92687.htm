ACC: Open Report or Form Based on a Table Locks the Table

 Last reviewed: May 7, 1997
Article ID: Q92687
The information in this article applies to:
  • Microsoft Access versions 1.0, 1.1, 2.0, 7.0, 97

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

When you try to open a table in Design view in Microsoft Access 7.0 or 97, you may receive the following error message: 

   You can't open the table '<tablename>' for modification.
In Microsoft Access 1.x and 2.0, you may receive the following error message: 

   Can't open table '<table name>' for modifications. Open it read-only?

CAUSE

This error message occurs when you attempt to open a table in Design view when the table is the record source for a form or report, and one of the following conditions is also true:

  • A report is open in Print Preview or a form is open in Form view that has the same record source or is based on a query with the same record source.
  • A form is opened in Form view first, and then switched to Design view.
  • A report is opened in Design view, then switched to Print Preview, and then switched back to Design view. (You cannot switch to Design view if you opened the report in Print Preview first.) This message does not occur when you open the form or report in Design view first (without ever switching to Form view or Print Preview), and then open the table in Design view.

RESOLUTION

You must close the Form or Report window to be able to open the table in Design view to make changes.

STATUS

This behavior is by design.

REFERENCES

For more information about this topic, search the Help Index for "editing data, multiuser environment," and then "Tips for editing database objects in a multiuser environment."
