OL97: New Field Button Is Unavailable in Field Dialog Boxes

 Last reviewed: January 19, 1998
Article ID: Q159857
The information in this article applies to:
  • Microsoft Outlook 97

SYMPTOM

When using the Field Chooser or Show Fields options in Microsoft Outlook 97, the resulting dialog boxes may display the New button as dimmed (unavailable).

CAUSE

You can only create New fields in MAPI folders to which you have sufficient permissions.

MORE INFORMATION

You can access the Field Chooser and Show Fields options on the View menu in Outlook. The New button is not available for one of the following reasons:

- The current folder is a file system folder. For example, if the folder 

  banner shows that the active folder is My Documents, then the New button
  will be dimmed.
- You do not have sufficient permissions to the selected MAPI folder. If 
  the current folder is a MAPI folder, such as a public folder, you must
  have sufficient permissions to that folder to allow the creation of
  fields. Have the folder owner ensure create permission is available to
  you for the active folder. The folder owner, or another user with
  sufficient permissions, can access the permission settings for the folder
  by right-clicking the folder in the Outlook Bar or Folder List, and
  then clicking Properties on the shortcut menu.
To validate that the New field button is working properly, follow these steps:

  1. Select the Inbox or another MAPI folder to which you know you have sufficient permissions.

  2. On the View menu, click Show Fields or Field Chooser.

  3. Confirm that the New button is available. :
