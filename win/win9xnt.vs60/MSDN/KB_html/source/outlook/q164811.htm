OL97: Err Msg: Outlook Setup Cannot Create Schdmapi.dll

 Last reviewed: January 19, 1998
Article ID: Q164811
The information in this article applies to:
  • Microsoft Outlook 97 version 8.01
  • Microsoft Office 97 for Windows

SYMPTOMS

When you attempt a "Run From Network Server" setup of Outlook 8.01 to run an Outlook workstation from a server, you may encounter the following error message: 

   Setup cannot create folder <Run from Server Path>\Office\Schdmapi.dll
If you choose Ignore, the setup continues without updating the Schdmapi.dll file in the Office 97 server folder.

CAUSE

Outlook setup is trying to update a read-only Schdmapi.dll file on the Office 97 administrative installation folder on the server.

RESOLUTION

To resolve this problem, rename the Schdmapi.dll file on the Office 97 administrative installation folder on the server and copy the Schdmapi.dll file from the Outlook 8.01 Compact Disc to the Office 97 administrative installation folder on the server.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Outlook 8.01. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

You should use the above resolution before attempting a Quiet mode Setup to ensure that the Quiet mode Setup is not interrupted by the error dialog.

Other Office programs, such as Word, also use the Schdmapi.dll file. Not updating this file may cause errors in other Office programs, especially when using address books.

REFERENCES

Microsoft Office 97 Resource Kit. Redmond, Washington: Microsoft Press, 1997.
