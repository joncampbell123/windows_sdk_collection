OL97: MPREXE Causes Invalid Page Fault When Checking for Mail

 Last reviewed: January 19, 1998
Article ID: Q171024
The information in this article applies to:
  • Microsoft Outlook 97

SUMMARY

If the Windows 95 password list file is damaged, you may receive the following error message when you try to check for new mail in Microsoft Outlook: 

   MPREXE caused an invalid page fault in module Kernel32.dll

MORE INFORMATION

Microsoft Outlook 97 stores mail server passwords in the Windows 95 password list file. When Outlook 97 checks for new mail, it must reference this password list file. The name of the password list file is generated from the name used as the Windows Logon ID, for example <Username>.pwl.

If your password list file becomes corrupted or damaged, you can rebuild it by following these steps:

  1. On the Windows taskbar, click Start, point to Find, and then click Files Or Folders.

  2. In the Named box, type the following: 

          *.pwl

  3. Click Find Now.

  4. Right-click the file called <username>.pwl, and then click Rename.

  5. Rename the file to <username>.old, and then press ENTER.

  6. Restart Windows 95.

  7. When prompted for a name and password, you must type in a name. This name will be your Logon ID and will also be used as a name for your new password list file. Entering a password is optional. Leaving the password blank will create what is called a null password. If you use the null password, Windows will not prompt you for a password again.

The first time you run any application that stores a password in the password list file, you will be prompted for a password. If you tell the application to remember the password, it will be stored in the newly created password list file.
