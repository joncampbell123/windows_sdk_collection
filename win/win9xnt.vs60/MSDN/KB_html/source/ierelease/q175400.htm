Mouse Pointer Automatically Moves to Upper-Left Corner of Screen

 Last reviewed: November 26, 1997
Article ID: Q175400
The information in this article applies to:
  • Microsoft Internet Explorer version 4.0 for Windows 95
  • Microsoft IntelliType software version 1.1

SYMPTOMS

When you try to click a command after clicking Start, the mouse pointer may move to the upper-left corner of the screen.

CAUSE

This behavior can occur if you are using IntelliType with the SnapTo option enabled.

RESOLUTION

To resolve this problem, disable the SnapTo option, and then download and install the 11p32upd.exe file. To do so, follow these steps:

  1. Press CTRL+ESC to open the Start menu.

  2. Use the UP ARROW key to select Settings, and then press the RIGHT ARROW key to select Control Panel. Press ENTER.

  3. Use the ARROW keys to select the Keyboard icon, and then press ENTER.

  4. Press SHIFT+TAB, press RIGHT ARROW, and then press RIGHT ARROW to select the Pointer Activity tab.

  5. Press TAB three times, and then press SPACE BAR to clear the "Snap pointer to default button in dialog boxes check box.

  6. Press TAB twice, and then press ENTER.

  7. Close Control Panel.

  8. Download and install the 11p32upd.exe file from the Microsoft Software Library. To do so, follow these steps:

    a. Download the 11p32upd.exe file from the Microsoft Software Library 

          to an empty folder on your hard disk.

    b. Double-click the 11p32upd.exe file you downloaded in step A.

    c. Follow the instructions on the screen.

NOTE: After you install the 11p32upd.exe file, you can turn the SnapTo option back on if you want.

The following file is available for download from the Microsoft Software Library: 

 ~ 11p32upd.exe (size: 162016 bytes)
For more information about downloading files from the Microsoft Software Library, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q119591
   TITLE     : How to Obtain Microsoft Support Files from Online Services

STATUS

Microsoft has confirmed this to be a problem in Microsoft IntelliType version 1.1.
