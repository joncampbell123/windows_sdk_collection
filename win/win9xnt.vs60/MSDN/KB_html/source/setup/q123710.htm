PRB: GP Fault Running Setup on DEC Alpha

 Last reviewed: September 9, 1996
Article ID: Q123710
The information in this article applies to:
  • Microsoft Setup, version 1.1
  • Microsoft Access version 2.0
  • Microsoft Windows NT operating system version 3.1

SYMPTOMS

When you try to install any upgrade version of Microsoft Access version 2.0 on a DEC Alpha computer running Microsoft Windows NT version 3.1, you may receive the following general protection (GP) fault error message: 

   ACMSETUP caused a general protection fault in module KRNL286.EXE

CAUSE

This error is caused by the compliance checking program (CCP) used in the Setup program in upgrade versions of Microsoft Access 2.0.

RESOLUTION

Obtain and install a retail (non-upgrade) version of Microsoft Access version 2.0. The retail version of Microsoft Access 2.0 does not call the CCP.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access version 2.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
