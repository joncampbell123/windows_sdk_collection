Err Msg: "Cannot Find <Filename>.EXE" During Installation

 Last reviewed: September 9, 1996
Article ID: Q119824
The information in this article applies to:
  • Microsoft Office for Windows, version 4.0
  • Microsoft Excel for Windows, version 5.0

SUMMARY

When you run the Setup program for one of the applications listed above, you receive one of the following error messages:

  • Microsoft Office version 4.0 

          Cannot find WINWORD.EXE

    -or-

  • Microsoft Excel version 5.0 

          Cannot find 'C:\EXCEL\EXCEL.EXE'

CAUSE

This error occurs if you run the Setup program while a virus-protection utility is running. Most virus-protection utilities do not allow the Setup program to combine the necessary files to create the WINWORD.EXE or EXCEL.EXE file, which results in the error.

WORKAROUND

To work around this problem, temporarily disable the virus-protection utility and then run the Setup program again.

For example, if you are using MS-DOS 6.0 or MS-DOS 6.2 and are running VSAFE.COM, you can work around the problem by disabling the line in your AUTOEXEC.BAT file that contains VSAFE, restarting your computer, and then running the Setup program. After you have installed the application, you can modify your AUTOEXEC.BAT file to run VSAFE.COM again.

MORE INFORMATION

Page 5 of the Microsoft Excel version 5.0 "User's Guide" mentions that you should disable any anti-virus software before installing Microsoft Excel. Also, the VSAFE.COM Help file states that VSAFE.COM should not be used when Microsoft Windows is running.

For more information about using VSAFE.COM, do one of the following:

  • Quit Windows, and at an MS-DOS command prompt, type "help" (without the quotation marks).

    -or-

  • From the Search menu, choose Find and type "vsafe" (without the quotation marks).

REFERENCES

"Microsoft Excel User's Guide," version 5.0, Chapter 1, "What You Need to Use Microsoft Excel"
