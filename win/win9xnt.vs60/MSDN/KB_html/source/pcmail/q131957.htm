PROFS: VMGATE.EXE Version 3.40.0042 Update

 Last reviewed: March 9, 1998
Article ID: Q131957
3.40a MS-DOS kb3rdparty kbfile

The information in this article applies to:

  • Microsoft Mail Gateway to IBM PROFS and OfficeVision, version 3.4a

SUMMARY

Microsoft provides a replacement for the VMGATE.EXE file that is included with version 3.4a of Microsoft Mail Gateway to IBM PROFS and OfficeVision.

For complete information about obtaining and installing the VMGATE.EXE file, see the following sections below:

  • To download the updated files
  • To update your VMGATE.EXE file

MORE INFORMATION

This update corrects the following problem:

  • The following errors occur in the VMGATE log:

    [15:24] - ERROR: scanning data fields in: L:MAI\MAC\0001594C.MAI 

       [15:24] -         ...will try again at NEXT time, processing continues

   [15:26] - ERROR: scanning data fields in: L:MAI\MAC\0001594C.MAI
   [15:26] -         ...will try again at NEXT time, processing continues

   [15:28] - ERROR: scanning data fields in: L:MAI\MAC\0001594C.MAI
   [15:28] -         ...SYSTEM.LOG entry written, diagnostic sent to:

   After the .MAI file fails for the third time, the mail is returned to
   the sender and the following message is logged in the system log:

      System log entry on: 12-15-94 15:28
      VMGATE program has detected a mail problem
      Message data:
        From: IES/TOWER/SZHJZG
        Date: 12-15-94
        Time: 15:29
        Subject: MMIS year end processing issues
      Mail retry count exceeded sending to: VMGATE

To update your VMGATE.EXE file

 


Decide which executable file matches your configuration:



   Executable file   3270 emulation card supported
   ---------------------------------------------------------------------
   NOVELL.EXE        Novell NetWare 3270
   DCA_CUT.EXE       DCA CUT-mode
   DCA_DFT.EXE       DCA Non-CUT (DFT) mode
   IBM_HIGH.EXE      IBM High Level


At the MS-DOS command prompt, type the following and press ENTER



   copy <path>:\VMGA.exe <destination>\vmgate.exe


where <path> is the drive and directory where you ran the self-extracting
VMGA.EXE file, <destination> is the drive and directory where your
VMGATE.EXE file currently resides, and VMGA is the corresponding
executable file for your software vendor. For example, if you ran the self-
extracting file from the TEST directory on drive D, your VMGATE.EXE file is
located in the MAILEXE directory on drive C, and you are using IBM High
Level, type the following command and press ENTER:



   copy d:\test\IBM_HIGH.EXE c:\mailexe\vmgate.exe
