PC Adm: Non-Alphanumeric Password Does Not Allow Connection

 Last reviewed: March 31, 1995
Article ID: Q128319
The information in this article applies to:
  • Microsoft Mail for PC Networks, versions 3.0, 3.0a, 3.2, and 3.2a

SYMPTOMS

When you enter a password for an external postoffice that is configured as Direct Via modem, the password field will not accept non-alphanumeric characters.

The password can be configured via the Config, Password option under ADMIN.EXE to contain non-alphanumeric characters. If this occurs, it will be impossible to connect directly to this postoffice via modem.

RESOLUTION

Change the postoffice password to only contain alphanumeric characters.

STATUS

Microsoft has confirmed this to be a problem in versions 3.0, 3.0a, 3.2, and 3.2a of Microsoft Mail for PC Networks. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
