FPD: Previous File Name Not Cleared When Printing to File

 Last reviewed: March 9, 1998
Article ID: Q119423
3.00a MS-DOS kbbug3.00a kb3rdparty

The information in this article applies to:

  • Microsoft Mail FAX Printer Driver, version 3.0a

SYMPTOMS

When you "Print to File" with the FAX Printer Driver selected as your default printer driver, the first filename you enter stays in the Name Output File Name dialog box. Later uses of the command do not replace it with new filenames.

Version 3.0a of the Microsoft FAX Printer Driver comes with versions 3.0 and 3.0a of the Microsoft Mail Gateway to FAX and with versions 3.0, 3.0a and 3.2 of Microsoft Mail for PC Networks.

CAUSE

Under investigation.

WORKAROUND

Manually clear the highlighted filename before you select OK in subsequent "Print to File" operations.

STATUS

Microsoft has confirmed this to be a problem with the Microsoft FAX Printer Driver, version 3.0a. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
