PC WFW: Users Don't Appear in the GAL After Upgrade

 Last reviewed: October 24, 1994
Article ID: Q91071
The information in this article applies to:
  • Microsoft Mail for PC Networks, version 3.0

SYMPTOMS

After you use Extensions for Windows for Workgroups to upgrade a Windows version 3.1 for Workgroups postoffice to a version 3.0 Microsoft Mail for PC Networks postoffice, those mail accounts created before the Extend program was run are not displayed in the Global Address List (GAL).

RESOLUTION

To add those users to the Global Address List, manually export the user list from the Mail Administrator program (ADMIN.EXE) by selecting Config, Dir-Sync, Requestor, Export.

STATUS

Microsoft has confirmed this to be a problem in version 3.0 of Extensions for Windows for Workgroups. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
