Maintain 8.3 Naming Standard to Avoid Branding Error

 Last reviewed: February 13, 1998
Article ID: Q172137
The information in this article applies to:
  • Microsoft Internet Explorer Administration Kit, version 4.0

SYMPTOMS

Branding errors may occur if you use long filenames when creating customized versions of Internet Explorer using the Internet Explorer Administration Kit (IEAK).

WORKAROUND

To avoid branding errors, maintain the 8.3 naming convention for all files specified in the IEAK wizard.
