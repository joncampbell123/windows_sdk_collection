DRW98: Draw 98 Setup Requires Mso97.dll and Mso7enu.dll

 Last reviewed: February 23, 1998
Article ID: Q181552
The information in this article applies to:
  • Microsoft Draw 98 for Windows

SYMPTOMS

When attempting to install Microsoft Draw 98, you may receive the following error message: 

   Setup could not find at least one Microsoft Office 97 application
   installed on your system. Please install at least one Microsoft Office
   97 application before installing this application.

   -or-

   Setup could not find a matching language version of Office 97 installed
   on your system. Please install a matching language version of Office 97
   before installing this application
followed by: 

  Microsoft Draw 98 was not completed successfully.

CAUSE

Draw 98 requires Mso97.dll and Mso7enu.dll. If the Draw 98 setup program is unable to find both of these dynamic-link library (DLL) files, it does not continue.

RESOLUTION

To resolve this problem, you must have Microsoft Office 97 (or at least one of the Office programs) installed on your computer prior to installing Draw 98.

MORE INFORMATION

Draw 98 only installs if it finds the Mso97.dll file and the matching language DLL (for example, Mso7enu.dll). Setup uses the following logic when attempting to locate the Mso97.dll file:

  1. Setup looks for the registry entry for Mso97.dll. If Setup does not find this registry entry, it fails.

  2. If Setup finds Mso97.dll, it then looks for Mso7enu.dll. It looks for Mso7enu.dll on all local disk drives.

NOTE: Setup also fails if the installation of Office 97 was an ADMIN installation or if it was installed onto a network drive or removable disk drive.
