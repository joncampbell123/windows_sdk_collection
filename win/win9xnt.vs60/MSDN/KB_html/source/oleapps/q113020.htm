ClipArt Gallery Cannot Find ARTGALRY.IDX When First Used

 Last reviewed: February 25, 1998
Article ID: Q113020
The information in this article applies to:
  • Microsoft ClipArt Gallery for Windows, versions 1.0, 1.1, 2.0

SUMMARY

The first time you try to insert a Microsoft ClipArt Gallery object, the following message appears: 

   The ClipArt Gallery cannot find <path>\artgalry.idx.

   This file contains information about thumbnail pictures for the ClipArt
   Gallery. Choose Browse to locate the file. Choose New to create a new
   database of thumbnail pictures.
If this is the first time you have tried to insert a Microsoft ClipArt Gallery object, click New. ClipArt Gallery then checks to see if any applications have provided information for importing their clip art images into the ClipArt Gallery. If any such information is found, you are usually prompted as to whether you want to add the images with a message similar to the following: 

   Would you like to add clip art from <application name> now?
   This may take a few minutes.
If you click Yes, the clip art thumbnails are added to ClipArt Gallery.

Depending on the number of applications that have been installed and the number of clip art collections to be added, you may be prompted several times before the ClipArt Gallery window appears. If you cancel any of the clip art importing, ClipArt Gallery still runs, but you receive the following message: 

   The ClipArt Gallery will not contain all of your new clip art.
And each time you attempt to insert a ClipArt Gallery object in the future, you will be prompted to add the new clip art that you previously chose not to add.

MORE INFORMATION

The Setup programs for the applications that install Microsoft ClipArt Gallery do not install or create this index file automatically because they might replace an existing index file from an earlier installation of another application that includes ClipArt Gallery.
