MSGraph: Empty Text Boxes in Format Text Dialog Box

 Last reviewed: February 19, 1998
Article ID: Q91104
3.00 WINDOWS kbtool

The information in this article applies to:

  • Microsoft Graph for Windows, version 3.0b
  • Microsoft PowerPoint for Windows, version 3.0
  • Microsoft Windows operating system version 3.1

SUMMARY

Microsoft Graph version 3.0b, which ships with version 3.0 of Microsoft PowerPoint for Windows, requires the MS Sans Serif screen font to properly display rotated text in the Format Text dialog box. Under some circumstances if you choose Text from the Format menu in Microsoft Graph, the text boxes in the Orientation section of the Format Text dialog may appear as empty rectangles.

MORE INFORMATION

This problem occurs under the following circumstances:

  1. The Helv= line in the [FontSubstitutes] section of WIN.INI references a font other then MS Sans Serif.

  2. The MS Sans Serif font is corrupted.

  3. The MS Sans Serif font has been deleted.

  4. A 3rd party TrueType font called "Helv" is added to the Windows environment.

Steps to Troubleshoot Problem

  1. Verify that the MS Sans Serif font is installed:

    a. Open Control Panel and choose the Fonts icon.

    b. Highlight the MS Sans Serif entry on the list of installed 

          fonts.

    c. If a font sample does not appear, or if the MS Sans Serif entry 

          is missing entirely, choose the Add button.

    d. In the Add Fonts dialog box, go to the WINDOWS\SYSTEM directory.

    e. A list of available fonts will appear. Choose MS Sans Serif and 

          choose the OK button.

    f. Highlight the MS Sans Serif entry again to verify that a font 

          sample displays.

  2. Check the [FontSubstitutes] section of WIN.INI. There must be a line in this section that reads as follows: 

          Helv=MS Sans Serif

    If this is not correct, correct or add this line, save the WIN.INI file and restart Windows.

  3. Verify the integrity of the MS Sans Serif font by using Character Map:

    a. Run Character Map.

    b. From the fonts list field, select MS Sans Serif.

    c. Verify that characters are showing up in the Character Map 

          matrix.

    Using Control Panel to check the sample display of a font may not correctly identify a corrupted font. The Fonts display may map to another font incorrectly which will make a corrupt font appear to be correct.

  4. Check the Fonts section of Control Panel to see if a third party TrueType font package containing a Helv font has been added to the Windows environment. If you see that a Helv TrueType font has been added, perform the following steps:

    a. Open Control Panel and choose the Fonts icon.

    b. Highlight the Helv entry on the list of installed fonts. c. Choose the Remove Fonts button.

    d. In the Remove Fonts dialog box, clear the check box labeled 

          Delete Font File from Disk.
