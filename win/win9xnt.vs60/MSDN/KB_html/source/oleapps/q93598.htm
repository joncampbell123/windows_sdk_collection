WordArt Err Msg: "Could Not Load Necessary Font Files..."

 Last reviewed: September 10, 1996
Article ID: Q93598
The information in this article applies to:
  • Microsoft WordArt version 1.0
  • Microsoft Windows operating system versions 3.0 and 3.1

SUMMARY

If you try to create an object with Microsoft WordArt (which ships with Microsoft Publisher version 1.0; Microsoft Word for Windows versions 2.0, 2.0a, 2.0b, and 2.0c; and Microsoft Publisher Design Pack version 1.0), you may receive the following error message: 

   Microsoft WordArt could not load necessary font files. Please ensure
   WordArt is properly installed.
This message is generated if the file FONTFX.FNT is damaged or missing. This file should be located in the WORDART subdirectory of the MSAPPS directory.

MORE INFORMATION

To correct this problem, use the following steps:

  1. Rename or delete the current copy of FONTFX.FNT (if it exists).

  2. Run the Setup program for the application with which you received WordArt. (To do this, place Disk 1 from the application in either drive A or B, choose Run from the File menu, and type "A:\SETUP" or B:\SETUP" (without the quotation marks) at the command prompt.

  3. The Setup program then prompts you to select either a default or a custom setup installation. Choose the custom installation.

  4. A checklist of various program components is displayed. If there is an option for WordArt by itself, select that option only. If not, select the Program Files option.

  5. Place the appropriate disk in the appropriate drive when prompted by the Setup program.

Some applications, such as Microsoft Publisher, have methods for manually decompressing files from the distribution disks. If the application supports this, FONTFX.FNT can simply be decompressed from the disks and copied to the WINDOWS\MSAPPS\WORDART directory. For more information, query on the following words in the Microsoft Knowledge Base: 

   <the name of the application> and manual and decompress and file
