Draw: Unable to Undo Last Operation After Changing View

 Last reviewed: September 9, 1996
Article ID: Q79688
The information in this article applies to:
  • Microsoft Draw version 1.0

SUMMARY

In Microsoft Draw version 1.0, the Undo command will be unavailable if you switch to a different view or application before attempting to choose Undo from the Edit menu.

Steps to Reproduce Problem

  1. In Microsoft Draw, choose a command that you can normally undo. For example, choose the arrow button, and move an object in your drawing.

  2. From the View menu, choose a different view, such as 75%.

  3. Make Draw the active window again by positioning the insertion point in your drawing.

  4. View the Edit menu.

The Undo command is unavailable.

MORE INFORMATION

Microsoft PowerPoint version 2.0 for Windows also exhibits this problem; however, other Windows applications that include an Undo command (such as Word for Windows, Excel for Windows, and Works for Windows) allow you to undo your last action even after switching to another view or application.

Microsoft is researching this problem and will post new information here as it becomes available.
