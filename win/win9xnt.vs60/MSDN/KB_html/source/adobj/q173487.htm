INFO: ADO 1.1 QFE Download Now Available (Updates ADO 1.0)

 Last reviewed: September 17, 1997
Article ID: Q173487
The information in this article applies to:
  • ActiveX Data Objects (ADO), version 1.0

SUMMARY

The ActiveX Data Objects (ADO) QFE includes redistributable files for ADO 1.1 and the OLE DB provider for ODBC, version 1.1. The QFE also provides a registry update so that ADO 1.1 and future versions of ADO can co-exist, and improved handling of DBCS data by the GetChunk method.

MORE INFORMATION

You should use this version of ADO 1.1 and Setup for redistributing ADO 1.0- enabled applications.

If you use the Microsoft Visual Basic Setup Wizard, you will need the Adoregex.dll file. This DLL invokes the ADO redistribution setup (provided in the download) from within a distribution package created with the Visual Basic Application Setup Wizard. For more information, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q172406
   TITLE     : FILE: Redistributing ADO 1.0/1.1 with Visual Basic 5.0
ADO 1.x runs on Windows 95 and Windows NT 4.0 and higher. It is not certified to run on Windows NT 3.51.

This setup includes redistributable files for ADO and OLE DB. It does not include any files for ODBC 3.0. ODBC 3.0 files are included in Office 97 and Visual Studio 97 and are available from http:\\www.microsoft.com\data\odbc\. Localized versions of the ADO QFE will be available early in Q4 of 1997 for the following languages:

  • Dutch
  • English
  • French
  • German
  • Japanese
  • Spanish
  • Swedish

There is no silent version for this installation.

This file is signed, so you can use it on a Web page. The HTML tag for downloading the ADO redistributable would look something like this (which also assumes you renamed the ADO 1.0 QFE to Ado1inst.exe): 

   <object id="ADO1INST"
   classid="CLSID:A21D8702-F6B2-11D0-9B1F-00AA00A102C4"
   codebase="ado1inst.exe#version=4.71.0819.0"
   align="baseline"
   border="0"
   width="50"
   height="50">
For additional information, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q167597
   TITLE     : FILE: Specifying FileVersion and #Version for Component
               Download
