VBE: No ScreenTips in Visual Basic Editor When Balloon Help On

The information in this article applies to:

  • Microsoft Office 98 Macintosh Edition
  • Microsoft Excel 98 Macintosh Edition
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

When you move the mouse cursor over a toolbar button in the Visual Basic Editor, neither Balloon Help nor ScreenTips appear.

Also, the first time you activate the Visual Basic Editor in Microsoft Excel 98 Macintosh Edition and move the mouse cursor over a toolbar button, you may receive the following error message: 

   Microsoft Excel cannot open the required Help file. Please run the
   installer application to restore any missing files.

CAUSE

These problems occur if you have Balloon Help turned on. Balloon Help is not available in the Visual Basic Editor, and if Balloon Help is turned on, ScreenTips do not appear.

RESOLUTION

If you want ScreenTips to appear in the Visual Basic Editor, you must turn off Balloon Help. To do this, click Hide Balloons on the Help menu.

STATUS

This behavior is by design of the programs listed at the beginning of this article.

MORE INFORMATION

There are two types of "instant help" available for toolbar buttons: ScreenTips and Balloon Help.

  • ScreenTips appear when you move the mouse cursor over a toolbar button for a couple of seconds. Usually, ScreenTips identify the toolbar button, but they do not explain the functionality of the button.
  • Balloon Help provides a more extended explanation of the functionality of a toolbar button or menu item. To turn on Balloon Help, click Show Balloons on the Guide menu. To see Balloon Help, move the mouse cursor over a button or menu item. Within a couple of seconds, a balloon containing Help information will appear.

When you turn on Balloon Help, ScreenTips are automatically disabled. Because Balloon Help is not available in the Visual Basic Editor, there is no "instant help" available in the Visual Basic Editor when Balloon Help is enabled.
