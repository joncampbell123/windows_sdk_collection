PRB: KEYCOMP = DOS Disables CTRL+F10 and CTRL+F5

 Last reviewed: June 27, 1995
Article ID: Q124326
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5, 2.5a, 2.5b, 2.6, 2.6a
  • Microsoft FoxPro for Macintosh, versions 2.5b, 2.5c, 2.6a

SYMPTOMS

Pressing CTRL+F10 does not maximize a window, and pressing CTRL+F5 does not minimize a window.

CAUSE

The KEYCOMP setting in the CONFIG.FPW file has been set to MS-DOS, or the SET KEYCOMP TO DOS command has been used.

RESOLUTION

Change the KEYCOMP setting to WINDOWS. If the setting was changed by using the CONFIG.FPW file use the following command to change the setting to WINDOWS: 

   KEYCOMP = WINDOWS
If the setting was changed by using the command SET KEYCOMP TO DOS use the following command to set it to WINDOWS: 

   SET KEYCOMP TO WINDOWS

STATUS

This behavior is by design.
