32-Bit .EXE Requires 32-Bit Development Version Installed

 Last reviewed: April 17, 1995
Article ID: Q99551
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.5 and 2.5a
  • Microsoft FoxPro Distribution Kit for MS-DOS, versions 2.5 and 2.5a

With FoxPro versions 2.5 and later for MS-DOS, you must have the Extended (32-bit) version of FoxPro for MS-DOS (FOXPROX.EXE) installed in order to build 32-bit stand-alone .EXE files. 32-bit stand-alone .EXE files can be built from the 16- or 32-bit product but only if FOXPROX.EXE is installed. If FOXPROX.EXE is not installed, the "Cannot create file \FoxProx.exe" error message is returned.

FOXPROX.EXE must be installed in this case because it contains components that are needed by FoxPro when building 32-bit stand-alone .EXE files and these components cannot be legally distributed any other way.
