Using Printer Driver Setup When Running FoxPro MS-DOS Report

 Last reviewed: August 28, 1995
Article ID: Q121053
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5x, 2.6, 2.6a

SYMPTOMS

When a report is created with a defined printer setup and invoked from the Run menu, the printer setup will not be used.

RESOLUTION

To use the report with its related printer driver, you must invoke the report either from the Database menu or by issuing the REPORT FORM <formname> PDSETUP command from the Command window. If you invoke the report from the Database menu, choose the Set Printer Driver option to use the report with the defined printer driver.

NOTE: The online Help topic for the Set Printer Driver option on the Database menu incorrectly lists the dialog box as the Printer Driver Setup dialog box, but the functionality is the same.
