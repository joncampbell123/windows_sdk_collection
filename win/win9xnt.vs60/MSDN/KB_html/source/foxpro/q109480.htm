PRB: "Feature Not Available" After Migrating FB+/Mac Screen

 Last reviewed: May 16, 1996
Article ID: Q109480
The information in this article applies to:
  • Microsoft FoxPro for Macintosh, version 2.5b

SYMPTOMS

After you migrate a FoxBASE+ version 2.01 for the Macintosh screen to FoxPro 2.5 for Macintosh, generate the newly migrated screen, and then execute this new code, the error "Feature Not Available" will be displayed with the "MODIFY WINDOW SCREEN" command highlighted in the .SPR file.

CAUSE

The FoxBASE+/Mac Migration utility creates a FoxPro screen "in desktop" with the "System" characteristic. The "SYSTEM" clause is not a valid clause when modifying the FoxPro desktop, or "SCREEN".

RESOLUTION

To solve this problem, do the following:

  1. Open the migrated screen.

  2. From the Screen menu, choose Screen Layout.

  3. Choose the Window Style button.

  4. Change the window type from Desktop to another type (for example, User).

  5. Change the type back to Desktop (if desired).

  6. Generate the screen again by choosing Generate from the Program menu.

MORE INFORMATION

Steps to Reproduce Problem

  1. From the Program menu, choose Do, and double-click FB+Migrate.App in the FoxPro:Goodies:FB+Migrate folder.

  2. Click the Convert Files button.

  3. Select the FoxBase+/Mac:Tutorial:Checkbook:Forms folder as the Source folder, and the FoxPro:Tutorial folder as the Destination folder.

  4. Add RECONCILIATION.SCX to the Selected Files list.

  5. Click the Convert button, and then the Done button.

  6. In the command window, type the following: 

          MODIFY SCREEN \FoxPro\Tutorial\Reconciliation.scx

  7. From the Program menu, choose Generate. Click the Generate button.

  8. Close the screen.

  9. From the Program menu, choose Do, and double-click RECONCILIATION.SPR in the FoxPro:Tutorial folder.

The error "Feature Not Available" will be displayed with the "MODIFY WINDOW SCREEN" command highlighted in the .SPR file.
