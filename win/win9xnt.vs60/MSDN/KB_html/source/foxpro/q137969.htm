BUG: Report's Graphic Boxes Intersection Displays Incorrectly

 Last reviewed: October 15, 1995
Article ID: Q137969
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.5, 2.5a, 2.5b, 2.6, 2.6a

SYMPTOMS

The intersection of graphical boxes grouped on a report created in FoxPro version 2.6x for MS-DOS displays incorrectly when you run or edit the report in FoxPro version 2.5x for MS-DOS running under MS-DOS version 5.0.

WORKAROUND

Steps to Workaround Problem

  1. Edit the report in FoxPro version 2.5x for MS-DOS.

  2. Select the Box Group that is displayed incorrectly.

  3. On the Report menu, click UnGroup.

  4. Select All the Boxes that were ungrouped in step 3.

  5. On the Report menu, click Group Boxes.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The intersections of graphical boxes drawn on a report should appear as crosses. However, under these specific circumstances, they appear in the form of an H.

Steps to Reproduce Problem

  1. Create a report in FoxPro version 2.6x for MS-DOS.

  2. Draw two intersecting graphical boxes on the report.

  3. Select both of the boxes, and on the Report menu, click Group Boxes.

  4. Save the report.

  5. Take the report to a computer that's running FoxPro version 2.5x for MS-DOS and MS-DOS version 5.0.

  6. Run the report.

NOTE: Where the graphical boxes intersect, they appear in the form of an H instead of a cross as expected.
