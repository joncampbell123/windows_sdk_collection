FIX: PROAPI16 Upper Memory Block (UMB) Use Problems

 Last reviewed: October 14, 1997
Article ID: Q87594
2.00 MS-DOS kbprg kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft FoxPro for MS-DOS, version 2.0

SYMPTOMS

The PROAPI16 application uses upper memory blocks (UMBs) above 640K to make additional memory available. This causes problems in some memory configurations.

STATUS

Microsoft has confirmed this to be a problem in Microsoft FoxPro version 2.0 for MS-DOS. This problem was corrected in Microsoft FoxPro version 2.0 for MS-DOS released July 19, 1991.
