PRB: Class Browser Shortcut Keys Interfere with VFP Shortcuts

 Last reviewed: September 17, 1995
Article ID: Q136847
The information in this article applies to:
  • Microsoft Visual FoxPro for Windows, version 3.0

SYMPTOMS

When the Class Browser is running, some shortcut keys interfere with Visual FoxPro shortcut keys. Specifically, the ALT+H (Hierarchical), ALT+T (Type), ALT+P (Protected), and ALT+E (Empty) conflict with the Help, Tools, Program, and Edit menus.

RESOLUTION

If the Command window is active while the Class Browser window is displayed, the Visual FoxPro hot keys function normally. Use the mouse, or press CTRL+F6 to switch to another window from the Class Browser.

STATUS

This behavior is by design.
