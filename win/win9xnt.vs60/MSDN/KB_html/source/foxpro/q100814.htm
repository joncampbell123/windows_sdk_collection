Problem with DO Program Example in Language Reference

 Last reviewed: April 29, 1996
Article ID: Q100814
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

Page L3-465 of the "Language Reference" gives a program example for the DO command that could cause the control panel, PANEL2, to be covered by the other window, CUSTOMER.

If this problem occurs, do one of the following to solve the problem:

  • Change the screen font to FoxFont by holding down the SHIFT key and choosing Screen Font from the Text menu.

    -or-

  • Modify the screen coordinates to reflect the font currently in use. To do this, modify the code example so that the control panel, PANEL2, has a larger vertical coordinate. Replace the window's second numeric value until the control panel window is no longer covered by the CUSTOMER window.
