BUG: Query Crashes ("File Does Not Exist") When Saving File

 Last reviewed: June 27, 1995
Article ID: Q112315
The information in this article applies to:
  • Microsoft FoxPro for Windows, version 2.6
  • Microsoft FoxPro for MS-DOS, version 2.6

SYMPTOMS

When you are saving a query to disk with a filename that includes an accented character (such as a letter with an umlaut), the Query Wizard will crash with the following error message: 

   ------------------------FoxPro Error--------------------------------
   Line Number:   261
   Program:           EMIT
   Error:                File does not exist.
   Source:              = ....

STATUS

Microsoft has confirmed this to be a problem in FoxPro version 2.6 for MS- DOS and FoxPro version 2.6 for Windows. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. From the File menu, choose Catalog Manager.

  2. Select the Query tab.

  3. Choose the New button.

  4. Select the Query Wizard.

  5. Select SQL Query Wizard and choose the Next button.

  6. Select Add then choose the Next button.

  7. Select Add then choose the Next button.

  8. Choose the Finish button.

  9. The next screen will have the Save Query For Later Use option button selected.

  10. Choose the Finish button.

  11. When prompted for the filename, type "TEST" (without the quotation marks), press ALT+0255 on the numeric keypad (which creates a "y" with an umlaut).

  12. Choose OK.

You will see the error message.
