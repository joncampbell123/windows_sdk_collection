PRB: "Maximum Record Locks Exceeded" on Novell 3.11

 Last reviewed: June 27, 1995
Article ID: Q98996
The information in this article applies to:
  • Microsoft FoxPro for Windows, version 2.5
  • Microsoft FoxPro for MS-DOS, versions 2.0 and 2.5

SYMPTOMS

When you are running FoxPro LAN version 2.0 or 2.5 from a Novell NetWare version 3.11 server, a record lock limitation is reached. FoxPro locks on a certain record and it will not lock any record after that record.

CAUSE

The default number of record locks on a Novell NetWare 3.11 server is 500 locks per connection. When FoxPro exceeds the 500 locks (depending on the amount of memory on the server) the problem described above occurs at a certain record. The file server will generate the following error message: 

   Maximum Record Locks Exceeded.

RESOLUTION

The Network Administrator needs to issue the following command on the file server: 

   Set Maximum Record Locks to 1000.

REFERENCES

Microsoft MS-DOS "User's Guide and Reference," version 5.0, pages 565-566
