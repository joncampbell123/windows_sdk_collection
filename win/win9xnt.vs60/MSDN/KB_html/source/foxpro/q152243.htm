MORE INFORMATION

Create "Enduring Variables" in FoxPro, page 1. Peter de Valenca. Creating user-defined equivalents of the FoxPro system memory variables. Useful when you want the safety and persistence of a system memory variable but the global accessibility of a PUBLIC variable.

README.1ST, Rockin' with FoxPro, page 2. Whil Hentzen.

Client/Server Focus: Do You Need Client/Server? Page 6. Robert Green. Tips on making the decision. SQL Server, Oracle, front end, back end.

Extend the Report Writer/Report Designer with UDFs, page 8. John Stepp.

Dr. FoxPro's Answer Clinic, page 14. John V. Petersen. Better calculator, limit number of times a form can be created in an application, work around Visual FoxPro's memory problems, changing fieldnames in a view after modifying table structure.

Use the Windows API to Print to Multiple Print Queues, page 16. Richard Aman. RegFn(), CallFn(), DECLARE, WNetGetConnection(), WNetAddConnection(), WNetCancelConnection(), WNetGetUser(), ExitWindows(), ExitWindowsEx().

Create Compound Boolean Searches, page 22. Jim Haentzschel.

Calling Win32 and 32-bit DLL Functions from FoxPro 2.x, page 25. Rick Strahl. DECLARE-DLL, #DECLARE, CALL32.DLL, SetFileAttributes, RegFP, registry, HKEY.

Cool Tip: XILIGHTS, page 30. Whil Hentzen. Colorize Your Editor with XILIGHTS. Shareware from Xitech to add color coding to both Visual FoxPro 3.0 and FoxPro for Windows 2.6x editors.

The Cutting Edge: Sleeping with the Enemy, page 31. Les Pinter. Editorial. Competition among database programs.

Fox World News, page 32. Bel Consulting ships FoxWord for Visual FoxPro. Allows the capture of report form output in a rich text format (.rtf) file. Azalea Software ships carrick, a Windows-based encryption tool.
