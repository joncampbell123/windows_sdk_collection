PRB: 286 Machine with QUADTEL.SYS Locks Up or Reboots Itself

 Last reviewed: April 17, 1995
Article ID: Q102464
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.5

SYMPTOMS

When the QUADTEL.SYS expanded memory manager (EMM) is used with FoxPro 2.5 for MS-DOS, problems may occur. For example, the machine may hang (lock up), reboot itself, or exhibit other strange behavior.

CAUSE

This behavior occurs because QUADTEL.SYS is not compatible with the Lotus/Intel/Microsoft (LIM) version 4.0 Expanded Memory Specification (EMS). EMMs must be LIM 4.0 compatible in order to work with FoxPro 2.5 for MS-DOS.

MORE INFORMATION

An EMM must be loaded in order for a 286 machine to access add-on expanded memory boards. The EMM enables the 286 machine to recognize and format any add-on memory as expanded memory.
