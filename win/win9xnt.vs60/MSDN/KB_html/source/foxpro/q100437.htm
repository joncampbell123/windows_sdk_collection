FIX: BUILD EXE Command Not Working in Command Window

 Last reviewed: September 22, 1997
Article ID: Q100437
2.50 WINDOWS kbprg kbfixlist kbbuglist kbdocerr

The information in this article applies to:

  • Microsoft FoxPro for Windows, version 2.5
  • Microsoft FoxPro Distribution Kit for Windows, version 2.5

SYMPTOMS

The BUILD EXE command does not work in the Command window of FoxPro 2.5 for Windows.

STATUS

Microsoft has confirmed this to be a problem in FoxPro 2.5 for Windows. This problem was corrected in FoxPro 2.5a for Windows.

MORE INFORMATION

A discrepancy exists in the FoxPro Distribution Kit for Windows documentation. On page 16 of the Distribution Kit's "User's Guide," under the heading "With the BUILD EXE Command," the text correctly states: 

   Instead of using the Project window, you can use the BUILD EXE
   command to create an .EXE file. With BUILD EXE, you can create
   an .EXE file in a program or the Command window.
However, on page L3-253 of the FoxPro 2.5 "Language Reference" manual, under "Remarks," the text states: 

   FoxPro for MS-DOS only
 The Help file in FoxPro for Windows directs the reader to the Distribution
Kit's "User's Guide."

The remark on page L3-253 of the FoxPro 2.5 "Language Reference" is incorrect.
