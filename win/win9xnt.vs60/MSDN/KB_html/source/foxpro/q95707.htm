PRB: DEFINE WINDOW <Custom Border String> Has Wrong Border

 Last reviewed: June 27, 1995
Article ID: Q95707
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

Page L3-432 of the "Language Reference" manual incorrectly describes the different types of border clauses of the DEFINE WINDOW command. The documentation states that in FoxPro for Windows a custom border string creates a window with the PANEL border. This information is incorrect. A custom border string actually creates a window with the SINGLE border.

Additional reference words: FoxWin 2.50 2.50a docerr
