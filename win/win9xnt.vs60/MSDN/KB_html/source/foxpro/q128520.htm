FIX: SET PRINT ON Before Printing a Report Produces Error

 Last reviewed: October 20, 1997
Article ID: Q128520
2.50b 2.50c MACINTOSH kbprint kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft FoxPro for Macintosh, versions 2.5b and 2.5c

SYMPTOMS

In FoxPro for Macintosh, if you issue the SET PRINT ON command prior to printing a report to a printer with the REPORT FORM <report filename> TO PRINTER ENVIRONMENT command, you will receive a System Error.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in FoxPro version 2.6a for Macintosh.

MORE INFORMATION

Steps to Reproduce Problem

  1. Create a quick report using SALESMAN.DBF.

  2. Issue the SET PRINT ON command before issuing the REPORT FORM salesman.frx TO PRINT ENVIRONMENT command.

    After a while, an error message dialog box will appear in FoxPro that says: 

          There is a printing problem. Please choose Print Monitor from the
      Application menu or click the Print Monitor window.

  3. Click the OK button in the error message dialog box to close down the dialog box.

  4. From Finder, choose Print Monitor from the Application Menu. Print Monitor will have this message in it: 

          "Unknown" from "Microsoft FoxPro" is damaged and cannot be printed.

  5. Click the OK button in the error message dialog box to close down the dialog box.

  6. From the Print Monitor, Click the upper left corner to close it down.
