PRB: Cannot Access General Fields in During BROWSE

 Last reviewed: April 18, 1995
Article ID: Q106162
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.5, 2.5a, and 2.5b

SYMPTOMS

You can not access general fields during a BROWSE command.

CAUSE

The ability to create general fields in FoxPro for MS-DOS through the CREATE TABLE command or the MODIFY STRUCTURE dialog box exists to allow developers working in MS-DOS to code applications that will run in FoxPro for Windows.

However, this feature in FoxPro for MS-DOS does not allow general fields to be manipulated. When you browse a table with a general field, the computer will beep each time you attempt to type into that field, paste into it, or erase it.

Additional reference words: FoxDos 2.50 2.50a 2.50b
