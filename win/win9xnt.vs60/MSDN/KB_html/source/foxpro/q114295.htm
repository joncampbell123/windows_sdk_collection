BUG: Date Fields Won't Show 4-Digit Year with SET CENTURY ON

 Last reviewed: June 27, 1995
Article ID: Q114295
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.6
  • Microsoft FoxPro for Windows, version 2.6

SYMPTOMS

The width of a date field in a screen created by the Screen Wizard is not large enough to display a year in the format "nnnn" (for example, "1994") even though SET CENTURY is ON.

CAUSE

The date field is large enough to accommodate only two digits for the year instead of four. The width of a date field created by the Screen Wizard is not affected by SET CENTURY.

RESOLUTION

After creating a screen with the Screen Wizard, you can manually increase the size of the date field by editing that screen with the Screen Builder. To modify the file, use the MODIFY SCREEN <screen name> command.

STATUS

Microsoft has confirmed this to be a problem in FoxPro version 2.6 for MS- DOS and FoxPro version 2.6 for Windows. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
