PRB: Error Type 7 When Running Wizard Created in FP for Win

 Last reviewed: June 3, 1996
Article ID: Q115406
The information in this article applies to:
  • Microsoft FoxPro for Macintosh, versions 2.5b, 2.5c

SYMPTOMS

When a screen that was created by the Screen Wizard in FoxPro for Windows is executed in FoxPro for Macintosh, the following error message occurs: 

    <Picture of a Bomb>

   Sorry, a system error occurred.
   "Microsoft FoxPro"
   <additional error text>
The <additional error text> may or may not include one of the following: 

   error type 7
   illegal instruction
The error window will also contain a button labeled Restart.

After you choose the Restart button, the Macintosh will reboot.

NOTE: This behavior does not occur under Visual FoxPro for Macintosh.

RESOLUTION

Do not copy the .SPX, .SCT, or .SCX files to FoxPro for Macintosh. If the .SPR file is executed, a series of error messages will be encountered, but it will not be necessary to restart the computer. Screens created by the Screen Wizard should not be used with FoxPro for Macintosh.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce

  1. Using FoxPro version 2.6 for Windows, create a screen with the Screen Wizard.

  2. Copy the screen files created in step 1 to a system with FoxPro version 2.5b or 2.5c for Macintosh.

  3. Execute the screen (DO <screen_name>.spr) on the Macintosh.
