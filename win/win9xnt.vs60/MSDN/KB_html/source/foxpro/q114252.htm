PRB: User-Procedure Check Box Dimmed on Printer Driver Setup

 Last reviewed: April 18, 1995
Article ID: Q114252
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.0

SYMPTOMS

When you are attempting to define a printer driver setup, the User Procedure check box may be unavailable (dimmed) when conventional memory is too low.

RESOLUTION

The routine required to set up a user procedure requires that more conventional memory be made available. If memory is less than the minimum needed, the check box for setting user procedures will be disabled. To enable this option under low-memory conditions, exit FoxPro and free more memory.

For information on how to perform a clean boot in order to free conventional memory, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q95314
   TITLE     : How to Perform a "Clean" Boot for FoxPro

MORE INFORMATION

This behavior does not occur in FoxPro version 2.5 or later for MS-DOS.
