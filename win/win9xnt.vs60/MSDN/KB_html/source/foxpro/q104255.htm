PRB: Single Column Report Has Options for Multicolumn Report

 Last reviewed: June 27, 1995
Article ID: Q104255
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

SYMPTOMS

If the Print Order button is set to printing across the page and the number of columns is one, the report will act as if it were a multicolumn report and options will be disabled in the Detail band.

RESOLUTION

Make sure the Print Order is set to "down" (top to bottom printing); with that order, the report will act normally for a single column report. (The "down" print order is the default for reports.)

MORE INFORMATION

Steps to Reproduce Problem

  1. Open a report and choose Page Layout from the Report menu.

  2. For Print Order, select "across" (the button that indicates printing from left to right horizontally across the page).

  3. Choose OK.

This procedure will cause the single column report to have the options available to a multicolumn report; for example, the Remove Line If Blank option will not work, the Position Relative To options will be disabled, and the Constant Band Height will be disabled.
