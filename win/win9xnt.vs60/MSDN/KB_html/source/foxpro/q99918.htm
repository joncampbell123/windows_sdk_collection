Packing Database on Novell Removes Sharable Attribute

 Last reviewed: June 27, 1995
Article ID: Q99918
The information in this article applies to:
  • Microsoft FoxPro for Windows, version 2.5
  • Microsoft FoxPro for MS-DOS, versions 1.02, 2.0, and 2.5

SUMMARY

If a database is located on a Novell server and is flagged with the "sharable" attribute, when the database is packed in FoxPro, the sharable attribute will be removed.

MORE INFORMATION

When a database on a Novell server is packed using the PACK command, FoxPro creates a new database with the same name and copies all the records that are not marked for deletion to the new database. FoxPro does not copy the sharable attribute to the new database. The older database is then deleted, making it appear that FoxPro is removing the sharable attribute.

There are currently no workarounds for this behavior. However, the database does not need the sharable attribute in order to be shared over the network. Losing this attribute has no effect in FoxPro.
