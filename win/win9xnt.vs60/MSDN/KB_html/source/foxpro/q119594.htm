PRB: Cannot Use Wizards to Modify Screen, Label, or Query

 Last reviewed: April 30, 1996
Article ID: Q119594
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.6
  • Microsoft FoxPro for Windows, version 2.6

SYMPTOMS

You cannot use a FoxPro wizard to modify a screen, label, or query even though that screen, label, or query was created by a FoxPro wizard.

CAUSE

The screen, label, or query has been modified by a FoxPro power tool.

STATUS

This behavior is by design.

MORE INFORMATION

Once a FoxPro power tool has been used to modify a screen, label, or query that was created by a FoxPro wizard, the item cannot subsequently be modified by a wizard. The Screen, Label, and Query Wizards were not designed to modify screens, labels, or queries that have been used with the Screen Builder, Label Designer, and Relational Query By Example (RQBE) power tools.

If you attempt to modify a screen, label, or query from within the Catalog Manager by choosing the Modify button and that screen, label, or query was created with a wizard (and not subsequently touched by a power tool), the Catalog Manager will produce a window in which you can choose to edit the screen, label, or query with a wizard or with a power tool.

If you attempt to modify a screen, label, or query from within the Catalog Manager by choosing the Modify button and that screen, label, or query was created with a wizard but has subsequently been touched by a power tool (or was created with a power tool), the Catalog Manager will launch the appropriate power tool. The intermediate window described above is not presented under these circumstances.
