SET CONFIRM OFF May Cause Data Loss in Browse Mode

 Last reviewed: June 27, 1995
Article ID: Q97169
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

When you are editing data in a Browse session in FoxPro for Windows with KEYCOMP = WINDOWS, moving from one field to the next by typing causes FoxPro to automatically overwrite the field contents of the second field if one exists. For example, typing 5 characters in a 4-character field causes any data in the next data field to be highlighted and deleted. This is the default behavior of FoxPro when the Windows-style editing keys are used.

This behavior occurs because the default setting for CONFIRM is OFF. To prevent data loss, issue SET CONFIRM ON or SET KEYCOMP TO DOS before entering Browse mode.
