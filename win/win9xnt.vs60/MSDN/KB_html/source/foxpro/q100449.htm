FIX: Color Boxes or Text Labels Appear as Black Boxes

 Last reviewed: September 22, 1997
Article ID: Q100449
2.5x 2.60 | 2.5x 2.60a 
WINDOWS   | MACINTOSH
kbtool kbprg kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft FoxPro Distribution Kit for Windows, versions 2.5x, 2.6
  • Microsoft FoxPro Distribution Kit for Macintosh, versions 2.5x, 2.6a

SYMPTOMS

Color boxes or text labels may appear as black boxes when you are printing or previewing a report in a FoxPro for Windows compact .EXE file. When the .EXE file is run in the development version of FoxPro, the report appears and prints as expected. However, when the .EXE file is run using the support library (FOXW2500.ESL), the black boxes appear.

RESOLUTION

NOTE: The first two resolutions may not always work for FoxPro 2.6a for Macintosh.

Do one of the following to solve the problem:

  1. Delete the labels and create them again. If colored boxes are used, replace the color with a pattern, using black as the foreground color and white as the background color.

    -or-

  2. Select the text label on the report form, choose Mode, and then choose Transparent from the Object menu.

    -or-

  3. Use the text string "TEST" as the contents of the field with a black colored fill in the .EXE file.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in FoxPro 2.6a for Windows.
