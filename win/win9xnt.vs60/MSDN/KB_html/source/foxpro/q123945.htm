BUG: Strange Characters in Landscape MS-DOS Report

 Last reviewed: December 15, 1994
Article ID: Q123945
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.6a

SYMPTOMS

When using the HP III printer driver to print a Landscape report that has a a line that stretches across the entire width of the page, FoxPro prints the line up to column 223, then prints a CHR(30) followed by CHR(64), and then continues to print the line to the edge of the page.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. From the File Menu, choose Printer Setup.

  2. Click in the box for Printer Driver Setup.

  3. Choose New from the Printer Driver Setups Screen.

  4. From the list of printers, choose the HP III.

  5. From the General options, choose Orientation Landscape.

  6. Put in a Setup Name, and Click OK.

  7. Click Set Default. Next, click Set. Then click OK.

  8. Enter USE C:\FPD26\TUTORIAL\OFFICES.DBF in the Command window.

  9. From the File Menu, choose New. Then select Report, and click OK.

  10. From the Report Menu, choose Page Layout.

  11. In Page Layout, set the Right Margin Column to 240, and click OK.

  12. Position the Cursor on a Detail line in the Report.

  13. From the Report Menu choose Box.

  14. Using the Mouse, Drag the Box to the Left Margin (Column 240).

  15. From the File Menu, choose File Save As, and save the Report.

  16. Enter REPORT FORM <Report Name> TO PRINTER.
