FIX: FOR Clause Is Ignored in REPORT FORM Command

 Last reviewed: April 17, 1995
Article ID: Q88639
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.0

SYMPTOMS

In Microsoft FoxPro version 2.0 for MS-DOS, the FOR clause is ignored if a PDSETUP clause, a FOR clause, and the PREVIEW option are all included in the REPORT FORM <file> command.

STATUS

Microsoft has confirmed this to be a problem in Microsoft FoxPro version 2.0 for MS-DOS. This problem was corrected in Microsoft FoxPro version 2.0 for MS-DOS dated March 25, 1992.

MORE INFORMATION

The REPORT FORM <file> command allows you to specify which report to print. The FOR clause allows you to specify the records from the database to be included in the report. The PDSETUP clause uses the printer driver setup saved with the report.

PDSETUP does not need to be issued with the PREVIEW option because PREVIEW directs output to the screen.
