PRB: ASSIST Does Not Invoke Catalog Manager

 Last reviewed: April 18, 1995
Article ID: Q116106
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.6

SYMPTOMS

The Catalog Manager is not invoked when the ASSIST command is entered in the Command window.

CAUSE

The Standard (16-bit) version of FoxPro for MS-DOS (FOXPRO.EXE) is being used.

RESOLUTION

The Catalog Manager cannot be used under the Standard version of FoxPro version 2.6 for MS-DOS. To use the Catalog Manager under FoxPro version 2.6 for MS-DOS, invoke the Extended (32-bit) version by typing "FOXPROX" (without the quotation marks) at the MS-DOS prompt.

STATUS

This behavior is by design. The Catalog Manager cannot be used with the Standard version of FoxPro version 2.6 for MS-DOS because the Standard version cannot provide sufficient memory to the Catalog Manager. The Catalog Manager can only be invoked under the Extended version of FoxPro version 2.6 for MS-DOS since only the Extended version can provide sufficient memory to the Catalog Manager.

MORE INFORMATION

For more information about the _ASSIST memory variable and the ASSIST command, please see the following Microsoft Knowledge Base article: 

   ARTICLE-ID: Q113513
   TITLE     : _ASSIST System Memory Variable
