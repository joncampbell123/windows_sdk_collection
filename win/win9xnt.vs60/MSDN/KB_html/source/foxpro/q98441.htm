Invalid Reference to PDUPDATE.PLB File in DRIVER2.C

 Last reviewed: June 27, 1995
Article ID: Q98441
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, and 2.5a
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

SUMMARY

PDUPDATE.PLB is referenced in the DRIVER2.C source code for DRIVER2.PLB, which is provided in the GOODIES\PDRIVERS subdirectory. This reference is invalid; no PDUPDATE.PLB file is shipped with the product.

MORE INFORMATION

PDUPDATE.PLB is referenced in the following comment in DRIVER2.C. It does not cause any errors. 

   The following code was added as an enhancement request. It
   stores the address of the procedure loadPDParms and a four byte
   checksum of this address into the last element of _PDPARMS. If
   the user wants to update the api's internal copy of _PDPARMS, it
   can now be done by loading the library PDUPDATE.PLB.

   Note: The address of loadPDParms() function must be placed in
   the last element in the _PDPARMS array in order for PDUPDATE to
   work.
