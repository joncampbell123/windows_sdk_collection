FIX: READKEY() Returns Value 1 After CLEAR READ

 Last reviewed: October 14, 1997
Article ID: Q88053
2.00 | 2.50 2.50a MS-DOS | WINDOWS kbprg kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft FoxPro for MS-DOS, version 2.0
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

SYMPTOMS

In FoxPro 2.0 for MS-DOS and in FoxPro 2.5 and 2.5a for Windows, the READKEY function returns the value 1 after a CLEAR READ.

STATUS

Microsoft has confirmed this to be a problem in releases of FoxPro 2.0 for MS-DOS dated prior to November 15, 1991. This problem was corrected in the release of FoxPro 2.0 for MS-DOS dated November 15, 1991.

Microsoft has also confirmed this to be a problem in FoxPro 2.5 and 2.5a for Windows. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
