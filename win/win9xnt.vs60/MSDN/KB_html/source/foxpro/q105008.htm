PRB: Database Not Opened in .SPR File When Run in Project

 Last reviewed: April 29, 1996
Article ID: Q105008
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

SYMPTOMS

When an .SPR file is run by itself and the environment is saved, the databases are opened properly. However, when the same .SPR file is run in a project, a dialog box appears asking for the database names.

CAUSE

The default generated code for the Generate command on the Program menu opens and closes files automatically. The default generated code for a screen in a project does NOT open and close files automatically.

RESOLUTION

Edit the generate options for the screen set in the project by choosing the More button and then selecting the Open Files and Close Files check boxes.

Additional reference words: FoxWin 2.50 2.50a use open database dbf
