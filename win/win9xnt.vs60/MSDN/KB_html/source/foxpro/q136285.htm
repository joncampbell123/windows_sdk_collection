FIX: Duplicate Entry with Copy and Paste

 Last reviewed: October 29, 1997
Article ID: Q136285
2.60a WINDOWS kbusage kbbuglist kbfixlist

The information in this article applies to:

  • Microsoft FoxPro for Windows, version 2.6a

SYMPTOMS

The field name appears twice in a new field.

CAUSE

This can occur when, under Windows 95, you create a table in FoxPro version 2.6a, and to save time when creating similar field names, you select the name of a field, press CTRL+C to copy it to the clipboard, move the pointer to the desired new entry location, and press CTRL+V to paste it.

WORKAROUND

Delete the extra copy of the field name.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft Visual FoxPro version 3.0 for Windows.

MORE INFORMATION

Steps to Reproduce Problem

  1. Start FoxPro version 2.6a under Windows 95.

  2. Type the following in the Command window:

    CREATE <table name>

  3. Enter a field name.

  4. Select the field name and press CTRL+C.

  5. Move to a new field name area, and press CTRL+V.

The copied entry is placed in the new area twice.
