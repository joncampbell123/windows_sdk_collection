PRB: DBF w/ Shared Windows NT Rights Can Be Opened Exclusively

 Last reviewed: December 12, 1996
Article ID: Q131365
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5, 2.5a, 2.5b, 2.6, 2.6a

SYMPTOMS

Under Windows NT Workstation version 3.5 and Windows NT Server version 3.5, FoxPro can open a .DBF file in Exclusive mode even when it should be denied access according to the access rights set in Windows NT.

CAUSE

When a FoxPro database file is stored on a Netware Server and accessed from a Windows NT Workstation or Server, it is possible to obtain an exclusive open of the file when it is not appropriate.

This happens when the attributes of the .DBF file are set to R, W and S (read, write, and share).

RESOLUTION

Delete the .DBF file's S attribute to prevent the problem.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.5. A fix to this problem is in development, but has not been regression-tested and may be destabilizing in production environments. Microsoft does not recommend implementing this fix at this time. Contact Microsoft Technical Support for more information on the availability of this fix.
