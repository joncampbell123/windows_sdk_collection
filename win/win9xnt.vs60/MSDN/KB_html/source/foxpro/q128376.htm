PRB: Cannot Connect to ODBC Drivers on Windows 95

 Last reviewed: October 18, 1996
Article ID: Q128376
The information in this article applies to:
  • Microsoft Visual FoxPro for Windows, version 3.0

SYMPTOMS

An error message occurs, or the operation fails, when attempting to use Open DataBase Connectivity (ODBC) under Windows 95.

Errors When Attempting to Connect to a FoxPro 2.5x .DBF File

One of the following error messages occur when attempting to connect to a FoxPro for Windows 2.5x .DBF file:

  • Connectivity Error
  • Microsoft ODBC .dll drivers SQLSETCONNECT option failed
  • Failed to get the expression service

CAUSE

ODBC was not available in your Windows environment when Visual FoxPro was installed.

STATUS

This behavior is by design. The Windows 95 ODBC drivers were not available when Visual FoxPro for Windows shipped.

RESOLUTION

Microsoft has made available an application note titled "Microsoft ODBC Drivers for 32-Bit Programs" (GE1263). For more information about GE1263, please see the following article(s) in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q140255
   TITLE     : GE1263: Microsoft ODBC Drivers for 32-Bit Programs

MORE INFORMATION

Upon install, Visual FoxPro detects which platform is running and installs adequate drivers for this platform. As ODBC drivers were not available for Windows 95 when Visual FoxPro was shipped, they are not installed with Visual FoxPro under Windows 95. As a result, there is no ODBC functionality with Visual FoxPro in Windows 95.

Because ODBC is not installed under Windows 95, you will not be able to access remote data. As a result, the functionality of SQL pass through and remote views is not available when Visual FoxPro is installed under Windows 95.
