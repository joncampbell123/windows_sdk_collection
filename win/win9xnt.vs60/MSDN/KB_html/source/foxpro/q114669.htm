PRB: Catalog Manager Behaves Inconsistently with SQL Queries

 Last reviewed: April 30, 1996
Article ID: Q114669
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.6, 2.6a
  • Microsoft FoxPro for MS-DOS, versions 2.6, 2.6a

SYMPTOMS

Catalog Manager doesn't parse an SQL SELECT statement when you are running a .QPR file created with the Query Wizard that was later modified using the Relational Query By Example (RQBE) tool.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Use Catalog Manager to create an SQL query using the Query Wizard.
2. Close the Catalog Manager and bring up the Command window. 3. Modify the resulting query with RQBE and direct the output to a report. 4. Open the Catalog Manager again. 5. Choose the Use button to open the query.

The report runs. Contrast this with the correct and expected behavior of the same query originally created with the RQBE (the report doesn't run).

Additional reference words: FoxDos FoxWin 2.60 2.60a
