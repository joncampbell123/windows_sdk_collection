PRB: "Not a Numeric Expression" When FOR...ENDFOR Using Array

 Last reviewed: June 27, 1995
Article ID: Q119100
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5x, 2.6
  • Microsoft FoxPro for MS-DOS, versions 2.5x, 2.6

SYMPTOMS

FoxPro returns a "Not a numeric expression" error message if a FOR ... ENDFOR construct is using an array as its memory variable. For example: 

   DIMENSION x(5)
   FOR x(1) = 1 TO 10
     * Program statements
   ENDFOR

CAUSE

Page L3-524 of the "Language Reference" manual says that "The memory variable or array element does not have to exist before FOR ... ENDFOR is executed." This statement is incorrect. The FOR ... ENDFOR construct cannot use an array element as its memory variable.
