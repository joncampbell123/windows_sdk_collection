PRB: APPEND FROM Causes "File <Name> Does Not Exist" Error

 Last reviewed: June 27, 1995
Article ID: Q97646
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, and 2.5a

SUMMARY

Issuing the command APPEND FROM <cursor file> causes the error message "FILE <cursor file> DOES NOT EXIST" to appear.

This error message occurs because the APPEND FROM command requires a database name. The cursor file is the alias for a database. Therefore, referring to the database name instead of the alias will resolve the error, as shown in this example: 

   APPEND FROM DBF('cursor file')

MORE INFORMATION

The Append From option on the Database menu is disabled when a cursor file is in the current work area.
