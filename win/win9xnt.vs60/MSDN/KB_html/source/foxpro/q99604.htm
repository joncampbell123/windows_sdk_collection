FIX: Syntax Error with TICKTOCK Variable in Screen Builder

 Last reviewed: September 22, 1997
Article ID: Q99604 
2.50    | 2.50
WINDOWS | MS-DOS kbprg kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft FoxPro for Windows, version 2.5
  • Microsoft FoxPro for MS-DOS, version 2.5

SYMPTOMS

When you generate a screen in FoxPro for Windows, the error message "syntax error" may occur. As a result, FoxPro brings up GENSCRN.PRG, with the line containing TICKTOCK highlighted.

RESOLUTION

To correct this problem, issue the RELEASE TICKTOCK command in the Command window.

Microsoft has confirmed this to be a problem in FoxPro version 2.5 for Windows and FoxPro version 2.5 for MS-DOS. This problem was corrected in FoxPro version 2.5a, both platforms.

MORE INFORMATION

Steps to Reproduce Problem

  1. Type "Public Ticktock" (without the quotation marks) in the Command window.

  2. Create and then generate a screen.

The "syntax error" message appears when screen generation begins.
