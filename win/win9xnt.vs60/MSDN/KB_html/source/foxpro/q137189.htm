PRB: Known Issues Using SCO UNIX 3.2v5.0 with FoxPro 2.6

 Last reviewed: September 29, 1995
Article ID: Q137189
The information in this article applies to:
  • Microsoft FoxPro for UNIX, version 2.6

SYMPTOMS

This article provides information about the following issues pertaining to FoxPro version 2.6 for UNIX running under SCO UNIX 3.2v5.0:

  • When you use the -e FoxPro startup switch under SCO UNIX 3.2v5.0, certain characters, such as the letter S, don't appear as they are typed.
  • Using scancode mode on a Wyse 60 terminal doesn't work because the terminfo file under SCO UNIX 3.2v5.0 for a Wyse 60 terminal has changed.

CAUSE

Due to an update of SCO's UNIX operating system to release 3.2v5.0, a couple of known issues have surfaced in using this new operating system version with FoxPro for UNIX.

NOTE: SCO UNIX is manufactured by the Santa Cruz Operation, a vendor independent of Microsoft; we make no warranty, implied or otherwise, regarding this product's performance or reliability.

RESOLUTION

At this time, there is no resolution or workaround to the -e switch issue.

For the Wyse 60 terminal issue, however, use the terminfo files compiled under an SCO UNIX version prior to 3.2v5.0.

STATUS

Microsoft is researching this behavior and will post new information here in the Microsoft Knowledge Base as it becomes available.
