FIX: SOUNDEX() Does Not Always Return a Four-Character String

 Last reviewed: September 22, 1997
Article ID: Q125788
2.60a MACINTOSH kbprg kbbuglist

The information in this article applies to:

  • Microsoft FoxPro for Macintosh, version 2.6a

SYMPTOMS

SOUNDEX() will not consistently return a four-character string when used on a Power Macintosh. There is no way to know in advance if SOUNDEX() will return a two-, three-, or four-character string.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Visual FoxPro 3.0b for Macintosh.

MORE INFORMATION

Steps to Reproduce Problem

In the Command window enter: 

   ? SOUNDEX("Test")
   * On a Power Macintosh it will not return a four character string.
