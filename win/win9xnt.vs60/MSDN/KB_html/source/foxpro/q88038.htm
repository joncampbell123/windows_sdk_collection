FIX: POINT Character Misplaced When Not Set to Default

 Last reviewed: October 14, 1997
Article ID: Q88038
2.00 MS-DOS kbprg kbfixlist kbbuglist 

 ----------------------------------------------------------------------
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.0

SYMPTOMS

In FoxPro version 2.0, if the SEPARATOR and POINT characters are each set to other than the default value, and width of a numeric field is not sufficient to hold the edited result, then the POINT character is misplaced.

STATUS

Microsoft has confirmed this to be a problem in Microsoft FoxPro version 2.0 for MS-DOS. This problem was corrected in Microsoft FoxPro version 2.0 dated September 26, 1991.

MORE INFORMATION

SET POINT TO [<expC>] changes the decimal point to another character. SET SEPARATOR TO [<expC>] specifies the character used as a numeric separator.
