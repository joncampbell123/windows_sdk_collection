FIX: Replacing a Numeric Field with 65534+1.6

 Last reviewed: April 17, 1995
Article ID: Q88641
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.0

SYMPTOMS

In FoxPro 2.0 for MS-DOS, replacing a numeric database field with the value 65534+1.6 results in a zero being placed in the field. The numeric field is defined as the correct size to hold the results.

STATUS

Microsoft has confirmed this to be a problem in Microsoft FoxPro version 2.0 for MS-DOS. This problem was corrected in Microsoft FoxPro version 2.0 for MS-DOS dated March 25, 1992.
