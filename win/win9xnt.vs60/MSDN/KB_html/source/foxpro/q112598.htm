PRB: Pressing BACKSPACE Key Deletes Entire Contents of Field

 Last reviewed: April 30, 1996
Article ID: Q112598
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5x, 2.6x
  • Microsoft FoxPro for Macintosh, version 2.5x, 2.6a

SYMPTOMS

When a screen is run, if you press the BACKSPACE key when you first move the cursor to a GET field, the entire contents of the field are deleted. Some users may regard this as unexpected behavior because pressing the BACKSPACE key generally moves the cursor back one space or returns the cursor to the previous field.

CAUSE

The Select Field On Entry check box in the Screen Builder is selected for the GET object.

NOTE: This behavior doesn't occur under Visual FoxPro because its Form Builder does not have a Select Field On Entry check box.

RESOLUTION

To keep the BACKSPACE key from deleting the entire field, clear the Select Field On Entry check box for the GET object.
