BUG: Import from Type RPD Causes GP Fault

 Last reviewed: September 21, 1995
Article ID: Q125375
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, 2.5a, 2.5b, 2.6, 2.6a
  • Microsoft FoxPro for Windows, versions 2.5, 2.5a, 2.5b, 2.6, and 2.6a

SYMPTOMS

Importing from a RapidFile file (type RPD) can cause a general protection (GP) fault in applications written in FoxPro for Windows and can cause applications written in FoxPro for MS-DOS to terminate.

WORKAROUND

You can work around the problem by exporting an ASCII file from RapidFile and appending the file to an existing structure in FoxPro.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

Type the following command in the command window: 

   IMPORT FROM ART.RPD TYPE rpd
ART.RPD is one the sample files that ships with RapidFile.
