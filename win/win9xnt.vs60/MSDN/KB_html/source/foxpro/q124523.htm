PRB: ERROR: FOXTOOLS.MLB must be in System:Extensions Folder

 Last reviewed: May 16, 1996
Article ID: Q124523
The information in this article applies to:
  • Microsoft FoxPro for Macintosh, version 2.6a

SYMPTOMS

You receive this error: 

   FOXTOOLS.MLB must be in System:Extensions Folder

CAUSE

FoxPro for Macintosh requires that FOXTOOLS.MLB be in the System:Extensions folder in order to use the Catalog Manager and the Wizards.

RESOLUTION

If FOXTOOLS.MLB is not in the System:Extensions folder, reinstall FoxPro by using the Reinstall option, or remove FoxPro from the hard drive and perform a new installation.

If FOXTOOLS.MLB is in the System:Extensions folder, follow these steps:

  1. If you are using System 7.0.1, please review the following article in the Microsoft Knowledge Base:

    ARTICLE-ID: Q114751 

       TITLE     : Contents of the FoxPro for Macintosh ASLM Release Notes

  2. Look for a possible Init conflict by following these steps:

    a. Start the Macintosh computer with extensions off by holding down the 

          SHIFT key as you restart the computer.

    b. Place all extensions in a place other then the Extensions folder 

          except the extensions that must be there for the task at hand such
      as a printer driver. Make sure the Shared Library Manager and the
      FOXTOOLS.MLB file are both left in the Extensions folder.

  3. If you receive the error "The Apple Shared Library Manager Must be Installed" when running the SET LIBRARY TO FOXTOOLS command in the FoxPro Command Window, you may need to replace the (operating system) System File with a new copy of the System File that works with this Macintosh. Then, reinstall FoxPro.
